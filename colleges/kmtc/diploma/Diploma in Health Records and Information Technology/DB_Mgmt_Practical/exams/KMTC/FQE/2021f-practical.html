Section Two: MS Access

1.
Create a database in your folder and name it 'Access'.
Activity
You have been posted to a newly established KMTC campus where the accountant needs help in creating a working database to manage students' fees. The requirements are as given below:
2.
Table 1: Student details
  • Admission number: should allow numerics only and it is automatically generated (2 mks)
  • Names: Capture three names and should not allow a null entry (1 mk)
  • Gender: Provide a drop down menu of male and female (2 mks)
  • Marital status: Provide a drop down menu for single, married, divorced, widow, widower and separated (2 mks)
  • Age: Only a number not exceeding 60 years (1 mark)
  • Department: Provide a drop down menu of "health records and information, clinical medicine, nursing and physiotherapy (1 mk)
  • Programme: Provide a drop down menu of HRI-certificate, HRI-preservice diploma, HRI-upgrading diploma, Clinical Medicine, Nursing-KRN, Nursing-KRCHN and Physiotherapy.
Show Practical Solution

Table Design Steps:

  • Admission Number: Data Type: AutoNumber (Automatically generated/Numeric).
  • Names: Data Type: Short Text. Field Property > Required: Yes (No null entry).
  • Gender: Data Type: Short Text. Lookup Wizard > "I will type values" > "Male", "Female".
  • Marital Status: Data Type: Short Text. Lookup Wizard > Type list (Single, Married, etc.).
  • Age: Data Type: Number. Validation Rule: <=60. Validation Text: "Age must not exceed 60".
  • Department: Data Type: Short Text. Lookup Wizard > Type list.
  • Programme: Data Type: Short Text. Lookup Wizard > Type list.
3.
Table 2: Fee Payment
  • Admission number
  • Fees Due: Equal to KShs. 90,000 by default. Any other figure entered should be rejected with a note "enter the approved fees" (2 mks)
  • Fees Paid: Must be equal to or greater than KShs. 40,000. A lesser amount entered shall be rejected with a note, "Note students must pay a minimum of KShs. 40,000 for admission to proceed. (2 mks)
  • Date: By default it should capture the current date automatically after an entry (2 mks)
  • Balance: Difference between fees due and feed paid (2 mks)
  • NB: All the currency fields should indicate a cumulative total at the bottom of the table (2 mks)
Show Practical Solution

Table Design Steps:

  • Admission number: Number (Foreign Key).
  • Fees Due: Currency. Default Value: 90000. Validation Rule: =90000. Validation Text: "enter the approved fees".
  • Fees Paid: Currency. Validation Rule: >=40000. Validation Text: "Note students must pay a minimum of KShs. 40,000 for admission to proceed".
  • Date: Date/Time. Default Value: =Date().
  • Balance: Calculated Field. Expression: [Fees Due] - [Fees Paid].
  • Cumulative Total: Open table in Datasheet View > Click "Totals" button on ribbon > At the bottom of Fees Paid/Due columns, select "Sum".
4.
Create a query for HRI and Nursing departments capturing students with a balance equal to or greater than KShs. 30,000 with the following fields, Admission Number, Names, Department, Programme, Fees paid and balance (8 marks)
Show Practical Solution

Procedure:

  1. Create Query Design > Add both tables (ensure relationship exists via Admission Number).
  2. Select Fields: Admission Number, Names, Department, Programme, Fees Paid, Balance.
  3. Criteria 1 (Dept): In Department column: Like "*HRI*" OR Like "*Nursing*" (or specifically select from dropdowns if exact match).
  4. Criteria 2 (Balance): In Balance column: >=30000.
  5. Run and Save.
5.
  1. Create a form for student details using the form wizard and choose the 'columnar' layout for your form (5 marks)
  2. Insert a button to close window with the appropriate picture on the button (2 marks)
  3. Insert a button to 'find record' in your form using the relevant picture (2 marks)
  4. Apply red colour to the field labels and size 'to fit' (2 marks)
  5. Insert a picture of your choice in the form (2 marks)
Show Practical Solution

Procedure:

  1. Create Form: Create > Form Wizard > Select Student Details table > Select All Fields > Layout: Columnar > Finish.
  2. Close Button: Design View > Button Control > Drag on form > Categories: Form Operations > Action: Close Form > Picture: Stop/Exit > Finish.
  3. Find Button: Button Control > Categories: Record Navigation > Action: Find Record > Picture: Binoculars > Finish.
  4. Formatting: Select all Labels (on the left) > Property Sheet > Fore Color: Red. Layout > Size > To Fit.
  5. Picture: Insert > Image > Browse > Place on form header or detail.
