Data Entry Steps:

  1. Create Table I (Employee biodata):
    • Go to Create > Table Design.
    • Fields: Employee No (Short Text), Employee Name (Short Text), Gender (Short Text), Age (Number).
    • Save as Employee biodata.
    • Enter the data provided in Datasheet View.
  2. Create Table II (Employee payment):
    • Go to Create > Table Design.
    • Fields: Employee No (Short Text), Gross pay (Currency/Number).
    • Save as Employee payment.
    • Enter the data provided.

Note:

b)
Add the column after gross pay and name payee.
Show Practical Solution

Procedure:

  1. Open Employee payment table in Design View.
  2. In the field list, click the empty row below Gross pay.
  3. Type payee.
  4. Set Data Type to Calculated (if using Access 2010 or later) OR Currency (if you will calculate via query later). Given instruction (c), "Register formula", usually implies a Calculated field in modern Access exams.
c)
Register formula to calculate the payee at 30% of the gross pay.
Show Practical Solution

Method 1: Calculated Field (Table Design)

  1. In Design View for Employee payment table.
  2. Set the Data Type for payee to Calculated.
  3. In the Expression Builder window that pops up, type: [Gross pay] * 0.30.
  4. Click OK.
  5. Save the table.

Method 2: Update Query (Alternative)

  1. Create > Query Design > Add Table II.
  2. Select Update Query type.
  3. Field: Payee. Update To: [Gross pay] * 0.3. Run.
d)
Add the field with the name & NSSF after the payee and register a formula to calculate it at 1% for each employee based on gross pay (8 marks)
Show Practical Solution

Procedure:

  1. Open Employee payment table in Design View.
  2. In the row below payee, type the field name: NSSF (Avoid using special characters like '&' in field names if possible, but if required, type & NSSF).
  3. Set Data Type to Calculated.
  4. In Expression Builder, type: [Gross pay] * 0.01.
  5. Click OK and Save.
  6. Switch to Datasheet view to verify the calculations appear.
e)
Set employee no. as the primary key (4 marks)
Show Practical Solution

Procedure:

  1. Open Employee biodata table in Design View.
  2. Right-click the Employee No row selector.
  3. Select Primary Key.
  4. Do the same for the Employee payment table to ensure unique records for payments.
  5. Save both tables.
f)
Create the relationship for the two tables (4 marks)
Show Practical Solution

Procedure:

  1. Go to Database Tools > Relationships.
  2. Add both tables.
  3. Drag Employee No from Employee biodata to Employee No in Employee payment.
  4. (Optional) Check "Enforce Referential Integrity".
  5. Click Create.
  6. Save the relationship layout.
g)
Create a query consisting of employee biodata and the salaries of employees of more than 40,000/= (6 marks)
Show Practical Solution

Procedure:

  1. Go to Create > Query Design.
  2. Add both tables.
  3. Select fields: Employee No, Employee Name (from biodata), and Gross pay (from payment).
  4. In the Criteria row under the Gross pay column, type: >40000.
  5. Run the query to verify results (Should show records for 60400, 59300, 70560, 44900).
  6. Save query as High Salary Query.
h)
Create form using form wizard to include biodata fields, gross pay, payee and NSSF. (10 marks)
Show Practical Solution

Procedure:

  1. Go to Create > Form Wizard.
  2. In the dropdown "Tables/Queries", select Table: Employee biodata.
  3. Move all fields (Employee No, Name, Gender, Age) to the "Selected Fields" box using the >> or > button.
  4. Change the dropdown to Table: Employee payment.
  5. Select Gross pay, payee, and NSSF. Move them to the selected fields box.
  6. Click Next.
  7. Choose layout (Columnar is standard). Click Next.
  8. Title the form (e.g., Employee Full Details).
  9. Click Finish.
