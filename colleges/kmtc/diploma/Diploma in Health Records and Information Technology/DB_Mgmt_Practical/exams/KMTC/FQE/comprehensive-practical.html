  5. Split the Blood Pressure column into systolic and diastolic components. Calculate the average systolic and diastolic blood pressure. (3 marks)

Procedures:

  1. Data Entry: Type the data exactly into cells A1:I6. Apply borders and bold headers.
  2. Calculations (Age):
    • Mean: =AVERAGE(B2:B6)
    • Median: =MEDIAN(B2:B6)
    • Std Dev: =STDEV.S(B2:B6)
    • Avg Heart Rate: =AVERAGE(E2:E6)
  3. Split Blood Pressure:
    • Select BP column (D2:D6).
    • Data Tab > Text to Columns.
    • Select "Delimited" > Next > Check "Other" and type / > Finish.
    • Rename columns to "Systolic" and "Diastolic".
    • Calculate Averages for new columns.
  4. Highlight High BP:
    • Select the Systolic and Diastolic data.
    • Home > Conditional Formatting > New Rule > Use a formula.
    • Formula: =OR($D2>140, $E2>90) (assuming D is Systolic, E is Diastolic).
    • Set Format (e.g., Red Fill).
  5. Scatter Plot: Select Lab Result 1 and 2 columns > Insert > Charts > Scatter.
  6. Histogram: Select Age column > Insert > Charts > Histogram (Statistic Chart).
  7. Bar Chart (Gender): First create a count summary (Male: 3, Female: 2) in a separate small table, then Insert Bar Chart based on that summary.

Key Design Steps:

  • Primary Keys: PatientID, AppointmentID, HistoryID, PrescriptionID.
  • Foreign Keys: PatientID in Appointments, MedicalHistory, and Prescriptions tables linking back to Patients table.
  • Relationships: One-to-Many from Patients to all other tables. Enforce Referential Integrity.
  • Data Entry: Enter all 5 records per table as shown in the images.
Queries
  1. Create a query to display all patient information with their appointment details.
  2. Create a query to find patients with specific medical conditions.
  3. Advanced Query: Create a query to find patients with a specific medical condition who have upcoming appointments. Provide the SQL code for this query. (2 Marks)
Show SQL Solution

SQL Code for Advanced Query:

SELECT Patients.FirstName, Patients.LastName, MedicalHistory.Diagnosis, Appointments.AppointmentDate
FROM (Patients INNER JOIN MedicalHistory ON Patients.PatientID = MedicalHistory.PatientID) 
INNER JOIN Appointments ON Patients.PatientID = Appointments.PatientID
WHERE (((MedicalHistory.Diagnosis)=[Enter Diagnosis]) AND ((Appointments.AppointmentDate)>=Date()));

Explanation: This SQL joins three tables. It asks the user for a diagnosis (e.g., "Hypertension") and filters for appointments occurring today or in the future.

Forms & Reports
  • Design a form for entering a new patient's information into the "Patients" table.
  • Design a form for scheduling a new appointment.
  • Generate a report listing all patients and their medical history.
  • Generate a report showing upcoming appointments.
Validation & Backup
  • Data Validation: Ensure prescription dosages are numeric and cannot be negative. (Validation Rule: >=0, though "10mg" is text, so question implies separating unit or checking field type logic. If text, skip rule. If Number, rule applies).
  • Backup: Save the database with password "FQE001". (File > Info > Encrypt with Password).
