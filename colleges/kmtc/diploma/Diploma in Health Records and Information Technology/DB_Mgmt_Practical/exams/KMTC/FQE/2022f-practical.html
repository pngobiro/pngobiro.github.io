Section 1: MS Access (30 Marks)

Your friend has asked you to help him design a database to help him track his patients.

1.
Create a new database and save it on the desktop and name it "clinic database" followed by your last digit of exam number ie "clinic database 001". (1 mark)
Show Practical Solution

Procedure:

  1. Open Microsoft Access.
  2. Click Blank Database.
  3. File Name: clinic database 001 (replace 001 with your digits).
  4. Location: Browse to Desktop.
  5. Click Create.
2.
Create a Table called PATIENT TABLE as show below; (5 marks)
Field NameData typeField size or font
ID NumberText10
NameText15
SurnameText15
Telephone numberNumberLong integer
D. O. BDate/timeMedium data
Payment methodText20
NationalityText25
NHIFYes/NoYes/No
Show Practical Solution

Procedure:

  1. Go to Create > Table Design.
  2. Enter fields and set properties in the bottom pane:
    • ID Number: Short Text, Field Size 10.
    • Name: Short Text, Field Size 15.
    • Surname: Short Text, Field Size 15.
    • Telephone number: Number, Field Size Long Integer.
    • D. O. B: Date/Time, Format Medium Date (Note: "Medium data" in QP likely means Medium Date).
    • Payment method: Short Text, Field Size 20.
    • Nationality: Short Text, Field Size 25.
    • NHIF: Yes/No, Format Yes/No.
  3. Save as PATIENT TABLE.
3.
Choose an appropriate primary key (1 mark)
Show Practical Solution

Answer:

The most unique identifier is ID Number.

Procedure:

  1. Right-click the ID Number row selector.
  2. Select Primary Key.

  1. Open PATIENT TABLE in Datasheet View.

5.
Create a lookup wizard for the payment field as shown in above table (2 marks)
Show Practical Solution

Procedure:

  1. Switch to Design View.
  2. Select Payment method field.
  3. In Data Type, select Lookup Wizard.
  4. Select "I will type in the values that I want".
  5. Type the values: NHIF, Cash, Credit, Insurance.
  6. Click Finish.
  7. Save the table.
6.
Sort the table in ascending order by surname (1 mark)
Show Practical Solution

Procedure:

  1. Open table in Datasheet View.
  2. Click the dropdown arrow on the Surname column header.
  3. Select Sort A to Z.
7.
Move the date of birth and telephone number fields so that the date of birth field is now directly after the surname field (1 mark)
Show Practical Solution

Procedure:

  1. In Datasheet View, click the D. O. B column header to select it.
  2. Click and drag the column header to the left, dropping it immediately after the Surname column.
  3. The order should be: ID, Name, Surname, D.O.B, Telephone...
8.
Create a form with all fields on the patients table. Name the form PATIENTS ENTRIES (4 marks)
Show Practical Solution

Procedure:

  1. Highlight PATIENT TABLE.
  2. Go to Create > Form.
  3. Save the form as PATIENTS ENTRIES.
9.
Insert a picture in the form and ensure that all the text is visible (2 marks)
Show Practical Solution

Procedure:

  1. Open PATIENTS ENTRIES form in Design View.
  2. Go to Form Design tab > Insert Image.
  3. Select an image from your computer.
  4. Draw a box in the Form Header or Detail section to place the image.
  5. Ensure it doesn't overlap text fields (resize sections if needed).
10.
Create report based on the patient table showing the fields ID number, name, telephone number and payment method (2 Marks)
Show Practical Solution

Procedure:

  1. Go to Create > Report Wizard.
  2. Select Table: PATIENT TABLE.
  3. Select fields: ID Number, Name, Telephone number, Payment method.
  4. Click Next > Next > Finish.
11.
Create a new table called "PAYMENT TABLE" using the Query method. The table should have the following from the patient table; ID No., surname, payment method and NHIF (2 Marks)
Show Practical Solution

Procedure:

  1. Go to Create > Query Design.
  2. Add PATIENT TABLE.
  3. Select fields: ID Number, Surname, Payment method, NHIF.
  4. In the Query Design ribbon, click Make Table.
  5. Table Name: PAYMENT TABLE. Click OK.
  6. Click Run (!). Confirm adding rows.
  7. Save Query as Make Payment Table Query.
12.
Create a query showing patients born after 1972. Save the query as "AGE QUERY" (2 marks)
Show Practical Solution

Procedure:

  1. Go to Create > Query Design.
  2. Add PATIENT TABLE.
  3. Add fields: ID Number, Name, D. O. B (and others if desired).
  4. In Criteria row under D. O. B, type: >#31/12/1972# (Access uses # for dates).
  5. Run to verify.
  6. Save as AGE QUERY.
13.
Create a Macro to exit the database (2 marks)
Show Practical Solution

Procedure:

  1. Go to Create > Macro.
  2. In the dropdown, select QuitAccess.
  3. Save as ExitMacro.
