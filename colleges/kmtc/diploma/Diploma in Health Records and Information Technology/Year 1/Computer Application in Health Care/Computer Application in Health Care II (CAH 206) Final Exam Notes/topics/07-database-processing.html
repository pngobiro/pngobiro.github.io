Database Processing

Database Fundamentals

1. Database

Database is an organized collection of data. It's considered organized because data is stored in categories that are accessible in a logical/particular manner.

2. Components of database

  • Files - a file is a large organized data structure that contains related data E.g. Records. A file stores data and programs in the secondary memory.
  • Records - it's any entry of data in a database. It can also be defined as a "group of related data about on item".
  • Field - a column of similar information /or/ a group of vertically arrange data that contains similar information of the same data type.

3. Database system

Database system is comprised of 4 major components namely:-

  • Data - values physically stored in the database
  • Database Hardware - this are computer hardware on which the database resides E.g. Secondary storage disk, together with associated devices e.g. CPU, channels ect.
  • Database Software - it contains DBMS which is an application program that assist users to store information in an orderly manner for timely and quick retrieval. It provide all the functions needed to define the Data Structure and the way they are store.
  • Database user - database users are classified into three broad categories namely:-
    1. Application program
    2. End user
    3. Database Administrator

5. Entity-relationship modeling

a. Entity

  • An entity is a thing or an object of significant, either real or imaginary, about which information need to be known and stored.
  • An entity maybe an object with physical existence E.g. person, house etc. or a conceptual existence E.g. Job, Company, Country etc.
  • An entity has partial properties/characteristics/attributes which describe it.

b. Attribute

  • An attribute is any information that serve to Qualify, Quantify, Identify, Classify, or Express the state of an Entity.
  • It can also be defined as "Any description of a thing of significance E.g. Text, Number, Picture, Feelings, Smell, and Sound etc."
  • Primary Key is an Attribute that is used to uniquely identify an entity/record(identify a row in a table) and is chosen from candidates Key.

6. Relationship among data

Relationship is "An association among entities". It may also associate an entity with itself. There are three types of relation namely:-

  • One - to - one relationship e.g. student and admin. No.
  • One - to - many relationship e.g. student and subject
  • Many - to - many relationship

7. Database model

Database model is a "Specific method for describing the structure and processing within a database". The three most used models are:-

  • Hierarchical model - establishes a parent-child relationship. The origin of data is called the Root.
  • Relational model - establishes relationship among entities. Data is represented in a simple Row-Column (tabular)format.
  • Network model

8. Features of relationship

  • Each Row/Record in a relation contain data about the same Object/Entity
  • Each Column/Field contains data about an attribute/property of an Object/Entity
  • Each Column/Field in a table has a unique Name/Label
  • The order of the Field/Column in the table is irrelevant
  • All entries in a Field/Column must be of the Same Data Type e.g. Name, date, number etc.
  • No Row/Record in the table can be identical (no repetition)

Microsoft Access

Ms-Access database consist of a single file that holds the data tables, forms, reports, queries and indexes.

10. Components/objects of Ms Access database

  • Tables - a table is a database object that consists of Row/Record and Column/Field used to store data in a database. It's the basic unit of storage.
  • Forms - is a database object used to simplify data entry, viewing, editing and deleting in a database.
  • Reports - are use for viewing and printing of some or all the data in one or more databases.
  • Queries - are use to select information that meet a given criteria in one or more databases. A select query is used to create subsets of data that you can use to answer specific questions.

Ms-Access provides a "WIZARDS" that help users design queries. Ms-Access can create a primary key if no single field have the properties required for it (i.e. should be Unique and not Null). Primary key is used to uniquely identify a Record/Row in a table. Indexing - an index file contains a subset of the data files based on one or more Key Fields.

11. Data integrity

Data integrity is the process of ensuring the correctness and completeness of data in the database. This is to ensure validity/validation of data values when the contents of the database are modified so that the integrity of the database is not lost. Some of the constraints of data integrity are(those that make data have errors):-

  • Validity integrity - ensure data entered matches the Field/column data type
  • Entity integrity - avoid duplication or omission of data/record by ensuring that no primary key is duplicated or is left Null
  • Referential integrity - it establishes a parent-child relationship i.e. any change in the parent table must and should be reflected in the child table.

The Access Interface and Tools

12. Access interface

The look and feel of Ms-Access interface is very similar to many of the other Ms-Office products. I.e. it has a Title Bar, Tool Bar, View Bar, Tab Bar, Status Bar, Etc.

13. Database toolbars

It contains all tools that are needed in access, but are in different Tab in Access 2007 e.g. Font formatting tools, Importation and exportation of data to/from external database, Creation of Tables, Forms, Reports and Queries, Database tools e.g. creation of relationship and analyzing data.

14. Tables

They are used to store your data in a database. To create a table, you must 1st create a database that will contain all the database objects. Table can be created either by using the:

  • Table wizard
  • Design view - used to create the table manually

Design view has 3 components:

  • Field name - they are user defined and depend on the application to be used for
  • Data type - there are different data type in Access e.g. text, memo, number, date and time, currency, yes/no, ole object and auto-number
  • Description - use to describe the field

15. Primary key

A good relational database contains tables where each record/data can be uniquely identified. One field in the table contains a unique value for each record in the table. The unique field is called primary key and is always recommended that you define a primary key for every table you create.

16. Field properties

It contains two Tabs that can be used to further define the data type and format. They include:-

  • General Tab - contain most of the information used to define the data type
  • Lookup Tab - to look up values from another query or table to supply information to supply information to this field

17. Linking tables

We link tables to create a relationship. Relationship is the way in which information in one table is related/associated to information in another table.

Forms, Queries, and Reports

18. Forms

It's used to provide a friendly interface between users and Access database. A Form can be used to:-

  • Display and modify data e.g. Add, delete etc.
  • Accept user inputs - accept certain values
  • Controlling application flow
  • Displaying messages e.g. error or warnings
  • For printing information

19. Controls tools

This are tools designed to help users navigate and utilize a Form e.g. Buttons, Radio buttons, text boxes etc. This controls are broken down into three categories namely:-

  • Bound - this are field of data that come from a table or a query and must have a bound control for each field that you display on the Form. Bound control cannot create a calculation
  • Unbound - contains a label or a text box. Calculation can be created from unbound control
  • Calculated - are values calculated in the Form e.g. Totals, Subtotals and average

20. Form layout

All forms must have a Detail section namely:-

  • Form Header - display information that remain the same for every record e.g. Title for the form
  • Page Header - display information e.g. title or header at the top of every printed page
  • Page Footer - display information e.g. date or page number at the bottom of every printed page
  • Form Footer - display information that remain the same for every record e.g. total and sub-total

You should be able to create a custom form, modify and edit a form, add graphics to a form, and create a sub-form.

21. Expression Builder

Access includes built-in functions similar to the function in Excel. One can also use it to develop owns calculation. To define an expression, ones need to:-

  1. Select unbound control(or group of instruction) to select, right click the control then select the properties and click on the expression builder button (...)
  2. Begin the expression by typing "=" or click the equal sign button
  3. Click the button that represent the calculation that you want to enter or double-click on one of the folders that is contained in the left box of the expression builder box
  4. Locate field from table, query, form or report for calculation by doubleclicking on the corresponding folder in the left box
  5. Type any value or constants you would like to include in the expression

22. Querying Data

A query is a question (or group of instruction) that you ask the database when you want to pull a specific records from the database. The results/respond is/are set of records that meet the criteria that you set. A query can also be define as - custom-designed view of data from one or more tables in a database. In Access, one can query by using:-

  • SQL statement
  • Query wizard
  • Designed a query using the design view

Query can also be used to view data, insert data, delete data, and create new table from data retrieved from one or more existing tables.

23. Common criteria

Examples include: Between value 1 and value 2, In, Null, Is not null, Like, Not.

24. Wildcard character

They are used as placeholders for other characters when you don't know exactly what you are looking for. They include:-

  • "*" - match any number of characters e.g. wh*
  • ? - match any single alphabetical character e.g. b?ll
  • [] - match any single alphabetical character within the brackets
  • ! - match any character not in the brackets e.g. b[ae]ll
  • "-" - matches any one of a range of characters e.g. d[a-d]d
  • # - matches any single numeric character e.g. 1#3

25. Sorting (retrieving records)

Sorting is - arranging data/record in order, according to a specific field, either in ascending or descending.

26. Reports

They are the best way to share data with others in a precise and understandable manner. Reports are very similar to forms, differences being that you can only use a report to print data and not anything else. One can create a report just like a form.
