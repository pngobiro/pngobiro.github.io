Data Security

1. Definitions

Data security is the protection of computer hardware, computer software (programs) and data in the computer and communication system against unauthorized modification, destruction, disclosure or transfer whether accidental or intentional.

Data control is the measures taken to enforce the security of the programs and data. In order to protect against data lose, control measures need to be put in place. These controls may be just procedures, software or hardware. Usually a combinations of all will be required to protect against all possible threats.

2. Various threats to data security

i. Physical damage and theft

Theft

In an event of computer theft, one will lose data and information stored in it plus computer itself. In most of the cases, data stored in the computer is by far more expensive that the computer itself.

System crash

This is the total computer failure that occurs when the computer is damaged.

Protection against theft and system crash

One should regularly backup his/her data so that you can restore the information to the point of last back up. Incomplete backup (infrequent, incomplete), may lead to loss of data stored in the system. In the case of computer theft, one will loss the hardware itself.

ii. Viruses

Definition

This is a hidden self-replicating computer program/software written with ill motives by mischievous individuals with bad intention and spread to other computers by sharing infected files or/and programs. Viruses are designed to "self replicate" or "clone" themselves, that is, secretly copy themselves from place to place. This process is called infection.

Virus detection

We detect a virus when we experience one or more of the following symptoms:

  • Slow down of computer programs or slow down of the computer general performance
  • Unusual and frequent error messages
  • Change in file size or replication of computer files
  • Loss or modification of data
  • System failure

Protection against viruses

  • Installing Anti-virus software in your computer
  • Files should be backup so that it can be restored back in the event of a system failure
  • One should avoid using programs of unknown origins
  • One should not open an attached file from unknown origin in an email
  • Always check for viruses on portable disks when using it to transfer files from one computer to another

iii. Software Bugs

These are errors in computer software that are usually unintentional. Running a computer with a bug can lead to serious malfunctioning to the computer and may lead to loss of data.

iv. Trojan horses

These are dangerous pieces of computer program that comes intentionally hidden or appears like a software that is well-behave and useful.

v. Software chameleons

This type of program is made to look harmless but in the background, it carries out illegal adjustments to your computer. E.g. it may be designed to record your secret password or steal a tiny amount of money over a long period of time.

vi. Software bombs

This is programs which cause destruction as soon as you try to execute or run it.

vii. Logical bombs

Works like software bombs but it triggered by some condition such as an attempt to carry out a certain operation such as saving a file.

viii. Time bombs

These are programs triggered by dates or after a certain number of a particular operations has been carried out.(e.g. on 12th dec 1999 live Mellissa)

ix. Replicators

This copies itself onto the disk &memory (ram) so many times that space is not available for other programs to run thus crippling the computer.

x. Worms

These travel across networks & are usually designed to cause alteration similar to software chameleons wherever they go. They are difficult to detect with antivirus.

3. Computer crimes

i. Hacking

This is when an individual gets unauthorized access into the computer system. Data being transmitted in a network can also be intercepted. Once an hacker breaks into the system, they can:

  • Take a look at the data and information stored in the system
  • Steal the information
  • Modify or destroy the information

Protection against hacking

  • Log off or shut down the computer when one is not using it
  • Choose passwords that are not obvious and keep them confidential and should not be written down anywhere
  • Change password frequently
  • Encrypt data before sending them over the network
  • One should also use other methods of authentication e.g. fingerprints, voice recognition, face recognition etc.

ii. Trespass

This is when a person is able to connect to your computer through a computer network and is able to control it in some way, e.g. accessing a file or running a program.

iii. Tracking

This is when a person can monitor what you are doing on your computer or your current location from a remote computer.

iv. Tapping

This is when a person can gain access to information that is being transmitted via communication channel.

v. Cracking

This is similar to hacking only that in cracking, one is doing so with the intention of causing damage or self gain.

vi. Sabotage

This is when a person renders another person's system unusable through a number of activities e.g. denial of service.

vii. Spam

This is when a person sends unsolicited junk email, called spam, by accessing a list of email address, to thousands of users.

4. Privacy, Fraud, and Protection

Privacy and fraud

Privacy is when a person is able to keep his/her secrets without other people knowing them. Cracking, trespassing, tracking, tapping and hacking are all a violation of one's privacy and are generally considered an offence/crime/fraud in most countries.

Detection and protection of computer crimes

One should take the following precautions:

  • Physical security
  • Audit trail
  • Data encryption
  • Firewalls
  • install antivirus
