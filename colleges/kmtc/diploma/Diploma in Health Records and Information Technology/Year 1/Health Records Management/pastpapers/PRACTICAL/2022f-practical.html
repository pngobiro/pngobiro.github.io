Section 1: MS Access (30 Marks)

Patient Database Design
1.

Your friend has asked you to help him design a database to help him track his patients.

Tasks:

  1. Create a new database and save it on the desktop. Name it "clinic database" followed by your last digit of exam number (e.g., "clinic database 001").
  2. Create a Table called PATIENT TABLE with the following structure: (5 marks)
    Field Name Data Type Field Size / Format
    ID NumberText10
    NameText15
    SurnameText15
    Telephone numberNumberLong Integer
    D.O.BDate/TimeMedium Date
    Payment methodText20
    NationalityText25
    NHIFYes/NoYes/No
  3. Choose an appropriate primary key. (1 mark)
  4. Enter the provided data (5 records) into the table. (5 marks)
  5. Create a lookup wizard for the payment field. (2 marks)
  6. Sort the table in ascending order by surname.
  7. Move the date of birth and telephone number fields so that the date of birth field is now directly after the surname field.
  8. Create a form with all fields on the patients table. Name the form PATIENTS ENTRIES.
  9. Insert a picture in the form and ensure that all the text is visible.
  10. Create report based on the patient table showing the fields ID number, name, telephone number and payment method. (2 Marks)
  11. Create a new table called "PAYMENT TABLE" using the Query method. The table should have the following from the patient table: ID No., Surname, Payment method, and NHIF.
  12. Create a query showing patients born after 1972. Save the query as "AGE QUERY".
  13. Create a Macro to exit the database.
Show Step-by-Step Guide

Practical Steps Guide:

1. Database Creation:

  • Open MS Access > Blank Database > File Name: "clinic database [digit]" > Click Create.

2. Table Design:

  • Go to Create tab > Table Design.
  • Enter Field Names and Data Types as specified.
  • In Field Properties (bottom pane), set "Field Size" or "Format" as required (e.g., for D.O.B, set Format to "Medium Date").

3. Primary Key:

  • Right-click "ID Number" row selector > Select Primary Key (ID Number is unique for every patient).
  • Save table as "PATIENT TABLE".

4. Data Entry:

  • Switch to Datasheet View. Enter the 5 records provided in the exam paper.

5. Lookup Wizard:

  • Switch back to Design View. Click "Payment method" data type > Select Lookup Wizard.
  • Select "I will type in the values that I want".
  • Type options: "NHIF", "Cash", "Credit", "Insurance". Finish.

6. Sorting & Moving Columns:

  • In Datasheet View, click the drop-down arrow on "Surname" > Sort A to Z.
  • Click and hold the "D.O.B" column header, drag it to the left of "Telephone number".

7. Forms & Reports:

  • Form: Select Table > Create > Form Wizard > Add all fields > Finish. Name it "PATIENTS ENTRIES". Open in Design View to insert an image (Design > Insert Image).
  • Report: Create > Report Wizard > Select specific fields (ID, Name, Tel, Payment) > Finish.

8. Queries:

  • Make Table Query: Create > Query Design > Add Patient Table > Drag required fields to grid. Click "Make Table" in ribbon > Name new table "PAYMENT TABLE" > Run (!).
  • Age Query: Create > Query Design > Add Patient Table > Add all fields. Under "D.O.B" criteria, type: > #31/12/1972# (assuming "born after 1972"). Save as "AGE QUERY".

9. Macro:

  • Create > Macro. Select action QuitAccess. Save Macro.

Section 2: Web Design (10 Marks)

HTML Programming
2.

You have been identified to create a webpage to help in COVID-19 vaccine distribution for Kitui County.

Task: Generate a HTML code for the following output:

KITUI COUNTY GOVERNMENT

COVID - 19 TRACKING

ENTER YOUR DETAILS FOR QUICK RESPONSE

Name:

ID No:

Location:

Sub location:

Telephone No:

Show HTML Code Solution

Solution Code:

Save this code as covid_tracking.html.

<!DOCTYPE html> <html> <head> <title>Kitui County COVID-19 Tracking</title> </head> <body> <!-- Centered Header Section --> <center> <h1>KITUI COUNTY GOVERNMENT</h1> <h2>COVID - 19 TRACKING</h2> <h3>ENTER YOUR DETAILS FOR QUICK RESPONSE</h3> <hr> </center> <!-- Form Section --> <form> <p>Name: <input type="text" name="name"></p> <p>ID No: <input type="text" name="idno"></p> <p>Location: <input type="text" name="location"></p> <p>Sub location: <input type="text" name="sublocation"></p> <p>Telephone No: <input type="text" name="tel"></p> </form> </body> </html>

Note: The exam output shows basic text inputs. You can use standard <input> tags inside paragraph tags <p> or break tags <br> to achieve the layout. The centering can be done using the legacy <center> tag (common in older exams) or CSS style="text-align:center".
