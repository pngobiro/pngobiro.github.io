Step-by-step MS Access Procedures:

1. Database Creation:

  • Open Access. Blank Database. Name: KK Salary Management System.

2. Table I (Employee Biodata):

  • Create Table Design. Fields: Employee No (Short Text/Number), Employee Name (Short Text), Gender (Short Text), Age (Number).
  • Set Employee No as Primary Key.
  • Enter the data provided in Datasheet View.

3. Table II (Employee Payment):

  • Create Table Design. Fields: Employee No, Gross_pay (Currency/Number).
  • Set Employee No as Primary Key.
  • Data Entry: Copy the Employee Numbers from Table I in order and assign the Gross Pay values sequentially (e.g., 1989049 gets 60400, 2000346 gets 59300, etc.).

4. Calculated Fields (Payee & NSSF):

  • Method A (Calculated Data Type - Access 2010+):
    • In Design View of Table II, add field payee. Data Type: Calculated. Expression: [Gross_pay] * 0.30.
    • Add field NSSF. Data Type: Calculated. Expression: [Gross_pay] * 0.01.
  • Method B (Update Query - Classic Method):
    • Create fields payee and NSSF as Currency/Number type.
    • Create Query Design > Add Table II. Change to Update Query.
    • Field: payee -> Update To: [Gross_pay] * 0.3. Run.
    • Field: NSSF -> Update To: [Gross_pay] * 0.01. Run.

5. Relationships:

  • Database Tools > Relationships. Add both tables.
  • Drag Employee No from Biodata table to Payment table.
  • Enforce Referential Integrity (One-to-One relationship since both are Primary Keys).

6. Query (> 40,000):

  • Create Query Design. Add both tables.
  • Select Biodata fields (Name, etc.) and Gross_pay.
  • Criteria under Gross_pay: > 40000.

7. Form:

  • Create > Form Wizard.
  • Select all fields from Biodata table and Payment table (Access handles the join).
  • Choose layout (Columnar/Tabular). Finish.
