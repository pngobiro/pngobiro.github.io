Question 1 (20 marks)

(a) Outline four guidelines followed when creating flowcharts. (4 marks)

(b) Differentiate between logical and physical system design models. (4 marks)

(c) Biltech, an ICT Company has been using project management software in its projects. Explain three benefits that the company would realise from this. (6 marks)

(d) John developed a terms of reference document during systems problem definition stage. Explain three benefits of this document. (6 marks)

Answer

(a) Guidelines for creating flowcharts:

  1. Use standard symbols
    • Follow flowchart conventions
    • Maintain consistency
  2. Flow direction
    • Top to bottom
    • Left to right
  3. Clear labeling
    • Descriptive text
    • Concise wording
  4. Single flow lines
    • No crossing lines
    • Clear connections

(b) Logical vs Physical design models:

Logical Design Physical Design
Abstract representation Implementation details
What the system does How the system does it
Technology independent Technology specific
Business process focus Technical solution focus

(c) Benefits of project management software:

  1. Resource management
    • Allocation tracking
    • Workload balancing
  2. Timeline monitoring
    • Schedule tracking
    • Milestone management
  3. Communication
    • Team collaboration
    • Progress reporting

(d) Benefits of terms of reference document:

  1. Clear scope definition
    • Project boundaries
    • Deliverables specification
  2. Stakeholder alignment
    • Common understanding
    • Expectations management
  3. Project control
    • Reference document
    • Change management

Question 2 (20 marks)

(b) Explain two advantages of managing an ICT project. (4 marks)

    • Follow-up questions
  3. Non-verbal cues
    • Body language
    • Emotional context
  4. Flexibility
    • Adaptable questions
    • Dynamic discussion

(b) Advantages of ICT project management:

  1. Resource optimization
    • Efficient allocation
    • Cost control
  2. Risk mitigation
    • Early identification
    • Proactive handling

(c) Risk minimization activities:

  1. Risk assessment
    • Regular reviews
    • Impact analysis
  2. Contingency planning
    • Backup strategies
    • Alternative solutions
  3. Team training
    • Risk awareness
    • Response procedures

(d) User training methods:

  1. Classroom training
    • Group instruction
    • Hands-on practice
  2. One-on-one training
    • Personal attention
    • Customized pace
  3. Computer-based training
    • Self-paced learning
    • Interactive tutorials

Question 3 (20 marks)

(a) Outline four advantages of using prototype during system development. (4 marks)

(b) Describe each of the following fact finding methods used in system development:

(i) observation;

(ii) research;

(iii) document inspection. (6 marks)

(c) State two differences between system design and system coding stages. (4 marks)

(d) A student opted to use bottom-up approach during systems analysis. Explain three possible reasons for this. (6 marks)

Answer

(a) Advantages of prototyping:

  1. Early feedback
    • User validation
    • Requirements refinement
  2. Risk reduction
    • Early testing
    • Design validation
  3. User involvement
    • Active participation
    • Better understanding
  4. Cost savings
    • Early problem detection
    • Reduced rework

(b) Fact finding methods:

(i) Observation:

  • Direct monitoring of activities
  • Real-time process understanding
  • Non-intrusive data collection

(ii) Research:

  • Literature review
  • Industry standards study
  • Similar systems analysis

(iii) Document inspection:

  • Existing documentation review
  • Process manual analysis
  • Forms and reports study

(c) System design vs. coding differences:

System Design System Coding
Planning phase Implementation phase
Abstract solution Concrete implementation

(d) Reasons for bottom-up approach:

  1. Detailed understanding
    • Component-level focus
    • Technical depth
  2. Module reusability
    • Independent components
    • Flexible integration
  3. Early testing
    • Unit-level verification
    • Incremental validation

(ii) outsourcing (4 marks)

    • Physical equipment
    • Infrastructure components
  2. Software
    • Application programs
    • System software
  3. Data
    • Stored information
    • Processing input

(b) System acquisition methods:

(i) Leasing:

  • Temporary system needs
  • Regular upgrades required

(ii) Outsourcing:

  • Specialized expertise needed
  • Cost reduction focus
  • Non-core business functions

(c) Structured English vs. Pseudo-code:

Structured English Pseudo-code
Natural language based Programming language like
Less technical More technical
Business audience Developer audience

(d) Signs of failing ICT projects:

  1. Schedule delays
    • Missed deadlines
    • Timeline extensions
  2. Budget overruns
    • Cost escalation

  3. Quality issues
    • Defect increase
    • Performance problems

Question 5 (20 marks)

(a) (i) Outline four advantages of decision tables as a system design tool. (4 marks)

(ii) Yamtech Company intends to design a new system. Explain three qualities that this design should have. (6 marks)

(b) Distinguish between deterministic and probabilistic systems. (4 marks)

(c) Explain two reasons that could make a company prefer direct change-over method during system implementation. (4 marks)

(d) Identify and state the function of dataflow symbol shown in Figure 1. (2 marks)

Answer

(a) (i) Advantages of decision tables:

  1. Clear representation
    • Organized format
    • Visual clarity
  2. Complex logic handling
    • Multiple conditions
    • Various outcomes
  3. Completeness verification
    • All scenarios covered
    • Gap identification
  4. Easy maintenance
    • Simple updates
    • Clear structure

(ii) System design qualities:

  1. Modularity
    • Independent components
    • Easy maintenance
  2. Scalability
    • Growth accommodation
    • Performance maintenance
  3. Reliability
    • Consistent operation
    • Error handling

(b) Deterministic vs. Probabilistic systems:

Deterministic Systems Probabilistic Systems
Fixed outcomes Variable outcomes
Predictable behavior Statistical behavior
Known conditions Uncertain conditions
Repeatable results Varied results

(c) Reasons for direct change-over:

  1. Limited resources
    • Budget constraints
    • Time constraints
  2. Simple system
    • Low complexity
    • Minimal risk

(d) Dataflow symbol:

  • Process symbol (rectangle)
  • Function: Shows data transformation or processing steps

Question 8 (20 marks)

(a) Outline four advantages of structured charts as a system design tool. (4 marks)

(b) Describe three types of system documentation. (6 marks)

(c) A student has opted to use questionnaires to gather data for an ICT project. Explain two limitations of this tool. (4 marks)

(d) A system analyst has been tasked with maintaining the company’s system. Explain three types of system maintenance he could carry out. (6 marks)

Answer

(a) Advantages of structured charts:

  1. Hierarchical representation
    • Clear module structure
    • Easy navigation
  2. Modularity emphasis
    • Independent components
    • Reusability promotion
  3. Top-down design support
    • Systematic breakdown
    • Structured approach
  4. Communication tool
    • Design overview
    • Team understanding

(b) Types of system documentation:

  1. User documentation
    • User manuals
    • Training materials
  2. System documentation
    • Design specifications
    • Technical manuals
  3. Operations documentation
    • Run procedures
    • Backup plans

(c) Limitations of questionnaires:

  1. Low response rate
    • Potential bias
    • Incomplete data
  2. Lack of clarification
    • Misinterpretation risk
    • No follow-up

(d) Types of system maintenance:

  1. Corrective maintenance
    • Fixing errors
    • Bug resolution
  2. Adaptive maintenance
    • Environment changes
    • New requirements
  3. Perfective maintenance
    • Performance improvement
    • Functionality enhancement
