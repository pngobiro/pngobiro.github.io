Question 1 (20 marks)

(a) Outline four characteristics of a system. (4 marks)

(b) Explain two circumstances under which observation method would be appropriate during data collection. (4 marks)

(c) Differentiate between logical and physical data flow diagrams as used in system analysis. (4 marks)

(d) A system analyst used prototyping methodology during system development.

(i) Describe the methodology. (2 marks)

(ii) Explain three advantages of this methodology. (6 marks)

Answer

(a) Characteristics of a System:

  1. Purpose: Every system has a specific objective or goal it aims to achieve.
  2. Components/Subsystems: Consists of interrelated parts working together.
  3. Interaction/Interdependence: Components interact and depend on each other to function.
  4. Boundary: Defines the scope of the system, separating it from its environment.

(b) Circumstances for Using Observation Method:

  1. Understanding Actual Processes: When needing to see how tasks are performed in reality, not just described.
  2. Verifying Information: To confirm data gathered through other methods like interviews.

(c) Logical vs. Physical Data Flow Diagrams (DFDs):

Logical DFDPhysical DFD
Focuses on *what* the system does (business logic).Focuses on *how* the system is implemented (technology, people).
Technology independent.Technology specific (shows files, programs, devices).

(d) Prototyping Methodology:

(i) Description: An iterative approach where a working model (prototype) is built quickly, shown to users for feedback, and refined until requirements are met.

(ii) Advantages:

  1. Improved User Feedback: Users interact with a tangible model early.
  2. Clarified Requirements: Helps uncover misunderstandings early.
  3. Reduced Development Time/Cost: Can shorten delivery by clarifying needs quickly.

Question 2 (20 marks)

(a) Outline four objectives of system analysis. (4 marks)

(b) Explain two circumstances under which payback period method would be appropriate during feasibility study. (4 marks)

(c) Differentiate between corrective and adaptive system maintenance types. (4 marks)

(d) Figure 1 shows symbols used in system documentation. Use it to answer the questions that follow.

     (i)   ┌─────┐
           │     │
           └─────┘
     (ii)  /─────/
          /     /
         /─────/
     (iii)   ▭ (Open Rectangle)
     (iv)    ▼ (Inverted Triangle)

Identify each symbol and state its function. (8 marks)

Answer

(a) Objectives of System Analysis:

  1. Understand the Existing System: Investigate current processes and data.
  2. Identify Problems and Opportunities: Pinpoint issues and areas for improvement.
  3. Define User Requirements: Document what the new system must do.
  4. Evaluate Alternatives: Assess different potential solutions.

(b) Circumstances for Using Payback Period Method:

  1. Liquidity Focus: When quick recovery of initial investment is crucial due to cash flow constraints.
  2. High Risk/Uncertainty: For projects where long-term forecasts are unreliable, favoring faster returns.

(c) Corrective vs. Adaptive System Maintenance:

Corrective MaintenanceAdaptive Maintenance
Fixing errors/bugs discovered after deployment.Modifying the system to cope with changes in the external environment (OS, hardware, regulations).
Reactive. Goal: Restore correct operation.Proactive/Reactive. Goal: Ensure compatibility.

(d) System Documentation Symbols:

  1. Symbol (i) Rectangle: Name: Process. Function: Represents a task or function transforming data.
  2. Symbol (ii) Parallelogram: Name: Input/Output. Function: Represents data entry or display.
  3. Symbol (iii) Open Rectangle: Name: Data Store. Function: Represents stored data (file, database).
  4. Symbol (iv) Inverted Triangle: Name: Merge / Offline Storage. Function: Represents merging paths or offline file storage.

Question 3 (20 marks)

(a) Outline four activities carried out during system implementation stage. (4 marks)

(b) Explain two benefits of Joint Application Development (JAD) approach during system development. (4 marks)

(c) Differentiate between decision tree and decision table as used in system analysis. (4 marks)

(d) A system analyst used observation and document analysis methods during fact finding stage.

(i) Outline two advantages of observation method. (2 marks)

(ii) Outline two advantages of document analysis method. (2 marks)

(iii) Explain two challenges the analyst could have experienced when using the two methods concurrently. (4 marks)

Answer

(a) Activities During System Implementation Stage:

  1. Coding/Programming: Writing the software code.
  2. Testing: Finding and fixing defects (unit, integration, system, acceptance).
  3. Installation: Setting up hardware and software.
  4. Data Conversion/Migration: Moving data from old to new system.

(b) Benefits of Joint Application Development (JAD):

  1. Improved Requirements Quality: Collaborative workshops lead to more accurate requirements.
  2. Reduced Development Time: Faster decision-making and issue resolution in focused sessions.

(c) Decision Tree vs. Decision Table:

Decision TreeDecision Table
Graphical (tree structure).Tabular (grid format).
Good for sequential logic.Good for complex logic with many conditions, ensures completeness.

(d) Fact Finding Methods:

(i) Advantages of Observation:

  1. First-hand Information: See actual processes.
  2. Contextual Understanding: Understand the environment and informal procedures.

(ii) Advantages of Document Analysis:

  1. Objective Data: Factual info on formal procedures, rules.
  2. Efficiency: Quick baseline understanding without using user time initially.

(iii) Challenges of Using Both Concurrently:

  1. Conflicting Information: Observed practice may contradict documentation.
  2. Time Allocation: Balancing time between observing and analyzing documents.

Question 4 (20 marks)

(a) Outline four challenges that may be experienced during system changeover. (4 marks)

(b) Explain two circumstances under which spiral model would be appropriate during system development. (4 marks)

(c) Differentiate between technical and operational feasibility studies as used in system development. (4 marks)

(d) A system analyst used questionnaires during data collection stage.

(i) Outline two advantages of this method. (2 marks)

(ii) Explain three ways the analyst could use to enhance the response rate. (6 marks)

Answer

(a) Challenges During System Changeover:

  1. Data Inconsistency: Errors during data migration leading to incorrect information in the new system.
  2. User Resistance: Staff may be reluctant to adopt the new system due to unfamiliarity or fear of change.
  3. Technical Glitches: Unexpected bugs or performance issues arising in the new system after going live.
  4. Inadequate Training: Users not properly trained, leading to errors and inefficiency.
  5. System Downtime: Potential disruption to business operations during the transition period.
*(Note: Only four were required)*

(b) Circumstances for Using Spiral Model:

  1. High-Risk Projects: When the project involves significant technical or requirement uncertainty, the model's emphasis on risk analysis in each iteration is beneficial.
  2. Large and Complex Systems: Suitable for large-scale projects where requirements evolve and phased development with risk assessment is necessary.
  3. Requirements are Unclear/Evolving: When it's difficult to define all requirements upfront, the iterative nature allows for refinement in each cycle.
*(Note: Only two were required)*

(c) Technical vs. Operational Feasibility:

Technical FeasibilityOperational Feasibility
Assesses if the required technology exists and if the organization has the technical expertise to build and maintain the system.Assesses if the proposed system will be used effectively within the organization and if it fits with existing business processes and culture.
Focus: Technology, skills, resources.Focus: User acceptance, process integration, organizational impact.
Question: Can we build it?Question: If we build it, will they use it effectively? Will it work in our environment?

(d) Questionnaires:

(i) Advantages:

  1. Cost-Effective: Can gather data from many people relatively cheaply and quickly.
  2. Anonymity: Can encourage more honest responses on sensitive topics.
  3. Standardization: Easy to compare and analyze responses due to consistent question format.
*(Note: Only two were required)*

(ii) Enhancing Response Rate:

  1. Clear Purpose & Instructions: Clearly explain why the questionnaire is needed and how to complete it. Keep it concise.
  2. Incentives: Offer a small reward (entry into a draw, small discount) for completion.
  3. Reminders & Follow-ups: Send polite reminders to non-respondents after a reasonable period.
  4. Ensure Anonymity/Confidentiality: Clearly state how the data will be used and protect respondent privacy.
  5. Pilot Testing: Test the questionnaire on a small group first to identify confusing questions or issues.
*(Note: Only three were required)*

Question 5 (20 marks)

(a) Outline four characteristics of a good system analyst. (4 marks)

(b) Explain two types of information systems based on the level of management they support. (4 marks)

(c) Differentiate between pilot and direct system changeover methods. (4 marks)

(d) A system analyst used interviews during data collection stage.

(i) Outline two advantages of this method. (2 marks)

(ii) Explain three guidelines the analyst should observe to ensure success of the interview. (6 marks)

Answer

(a) Characteristics of a Good System Analyst:

  1. Analytical Skills: Ability to break down complex problems, understand processes, and identify requirements.
  2. Communication Skills: Ability to listen effectively, ask probing questions, explain technical concepts clearly (verbally and written), and interact with diverse stakeholders.
  3. Technical Knowledge: Understanding of IT concepts, databases, programming principles, hardware, and software.
  4. Business Acumen: Understanding of business operations, processes, and objectives to align IT solutions with business needs.
  5. Problem-Solving Skills: Ability to identify issues, evaluate alternatives, and propose effective solutions.
*(Note: Only four were required)*

(b) Information Systems Based on Management Level:

  1. Transaction Processing Systems (TPS): Support operational management by recording daily routine transactions (e.g., sales order entry, payroll). Focus on efficiency and accuracy.
  2. Management Information Systems (MIS): Support middle management (tactical level) by providing summary and exception reports based on data from TPS, aiding monitoring and control.
  3. Decision Support Systems (DSS): Support middle and sometimes senior management by providing analytical tools and models to help with semi-structured decisions (e.g., "what-if" analysis).
  4. Executive Support Systems (ESS) / Executive Information Systems (EIS): Support senior management (strategic level) by providing high-level, summarized information (often graphical) about overall organizational performance and external trends to aid strategic decision-making.
*(Note: Only two were required)*

(c) Pilot vs. Direct System Changeover:

Pilot ChangeoverDirect Changeover (Big Bang)
The new system is introduced in only one part of the organization (e.g., one department, one location) first.The old system is completely shut down, and the new system is started simultaneously for the entire organization.
Allows testing and refinement in a limited environment before full rollout. Lower risk.Faster implementation if successful, but higher risk as there is no fallback if the new system fails.
Experience gained in the pilot phase helps with wider implementation.Requires careful planning and extensive testing beforehand.

(d) Interviews:

(i) Advantages:

  1. In-depth Information: Allows for probing questions and detailed responses.
  2. Clarification: Ambiguities can be resolved immediately.
  3. Build Rapport: Helps establish a relationship with users.
  4. Non-verbal Cues: Observer can gauge reactions and understanding.
*(Note: Only two were required)*

(ii) Guidelines for Successful Interview:

  1. Preparation: Define clear objectives, research the interviewee's role, and prepare specific, open-ended questions in advance.
  2. Conducting the Interview: Start with introductions, explain the purpose, listen actively, avoid leading questions, take notes concisely, and manage time effectively.
  3. Follow-up: Summarize key points at the end, thank the interviewee, and send a follow-up summary for verification if appropriate.
  4. Environment: Choose a quiet, comfortable location free from interruptions.
*(Note: Only three were required)*

Question 6 (20 marks)

(a) Outline four benefits of Information Systems to an organization. (4 marks)

(b) Explain two circumstances under which parallel changeover method would be appropriate during system implementation. (4 marks)

(c) Differentiate between functional and non-functional requirements as used in system analysis. (4 marks)

(d) A system analyst used structured English during system design.

(i) Describe the tool. (2 marks)

(ii) Explain three advantages of using the tool. (6 marks)

Answer

(a) Benefits of Information Systems:

  1. Improved Efficiency: Automating repetitive tasks and streamlining workflows.
  2. Enhanced Decision Making: Providing timely, accurate information to managers.
  3. Competitive Advantage: Enabling new products, services, or business models.
  4. Better Communication & Collaboration: Facilitating information sharing within and outside the organization.
  5. Cost Reduction: Reducing manual labor, errors, and resource wastage.
*(Note: Only four were required)*

(b) Circumstances for Using Parallel Changeover:

  1. High-Risk Systems: When system failure would have severe consequences (e.g., financial systems, critical infrastructure), running both systems allows verification and a fallback.
  2. Need for Verification: When absolute accuracy is critical, running both systems allows direct comparison of outputs to ensure the new system is correct before discarding the old one.
  3. Complex Systems: Where the risk of unforeseen errors during initial operation is high.
*(Note: Only two were required)*

(c) Functional vs. Non-Functional Requirements:

Functional RequirementsNon-Functional Requirements
Define *what* the system must do (features, functions, processes).Define *how* the system must perform (quality attributes).
Example: "System shall calculate sales tax."Example: "System response time must be under 2 seconds."
Verifiable: Can test if the function works or not.Measurable/Testable: Assessed against benchmarks (performance, security, usability).

(d) Structured English:

(i) Description:

  • A tool used in system analysis and design to describe process logic using a limited set of keywords (like IF-THEN-ELSE, DO-WHILE) and indentation within natural English statements. It aims to be precise like code but readable like plain English.

(ii) Advantages:

  1. Clarity & Precision: More precise than narrative descriptions, reducing ambiguity in process logic.
  2. Readability: Easier for non-technical stakeholders (like users or managers) to understand compared to pseudocode or actual code.
  3. Bridge to Coding: Provides a structured format that is relatively easy for programmers to translate into actual code.
  4. Standardization: Using limited keywords enforces a consistent way of describing logic.
*(Note: Only three were required)*

Question 7 (20 marks)

(a) Outline four contents of a feasibility study report. (4 marks)

(b) Explain two challenges that may be experienced during system analysis stage. (4 marks)

(c) Differentiate between cohesion and coupling as used in system design. (4 marks)

(d) A company intends to develop an Information System. The project manager prepared the following project network diagram.

      (Diagram similar to Q8(d) in 2022j, but potentially different activities/durations)
      Example:
          A(3) -> B(4) -> D(5) --┐
           \                     | -> F(4)
            `-> C(2) -> E(3) --┘

(i) Identify the critical path. (2 marks)

(ii) Determine the project duration. (2 marks)

(iii) Explain two benefits of using the diagram during project management. (4 marks)

Answer

(a) Contents of a Feasibility Study Report:

  1. Executive Summary: Brief overview of the project, findings, and recommendations.
  2. Problem/Opportunity Definition: Description of the issue or opportunity the proposed system addresses.
  3. Analysis of Alternatives: Description and evaluation of different potential solutions considered.
  4. Feasibility Assessment: Detailed analysis of technical, economic, operational, schedule, and legal feasibility.
  5. Recommendations: Suggestion on whether to proceed with the project and the preferred alternative.
  6. Cost-Benefit Analysis: Detailed breakdown of expected costs and benefits.
*(Note: Only four were required)*

(b) Challenges During System Analysis Stage:

  1. Incomplete/Ambiguous Requirements: Difficulty in getting clear, complete, and consistent requirements from users or stakeholders.
  2. Changing Requirements: Users changing their minds or business needs evolving during the analysis phase, leading to rework (scope creep).
  3. Communication Gaps: Misunderstandings between analysts and users/stakeholders due to technical jargon or differing perspectives.
  4. Resistance to Change: Users or managers may be unwilling to cooperate or provide accurate information if they feel threatened by the proposed system.
*(Note: Only two were required)*

(c) Cohesion vs. Coupling:

CohesionCoupling
Measures how strongly related the functions or elements *within* a single module are.Measures the degree of interdependence *between* different modules.
High cohesion (module performs one well-defined task) is desirable.Low coupling (modules are independent) is desirable.
Focus: Internal module strength.Focus: External module dependencies.
Example: A module calculating only sales tax has high cohesion.Example: Two modules accessing the same global variable have high coupling.

(d) Project Network Diagram Analysis (Using Example):

*(Calculations based on the example diagram: A(3) -> B(4) -> D(5) --┐ | -> F(4) `-> C(2) -> E(3) --┘)*

(i) Critical Path Identification:

  • Path 1: A -> B -> D -> F = 3 + 4 + 5 + 4 = 16 weeks
  • Path 2: A -> C -> E -> F = 3 + 2 + 3 + 4 = 12 weeks
  • Critical Path: A -> B -> D -> F

(ii) Project Duration:

  • The duration of the critical path determines the minimum project duration.
  • Project Duration: 16 weeks

(iii) Benefits of Using the Diagram:

  1. Identify Critical Activities: Highlights tasks that cannot be delayed without delaying the entire project, allowing focus on managing them.
  2. Visualize Dependencies: Clearly shows the relationships and order in which tasks must be performed.
  3. Estimate Project Duration: Provides a basis for calculating the shortest possible time to complete the project.
  4. Resource Planning: Helps in scheduling resources based on task dependencies and durations.
*(Note: Only two were required)*

Question 8 (20 marks)

(a) Outline four advantages of database management systems. (4 marks)

(b) Explain two circumstances under which outsourcing would be an appropriate method of system acquisition. (4 marks)

(c) Differentiate between alpha testing and beta testing as used in system development. (4 marks)

(d) A college library offers borrowing services to students and staff. A member requests for a book which is checked against the books file. If the book is available and the member has no overdue book, the book is issued and the transaction stored in the issues file. The member is notified through SMS. If the book is not available, a message is displayed to the member. If the member has an overdue book, an overdue notification is sent to the member.

Draw a level 1 data flow diagram to represent this logic. (8 marks)

Answer

(a) Advantages of Database Management Systems (DBMS):

  1. Data Redundancy Control: Centralized management helps minimize unnecessary duplication of data.
  2. Data Consistency & Integrity: Enforces rules (constraints) to ensure data accuracy and validity.
  3. Data Sharing: Allows multiple users and applications to access the same data concurrently.
  4. Data Security: Provides mechanisms for access control, user authentication, and backups.
  5. Improved Data Access: Offers query languages (like SQL) for efficient data retrieval and manipulation.
*(Note: Only four were required)*

(b) Circumstances for Using Outsourcing:

  1. Lack of In-house Expertise: When the organization doesn't have the necessary skills or experience to develop or manage a specific system.
  2. Cost Reduction: When an external provider can offer the service (e.g., payroll, hosting) more cheaply due to economies of scale or specialization.
  3. Focus on Core Business: To allow the organization to concentrate its resources on its primary activities rather than non-core IT functions.
  4. Faster Implementation: When an external vendor can deliver a solution more quickly than developing it internally.
*(Note: Only two were required)*

(c) Alpha Testing vs. Beta Testing:

Alpha TestingBeta Testing
Performed by internal teams (developers, testers, QA) within the development organization.Performed by a limited number of real end-users in their own environment, outside the development organization.
Conducted in a controlled lab or simulated environment.Conducted in the actual user environment.
Focus is often on finding bugs, usability issues, and ensuring functionality before external release.Focus is on gathering feedback on usability, performance, and satisfaction from real users before final release.
Developers may be present or readily available.Developers are generally not present.
