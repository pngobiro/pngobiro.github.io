Question 1 (20 marks)

(a) (i) State two types of system controls as used in information systems. (1 mark)

(ii) Outline two roles of information in organizations. (2 marks)

(b) Differentiate between direct and pilot changeover strategies. (4 marks)

(c) Wasch, a rural self help Sacco intends to implement an add-ons module to an existing system.

(i) State the most appropriate method of training the Sacco could use justifying your answer. (2 marks)

(ii) Explain two challenges with the method in (i). (4 marks)

(d) Table 1 shows details of project A and project B. Given that initial costs for the projects was Kshs.50,000 and Kshs.150,000 respectively determine the:

(i) most worthwhile project using the return on investment technique; (4 marks)

(ii) payback period for each project. (3 marks)

Answer

(a)(i) System Controls:

  • Preventive Controls
  • Detective Controls
  • Corrective Controls

(a)(ii) Roles of Information:

  • Decision Making
  • Operational Support
  • Strategic Planning

(b) Direct vs Pilot Changeover:

Direct ChangeoverPilot Changeover
Old system stopped, new system started immediately.New system implemented in one part/department first.
High risk, quick implementation.Lower risk, allows testing before full rollout.

(c)(i) Training Method:

  • Phased training or On-site training. Justification: Allows users to learn the add-on module in context of the existing system with minimal disruption.

(c)(ii) Challenges:

  • Scheduling conflicts for staff.
  • Resistance to change or learning new features.

(d)(i) Return on Investment (ROI):

Project A: Total Profit = (10*5)*1000 = 50,000. Avg Profit = 50,000/5 = 10,000. ROI = (10,000 / 50,000) * 100% = 20%
Project B: Total Profit = (50*4 + 30)*1000 = 230,000. Avg Profit = 230,000/5 = 46,000. ROI = (46,000 / 150,000) * 100% = 30.67%
Project B is more worthwhile based on ROI.

(d)(ii) Payback Period:

Project A: 50,000 / 10,000 = 5 years.
Project B: Year 1: 50k, Year 2: 100k, Year 3: 150k. Payback = 3 years.

Question 2 (20 marks)

(a) (i) State four usability considerations when designing a new system. (2 marks)

(ii) State four physical threats that can affect an information system. (2 marks)

(b) Differentiate between indexed-sequential and sequential file organization methods. (4 marks)

(c) Jane, a systems analyst with a certain company carried out a static testing on a proposed system. After implementation the system users realized that the system was generating incorrect output.

(i) Outline two scope of the testing strategy used. (4 marks)

(ii) Identify two limitations of the adopted testing strategic. (2 marks)

(d) A customer with a certain bank is issued an ATM card after opening an account with a bank. At the ATM machine, he/she is supposed to enter correct ATM pin number and then select appropriate transaction. The ATM verifies the status of the customer’s account and if information is satisfactory, the transaction is executed, the ATM card issued back and receipt is generated. Otherwise ATM card is rejected. The customer’s account file is then updated appropriately. The transaction may include balance inquiry and withdraw.

Answer

(a)(i) Usability Considerations:

  • Learnability
  • Efficiency
  • Memorability
  • Error prevention/handling
  • User Satisfaction

(a)(ii) Physical Threats:

  • Fire
  • Water Damage (Flooding)
  • Power Outages
  • Theft/Vandalism
  • Natural Disasters (Earthquake, etc.)

(b) Indexed-Sequential vs Sequential:

SequentialIndexed-Sequential

Good for batch processing, slow for random access.Allows both sequential and direct (random) access.

(c)(i) Scope of Static Testing:

  • Code Reviews/Walkthroughs: Examining source code without execution.
  • Documentation Review: Checking requirements, design documents for errors/consistency.

(c)(ii) Limitations of Static Testing:

  • Cannot find runtime errors (e.g., memory leaks, performance issues).
  • Does not verify the actual behavior of the system under real conditions.

Question 3 (20 marks)

(a) Explain each of the following terms in relation to system documentation: (i) accuracy; (ii) exhaustive. (4 marks)

(b) Differentiate between investigation and fact recording as used in feasibility studies. (4 marks)

(c) Dominic, a business man who owns a big carpentry industry decided to consult a systems analyst about development of a system that would maximize efficiency of his carpentry industry.

(i) Identify two appropriate system thinking methods the consultant could use. (4 marks)

(ii) Outline two advantages of using appropriate system thinking. (2 marks)

(d) Read the following narrative and answer the question that follows. A 15-aside rugby male team is to be constituted in a mixed technical training Institute. A team member should be more than 5ft tall and not less than 50kgs. Construct a limited entry decision table to represent the information. (6 marks)

Answer

(a) System Documentation Terms:

  • Accuracy: Documentation correctly reflects the system's actual state, functionality, and processes without errors or misrepresentations.
  • Exhaustive: Documentation covers all relevant aspects of the system comprehensively, leaving no significant gaps in information needed by its intended audience (users, developers, maintainers).

(b) Investigation vs Fact Recording:

InvestigationFact Recording
Active process of gathering information through methods like interviews, observation.Passive process of documenting the gathered information systematically.
Focuses on understanding the problem/system.Focuses on creating a reliable record of findings.

(c)(i) System Thinking Methods:

  • Soft Systems Methodology (SSM): Good for understanding complex human activity systems and differing perspectives.
  • System Dynamics: Useful for modeling feedback loops and understanding system behavior over time, especially for efficiency improvements.

(c)(ii) Advantages of System Thinking:

  • Holistic Understanding: Provides a broader view of the system and its interactions.
  • Identifies Leverage Points: Helps find areas where small changes can have significant impact.

Question 4 (20 marks)

(a) State four possible causes of corrective maintenance. (2 marks)

(b) (i) Figure 1 shows a design tool.

(I) Identify the design tool justifying your answer. (2 marks)

(II) Explain two advantages of the design tool identified in (I). (4 marks)

(ii) Explain two limitations of using structured charts in systems analysis. (4 marks)

(c) Philip, a systems analyst with a new company has a task of reviewing the information system.

(i) Identify the most appropriate type of documentation that he would require to efficiently carry out the task justifying your answer. (2 marks)

(ii) Outline six possible contents of the type of documentation identified in (i). (6 marks)

Answer

(a) Causes of Corrective Maintenance:

  • Software defects (bugs) discovered after deployment.
  • Logic errors in the code.
  • Design flaws leading to incorrect functionality.
  • Failure to meet specified requirements.

(b)(i)(I) Design Tool Identification:

  • Menu/Screen Mockup/Prototype. Justification: It visually represents a user interface screen with selectable options (Add Stud, Search, Disp_all, Exit).

(b)(i)(II) Advantages of Mockup/Prototype:

  • User Feedback: Allows users to visualize and provide early feedback on the interface design and flow.
  • Requirement Clarification: Helps clarify and validate user interface requirements before coding begins.

(b)(ii) Limitations of Structured Charts:

  • Limited Data Representation: Primarily focus on process flow and modular structure, not detailed data structures or relationships.
  • Can Become Complex: For large systems, structured charts can become overly complex and difficult to read/maintain.

(c)(i) Required Documentation:

  • System Documentation (or Technical Documentation). Justification: This provides the detailed technical information about the system's design, architecture, code, and operation needed for a thorough review.

(c)(ii) Contents of System Documentation:

  • System Overview/Architecture
  • Data Flow Diagrams (DFDs)
  • Entity Relationship Diagrams (ERDs)
  • Data Dictionary
  • Program Code/Modules Description
  • Test Plans and Results
  • Hardware/Software Requirements

Question 5 (20 marks)

(a) (i) Explain two rules used in construction of data flow diagram. (4 marks)

(ii) Differentiate between entity relation diagrams and entity life history diagrams. (4 marks)

(iii) Assuming that you are a systemns analyst at Exsys Company Ltd., and that the company intends to develop an information system for its human resource operations. Outline four specific objectives for the proposed system. (4 marks)

(b) Explain two factors that would necessitate the use of prototyping during system development. (4 marks)

(c) Agnes intends to gather data for her trade project. Outline four data gathering methods she could use clearly stating their possible use. (4 marks)

Answer

(a)(i) DFD Construction Rules:

  • Process Naming: Processes must have verb-phrase names (e.g., "Validate Order", "Calculate Pay").
  • Data Flow Naming: Data flows must have noun-phrase names representing the data moving (e.g., "Customer Order", "Payment Details").
  • Conservation of Data: Data entering a process must be related to data leaving it (no "black holes" where data disappears or "miracles" where data appears from nowhere).
  • External Entities: Data flows must start or end at a process; they cannot go directly between two external entities or two data stores.

(a)(ii) ERD vs ELH:

Entity Relationship Diagram (ERD)Entity Life History (ELH)
Models relationships between different entities (data structures).Models the sequence of events/states an entity instance goes through from creation to deletion.
Static view of data structure.Dynamic view of entity behavior over time.
Focuses on data storage and links.Focuses on process triggers and state changes.

(a)(iii) HR System Objectives:

  • To reduce employee record processing time by 30%.
  • To improve accuracy of payroll calculation to 99.9%.
  • To provide managers with online access to employee leave balances within 5 seconds.
  • To automate the generation of monthly HR reports (e.g., headcount, turnover).

(b) Factors Necessitating Prototyping:

  • Unclear User Requirements: When users find it difficult to articulate their needs or visualize the final system.
  • High-Risk/Complex Features: To test the feasibility and usability of complex or novel system components before full development.

(c) Data Gathering Methods:

  • Interviews: Use - Gathering in-depth qualitative information, opinions, and specific details from key stakeholders.
  • Questionnaires/Surveys: Use - Collecting quantitative data or opinions from a large number of people efficiently.
  • Observation: Use - Understanding current processes and workflows by watching users perform tasks in their natural environment.
  • Document Analysis: Use - Reviewing existing forms, reports, manuals, and system documentation to understand current data and procedures.

Question 6 (20 marks)

(a) (i) Explain the term ergonomic as used in systems design. (2 marks)

(ii) Explain two significances of system development life cycle. (4 marks)

(iii) Describe the following terms as used in systems analysis: (I) workflow; (II) conceptual data model. (4 marks)

(b) Outline two objectives of good system design. (2 marks)

(c) A module II student in a certain college was given an assignment to design two modules in system that would:

(i) calculate determine and display the difference of two numbers if the first number is greater than the second one, else calculate the sum of the two numbers;

(ii) determine and display the largest number in a set of three numbers.

Draw a system flowchart to represent each of the module. (8 marks)

Answer

(a)(i) Ergonomics:

  • Designing systems, interfaces, and workspaces to optimize human well-being and overall system performance. Focuses on usability, comfort, safety, and efficiency for the user.

(a)(ii) Significance of SDLC:

  • Structured Approach: Provides a systematic and phased framework for managing the complexity of system development.
  • Control and Monitoring: Enables better project planning, tracking, and control over deliverables, timelines, and resources at each stage.

(a)(iii) Systems Analysis Terms:

  • Workflow: The sequence of steps, tasks, or activities involved in a particular business process, showing how work flows through the organization or system.
  • Conceptual Data Model: A high-level representation of the data requirements of a system, showing major entities and their relationships, independent of specific database technology.

(b) Objectives of Good System Design:

  • Meet User Requirements: The design should accurately and effectively fulfill the needs identified during analysis.
  • Reliability and Robustness: The system should perform consistently and handle errors gracefully.
  • Maintainability and Flexibility: The design should allow for easy modifications, updates, and future enhancements.

(c) System Flowcharts:

Module (i): Difference or Sum

[Start] --> [Input Num1, Num2] --> [Is Num1 > Num2?] --Yes--> [Calculate Diff = Num1 - Num2] --> [Display Diff] --> [End]
                                      |
                                      +--No--> [Calculate Sum = Num1 + Num2] --> [Display Sum] --> [End]

Module (ii): Largest of Three

[Start] --> [Input A, B, C] --> [Is A > B?] --Yes--> [Is A > C?] --Yes--> [Largest = A] --> [Display Largest] --> [End]
                 |                     |                |
                 |                     No               +--No--> [Largest = C] --> [Display Largest] --> [End]
                 |                     |
                 +--------------------No--> [Is B > C?] --Yes--> [Largest = B] --> [Display Largest] --> [End]
                                       |
                                       +--No--> [Largest = C] --> [Display Largest] --> [End]

Question 7 (20 marks)

(a) Outline two types of data input methods that can be used in information systems. (2 marks)

(b) Figure 2 shows an organization chart for a particular company. State two characteristics for the information system used at each management level. (6 marks)

(c) RST Company Ltd., intends to recruit a systems analyst. Explain two technical skills that the company would expect the system analyst to posses. (4 marks)

(d) The following are characteristics of different systems:

(i) Does not interact with the system environment;

(ii) Its behavior is dictated by the changes in the environment;

(iii) The output can be predicted based on the input;

(iv) The system does not have a definite boundary.

Identify each of the system. (8 marks)

Answer

(a) Data Input Methods:

  • Manual Keyboarding (e.g., typing data into forms).
  • Source Data Automation (e.g., Barcode scanning, Optical Character Recognition - OCR).

(b) Information System Characteristics by Management Level:

  • Level 1 (Strategic/Top Management): Characteristics: Highly summarized information, external focus, long-term perspective, unstructured decision support (e.g., Executive Support Systems - ESS).
  • Level 2 (Tactical/Middle Management): Characteristics: Semi-structured information, internal focus with some external data, medium-term perspective, control and planning oriented (e.g., Management Information Systems - MIS, Decision Support Systems - DSS).
  • Level 3 (Operational/Lower Management): Characteristics: Detailed transaction data, internal focus, short-term perspective, highly structured, routine decision support (e.g., Transaction Processing Systems - TPS).

  • Modeling Skills: Ability to create DFDs, ERDs, UML diagrams, flowcharts to represent system requirements and design.

  • Programming Fundamentals: Basic understanding of programming logic and structures, even if not coding directly.

(d) System Identification:

Question 8 (20 marks)

(a) Differentiate between hard and soft systems. (4 marks)

(b) (i) Ann, a system analyst with a certain company is required to develop a term of reference document which is to be used for a proposed system. Outline four roles of this document. (4 marks)

(ii) A module II student would like to carry out a feasibility study for a term project. Outline four aspects that the study should address. (4 marks)

(c) Tom, a systems analyst with a software development company was required to design a database management system.

(i) Outline two types of data files that he would use in the proposed system. (2 marks)

(ii) Outline two principle requirements he would consider when selecting the storage media for the proposed database system. (2 marks)

(d) Outline four factors that would affect system maintenance. (4 marks)

Answer

(a) Hard vs Soft Systems:

Hard SystemsSoft Systems
Well-defined problems and objectives.Ill-defined, complex problems with multiple perspectives.
Focus on technical efficiency and optimization.Focus on human activity, culture, and politics.
Quantitative analysis and modeling are primary tools.Qualitative analysis, discussion, and learning are key.
Clear boundaries, often technical systems.Fuzzy boundaries, involve social and organizational aspects.

(b)(i) Roles of Terms of Reference (ToR):

  • Defines Project Scope: Clearly outlines what is included and excluded from the project.
  • Sets Objectives: States the specific goals the proposed system aims to achieve.
  • Identifies Constraints: Lists limitations (budget, time, resources) affecting the project.
  • Establishes Authority/Responsibilities: Defines who is responsible for what and the project governance structure.

(b)(ii) Feasibility Study Aspects:

  • Technical Feasibility: Can the system be built with existing technology? Are the skills available?
  • Economic Feasibility: Do the benefits outweigh the costs? Is it financially viable?
  • Operational Feasibility: Will the system work within the organization's culture and processes? Will users accept it?
  • Schedule Feasibility: Can the system be built within the required timeframe?

(c)(i) Types of Data Files:

  • Master Files: Contain permanent or semi-permanent reference data (e.g., Customer Master File, Product Master File).
  • Transaction Files: Contain records of events or activities that update master files (e.g., Sales Transaction File, Payment Transaction File).
  • Report Files: Contain formatted data ready for printing or display.
  • Backup Files: Copies of other files for recovery purposes.

(c)(ii) Storage Media Selection Requirements:

  • Capacity: Must be large enough to hold the expected volume of data.
  • Access Speed: Needs to be fast enough to meet performance requirements for data retrieval and updates.
  • Cost: The cost per unit of storage should be within budget.
  • Reliability/Durability: The media should be reliable to prevent data loss.

(d) Factors Affecting System Maintenance:

  • Quality of initial development (better quality = less corrective maintenance).
  • Quality and completeness of documentation.
  • Skills and availability of maintenance staff.
  • Frequency of changes in user requirements or the operating environment.
  • Tools and techniques used for development and maintenance.
