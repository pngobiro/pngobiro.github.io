DIPLOMA IN INFORMATION COMMUNICATION TECHNOLOGY

MODULE II

SYSTEMS ANALYSIS AND DESIGN

INSTRUCTIONS TO CANDIDATES

This paper has EIGHT questions.

Answer any FIVE in the answer booklet provided.

All questions carry equal marks.

Candidates should answer the questions in English.

Question 1 (20 marks)

(a) (i) Outline two characteristics of decision support systems. (2 marks)

(ii) Outline two limitations of an expert system. (2 marks)

(b) Explain each of the following system development methodologies:

(i) structured; (2 marks)

(ii) traditional; (2 marks)

(iii) object oriented. (2 marks)

(c) Differentiate between structured and unstructured interviews as applied in systems analysis. (4 marks)

(d) Raymond would like to apply risk management techniques to avoid derailment of the information system project. Explain three risk management techniques that he could use. (6 marks)

Answer

(a)(i) Characteristics of Decision Support Systems:

  • Interactive and User-Friendly: Allows users to easily interact with the system and modify parameters
  • Flexibility in Data Analysis: Supports various analytical models and what-if scenarios
  • Support for Semi-Structured Decisions: Helps in decisions where human judgment is still needed

(a)(ii) Limitations of Expert Systems:

  • Lack of Common Sense: Cannot apply general knowledge like humans can
  • Limited Domain: Only effective within their specific area of expertise
  • High Development Costs: Expensive to develop and maintain
  • Cannot Learn from Experience: Need manual updates to knowledge base

(b) System Development Methodologies:

(i) Structured:

  • Uses formal, systematic approach with defined phases
  • Employs structured tools like DFDs, ERDs
  • Emphasizes top-down design and modular development

(ii) Traditional:

  • Sequential, linear approach (Waterfall model)
  • Completes each phase before moving to next
  • Heavy documentation focus

(iii) Object-Oriented:

  • Based on objects that contain both data and methods
  • Uses concepts like inheritance, encapsulation, and polymorphism
  • Promotes code reuse and modularity

(c) Structured vs Unstructured Interviews:

Structured Interviews Unstructured Interviews
Follows predetermined set of questions Flexible, conversational approach
Consistent format across all interviews Questions evolve based on responses
Easier to analyze and compare responses Can uncover unexpected information

(d) Risk Management Techniques:

  • Risk Identification and Assessment
    • Systematically identify potential risks
    • Evaluate probability and impact
  • Risk Mitigation Planning
    • Develop specific strategies for each risk
    • Create contingency plans
  • Risk Monitoring and Control
    • Regular review of risk status
    • Update mitigation strategies as needed

Question 2 (20 marks)

(a) State the function of each of the following data validation checks:

(i) range check; (1 mark)

(ii) validity check; (1 mark)

(iii) format check. (1 mark)

(b) Explain each of the following system design tools:

(i) decision table; (2 marks)

(ii) Structured English. (2 marks)

(c) Ken intends to organize a structured walkthrough after system design.

(i) Outline three members of the project team that should participate in the exercise. (3 marks)

(ii) Explain two benefits of this exercise. (4 marks)

(d) With the aid of a diagram in each case, describe three types of relationships used in ERDs. (6 marks)

Answer

(a) Data Validation Checks:

  • Range Check: Verifies if input falls within acceptable minimum and maximum values.
  • Validity Check: Ensures data meets predefined rules or criteria for being valid.
  • Format Check: Verifies if data follows required pattern or structure (e.g., date format).

(b) System Design Tools:

(i) Decision Table:

  • Tabular representation of complex business logic
  • Shows conditions and corresponding actions
  • Helps verify completeness of decision logic

(ii) Structured English:

  • Pseudo-code using simple English statements
  • Uses basic control structures (IF-THEN-ELSE, WHILE, etc.)
  • Bridge between system design and actual coding

(c)(i) Structured Walkthrough Participants:

  • System Analyst/Designer: Presents the design
  • Project Manager: Oversees process and timeline
  • Developer Representative: Technical perspective
  • End User Representative: User perspective
  • Quality Assurance Team Member: Testing perspective

(c)(ii) Benefits of Structured Walkthrough:

  • Early Error Detection
    • Identifies design flaws before implementation
    • Reduces cost of fixes
  • Knowledge Sharing
    • Team members understand design decisions
    • Facilitates future maintenance
  • Quality Improvement
    • Multiple perspectives help improve design
    • Ensures requirements are met

(d) ERD Relationships:

1. One-to-One (1:1)
   Entity A ────────1─────────1───── Entity B

2. One-to-Many (1:N)
   Entity A ────────1─────────N───── Entity B

3. Many-to-Many (M:N)
   Entity A ────────M─────────N───── Entity B

Question 3 (20 marks)

(a) Outline two roles of each of the following information system stakeholders:

(i) system owner; (2 marks)

(ii) system analyst. (2 marks)

(b) (i) The following describes elements of an examinations information system. Classify them as either process, outputs, environment or control: Exam supervisors, mean score, examination room, marking scheme, grading criteria, ranking. (3 marks)

(ii) Jackson System Development (JSD) is a linear development method. Describe three steps that could be followed when using the methodology. (3 marks)

(c) A certain Non Governmental Organization (NGO) based in Canada intends to use online survey technique to carry out a feasibility study in Kenya. Explain three advantages of using this technique. (6 marks)

(d) Explain two circumstances that would warrant changes to a newly implemented system. (4 marks)

Answer

(a)(i) System Owner Roles:

  • Provides funding and resources for the project
  • Makes key decisions about system scope and objectives
  • Approves major project milestones and changes
  • Ensures alignment with business strategy

(a)(ii) System Analyst Roles:

  • Gathers and analyzes system requirements
  • Designs system solutions
  • Coordinates between stakeholders
  • Documents system specifications

(b)(i) Classification of Elements:

Element Classification
Exam supervisors Environment
Mean score Output
Examination room Environment
Marking scheme Control
Grading criteria Control
Ranking Output

(b)(ii) JSD Steps:

  1. Entity Action Step
    • Identify real-world entities
    • List actions performed by or on entities
  2. Entity Structure Step
    • Model sequential ordering of actions
    • Create entity structure diagrams
  3. Initial Model Step
    • Define system functions
    • Create initial specification

(c) Advantages of Online Survey Technique:

  • Wide Geographic Coverage
    • Can reach respondents across Kenya easily
    • No physical travel required
  • Cost-Effective
    • Eliminates printing and distribution costs
    • Reduces data entry costs
  • Quick Data Collection
    • Instant response submission
    • Real-time data analysis possible
  • Better Data Quality
    • Automated validation checks
    • Reduced data entry errors

(d) Circumstances Warranting System Changes:

  • Business Environment Changes
    • New regulations or compliance requirements
    • Changes in business processes
  • Technical Issues
    • Performance problems discovered in production
    • Security vulnerabilities identified
  • User Requirements
    • Missing functionality identified after implementation
    • Usability issues reported by users

Question 4 (20 marks)

(a) Differentiate between data dictionary and data store as used in systems analysis and design. (4 marks)

(b) For each of the following SDLC stages, outline three objectives that the system analyst strives to achieve:

(i) feasibility study; (3 marks)

(ii) system design. (3 marks)

(c) Table 1 shows net profits for projects X and Y where initial investment was Kes 150,000 and 198,000 respectively. Use it to answer the questions that follow.

Year    Project X    Project Y
        (Kes)        (Kes)
1       45,000       51,000
2       55,000       51,000
3       35,000       51,000
4       43,000       51,000

(i) Calculate the payback period for each of the projects. (4 marks)

(ii) Explain three limitations of using the payback technique on project evaluation. (6 marks)

Answer

(a) Data Dictionary vs Data Store:

Data Dictionary Data Store
Contains metadata about all data elements in the system Physical location where data is held in the system
Describes data structure, type, and relationships Actual repository of data (files, databases)
Used for documentation and reference Used for data storage and retrieval

(b)(i) Feasibility Study Objectives:

  • Determine Technical Feasibility
    • Assess available technology
    • Evaluate technical skills needed
  • Evaluate Economic Feasibility
    • Cost-benefit analysis
    • Return on investment
  • Assess Operational Feasibility
    • User acceptance
    • Organizational impact

(b)(ii) System Design Objectives:

  • Create Detailed Specifications
    • User interface design
    • Database design
    • Process design
  • Ensure Requirements Coverage
    • Map design to requirements
    • Validate completeness
  • Optimize System Performance
    • Efficiency considerations
    • Resource utilization

(c)(i) Payback Period Calculation:

Project X:

Investment: 150,000
Year 1: 45,000 (Cum: 45,000)
Year 2: 55,000 (Cum: 100,000)
Year 3: 35,000 (Cum: 135,000)
Year 4: 43,000 (Cum: 178,000)

Remaining: 15,000 needed after Year 4
Payback Period = 3 + (15,000/43,000) = 3.35 years

Project Y:

Investment: 198,000
Year 1: 51,000 (Cum: 51,000)
Year 2: 51,000 (Cum: 102,000)
Year 3: 51,000 (Cum: 153,000)
Year 4: 51,000 (Cum: 204,000)

Payback Period = 3 + (45,000/51,000) = 3.88 years

(c)(ii) Limitations of Payback Technique:

  • Ignores Time Value of Money
    • Doesn't consider inflation
    • Treats all cash flows equally
  • Ignores Post-Payback Returns
    • Doesn't consider profits after payback period
    • May reject profitable long-term projects
  • Arbitrary Cutoff Period
    • No scientific basis for acceptable payback period
    • May vary by industry or company

Question 5 (20 marks)

(a) William intends to use direct change over during systems implementation stage. Explain two reasons for his choice of this method. (4 marks)

(b) Jack has been assigned the responsibility of acquiring an information system for his company. Explain two methods that he could use to achieve the objective. (4 marks)

(c) Figure 1 shows part of a dataflow diagram. Use it to answer the question that follows:

[DFD diagram showing payroll processing with errors]

(i) Identify four errors in the DFD. (4 marks)

(ii) Outline two advantages of using DFD's in system design. (2 marks)

(d) Jacob intends to write a report on factors that influence system maintenance costs. Explain three such factors that he could include in the report. (6 marks)

Answer

(a) Reasons for Direct Changeover:

  • Cost-Effective
    • No parallel running costs
    • Minimal transition period
  • Clear Cutoff Point
    • Clean break from old system
    • Avoids confusion of multiple systems
  • Resource Efficiency
    • No need to maintain two systems
    • Staff focus on single system

(b) System Acquisition Methods:

  • Off-the-Shelf Software Purchase
    • Buy ready-made solution
    • Customize if needed
  • Custom Development
    • In-house development team
    • Outsourced development
  • Software as a Service (SaaS)
    • Cloud-based subscription
    • Managed by vendor

(c)(i) DFD Errors:

  • Missing flow direction arrows
  • Incorrect data store notation
  • Process without input/output flows
  • Inconsistent naming conventions

(c)(ii) Advantages of DFDs:

  • Visual Representation
    • Easy to understand system flow
    • Clear process relationships
  • Communication Tool
    • Bridge between technical and non-technical stakeholders
    • Facilitates requirement validation

(d) Factors Influencing Maintenance Costs:

  • System Complexity
    • Number of modules and interfaces
    • Technology stack diversity
  • Documentation Quality
    • Completeness and accuracy
    • Accessibility and organization
  • Code Quality
    • Coding standards adherence
    • Modularity and reusability
  • User Support Requirements
    • Training needs
    • Help desk support

Question 6 (20 marks)

(a) Define each of the following terms as used in system analysis and design:

(i) system procedures; (2 marks)

(ii) system features; (2 marks)

(iii) system boundaries. (2 marks)

(b) State the purpose of each of the following systems development tools:

(i) checklist; (2 marks)

(ii) observation; (2 marks)

(iii) interviewing. (2 marks)

(c) Explain four activities that are carried out during system maintenance. (8 marks)

Answer

(a) Definitions:

  • System Procedures:
    • Step-by-step instructions for performing system operations
    • Define how tasks should be executed within the system
  • System Features:
    • Distinct characteristics or functionalities of a system
    • Capabilities that define what the system can do
  • System Boundaries:
    • Defines the scope and limits of the system
    • Separates the system from its environment

(b) Purpose of Development Tools:

  • Checklist:
    • Ensures all requirements are captured
    • Provides structured way to verify completeness
  • Observation:
    • Captures actual user behavior and workflows
    • Identifies unstated requirements and inefficiencies
  • Interviewing:
    • Gathers detailed information from stakeholders
    • Clarifies requirements and expectations

(c) System Maintenance Activities:

  1. Corrective Maintenance
    • Fix bugs and errors in the system
    • Address system crashes and performance issues
  2. Adaptive Maintenance
    • Modify system to handle environmental changes
    • Update system for new hardware/software requirements
  3. Perfective Maintenance
    • Enhance system performance and functionality
    • Add new features requested by users
  4. Preventive Maintenance
    • Update system to prevent future problems
    • Improve system reliability and maintainability

Question 7 (20 marks)

(a) Explain each of the following file organization methods:

(i) sequential; (2 marks)

(ii) indexed sequential; (2 marks)

(iii) random. (2 marks)

(b) With the aid of a diagram, illustrate how indexed sequential file organization method works. (6 marks)

(c) Explain four advantages of modular system design. (8 marks)

Answer

(a) File Organization Methods:

  • Sequential:
    • Records stored in physical sequence
    • Must read through all previous records to access a specific record
  • Indexed Sequential:
    • Combines sequential access with index for direct access
    • Uses index to locate records quickly
  • Random:
    • Records stored at random locations
    • Uses hashing function to determine record location

(b) Indexed Sequential File Organization:

Index Table          Data File
-----------         ----------
Key  Address        Records
 10    →            Record 10
 20    →            Record 20
 30    →            Record 30
 ...                ...
 
1. Index provides quick lookup
2. Records stored sequentially
3. New records use overflow area

(c) Advantages of Modular Design:

  • Easier Maintenance
    • Modules can be modified independently
    • Localized impact of changes
  • Reusability
    • Modules can be reused in other systems
    • Reduces development time and cost
  • Better Testing
    • Modules can be tested independently
    • Easier to isolate and fix problems
  • Parallel Development
    • Multiple teams can work simultaneously
    • Faster project completion

  • Project Constraints
    • Development time
    • Budget limitations
    • Long-term maintenance considerations

  • Technical Documentation
    • System architecture and design
    • Code documentation
    • Database schemas
  • User Documentation
    • User manuals
    • Training materials
