Question 1 (20 marks)

(a) (i) Explain term meta data as used in system analysis and design. (2 marks)

(ii) Explain each of the following terms as used in project scheduling: I optimistic time; (2 marks) II early start time. (2 marks)

(b) Distinguish between project task and milestone as used in computer project management. (4 marks)

(c) Zep intends to develop a system testing policy for her company. Outline four testing rules that she is likely to include in the policy. (4 marks)

(d) (i) Outline two benefits of carrying out system maintenance policy in organizations. (2 marks)

(ii) Parallel changeover is widely used by system developers during systems implementation. Explain two advantages that are associated with this changeover scheme. (4 marks)

Answer

(a)(i) Meta Data:

  • Data about data. It describes the properties, characteristics, structure, and context of other data (e.g., data type, field length, source, creation date).

(a)(ii) Project Scheduling Terms:

  • Optimistic Time (To): The minimum possible time required to accomplish a task, assuming everything proceeds better than is normally expected.
  • Early Start Time (ES): The earliest possible point in time at which an activity can begin, based on the completion times of its predecessors.

(b) Project Task vs Milestone:

Project TaskMilestone
A specific piece of work or activity with a defined duration and resource requirement.A significant point or event in a project timeline, marking the completion of a major phase or deliverable. Has zero duration.
Consumes time and resources.Represents an achievement or checkpoint.
Example: "Code login module".Example: "Login Module Complete".

(c) System Testing Policy Rules:

  • All code must pass unit tests before integration.
  • Integration testing must verify interfaces between modules.
  • System testing must cover all specified functional and non-functional requirements.
  • User Acceptance Testing (UAT) must be performed and signed off by end-users before deployment.
  • Regression testing must be performed after any bug fix or modification.

(d)(i) Benefits of System Maintenance Policy:

  • Standardized Procedures: Ensures consistency in how maintenance is requested, approved, performed, and documented.
  • Improved Control & Prioritization: Helps manage maintenance workload, prioritize requests, and control costs.
  • Enhanced System Stability: Reduces unplanned downtime through scheduled preventive maintenance and controlled changes.

(d)(ii) Advantages of Parallel Changeover:

  • Lower Risk: The old system runs alongside the new system, providing a fallback if the new system fails.
  • Comparison: Allows direct comparison of outputs from both systems to ensure the new system is accurate.
  • User Confidence: Users can learn the new system while still having the safety net of the old one.

Question 2 (20 marks)

(a) (i) Outline two strategies that could be used by systems analyst to mitigate emerging trends in SAD. (2 marks)

(ii) Explain two methods that could be used to acquire new information system in an organization. (4 marks)

(b) Distinguish between cardinality and relationship as used in entity relation diagrams. (4 marks)

(c) For each of the following SDLC stages, outline three objectives that the system analysis strive to achieve:

(i) feasibility studies; (3 marks)

(ii) maintenance. (3 marks)

(d) Tracey, an intern student with a certain company was required to carry out problem identification for a proposed information system. Outline four rules that she is likely to use. (4 marks)

Answer

(a)(i) Mitigating Emerging Trends:

  • Continuous Learning: Staying updated with new technologies, methodologies, and tools.
  • Using Flexible/Agile Methodologies: Adopting approaches that can accommodate changing requirements and technologies.
  • Focusing on Architectural Principles: Designing systems with modularity and interoperability to allow easier integration of new trends.

(a)(ii) Acquiring New Information Systems:

  • In-house Development: Building the system using the organization's own IT staff.
  • Purchasing Off-the-Shelf Software (COTS): Buying a pre-built software package from a vendor.
  • Outsourcing: Hiring an external company to develop the system.
  • Cloud Services (SaaS): Subscribing to software hosted by a third-party provider.

(b) Cardinality vs Relationship (ERD):

RelationshipCardinality
An association or link between two or more entities.Specifies the number of instances of one entity that can (or must) be associated with each instance of another entity.
Describes *that* entities are connected.Describes *how many* instances are connected (e.g., 1:1, 1:M, M:N).
Example: "Employee *Works In* Department".Example: "One Department can have *Many* Employees".

(c) SDLC Stage Objectives:

  • (i) Feasibility Studies:
    • Determine if the project is technically possible.
    • Assess if the project is economically justifiable (benefits vs. costs).
    • Evaluate if the project is operationally viable within the organization.
    • Check if the project aligns with legal and ethical requirements.
  • (ii) Maintenance:
    • Correct errors (Corrective).
    • Adapt the system to environmental changes (Adaptive).
    • Enhance system performance or usability (Perfective).
    • Prevent future problems (Preventive).
    • Ensure continued operation and user support.

(d) Rules for Problem Identification:

  • Focus on the Problem, Not the Solution: Understand the root cause before jumping to solutions.
  • Involve Stakeholders: Gather perspectives from users, managers, and others affected by the problem.
  • Use Multiple Fact-Finding Techniques: Combine interviews, observation, document analysis, etc., for a comprehensive view.
  • Quantify the Problem: Where possible, measure the impact of the problem (e.g., cost, time lost, error rate).
  • Define Clear Boundaries: Understand the scope of the problem being investigated.

Question 3 (20 marks)

(a) (i) Explain the term environment as used in information systems. (2 marks)

(ii) Distinguish between adaptive and probabilistic systems. (4 marks)

(b) Joseph intends to develop a political information system. From the preliminary studies the system had undefined requirements.

(i) Identify the most appropriate system development approach he could use Justifying your answer. (2 marks)

(ii) Explain two advantages of the approach identified in (i). (4 marks)

(c) (i) Differentiate between static and dynamic testing as used in systems implementation. (4 marks)

(ii) The following is description of a diagnostic module used in a clinic system. The system would prompt the nurse to enter the patient temperature and then check whether the temperature is positive or negative. If the temperature is negative then system will terminate else will convert the temperature to Fahrenheit and prints the diagnosis report.

Draw a program flowchart to represent the logic of the module. (4 marks)

Answer

(a)(i) Environment (Information Systems):

  • Everything outside the system's boundary that interacts with or influences the system. This includes users, other systems, organizational factors, regulations, competitors, etc. Open systems interact with their environment.

(a)(ii) Adaptive vs Probabilistic Systems:

Adaptive SystemProbabilistic System
Can modify its own behavior or structure in response to changes in its environment or its own performance.Its behavior or output cannot be precisely predicted for a given input, involving an element of randomness or uncertainty.
Focuses on learning and self-adjustment.Focuses on likelihoods and statistical outcomes.
Example: A spam filter that learns from user feedback.Example: A weather forecasting system.

(b)(i) Development Approach:

  • Prototyping or Agile Methodologies (like Scrum). Justification: These approaches are well-suited for situations with unclear or evolving requirements, as they allow for iterative development and frequent feedback from stakeholders to refine understanding.

(b)(ii) Advantages:

  • Handles Vague Requirements: Allows requirements to emerge and be clarified through iterations.
  • User Involvement: Encourages active user participation and feedback throughout development.
  • Flexibility: Easier to adapt to changes during the project lifecycle.

(c)(i) Static vs Dynamic Testing:

Static TestingDynamic Testing
Testing performed without executing the code.Testing performed by executing the code.
Involves reviews, walkthroughs, inspections of code or documents.Involves running the software with test cases to observe its behavior.
Finds defects early in the SDLC (e.g., requirement errors, design flaws, coding standard violations).Finds defects that occur during execution (e.g., runtime errors, incorrect output, performance issues).

Question 4 (20 marks)

(a) (i) Outline three reasons of using data dictionary in system development. (3 marks)

(ii) Agnes intends to use structured analysis approach for her research project. Explain two limitations that she is likely to experience with the approach. (4 marks)

(b) Maureen intends to use observation method to investigate a proposed system. Outline five disadvantages of her method of data collection. (5 marks)

(c) Crest Hotel Ltd. intends to use a self-service system to allow its customers to order from their menu that consist of food, beverages and drinks. The order details are then stored on a pending order file which is then assigned to one of the hotel chefs for processing; the chef details are stored on the chef file. The chef is required to verify the order based on the details in the inventory file. When the food is ready the chef updates both the waiter and pending files. The waiter then serves the customer with the order and receives a payment which is forwarded to the cashier who then issues the customer with a receipt and updates the waiter and order files respectively.

Draw a Level 1 dataflow diagram to represent the hotel system. (8 marks)

Answer

(a)(i) Reasons for Using Data Dictionary:

  • Standardization: Provides consistent definitions and formats for all data elements across the system.
  • Communication: Acts as a central reference for developers, analysts, and users regarding data meanings and structures.
  • Impact Analysis: Helps identify where data elements are used, making it easier to assess the impact of changes.
  • Documentation: Serves as crucial documentation for system understanding and maintenance.

(a)(ii) Limitations of Structured Analysis:

  • Difficulty with Changing Requirements: Can be rigid and less adaptable to frequent requirement changes compared to agile methods.
  • Process-Oriented Focus: May sometimes neglect the data aspect or complex data relationships compared to data-centric approaches.
  • Potential for Communication Gaps: Heavy reliance on formal diagrams (DFDs, ERDs) might lead to misunderstandings if not well-explained or if stakeholders are not familiar with them.

(b) Disadvantages of Observation Method:

  • Hawthorne Effect: People may behave differently when they know they are being observed.
  • Observer Bias: The observer's interpretation of events might be subjective.
  • Time-Consuming: Can require significant time to observe processes thoroughly.
  • Limited Scope: May only capture routine activities and miss exceptions or infrequent events.
  • Interpretation Challenges: Observed actions might not fully reveal the underlying reasons or decision processes.

Question 5 (20 marks)

(a) Describe each of the following types of nodes as used in decision trees:

(i) chance/event; (2 marks)

(ii) terminal/edge; (2 marks)

(iii) decision. (2 marks)

(b) (i) Explain the term super key as used in entity relation diagram. (2 marks)

(ii) Structured English seeks to combine both system logic and natural language in system design. Outline four guidelines that could be used when using this design tool. (4 marks)

(c) Table 1 shows details for a proposed system project in a certain organization. Use it to answer the question that follows.

Table 1
ActivityDescriptionDuration (Weeks)Predecessor
AHardware selection6-

EFile conversion3B

Draw a labeled Gantt chart for the proposed project schedule. (8 marks)

Answer

(a) Decision Tree Nodes:

  • (i) Chance/Event Node: Represents an uncertain event with multiple possible outcomes, each having a probability. Often depicted as a circle.
  • (ii) Terminal/Leaf Node (Edge implies connection): Represents the final outcome or result of a sequence of decisions and events. Has no further branches. Often depicted as a triangle or just the outcome value.
  • (iii) Decision Node: Represents a point where a choice must be made between different alternatives. Often depicted as a square.

(b)(i) Super Key:

  • A set of one or more attributes within an entity that, taken together, uniquely identifies each instance (row) of that entity. A primary key is a minimal super key (no subset of its attributes is also a super key).

(b)(ii) Structured English Guidelines:

  • Use limited vocabulary: Primarily action verbs, nouns from the data dictionary, and specific keywords for logic (IF, THEN, ELSE, DO WHILE, REPEAT UNTIL, etc.).
  • Use indentation to show logical structure (sequence, selection, iteration).
  • Express logic clearly and unambiguously.
  • Avoid excessive jargon or overly complex sentence structures.

Question 6 (20 marks)

(a) (i) Outline two uses of a user manual. (2 marks)

(ii) Outline three criteria that should be considered when selecting system development methodologies. (3 marks)

(b) (i) Outline two advantages of rapid system development. (2 marks)

(ii) Poly intends to use prototyping to develop an information system for her company. Outline five strengths of the approach that could have influenced her. (5 marks)

(c) Abex Systems Company Ltd. intends to use computer-based training sessions for the users of a new information system.

(i) Explain two advantages of the training method that could have influenced the company. (4 marks)

(ii) Explain two limitation of the method. (4 marks)

Answer

(a)(i) Uses of User Manual:

  • Training: Helps new users learn how to operate the system.
  • Reference: Provides ongoing support for users performing tasks or troubleshooting problems.

(a)(ii) Criteria for Selecting Methodology:

  • Project Size and Complexity
  • Clarity and Stability of Requirements
  • Technical Environment / Tools Available
  • Team Skills and Experience
  • Time and Budget Constraints

(b)(i) Advantages of Rapid System Development (RAD):

  • Faster Delivery: Reduced development time compared to traditional methods.
  • Flexibility: More adaptable to changing requirements during development.
  • User Involvement: Typically involves users more closely through prototyping or iterations.

(b)(ii) Strengths of Prototyping:

  • Improved Requirements Gathering: Helps clarify unclear or ambiguous user needs.
  • Early User Feedback: Allows users to interact with a model early and provide input.
  • Reduced Risk: Identifies potential design flaws or usability issues before major investment.
  • Increased User Acceptance: Users feel more ownership and are more likely to accept a system they helped shape.
  • Can Serve as Specification: The final prototype can sometimes act as a basis for the final system specification.

(c)(i) Advantages of Computer-Based Training (CBT):

  • Flexibility: Users can learn at their own pace and schedule.
  • Consistency: Delivers standardized training content to all users.
  • Cost-Effective (Large Scale): Can reduce costs associated with instructors and travel for large numbers of trainees.
  • Interactive Elements: Can include simulations, quizzes, and multimedia for better engagement.

(c)(ii) Limitations of CBT:

  • Lack of Personal Interaction: No immediate instructor feedback or ability to ask nuanced questions easily.
  • Requires Self-Discipline: Users need motivation to complete the training independently.
  • Development Cost: Creating high-quality CBT content can be expensive and time-consuming initially.
  • Technology Dependent: Requires users to have access to appropriate hardware and software.

Question 7 (20 marks)

(a) Outline four guidelines that should be followed when developing system documentation. (4 marks)

(b) Table 2 shows net profits for project A and B respectively. Use it to answer the questions that follow.

Table 2
YearProject A Net profit value (Ksh.)Project B Net profit value (Ksh.)
130,00046,000
235,00046,000
340,00046,000
445,00046,000

(i) Using return on investment technique (ROI) determine the most worthwhile project given that the initial cost for the two projects was Kes 150,000 and Kes 195, 000 respectively. (4 marks)

(ii) Calculate the payback time for each of the project. (3 marks)

(iii) Explain two limitations of using the payback technique on the project. (4 marks)

(c) (i) State two types of stakeholders in a system project. (1 mark)

(ii) Fiona would like to apply risk management techniques to avoid derailment of an information system. Explain two risk management techniques that she could use. (4 marks)

Answer

(a) Documentation Development Guidelines:

  • Know the Audience: Tailor content, language, and level of detail to the intended readers (users, developers, managers).
  • Be Consistent: Use standard terminology, formatting, and structure throughout all documents.
  • Keep it Updated: Documentation must be maintained and revised as the system changes.
  • Use Clear Language: Avoid jargon where possible; explain technical terms if necessary. Use diagrams and examples.
  • Organize Logically: Structure the documentation for easy navigation and information retrieval (e.g., use indexes, tables of contents).

(b)(i) Return on Investment (ROI):

Project A: Total Profit = 30+35+40+45 = 150,000. Avg Profit = 150,000/4 = 37,500. ROI = (37,500 / 150,000) * 100% = 25%
Project B: Total Profit = 46*4 = 184,000. Avg Profit = 184,000/4 = 46,000. ROI = (46,000 / 195,000) * 100% = 23.59%
Project A is more worthwhile based on ROI.

(b)(ii) Payback Period:

Project A (Cost 150k):
Year 1: 30k (Cum: 30k)
Year 2: 35k (Cum: 65k)
Year 3: 40k (Cum: 105k)
Year 4: 45k (Cum: 150k). Payback = 4 years.

Project B (Cost 195k):
Year 1: 46k (Cum: 46k)
Year 2: 46k (Cum: 92k)
Year 3: 46k (Cum: 138k)
Year 4: 46k (Cum: 184k)
Year 5: Need 195 - 184 = 11k. Assuming profit continues at 46k/year (or need more data), Payback = 4 + (11,000 / 46,000) = 4.24 years.

(b)(iii) Limitations of Payback Technique:

  • Ignores Time Value of Money: Treats cash flows received in later years as equally valuable as those received earlier.
  • Ignores Cash Flows After Payback: Does not consider profitability or cash flows occurring after the initial investment is recovered.

(c)(i) Types of Stakeholders:

  • Users (End-users, Operators)
  • Management (Project Sponsor, Department Heads)
  • Development Team (Analysts, Programmers, Testers)
  • External Parties (Customers, Suppliers, Regulators)

(c)(ii) Risk Management Techniques:

  • Risk Avoidance: Changing the project plan to eliminate the risk or its impact (e.g., removing a high-risk feature).
  • Risk Mitigation: Taking steps to reduce the probability or impact of the risk (e.g., adding more testing for a complex module).
  • Risk Transfer: Shifting the risk to a third party (e.g., buying insurance, outsourcing a risky component).
  • Risk Acceptance: Acknowledging the risk and not taking action, often because the cost of mitigation outweighs the potential impact (usually for low-impact risks).

Question 8 (20 marks)

(a) Outline four qualities for a good output design in an information system. (4 marks)

(b) Jackson System Development (JSD) is a linear system development method. Explain three typical steps/stages that could be followed when using the methodology. (6 marks)

(c) Afueni Rank Ltd. intends to develop an agency banking system to carter for its growing number of customers. The proposed system will allow agents to register new bank accounts, deposits, withdraws and fund transfers for the bank customers. Draw a use-case diagram to represent the logical design of the described system. (5 marks)

(d) Arc Sport Club Ltd. intends to carry out a raffle competition in aid of its community project. The winner of the raffle must have been a club member for at least five years and must have bought raffles worth more than Kes 200,000. In addition the winner must be a Kenyan citizen and possess a good track record in sport.

Draw a decision tree to represent the logical of the narrative. (5 marks)

Answer

(a) Qualities of Good Output Design:

  • Accuracy: Output reflects correct data and calculations.
  • Timeliness: Output is available when needed for decision-making or operations.
  • Relevance: Output contains information pertinent to the user's needs, without unnecessary clutter.
  • Clarity and Readability: Output is well-formatted, easy to understand, and uses appropriate labels and summaries.
  • Appropriate Medium/Format: Output is delivered in a suitable format (e.g., screen display, printed report, file export).

(b) Jackson System Development (JSD) Stages:

  • Entity/Action Step: Identify entities in the real world and the actions they perform or suffer, creating an initial model based on these actions ordered by time.
  • Entity Structure Step: Define the structure of each entity's life history as a sequence, selection, or iteration of actions using structure diagrams (similar to Jackson Structured Programming - JSP).
  • Initial Model Step: Combine entity structures and actions, defining connections and data streams between them to form a system specification diagram (network model). (Followed by Function and System Timing steps for implementation details).
