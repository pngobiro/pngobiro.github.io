SYSTEMS ANALYSIS AND DESIGN

Write your name and index number in the answer booklet provided.

Question 1 (20 marks)

(a) (i) Outline two responsibilities of a system analyst in an organization. (2 marks)

(ii) Differentiate between structured and unstructured interviews. (4 marks)

(b) (i) Describe a knowledge base system. (2 marks)

(ii) State two desirable characteristics of internal documentation of a program. (2 marks)

(c) With the aid of a diagram, describe a closed control system. (4 marks)

(d) Transaction processing systems play a vital role in organizations. Explain three typical features of these systems that make them inevitable in organizations. (6 marks)

Answer

(a)(i) Responsibilities of a System Analyst:

  • Requirements Gathering and Analysis: Collecting and analyzing user needs and system requirements
  • System Design: Creating system specifications and design documents

(a)(ii) Structured vs Unstructured Interviews:

Structured Interviews Unstructured Interviews
Uses predetermined questions in fixed order Follows flexible, conversational approach
Easy to compare responses across interviewees Allows exploration of emerging topics
More formal and standardized More informal and adaptable
Limited follow-up questions Encourages spontaneous discussion

(b)(i) Knowledge Base System:

  • A specialized database that stores expert knowledge and rules
  • Used for automated problem-solving and decision support

(b)(ii) Characteristics of Internal Documentation:

  • Clear and readable: Easy to understand by other developers
  • Up-to-date: Reflects current version of the code

(c) Closed Control System:

+------------+     +------------+     +------------+     +------------+
| Input/     |     | Compare/   |     | Process/   |     | Output/    |
| Reference  |---->| Controller |---->| System     |---->| Result     |
+------------+     +------------+     +------------+     +------------+
                         ^                                     |
                         |                                     |
                         +-------------------------------------+
                                    Feedback Loop

Components:

  • Input/Reference: Desired state or value
  • Compare/Controller: Evaluates actual vs desired state
  • Process/System: Performs required actions
  • Output/Result: Actual system output
  • Feedback Loop: Returns information to controller

(d) Features of Transaction Processing Systems:

  • Real-time Processing:
    • Immediate transaction handling
    • Instant response to users
  • High Reliability:
    • Ensures data integrity
    • Maintains system availability
  • Data Consistency:
    • Maintains accurate records
    • Ensures synchronized data across system

Question 2 (20 marks)

(a) (i) Explain the importance of pseudo code in system design. (2 marks)

(ii) Outline three causes of project schedule slippage. (3 marks)

(b) Outline three limitations of using waterfall model in systems development. (3 marks)

(c) With the aid of diagrams, describe two types of relationships used in Entity relationship diagram (ERD). (6 marks)

(d) Most software developers are adopting objects oriented analysis and design. Explain three typical features associated with this methodology. (6 marks)

Answer

(a)(i) Responsibilities of a System Analyst:

  • Requirements Gathering and Analysis: Collecting and analyzing user needs and system requirements.
  • System Design: Creating system specifications and design documents.
  • Communication Bridge: Acting as liaison between users and technical team.
  • Project Management: Planning, coordinating, and monitoring system development activities.

(a)(ii) Structured vs Unstructured Interviews:

Structured Interviews Unstructured Interviews
Uses predetermined questions in a fixed order Follows a flexible, conversational approach
Easier to compare responses across interviewees Allows deeper exploration of emerging topics
More formal and standardized More informal and adaptable
Limited opportunity for follow-up questions Encourages spontaneous discussion

(b)(i) Knowledge Base System:

  • A specialized database system that stores and manages knowledge in a specific domain
  • Contains facts, rules, and expert knowledge used for problem-solving and decision-making
  • Often part of expert systems, providing the knowledge foundation for automated reasoning

(b)(ii) Desirable Characteristics of Internal Documentation:

  • Clarity and Readability: Easy to understand by other developers
  • Completeness: Covers all essential aspects of the code
  • Currency: Kept up-to-date with code changes
  • Consistency: Follows standard documentation format

(c) Closed Control System:

+------------+     +------------+     +------------+     +------------+
| Input/     |     | Compare/   |     | Process/   |     | Output/    |
| Reference  |---->| Controller |---->| System     |---->| Result     |
+------------+     +------------+     +------------+     +------------+
                         ^                                     |
                         |                                     |
                         +-------------------------------------+
                                    Feedback Loop

Components:

  • Input/Reference: Desired state or value
  • Compare/Controller: Compares actual vs desired state
  • Process/System: Performs the required actions
  • Output/Result: Actual system output
  • Feedback Loop: Returns output information to controller

(d) Features of Transaction Processing Systems:

  • Real-time Processing: Handles transactions immediately as they occur, providing instant responses to users.
  • High Reliability: Ensures data integrity and system availability through ACID properties (Atomicity, Consistency, Isolation, Durability).
  • High Volume Capability: Designed to handle large numbers of transactions efficiently.
  • Data Consistency: Maintains accurate and up-to-date data across all transactions.
  • Security Controls: Implements robust security measures to protect sensitive transaction data.

Question 3 (20 marks)

(a) Define the term resource as used in project management. (2 marks)

(b) (i) Describe two key components of a terms-of-reference document in information system projects. (4 marks)

(ii) Differentiate between portability and compatibility qualities as used in system design. (4 marks)

(c) Sarah was required to select a system development method for a proposed information system for her company. Outline four factors that she should consider in order to select an appropriate methodology. (4 marks)

(d) Charlie intends to validate an information system by tracing the system code.

(i) Identify the most appropriate type of test that he could apply. (2 marks)

(ii) Explain two challenges he could likely encounter while using the type of test identified in (i). (4 marks)

Answer

(a)(i) Importance of Pseudo Code:

  • Acts as a bridge between human language and programming code
  • Helps designers plan and structure program logic before actual coding
  • Makes it easier to communicate algorithms to non-technical stakeholders
  • Facilitates code review and debugging through clear logic representation

(a)(ii) Causes of Project Schedule Slippage:

  • Unrealistic Initial Estimates: Underestimating task complexity or effort required
  • Scope Creep: Uncontrolled changes or additions to project requirements
  • Resource Constraints: Unavailability of key personnel or equipment
  • Technical Challenges: Unexpected technical difficulties or integration issues
  • Poor Risk Management: Failure to identify and mitigate potential risks

(b) Limitations of Waterfall Model:

  • Inflexibility: Difficult to accommodate changes once a phase is complete
  • Late Testing: Problems aren't discovered until late in the development cycle
  • High Risk: Assumes requirements can be completely specified upfront
  • Limited User Feedback: Users don't see the system until it's almost complete

(c) ERD Relationship Types:

1. One-to-One (1:1)
+----------+     1     1     +----------+
| Entity A |---------------- | Entity B |
+----------+                 +----------+
Example: Employee ---- PersonalLocker

2. One-to-Many (1:N)
+----------+     1     N     +----------+
| Entity A |---------------- | Entity B |
+----------+                 +----------+
Example: Department ---- Employee

3. Many-to-Many (M:N)
+----------+     M     N     +----------+
| Entity A |---------------- | Entity B |
+----------+                 +----------+
Example: Student ---- Course

Key symbols used:

  • Crow's foot (branches) represents "many" side
  • Single line represents "one" side
  • Optional relationships shown with circle
  • Mandatory relationships shown with vertical line

(d) Features of Object-Oriented Analysis and Design:

  • Encapsulation:
    • Bundles data and methods that operate on that data within a single unit
    • Hides internal implementation details from outside access
    • Promotes data security and reduces system complexity
  • Inheritance:
    • Allows creation of new classes based on existing classes
    • Promotes code reuse and establishes relationships between classes
    • Supports hierarchical classification of objects
  • Polymorphism:
    • Allows objects to respond differently to the same message
    • Enables flexibility in implementing methods
    • Supports extensibility and maintenance

(b) Differentiate between legal and operational feasibility studies. (4 marks)

(c) Mobile application developers employ prototyping to solve user problems. Describe three types of prototypes used in this industry. (6 marks)

(a)(i) Use Case Diagram Symbols:

(a)(ii) Advantages of Use Case Diagrams:

  • Clear Communication:
    • Visual representation easily understood by stakeholders
    • Bridges gap between technical and non-technical audiences
  • Requirements Capture:
    • Clearly shows system functionality from user perspective
    • Helps identify and validate system requirements
  • Scope Definition:
    • Clearly shows system boundaries
    • Helps identify external entities and their interactions

(b) Legal vs Operational Feasibility:

Legal Feasibility Operational Feasibility
Focuses on legal compliance and regulations Focuses on how well the system will work in practice
Examines potential legal issues (patents, data protection, contracts) Examines user acceptance and organizational impact
Considers regulatory requirements and restrictions Considers workflow efficiency and business processes

(c) Types of Prototypes in Mobile Development:

  • 1. Paper Prototypes:
    • Hand-drawn sketches of screens and layouts
    • Quick and inexpensive to create
    • Used for early design feedback
  • 2. Wireframe Prototypes:
    • Digital mockups showing basic layout and navigation
    • No visual design elements
    • Focus on structure and functionality
  • 3. Interactive Prototypes:
    • Clickable mockups with simulated functionality
    • High-fidelity visual design
    • Tests user experience and interactions

  • Contrasts with off-the-shelf software solutions
  • Tailored to exact business processes and needs

(a)(ii) Uses of System Documentation:

    • Integration problems with other systems
  • Performance Issues:

Question 6 (20 marks)

(a) Outline two goals of usability in system design. (2 marks)

(b) Explain two factors that should be considered when formulating a risk plan in a project. (4 marks)

(c) Barbra was hired to develop and thoroughly test an information system for a client that did not have an existing information system.

(i) Identify the changeover strategy that she could use. (2 marks)

(ii) Outline four advantages of the changeover strategy identified in (i). (4 marks)

(d) Read the following narrative to answer the question that follows.

Mavao watering company intends to develop a system that would automate its operations. Customers are required to sign up for an account in the company's website which prompts them to fill in registration details, the submitted forms are stored in members file in the server located in the head office.

Members are then required to verify their account using the link provided in an email that was sent by the server. Upon verification their details is then stored in the verification file.

The verified members login the company's website using their credentials and select the types of transaction such as payment, print statement, customer care and suggestions box.

After the completion of the transaction, a transaction receipt is printed and transaction file updated.

Draw a level 1 dataflow diagram to model the described system. (8 marks)

Answer

(a) Goals of Usability in System Design:

  • Efficiency: Users can complete tasks with minimum effort and time
  • Effectiveness: Users can achieve their goals accurately and completely
  • User Satisfaction: System is pleasant and comfortable to use
  • Learnability: New users can quickly learn how to use the system

(b) Risk Plan Factors:

  • Risk Probability:
    • Likelihood of risk occurrence
    • Historical data and expert judgment
  • Risk Impact:
    • Potential consequences on project objectives
    • Cost, schedule, and quality implications
  • Risk Response Strategy:
    • Methods to address identified risks
    • Mitigation, avoidance, or acceptance plans

(c)(i) Recommended Changeover Strategy:

  • Direct Cutover/Direct Changeover Strategy
    • Immediate switch to new system
    • Most appropriate when no existing system exists

(c)(ii) Advantages of Direct Cutover:

  • Cost-Effective: No need to maintain multiple systems simultaneously
  • Clear Transition: No confusion about which system to use
  • Quick Implementation: Fastest way to get new system operational
  • Resource Efficiency: No need to duplicate data entry or operations

|           |    Login

  • External Entity: Customer
  • Processes:

Question 7 (20 marks)

(a) (i) Outline two data capturing methods that could be used in an information system. (2 marks)

(ii) Differentiate between user guide manual and technical manual as used in systems documentation. (4 marks)

(b) (i) Outline two participants that could be invited for a structured walkthrough meeting. (2 marks)

(ii) Outline three reasons that could influence a system designer to use functional decomposition. (3 marks)

(c) Decision trees are commonly used by system analysts. Outline three strengths of this tool that could be influencing this trend. (3 marks)

(d) Robert intends to train users of a new system using an online program.

(i) Identify the training method that he would be using. (2 marks)

(ii) Explain two benefits of using the training method identified in (i). (4 marks)

Answer

(a)(i) Data Capturing Methods:

  • Direct Entry:
    • Manual input through keyboards, touch screens
    • Form-based data entry systems
  • Automated Capture:
    • Barcode scanners and RFID readers
    • Optical Character Recognition (OCR)

(a)(ii) User Guide Manual vs Technical Manual:

User Guide Manual Technical Manual
Written for end-users Written for technical staff
Focus on how to use the system Focus on system architecture and maintenance
Simple, non-technical language Technical terminology and detailed specifications
Task-oriented instructions System-oriented documentation

(b)(i) Structured Walkthrough Participants:

  • Presenter: Developer/designer who created the work
  • Moderator: Facilitates the review process
  • Recorder: Documents findings and action items
  • Reviewers: Technical experts examining the work

(b)(ii) Reasons for Using Functional Decomposition:

  • Complexity Management:
    • Breaks down complex systems into manageable components
    • Makes system structure easier to understand
  • Modular Design:
    • Promotes reusable components
    • Facilitates parallel development
  • Maintenance:
    • Easier to locate and fix problems
    • Simpler to modify individual functions

(c) Strengths of Decision Trees:

  • Visual Clarity: Presents complex decision logic in an easy-to-follow format
  • Comprehensive Analysis: Shows all possible outcomes and decision paths
  • Easy Communication: Can be understood by both technical and non-technical stakeholders

(d)(i) Training Method:

  • Computer-Based Training (CBT) / E-Learning

(d)(ii) Benefits of Computer-Based Training:

  • Flexibility:
    • Users can learn at their own pace
    • Training available anytime, anywhere
  • Cost-Effectiveness:
    • Reduces travel and instructor costs
    • Can train multiple users simultaneously

Question 8 (20 marks)

(a) Outline two requirements for the Software as a Service (SaaS) in an organization. (2 marks)

(b) Explain two types of time estimates that could be used in a project network diagram. (4 marks)

(c) Rose intends to collect data about an organization without the knowledge of the workers.

(i) Explain the most appropriate data gathering technique that she could use. (2 marks)

(ii) Explain two problems that she could experience while using the fact-finding method identified in (i). (4 marks)

(d) (i) Jude intends to select a file organization method for an information system that he was designing for a client. Outline four factors that he should consider in the selection, other than cost. (4 marks)

(ii) Research statistics indicates high rate of system project failures. Outline four possible causes of the failures. (4 marks)

Answer

(a) SaaS Requirements:

  • Technical Requirements:
    • Reliable internet connectivity
    • Compatible web browsers/devices
  • Security Requirements:
    • Data encryption capabilities
    • Access control mechanisms

(b) Time Estimates in Project Network Diagrams:

  • Optimistic Time (O):
    • Best-case scenario estimate
    • Shortest possible completion time
  • Pessimistic Time (P):
    • Worst-case scenario estimate
    • Longest expected completion time

(c)(i) Data Gathering Technique:

  • Observation Method:
    • Unobtrusive monitoring of work processes
    • Direct observation without worker knowledge

(c)(ii) Problems with Observation Method:

  • Hawthorne Effect:
    • Workers may behave differently if they discover they're being observed
    • May not get accurate picture of normal operations
  • Ethical Concerns:
    • Privacy issues with covert observation
    • Potential breach of trust with employees

(d)(i) File Organization Selection Factors:

  • Access Speed: Required response time for data retrieval
  • Data Volume: Amount of data to be stored and processed
  • Update Frequency: How often data needs to be modified
  • Security Requirements: Data protection and access control needs

(d)(ii) Causes of System Project Failures:

  • Poor Requirements Analysis:
    • Incomplete or misunderstood requirements
    • Scope creep and changing requirements
  • Inadequate Planning:
    • Unrealistic timelines and budgets
    • Poor resource allocation
  • Communication Issues:
    • Poor stakeholder engagement
    • Lack of clear project objectives
  • Technical Challenges:
    • Complex integration issues
    • Technology limitations or incompatibilities
