Answer any FIVE questions in the answer booklet provided.

Question 1 (20 marks)

(a) Outline four rules that should be observed when drawing a data flow diagram. (4 marks)

(b) Explain three responsibilities of a project manager in an Information Communication Technology project. (6 marks)

(c) A system analyst at Faida Company developed a system documentation. Explain three purposes of this documentation. (6 marks)

(d) Techbriz Company is undertaking an Information Communication Technology project that is almost failing. Explain two signs exhibited by such projects. (4 marks)

Answer

(a) Rules for drawing DFDs:

  1. Process names should be meaningful and describe the transformation
  2. All flows must be labeled with appropriate data names
  3. Each process should have at least one input and one output flow
  4. Data stores must have at least one input and one output flow

(b) Project manager responsibilities:

  1. Planning and scheduling
    • Developing project timelines
    • Allocating resources
    • Setting milestones
  2. Team management
    • Assigning tasks
    • Monitoring performance
    • Resolving conflicts
  3. Stakeholder communication
    • Regular status updates
    • Managing expectations
    • Addressing concerns

(c) Purposes of system documentation:

  1. System maintenance
    • Guide for future modifications
    • Troubleshooting reference
  2. Knowledge transfer
    • Training new team members
    • Reference for developers
  3. System understanding
    • Architecture overview
    • Design decisions record

(d) Signs of failing ICT projects:

  1. Schedule delays
    • Missed deadlines
    • Extended timelines
  2. Budget overruns
    • Unexpected costs
    • Resource depletion

Question 3 (20 marks)

(a) Outline four advantages of using a Gantt chart during project management. (4 marks)

(b) Baraka Company limited has acquired a Management Information System (MIS). Explain two benefits that the company would realise from this system. (4 marks)

(c) (i) With the aid of a diagram, describe the waterfall model as used in system design. (4 marks)

(ii) Explain two benefits of carrying out a feasibility study in system development. (4 marks)

(d) Hillary used top down analysis methodology on an ICT project. Outline four reasons for his choice. (4 marks)

Answer

(a) Advantages of Gantt charts:

  1. Visual timeline representation
    • Clear project schedule
    • Easy to understand format
  2. Resource allocation tracking
    • Monitor workload distribution
    • Identify resource conflicts
  3. Progress monitoring
    • Track task completion
    • Identify delays
  4. Schedule management
    • Deadline tracking
    • Timeline adjustments

(b) Benefits of MIS:

  1. Improved decision making
    • Data-driven insights
    • Real-time information access
  2. Enhanced operational efficiency
    • Automated processes
    • Reduced manual work

(c) (i) Waterfall model:

    Requirements
         ↓
    Design
         ↓
    Implementation
         ↓
    Testing
         ↓
    Maintenance

Each phase must be completed before moving to the next phase.

(ii) Benefits of feasibility study:

  1. Risk assessment
    • Identify potential problems
    • Evaluate project viability
  2. Resource planning
    • Cost estimation
    • Resource requirements

(d) Reasons for top-down analysis:

  1. Better problem understanding
    • High-level view first
    • Systematic breakdown
  2. Easier management
    • Structured approach
    • Clear hierarchy
  3. Reduced complexity
    • Break down large problems
    • Manageable components
  4. Better documentation
    • Clear structure
    • Easy to maintain

Question 5 (20 marks)

(a) Outline four advantages of using structured narrative as a system design tool. (4 marks)

(b) Describe each of the following types of systems:

(i) open;

(ii) probabilistic;

(iii) adaptive. (6 marks)

(c) Explain a circumstance under which each of the following system change-over techniques would be appropriate:

(i) parallel;

(ii) direct. (4 marks)

(d) A system designer used functional decomposition in a system. Explain three advantages of this tool. (6 marks)

Answer

(a) Advantages of structured narrative:

  1. Clear documentation
    • Organized format
    • Easy to follow
  2. Detailed description
    • Complete process coverage
    • Step-by-step explanation
  3. Easy maintenance
    • Simple updates
    • Version control
  4. Communication tool
    • Stakeholder understanding
    • Reference document

(b) Types of systems:

(i) Open system:

  • Interacts with environment
  • Exchanges information/resources
  • Adapts to external changes

(ii) Probabilistic system:

  • Outcome not certain
  • Based on probability
  • Statistical prediction

(iii) Adaptive system:

  • Self-modifying behavior
  • Learns from experience
  • Responds to changes

(c) System change-over techniques:

(i) Parallel:

  • High-risk systems
  • Critical operations
  • Data verification needed

(ii) Direct:

  • Simple systems
  • Low-risk operations
  • Limited resources

(d) Advantages of functional decomposition:

  1. Simplified complexity
    • Break down large functions
    • Manageable components
  2. Better organization
    • Hierarchical structure
    • Clear relationships
  3. Easier maintenance
    • Modular design
    • Isolated updates

    • Process gaps
    • Logical errors

    • Stakeholder discussion
    • Process documentation

(ii) New system development trends:

  1. Agile development
  2. DevOps practices
  3. Cloud-based development

(b) System designer vs. developer:

System Designer System Developer
Creates system architecture Implements the design
Focus on planning Focus on coding
High-level design Detailed implementation
Conceptual work Technical work

(c) Reasons for employee training:

  1. Efficient system use
    • Proper functionality
    • Reduced errors
  2. Increased productivity
    • Better performance
    • Time savings

    Risk Analysis    Planning
          ↖️         ↗️
           \       /
            \     /
             \   /
    Evaluation ← Development

  1. Planning phase
    • Objectives setting
    • Alternative identification
  2. Risk analysis phase
    • Risk assessment
    • Risk mitigation
  3. Development phase
    • Product development
    • Testing
  4. Evaluation phase

Question 8 (20 marks)

(a) Outline four disadvantages of a structured chart design tool. (4 marks)

(b) Explain three components of system design. (6 marks)

(c) A student intends to use interview technique to collect data for a project. Explain two disadvantages of this technique. (4 marks)

(d) Paul developed a system using Structural Systems Analysis and Design Methodology (SSADM). Describe three models of this methodology that he could have used. (6 marks)

Answer

(a) Disadvantages of structured chart:

  1. Complex for large systems
    • Difficult to manage
    • Space constraints
  2. Limited detail representation
    • Basic relationships only
    • Missing complex interactions
  3. Maintenance challenges
    • Difficult updates
    • Version control issues
  4. Learning curve
    • Notation understanding
    • Interpretation skills

(b) System design components:

  1. User interface design
    • Screen layouts
    • User interaction
  2. Database design
    • Data structure
    • Relationships
  3. Process design
    • System functions
    • Business logic

(c) Disadvantages of interviews:

  1. Time consuming
    • Scheduling difficulties
    • Long duration
  2. Subjective responses
    • Personal bias
    • Inconsistent information

(d) SSADM models:

  1. Logical data model
    • Data structure
    • Entity relationships
  2. Data flow model
    • Process flow
    • Data movement
  3. Entity life history
    • State changes
    • Event sequences
