üìö System development methods

There are several methods of system development e.g:

  1. System Development Life Cycle (SDLC)
  2. Structured System Analysis Design methods (SSADM) method.
  3. Prototyping method.
  4. Rapid Application Development (RAD) etc.

‚ö° Brief Overview of Methods

System Development Life Cycle (SDLC): SDLC, also known as the traditional system development method, is a function-driven or process-driven approach. It requires the analyst to follow a sequence of phases during the development and implementation of an information system. This is a structured, step-by-step approach.

Structured System Analysis and Design Method (SSADM): SSADM is a widely used computer application development method in the UK. It's a structured approach that focuses on detailed documentation and clearly defined stages.

Prototyping: Prototyping involves creating a working model (prototype) of the system early in the development process. This allows users to interact with the system and provide feedback, leading to iterative refinements. This is an excellent way to handle user requirements that are not fully understood at the beginning.

Rapid Application Development (RAD): RAD is a methodology that emphasizes rapid prototyping and iterative development. It focuses on building a system quickly, often using tools that facilitate rapid development, like CASE (Computer-Aided Software Engineering) tools.
