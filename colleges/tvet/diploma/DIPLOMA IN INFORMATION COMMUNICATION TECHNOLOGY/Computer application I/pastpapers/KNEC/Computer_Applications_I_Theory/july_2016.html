1. (b) (4 marks)

Distinguish between report footer and page footer as used in database applications.

Answer

Page Footer:

A page footer appears at the bottom of each page in a printed database report. It is designed to include information that repeats on every page, such as page numbers, dates, or report titles. Page footers ensure consistent information across all pages of a multi-page report, aiding in navigation and identification.

Report Footer:

A report footer appears once at the very end of a database report, after all data records have been processed. It typically contains summary information for the entire report, such as grand totals, averages, or concluding remarks. Report footers provide a summary overview of the data presented in the whole report.

Distinction:

The key distinction is repetition and placement. Page footers repeat on every page for consistent page-level information, while report footers appear only once at the end of the entire report for overall summary.

1. (c) (4 marks)

The operating system performs the function of file management. Explain two such functions.

Answer

Two file management functions of an operating system are:

File and Folder Organization: The operating system allows users to create a hierarchical structure of directories (folders) and subdirectories to organize files. This function enables users to group related files together, making it easier to locate, access, and manage them. Users can create, rename, delete, and move files and folders within this structure, maintaining a logical and structured file system.

File Access and Permissions: The operating system manages access to files and folders by controlling user permissions. It defines who can access, modify, or execute specific files, ensuring data security and privacy. Access control mechanisms prevent unauthorized access, protect sensitive information, and allow for controlled sharing of files among users or processes based on defined permissions.

1. (d) (2 marks)

Table 1 shows an extract of records of students in the database of a college. Use it to answer the questions that follow.

Write query criteria that would be used to extract each of the following statements: (i) all students whose names' start with letter 'K' and come from Uganda;

Answer

Query criteria (i):

To extract all students whose names start with 'K' and are from Uganda, the criteria would be:

WHERE StudentName LIKE 'K%' AND Country = 'Uganda'

This SQL-like WHERE clause uses the LIKE operator to find names starting with 'K' and ensures the country is 'Uganda'.

1. (d) (2 marks)

(ii) students born before 6/5/1995 with fees balance less than 50,000.00;

Answer

Query criteria (ii):

To extract students born before 6/5/1995 with a fees balance less than 50,000.00, the criteria would be:

WHERE DateOfBirth < #1995-05-06# AND FeesBalance < 50000

This SQL-like WHERE clause filters for birth dates before May 6, 1995, and fees balance under 50,000. Date format might need adjustment based on database system.

2. (d) (3 marks)

Kabana had the following formulas entered in different cells of a worksheet: (i) =D29/$D$36*160;

Answer

Formula (i) =D29/$D$36*160:

This formula calculates a value by dividing the content of cell D29 by the absolute reference cell D36, and then multiplying the result by 160. The `$` symbols in `$D$36` indicate an absolute cell reference, meaning that when this formula is copied to other cells, the reference to D36 will remain fixed, while the reference to D29 will adjust relatively.

2. (d) (3 marks)

(ii) $D$3 $F$13; Explain the cell reference used in each of the formulas.

Answer

Formula (ii) $D$3 $F$13:

This formula is syntactically incorrect in standard spreadsheet applications. It appears to be attempting to use cell ranges or multiple cell references in a way that is not directly interpretable as a valid formula. In typical spreadsheet software, ranges are specified with a colon (e.g., D3:F13), and multiple cells are usually combined using operators or functions, not just spaces. It's likely a typographical error or an attempt to represent a conceptual or pseudo-formula rather than a working spreadsheet formula.

Cell References Explanation:

In formula (i), `$D$36` is an absolute cell reference, fixed to column D and row 36 regardless of where the formula is copied. `D29` is a relative cell reference, which will adjust its row and/or column based on the relative position when the formula is copied. In formula (ii), if it were intended to be a valid formula, `$D$3` and `$F$13` would both be interpreted as absolute cell references due to the `$` signs, referring to fixed cells D3 and F13 respectively. However, the syntax as presented is not standard for spreadsheet formulas.

3. (b) (3 marks)

Paul noticed the following errors displayed in different cells of a worksheet. #DIV/0! #REF! #VALUE!

(i) Explain the cause of each of the errors.

Answer

(i) Causes of the errors:

#DIV/0!: This error occurs when a formula attempts to divide a number by zero or an empty cell. Division by zero is mathematically undefined and results in this error in spreadsheet applications.

#REF!: This error indicates an invalid cell reference. It typically happens when a formula refers to a cell that has been deleted, or when a reference is no longer valid due to structural changes in the worksheet, like deleting rows or columns.

#VALUE!: This error occurs when a formula expects a specific type of argument (like a number or text) but receives a different type. It often arises from using text where a number is expected in a mathematical operation, or when incompatible data types are used in a formula.

3. (b) (3 marks)

(ii) Explain the remedy for each of the errors identified in (i).

Answer

(ii) Remedies for the errors:

#DIV/0! Remedy: To fix #DIV/0! error, ensure that the denominator in the division formula is not zero or an empty cell. Check the cell being used as the divisor and correct it to a non-zero value or use error handling functions like IFERROR to display a different result when division by zero occurs.

#REF! Remedy: To resolve #REF! error, review the formula and identify the invalid cell reference. Check if the referenced cell or range was accidentally deleted or moved. Correct the formula to refer to valid and existing cells or ranges within the worksheet. If cells were deleted intentionally, reconsider the formula's logic.

#VALUE! Remedy: To fix #VALUE! error, verify the data types used in the formula. Ensure that all arguments in the formula are of the correct type (e.g., numbers for arithmetic operations, text for text functions). Check for text values where numbers are expected and correct the input data or adjust the formula to handle different data types appropriately, possibly using functions to convert data types if needed.

4. (b) (ii) (4 marks)

Differentiate between MEMO and OLE data types as in database application, giving an example in each case.

Answer

MEMO Data Type:

MEMO data type is used to store large blocks of text in a database field. It is designed for variable-length text data such as notes, descriptions, or comments, which can exceed the character limits of standard text or string data types. MEMO fields are suitable for storing extensive textual information without fixed length constraints.

Example for MEMO: A 'ProductDescription' field in a product database table, where each product description can be lengthy and detailed, requiring a MEMO data type to accommodate variable text lengths.

OLE Data Type:

OLE (Object Linking and Embedding) data type is used to embed or link objects created in other applications (like documents, spreadsheets, images, sounds, or videos) into a database record. It allows storing complex, non-textual data within a database, integrating multimedia and document files directly into database records.

Example for OLE: An 'EmployeePhoto' field in an employee database table, where employee photographs (image files) are embedded directly into each employee's record using the OLE data type.

Differentiation:

MEMO data type is for storing large text amounts, focusing on textual data. OLE data type is for embedding or linking objects from other applications, handling binary data and multimedia content within the database.

4. (c) (4 marks)

Distinguish between slide sorter and slide layout as applied in presentation programs.

Answer

Slide Sorter:

Slide sorter is a presentation view that displays thumbnail previews of all slides in a presentation in a grid or sequence. It is primarily used for organizing and managing slides in a presentation. Users can reorder slides, delete slides, and get an overview of the entire slideshow flow in slide sorter view. It focuses on the overall structure and sequence of slides.

Slide Layout:

Slide layout refers to the arrangement of content placeholders (for titles, text, images, charts, etc.) on a single slide. It defines the structure and positioning of elements within an individual slide. Presentation programs offer various predefined slide layouts (e.g., title slide, title and content, section header) that users can choose from and customize to structure the content of each slide. It focuses on the internal structure and content arrangement of a single slide.

Distinction:

Slide sorter is for managing the sequence and overall structure of multiple slides in a presentation. Slide layout is for designing the internal structure and content arrangement within a single slide.

5. (c) (2 marks)

With the aid of a diagram in each case, describe the following database models: (i) hierarchical;

Answer

Hierarchical Database Model:

Hierarchical database model organizes data in a tree-like structure where each record has a single parent and multiple children. Relationships are one-to-many, forming a hierarchy from a root record down to subordinate records. Data is accessed by navigating down the hierarchy.

5. (c) (2 marks)

(ii) network.

Answer

Network Database Model:

Network database model is an extension of the hierarchical model, allowing records to have multiple parent records and multiple child records, forming a graph-like structure. This model uses pointers to connect related records, enabling more complex many-to-many relationships.

5. (d) (3 marks)

Distinguish between design view and datasheet view as applied in databases.

Answer

Design View:

Design view in databases is used to create and modify the structure of database objects like tables, forms, queries, and reports. In design view, users can define fields, set data types, establish relationships, and configure properties of database elements. It focuses on the structural and design aspects of database objects.

Datasheet View:

Datasheet view displays the actual data stored in a table in a row-and-column format, similar to a spreadsheet. It is used for viewing, adding, editing, and deleting data records within a table. Datasheet view provides a user-friendly interface to interact with the data content of a database table.

Distinction:

Design view is for structuring and defining database objects, focusing on schema and properties. Datasheet view is for interacting with the actual data content within tables, focusing on data manipulation and viewing.

Write a validation rule for each of the following conditions to be used during the design of database tables: (i) accept any characters between (a-z) only;

Answer

Validation rule (i):

Like "[a-z]*"

This rule uses the `Like` operator with a pattern that allows only lowercase letters from 'a' to 'z'. The `*` wildcard allows for any number of characters following the lowercase letters.

6. (b) (2 marks)

(ii) accept any 8 numeric characters only;

Answer

Validation rule (ii):

Like "########"

This rule uses the `Like` operator with a pattern that requires exactly 8 numeric characters. Each `#` symbol represents a required digit. Alternatively, depending on the database system, `Is Numeric` constraint and length check may be used.

Bcc (Blind Carbon Copy): Bcc is used when sending an email to multiple recipients and the sender wants to keep the recipients' email addresses private from each other. This is necessary for transparency and collaboration, allowing all recipients to see each other's email addresses and facilitating group communication and shared awareness of the conversation.
