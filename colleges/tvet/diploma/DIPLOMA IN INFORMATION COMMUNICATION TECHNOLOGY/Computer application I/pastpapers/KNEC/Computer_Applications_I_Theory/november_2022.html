1. (a) (3 marks)

Outline three advantages that an organization would get for using email for their communication.

Answer

Email is cost-effective as it reduces expenses associated with postage, printing, and long-distance calls.

It provides a fast and efficient way to send and receive messages, enabling quick communication across geographical barriers.

Email facilitates record-keeping and documentation, as sent and received messages are automatically stored and can be easily archived and retrieved.

1. (b) (4 marks)

Describe each of the following terms as used in desktop publishing applications:

(i) bleed;

(ii) crop.

Answer

Bleed:

Bleed refers to the area of a document that extends beyond the trim edge. It ensures that after trimming, no white edges are visible, especially for images or colored backgrounds that are intended to reach the edge of the printed page. Bleed is essential for professional printing to achieve edge-to-edge printing without any margin of error during the cutting process.

Crop:

Crop is the process of removing unwanted outer areas of an image to improve framing, emphasize the subject, or change the aspect ratio. In desktop publishing, cropping tools allow users to trim images directly within the layout software, focusing on the desired portion of the graphic and eliminating distracting or unnecessary elements. This is important for visual composition and efficient use of space in design layouts.

1. (c) (4 marks)

Distinguish between device drivers and firmware as used in computers.

Answer

Device Drivers:

Device drivers are software programs that enable the operating system to interact with hardware devices. They act as translators between the OS and the hardware, providing instructions on how to communicate and operate the specific device. Drivers are OS-specific and can be updated or changed independently of the hardware itself. They are typically installed after the OS and hardware are in place, allowing new or updated hardware to function correctly with the system.

Firmware:

Firmware is a specific type of software that is embedded directly into hardware devices. It provides the basic operational instructions for the hardware and is usually stored in non-volatile memory like ROM or flash memory. Firmware is more closely tied to the hardware than drivers, often controlling fundamental functions at a lower level. Updating firmware involves rewriting the software directly on the hardware's memory chip, and it is less frequently changed compared to device drivers. Firmware is essential for the initial boot process and basic hardware operations before the operating system fully takes over.

Key Differences:

Drivers are software that allows the OS to communicate with hardware, while firmware is software embedded in the hardware itself to control its basic operations. Drivers are OS-dependent and frequently updated, whereas firmware is hardware-dependent and updated less often. Drivers operate at a higher software level, while firmware operates at a lower, more fundamental hardware level.

1. (d) (4 marks)

Nancy was tasked to design a database for data storage. Outline four properties she could apply to fields during the design.

Answer

Data Type: Specify the type of data a field can hold, such as text, number, date, or currency, ensuring data integrity and appropriate storage.

Field Size/Length: Define the maximum number of characters or digits a field can store, optimizing storage space and preventing overflow.

Format: Set a specific display format for the data, like date format (DD/MM/YYYY), currency format (with decimal places), or text case (uppercase/lowercase), to ensure consistency and readability.

Validation Rules: Implement rules to restrict the values entered into a field, such as range checks (e.g., age must be between 0 and 120), format checks (e.g., email address format), or list of allowed values, maintaining data accuracy and reliability.

2. (a) (3 marks)

Outline the meaning of each of the following database concepts:

(1) data security;

(ii) data quality:

(iii) data integrity.

Answer

Data security:

Data security refers to the protection of data from unauthorized access, use, disclosure, disruption, modification, or destruction. It involves implementing measures and techniques to ensure confidentiality, integrity, and availability of data assets, safeguarding sensitive information from threats and vulnerabilities.

Data quality:

Data quality describes the overall usability of data for its intended purposes. It encompasses characteristics such as accuracy, completeness, consistency, validity, and timeliness. High data quality ensures that data is fit for use in operations, decision-making, and planning, leading to reliable insights and outcomes.

Data integrity:

Data integrity refers to the accuracy and consistency of data over its lifecycle. It ensures that data remains unaltered and reliable as it is stored, processed, and retrieved. Maintaining data integrity involves implementing validation rules, access controls, and error detection mechanisms to prevent data corruption and ensure data trustworthiness.

2. (b) (4 marks)

Rose was tasked to design a database for an organizations' data storage. Outline four types of keys she may use while creating the database fields.

Answer

Primary Key: A primary key uniquely identifies each record in a database table. It must contain unique values and cannot be null. It ensures that each record is distinguishable and serves as the main reference point for data retrieval and relationships.

Foreign Key: A foreign key is a field in one table that refers to the primary key in another table. It establishes a link between the two tables, creating a relationship and enabling data to be joined and accessed across related tables. Foreign keys enforce referential integrity.

Composite Key: A composite key is a primary key made up of two or more attributes (fields) within a table that, when combined, uniquely identify each record. It is used when a single attribute is insufficient to guarantee uniqueness, and the combination of attributes is necessary for unique identification.

Candidate Key: A candidate key is any attribute or set of attributes that can uniquely identify a record in a table. A table can have multiple candidate keys, but one is chosen as the primary key. The remaining candidate keys are considered alternate keys. They possess the properties of uniqueness and non-null values, making them suitable for primary key selection.

2. (c) (4 marks)

A company's technician was required to design a network operating system. Explain two types of operating system design he could apply.

Answer

Client-Server Design:

In a client-server design, the network operating system is structured around servers that provide resources and services, and clients that request and utilize those services. Servers are centralized and manage functionalities such as file storage, printing, email, and database access. Clients are typically individual workstations that rely on the server for shared resources and applications. This model offers centralized management, enhanced security, and efficient resource sharing, making it suitable for organizations with structured IT infrastructure.

Peer-to-Peer Design:

In a peer-to-peer design, all computers on the network act as both clients and servers. There is no central server; instead, each computer can share its resources directly with others. This design is simpler to set up and manage, often used in small networks without dedicated IT staff. Peer-to-peer networks are less centralized, offering more distributed control and potentially lower costs, but they can be less secure and harder to manage as the network scales up in size and complexity.

2. (d) (4 marks)

Joel was tasked to source for an Internet service provider to supply his college with Internet. Outline fours factor other than cost that he should consider during the selection.

Answer

Bandwidth and Speed: Consider the download and upload speeds offered by the ISP to ensure they meet the college's internet usage needs, including student access, online learning platforms, and administrative operations. Higher bandwidth and faster speeds are crucial for a smooth and efficient internet experience, especially with multiple users.

Reliability and Uptime: Evaluate the ISP's track record for service reliability and uptime. A reliable ISP should have minimal service interruptions and consistent performance. High uptime is essential for uninterrupted access to online resources and services for the college community.

Customer Support: Assess the quality and availability of customer support offered by the ISP. Responsive and effective technical support is important for resolving any connectivity issues or service disruptions promptly. Consider factors such as support hours, contact methods (phone, email, chat), and service level agreements.

Scalability and Future Needs: Determine if the ISP can accommodate future growth in internet usage and bandwidth requirements as the college expands or technology demands increase. Scalability ensures that the internet service can adapt to the evolving needs of the institution without requiring frequent and disruptive upgrades or provider changes.

3. (a) (3 marks)

Outline the function of each formula in a spreadsheet program:

(i) IF;

(ii) Len;

(iii) Concatenate.

Answer

IF:

The IF function performs a logical test and returns one value if the condition is true and another value if it is false. It is used for decision-making within a spreadsheet, allowing for different calculations or outputs based on whether a specified condition is met or not. It is fundamental for creating conditional logic in spreadsheets.

Len:

The Len function calculates and returns the number of characters in a text string. This includes letters, numbers, spaces, and special characters. It is useful for determining the length of text entries, validating data input lengths, or manipulating text strings based on their size.

Concatenate:

The Concatenate function joins two or more text strings into one string. It allows users to combine text from different cells or to add custom text to existing cell content. This is beneficial for creating combined labels, full names from separate first and last names, or constructing dynamic text messages within spreadsheet formulas.

3. (b) (4 marks)

Explain one circumstance in each case, that would necessitate the use of the following features in a word processing program:

(i) split window;

(ii) zoom.

Answer

Split window:

A circumstance that necessitates the use of a split window is when a user needs to view and edit two different parts of a long document simultaneously. For example, when cross-referencing information from a section at the beginning of a document to a section at the end, splitting the window allows for both sections to be visible on screen at the same time. This eliminates the need to scroll back and forth repeatedly, enhancing efficiency in tasks like comparing sections, copying and pasting between distant parts of the document, or maintaining consistency across the document.

Zoom:

A circumstance that necessitates the use of zoom is when a user needs to adjust the viewing scale of a document to improve readability or detail visibility. For instance, when working with a document that contains fine details, such as intricate diagrams or small font sizes, zooming in allows for a closer and clearer view of these elements. Conversely, zooming out can be necessary to get an overview of the document layout, check page breaks, or assess the overall structure and formatting of the document on screen. Zoom functionality is essential for accommodating different viewing preferences and document complexities.

3. (c) (4 marks)

Distinguish between ruler guides and column guides as applied in desktop publishing programs.

Answer

Ruler Guides:

Ruler guides are horizontal and vertical lines that users can drag from the rulers (typically located at the top and left edges of the document window) onto the page layout in desktop publishing software. They are non-printing lines used for precise alignment of objects and text boxes on the page. Ruler guides are manually positioned by the user and are flexible, allowing for custom placement anywhere on the page to aid in the visual arrangement and alignment of design elements across the entire page.

Column Guides:

Column guides are predefined vertical lines that divide the page into columns for text and other content in desktop publishing. They are typically set up in the document setup or master page settings and are used to create a structured grid layout for text flow and element placement within columns. Column guides help maintain consistency in text width and spacing across pages, particularly in multi-column documents like newspapers, magazines, or brochures. They provide a framework for organizing content into readable and visually balanced columns.

Key Differences:

Ruler guides are manually placed for general alignment across the page, while column guides are predefined for structuring content into columns. Ruler guides are flexible and can be positioned anywhere, whereas column guides are typically set up for consistent column layout throughout the document. Ruler guides assist in aligning individual elements, and column guides structure the overall page content flow.

3. (d) (4 marks)

Agnes intends to align text in a slide created using a presentation program. Outline four text alignment options she could apply.

Answer

Left Align: Text is aligned along the left margin, creating a straight left edge and a ragged right edge. This is commonly used for body text as it enhances readability by providing a consistent starting point for each line.

Center Align: Text is centered within the text box or slide width, with both left and right edges being ragged. Center alignment is often used for titles, headings, and short lines of text to create a balanced and formal appearance.

Right Align: Text is aligned along the right margin, creating a straight right edge and a ragged left edge. Right alignment is less common for body text in left-to-right languages but can be used for specific design elements or in right-to-left languages.

Justify Align: Text is aligned along both the left and right margins, creating straight edges on both sides. Spaces between words are adjusted to achieve this effect. Justification is often used in formal documents and publications to create a neat and structured block of text, but it may sometimes lead to uneven word spacing if not carefully managed.

4. (a) (3 marks)

State three reasons for using queries in a database management system.

Answer

Data Retrieval: Queries are used to retrieve specific data from one or more tables based on specified criteria. They allow users to extract only the necessary information, rather than the entire dataset, making data access efficient and targeted.

Data Manipulation: Queries can be used to manipulate data, such as updating records, deleting records, or inserting new records. This enables users to manage and modify the database content dynamically, ensuring data is current and accurate.

Data Analysis and Reporting: Queries facilitate data analysis by allowing users to perform calculations, aggregations, and filtering on the retrieved data. The results from queries can be used to generate reports, summaries, and insights, supporting decision-making and data-driven analysis.

4. (b) (4 marks)

Explain one function of each of the following database objects:

(1) forms:

(ii) reports.

Answer

Forms:

Forms in a database are used to provide a user-friendly interface for entering, editing, and viewing data in database tables. 4. (c) (4 marks)

Lydia intends to use tab stop feature to align text in a document created using a word processing program. Outline four examples of tab stops that she could apply.

Answer

Left Tab: Left tab stop aligns the start of the text at the tab stop position. As you type, the text extends to the right of the tab stop. This is commonly used for aligning lists of items or creating simple columns of text where the left edges of the items are aligned.

Center Tab: Center tab stop centers the text around the tab stop position. As you type, the text expands equally to the left and right of the tab stop. This is useful for centering headings or text within columns or for creating symmetrically aligned lists.

Right Tab: Right tab stop aligns the end of the text at the tab stop position. As you type, the text extends to the left of the tab stop. This is often used for aligning numbers in columns, especially in financial documents, where the right edges of the numbers need to be aligned for readability.

Decimal Tab: Decimal tab stop aligns numbers at the decimal point. It is a specialized type of tab stop primarily used for aligning columns of numbers with decimal points, ensuring that the decimal points are vertically aligned, making it easier to compare numerical data in columns, particularly in financial or statistical reports.

5. (a) (3 marks)

Outline the function of each of the following data types in a database:

(1) AutoNumber;

(ii) Memo;

(iii) LookUp Wizard.

Answer

AutoNumber:

The AutoNumber data type automatically assigns a unique sequential number to each new record added to a database table. It is primarily used to create primary keys that are automatically incremented, ensuring each record has a unique identifier without manual input. AutoNumber simplifies record management and guarantees uniqueness, essential for database integrity and relationships.

Memo:

The Memo data type is used to store large blocks of text, such as notes, descriptions, or comments, that exceed the character limit of a standard text field. It is designed to accommodate variable-length text data, from short sentences to lengthy paragraphs, without imposing a fixed length constraint. Memo fields are suitable for storing detailed textual information that does not fit into regular text or character-limited fields.

LookUp Wizard:

The LookUp Wizard is not a data type itself, but a tool used to create a lookup field in a database table. It allows a field to display values from another table or a predefined list. When creating a field using the LookUp Wizard, it guides the user to set up a relationship with another table or specify a value list. This simplifies data entry by providing users with selectable options, ensuring data consistency and referential integrity. It is used to create dropdown lists or combo boxes in forms and tables, linking fields to predefined sets of values.

5. (b) (4 marks)

Distinguish between rotate and flip operations as used in desktop publishing applications.

Answer

Rotate:

Rotation in desktop publishing refers to turning an object or image around a central point by a specified angle. Rotation changes the orientation of the object without mirroring it. The object maintains its shape and content but is presented at a different angle relative to its original position. Rotation is commonly used for adjusting the slant of text, graphics, or shapes to fit design layouts or create visual effects. The orientation change is in a circular direction, either clockwise or counterclockwise.

Flip:

Flipping, also known as mirroring, creates a reversed image of an object or graphic, either horizontally or vertically. Horizontal flip creates a mirror image from left to right, while vertical flip creates a mirror image from top to bottom. Flipping changes the object's orientation by creating a reflection across an axis, reversing its content along that axis. It is used for creating reflections, reversing object direction, or achieving symmetrical designs. The object's content is reversed, but its shape remains fundamentally the same.

Key Differences:

Rotation turns an object around a point, changing its angle, while flipping mirrors an object across an axis, reversing its content. Rotation maintains the object's orientation relationship but changes its angle, whereas flipping reverses the orientation. Rotation changes the angle, and flip creates a mirror image.

5. (c) (4 marks)

Figure 1 is a spreadsheet extract showing the revenue and expenditure for a company for a period of 5 years. Use it to answer the questions that follow.

(1) Using cell addresses, write a formula that would be used to compute the total revenue for the five years in cell G2.

(ii) Use the IF function to write a formula in cell B5 that would display the comment "PROFIT" if the revenue exceeds the expenditure otherwise to displays "LOSS" for the five years.

Answer

(i) Formula to compute total revenue in cell G2:

=SUM(B2:F2)

This formula uses the SUM function to add up the values in cells B2, C2, D2, E2, and F2, which represent the revenue for the five years from 2014 to 2018. The result, which is the total revenue, will be displayed in cell G2.

(ii) Formula using IF function in cell B5 to display "PROFIT" or "LOSS":

=IF(B2>B3,"PROFIT","LOSS")

This formula uses the IF function to compare the revenue in cell B2 (for the year 2014) with the expenditure in cell B3 (for the year 2014). If the revenue (B2) is greater than the expenditure (B3), the formula will display "PROFIT" in cell B5. Otherwise, if the revenue is not greater than the expenditure (meaning expenditure is greater than or equal to revenue), the formula will display "LOSS" in cell B5. This formula checks the profitability for the year 2014 and provides a textual comment accordingly.

5. (d) (4 marks)

Patrick created a presentation which he wishes to print. Outline four types of print layout options he could use to print the presentation.

Answer

Full Page Slides: Prints each slide on a separate full page. This layout is suitable for presentations where detailed visuals and text need to be clearly visible on printouts, maximizing the size of each slide on paper.

This option is useful for reviewing the presentation structure, content flow, and text-based information. 6. (a) (4 marks)

Explain each of the following features as used in a spreadsheet program:

(1) sort;

(ii) filter.

Answer

Sort:

The sort feature in a spreadsheet program rearranges the rows of data in a worksheet based on the values in one or more columns. Sorting can be done in ascending or descending order, either alphabetically for text data or numerically for numbers and dates. It allows users to organize data to easily find specific items, identify trends, or present data in a logical order. Sorting can be applied to a selected range of cells or the entire dataset, enabling efficient data organization and analysis.

Filter:

The filter feature in a spreadsheet program temporarily hides rows that do not meet specified criteria, displaying only the rows that match the defined conditions. Filtering allows users to focus on a subset of data that is relevant to their analysis or task. Criteria can be set based on values in one or more columns, using conditions such as equal to, greater than, contains, or begins with. Filters are non-destructive, meaning they do not delete or alter the original data, and they can be easily applied and removed, providing dynamic data viewing and analysis capabilities.

6. (c) (6 marks)

With the aid of a sketch in each case, describe the following components of an entity data model in databases:

(1) attribute;

(ii) relationship.

Answer

(i) Attribute:

An attribute is a characteristic or property of an entity in an entity-relationship model. It describes a detail or feature of an entity. Attributes are represented as ovals connected to their respective entity rectangles in an ER diagram. Each attribute has a name and a data type, defining the kind of information it holds. Attributes can be simple (atomic), composite (composed of multiple attributes), single-valued, multi-valued, or derived. They provide specific details about each instance of an entity.

Sketch Description: Imagine an entity called "STUDENT" represented by a rectangle. Attributes of STUDENT could be "StudentID", "Name", "Course", and "DateOfBirth". Each of these attributes would be represented by an oval connected to the STUDENT rectangle. "StudentID" could be underlined to denote it as a key attribute. For example:

[Rectangle: STUDENT] -- (Oval: StudentID) <--(Key Attribute)
                    -- (Oval: Name)
                    -- (Oval: Course)
                    -- (Oval: DateOfBirth)

In this sketch, StudentID, Name, Course, and DateOfBirth are attributes that describe the entity STUDENT. StudentID is the key attribute uniquely identifying each student.

(ii) Relationship:

A relationship represents an association or link between two or more entities in an entity-relationship model. Relationships describe how entities are related to each other in the database. Relationships are represented by diamond shapes connected to the entities they relate in an ER diagram. Each relationship has a name and a cardinality, defining the type and degree of association between entities (e.g., one-to-one, one-to-many, many-to-many). Relationships are crucial for structuring data and enabling meaningful connections between different types of data.

Sketch Description: Consider two entities, "STUDENT" and "COURSE", represented by rectangles. A relationship between them could be "ENROLLS_IN", indicating that students enroll in courses. This relationship would be represented by a diamond shape labeled "ENROLLS_IN" connected to both the STUDENT and COURSE rectangles. Cardinality can be indicated next to the lines connecting the relationship to the entities, for example, '1' for one and 'N' for many to show a one-to-many or many-to-many relationship if needed. For example:

[Rectangle: STUDENT] -- (Line) -- (Diamond: ENROLLS_IN) -- (Line) -- [Rectangle: COURSE]

In this sketch, "ENROLLS_IN" is the relationship connecting the STUDENT and COURSE entities, indicating how students are associated with courses. The cardinality (not shown in simple sketch) would further specify if one student can enroll in many courses (one-to-many from STUDENT to COURSE) or if many students can enroll in many courses (many-to-many).
