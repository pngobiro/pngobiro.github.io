Question 1

(a) (i) Explain a reason that make Visual Basic not suitable for developing expert systems.

(ii) Figure 1 shows two features in a Visual Basic Integrated Development Environment. Use it to answer the question that follows.

Describe the features labelled II and I.

(b) (i) Explain each of the following terms as used in Visual Basic programming:

  • (I) Event;
  • (II) Method.

(ii) Distinguish between pass by value and pass by reference as used in Visual Basic programming.

(c) Write a Visual Basic program that would compute the average of odd integers from 61 to 150 and display the output on a form. Use for.. loop. Attach the code to a command button.

Answer

(a) (i) Reason VB not suitable for expert systems:

Visual Basic is primarily designed for GUI development and event-driven programming, lacking strong built-in support for symbolic reasoning, knowledge representation, and inference engines typically required for complex expert systems.

(ii) Features labelled:

  • I: Project Explorer Window - Displays a hierarchical view of the forms, modules, and other components in the current project.
  • II: Properties Window - Displays the properties of the currently selected object (form or control) and allows modification of these properties.

(b) (i) Terms:

  • (I) Event: An action or occurrence recognized by an object, such as a mouse click or key press, to which the program can respond.
  • (II) Method: A procedure associated with an object that performs an action on that object.

(ii) Pass by value vs Pass by reference:

  • Pass by value (ByVal): A copy of the variable's value is passed to the procedure. Changes made to the parameter inside the procedure do not affect the original variable.
  • Pass by reference (ByRef): The memory address of the variable is passed to the procedure. Changes made to the parameter inside the procedure directly affect the original variable.

(c) Visual Basic program (Average of odd integers):


Private Sub Command1_Click()
    Dim i As Integer
    Dim sum As Long
    Dim count As Integer
    Dim average As Double

    sum = 0
    count = 0

    For i = 61 To 150 Step 2 ' Start at 61, increment by 2 to get odd numbers
        sum = sum + i
        count = count + 1
    Next i

    If count > 0 Then
        average = sum / count
        Form1.Print "Average of odd integers from 61 to 150: " & average
    Else
        Form1.Print "No odd integers found in the range."
    End If
End Sub

(c) Binary search steps (key = 28):

Question 3

(a) (i) Define each of the following terms as used in a Visual Basic Programming language:

  • I. form module
  • II. application
  • III. project.

(ii) Explain a challenge that companies with applications built using Visual Basic 6.0 compiler may have with the changing software technology.

(b) (i) Outline four limitations of using Visual Programming languages.

(ii) List two database objects in Visual Basic other than ADO that may be used to manipulate a database file.

(c) Distinguish between method and a function as used in a Visual Basic Programming language.

(d) Write a Visual Basic program that would display the sequence of numbers; \(50,45,40, \ldots 0\) on a listbox together with their cumulative sum. use while...wend loop. Attach the code to a command button.

Answer

(a) (i) Definitions:

  • I. Form module: A file containing the code associated with a specific form, including its event procedures and any general procedures specific to that form.
  • II. Application: A complete, executable program created using Visual Basic.
  • III. Project: A collection of files (forms, modules, controls, etc.) that make up a Visual Basic application.

(ii) Challenge with VB 6.0:

Visual Basic 6.0 is an older technology and is no longer supported by Microsoft. Applications built with it may face compatibility issues with modern operating systems and hardware. Migrating these applications to newer platforms like .NET can be complex and costly.

(b) (i) Limitations of Visual Programming languages:

  • Performance limitations compared to lower-level languages for certain tasks.
  • Can sometimes abstract too much, making low-level control difficult.
  • Dependency on specific IDEs and runtime environments.
  • Visual clutter can become an issue for very large projects.

(ii) Other database objects:

  • DAO (Data Access Objects)
  • RDO (Remote Data Objects)

(c) Method vs Function:

  • Method: A procedure associated with an object that performs an action. It does not explicitly return a value (though it might modify object properties or have output parameters). Declared using `Sub`.
  • Function: A procedure that performs a task and returns a value. Declared using `Function`.

(d) Visual Basic program (Sequence and sum):


Private Sub Command1_Click()
    Dim num As Integer
    Dim cumulativeSum As Long

    num = 50
    cumulativeSum = 0
    List1.Clear

    While num >= 0
        cumulativeSum = cumulativeSum + num
        List1.AddItem num & vbTab & cumulativeSum
        num = num - 5
    Wend
End Sub

(d) Visual Basic program (Commission calculation):


Private Sub Command1_Click()
    Dim agentName As String
    Dim salesVolume As Double
    Dim commissionRate As Double
    Dim commissionAmount As Double

    agentName = txtAgentName.Text
    salesVolume = Val(txtSalesVolume.Text)

    Select Case salesVolume
        Case Is >= 5000
            commissionRate = 0.15
        Case 4000 To 4999
            commissionRate = 0.12

            commissionRate = 0.09
        Case 2000 To 2999

    Picture1.Print "Commission: $" & Format(commissionAmount, "0.00")
End Sub

Question 5

(a) Outline four characteristics of a Visual Programming language.

(b) Write the standard prefix for each of the following Visual Basic programming language data types:

  • (i) Boolean;
  • (ii) Currency;
  • (iii) Byte;
  • (iv) Object.

(c) (i) A student used Textboxes to enter two integers into a Visual Basic program in order to get their sum. When the program was executed the result generated was a concatenation of the two integers. Explain a reason that could have led to this observation.

(ii) Distinguish between static and module level variables as used in Visual Basic programming language.

(d) A programmer designed a Visual Basic interface that would capture details of a student as follows:

  • Registration Number and Name through the use text boxes.
  • Age group in years (15-17,18-20,21-23) through the use of option buttons
  • Gender (Male or Female) through the use of option buttons

When the fields are entered and a command button is clicked, the program displays the Registration Number, Name, Gender and the statement "You are under age" if the age group is \(15-17\) otherwise "You are Mature" through the use of a message box.

  • (i) Sketch the interface;
  • (ii) Write a Visual Basic program to implement the interface.

Answer

(a) Characteristics of Visual Programming language:

  • Graphical User Interface (GUI) based development.
  • Event-driven programming model.
  • Use of visual components and controls.
  • Rapid Application Development (RAD) capabilities.

(b) Standard prefixes:

  • (i) Boolean: bln
  • (ii) Currency: cur
  • (iii) Byte: byt
  • (iv) Object: obj

(c) (i) Reason for concatenation:

The program treated the values entered into the Textboxes as strings rather than numbers. The '+' operator in Visual Basic performs string concatenation if both operands are strings. The student needed to convert the Text property of the Textboxes to a numeric type (e.g., using `Val()`) before performing addition.

(ii) Static vs Module level variables:

  • Static variable: A local variable declared with the `Static` keyword. It retains its value between calls to the procedure in which it is declared. Its scope is local to the procedure.
  • Module level variable: Declared in the Declarations section of a module. Its scope can be the entire module (Private) or the entire project (Public). Its lifetime is the duration the module is loaded.

(d) (i) Interface Sketch:

(A sketch showing a form with labels and textboxes for "Registration Number" and "Name", three option buttons for age groups (15-17, 18-20, 21-23) likely within a frame, two option buttons for gender (Male, Female) likely within another frame, and a command button labelled "Submit" or similar.)

(ii) Visual Basic program:


Private Sub Command1_Click()
    Dim regNo As String, studentName As String, gender As String, ageGroupMsg As String

    regNo = txtRegNo.Text ' Assuming textbox name is txtRegNo
    studentName = txtName.Text ' Assuming textbox name is txtName

    ' Assuming Option buttons for gender are named optMale and optFemale
    If optMale.Value = True Then
        gender = "Male"
    ElseIf optFemale.Value = True Then
        gender = "Female"
    Else
        gender = "Not Specified" ' Or handle error
    End If

    ' Assuming Option buttons for age group are named optAge1, optAge2, optAge3

Question 6

(a) (i) Outline the stage in programing when each of the following errors can be detected.

  • I. syntax errors
  • III. logical errors.

(ii) Explain the error in each of the following Visual Basic statements.

  • I. \(a / x=y / \theta\)
  • II. for ( \(x=0 ; x<10 ; x++\) ) \{ msgbox("Hello word") \}

(b) With the aid of an example, explain a way in which a public procedure can be accessed from a form in Visual Basic programming.

(c) State the Visual Basic control represented by each of the following names:

  • (i) tmrTask
  • (ii) shpTask
  • (iii) fraTask
  • (iv) drvTask.

(d) Every month, a salesperson is paid a fixed amount of Ksh 10,000 . The salesperson is paid an additional commission at the rate of \(5 \%\) of the individual sales greater or equal to Ksh 30.000 or suffers a pay cut of Ksh 1000 if the sales fall below Ksh. 30,000 . The total earning is taxed at a rate of \(15 \%\).

Write a Visual Basic program that allows a user to enter the amount of sales through the use of a text box. The program then computes the commission, the tax deducted and the net pay that the salesperson will receive. The net pay for the month should then be displayed on a label.

Answer

(a) (i) Stage when errors can be detected:

  • I. Syntax errors: Detected during compilation.
  • III. Logical errors: Detected during testing or runtime.

(ii) Errors in Visual Basic statements:

  • I. a / x=y / \theta: Invalid syntax. Division by zero is possible if x or theta is zero. Also, theta is not a valid variable name.
  • II. for ( x=0 ; x<10 ; x++ ) \{ msgbox("Hello word") \}: Invalid syntax. This is C++ syntax, not Visual Basic.

(b) Accessing a public procedure from a form:

Declare the procedure as Public in a module. Then, call the procedure from the form using the module name and procedure name. Example:


'In Module1.bas
Public Sub MyPublicProcedure()
    MsgBox "Hello from Module1!"
End Sub

'In Form1.frm
Private Sub Command1_Click()
    Module1.MyPublicProcedure ' Call the public procedure
End Sub

(c) Visual Basic controls represented by names:

  • (i) tmrTask: Timer control
  • (ii) shpTask: Shape control
  • (iii) fraTask: Frame control
  • (iv) drvTask: DriveListBox control

(d) Visual Basic program:


Private Sub Command1_Click()
    Dim sales As Double, commission As Double, tax As Double, netPay As Double, totalEarning As Double

    sales = Val(Text1.Text)

    If sales >= 30000 Then
        commission = sales * 0.05
    Else
        commission = -1000 ' Pay cut
    End If

    totalEarning = 10000 + commission
    tax = totalEarning * 0.15
    netPay = totalEarning - tax

    Label1.Caption = "Net Pay: Ksh " & netPay
End Sub

(c) Visual Basic program (Prime number check):


Private Sub Command1_Click()
    Dim num As Integer, i As Integer, isPrime As Boolean

    num = Val(InputBox("Enter an integer value:"))

    If num <= 1 Then
        MsgBox "Not a prime number"
        Exit Sub
    End If

    isPrime = True
    For i = 2 To num - 1
        If num Mod i = 0 Then
            isPrime = False
            Exit For
        End If
    Next i

    If isPrime Then
        MsgBox num & " is a prime number"
    Else
        MsgBox num & " is not a prime number"
    End If
End Sub
