Question 1

(a) Explain two ways in which arrays could be used in programming.

(b) Distinguish between a compiler and a linker as used in programming.

(c) With the aid of a diagram in each case, describe the following Pascal control structures:

  • (i) Repeat ...until;
  • (ii) While...do.

(d) Table 1 shows the Tax and NHIF computations used by Jazzy Company Ltd. Use it to answer the question that follow.

Write a Pascal program that would:

  • (i) accept the basic pay of an employee;
  • (ii) compute the income tax, NHIF and net pay;
  • (iii) output the computed results to two decimal places.

Hint : netpay = basicpay - (income tax + NHIF )

Answer

(a) Ways in which arrays could be used in programming:

  • Storing a collection of related data items of the same type.
  • Implementing data structures such as stacks, queues, and hash tables.

(b) Compiler vs Linker:

  • Compiler: Translates source code into object code.
  • Linker: Combines object code with library routines to create an executable file.

(c) Pascal control structures:

  • (i) Repeat ...until: Executes a block of code repeatedly until a condition becomes true.
  • (ii) While...do: Executes a block of code repeatedly as long as a condition is true.

(d) Pascal program:


program JazzyCompany;
uses crt;
var
  basicpay, incometax, nhif, netpay: real;
begin
  clrscr;
  write('Enter basic pay: ');
  readln(basicpay);

  if basicpay <= 1000 then
    incometax := 0
  else if basicpay <= 2000 then
    incometax := basicpay * 0.01
  else if basicpay <= 3000 then
    incometax := basicpay * 0.02
  else
    incometax := basicpay * 0.04;

  if basicpay <= 2500 then
    nhif := basicpay * 0.024
  else
    nhif := basicpay * 0.03;

  netpay := basicpay - (incometax + nhif);

  writeln('Income Tax: ', incometax:0:2);
  writeln('NHIF: ', nhif:0:2);
  writeln('Net Pay: ', netpay:0:2);

  readln;
end.

Question 2

(a) Outline four disadvantages of low level programming languages.

(b) Explain the circumstances under which each of the following statements would be used in a program:

  • (i) break;
  • (ii) continue.

(c) (i) With the aid of an example, describe null pointer as used in programming.

(ii) Write a Pascal program that would copy information from a file named file_- $/$ to a file named file_2.

(d) Distinguish between algorithm and pseudo code as used in programming.

Answer

(a) Disadvantages of low-level programming languages:

  • Difficult to learn and use.
  • Machine-dependent.
  • Time-consuming to write and debug.
  • Require a deep understanding of the hardware.

(b) Circumstances for using statements:

  • (i) break: Used to terminate a loop or switch statement prematurely.
  • (ii) continue: Used to skip the rest of the current iteration of a loop and proceed to the next iteration.

(c) (i) Null pointer:

A null pointer is a pointer that does not point to any valid memory location. It is often used to indicate that a pointer is not currently pointing to anything.

Example:


var
  ptr: ^Integer;
begin
  ptr := nil; // ptr is now a null pointer
end.

(ii) Pascal program:


program CopyFile;
var
  infile, outfile: text;
  ch: char;
begin
  assign(infile, 'file_1');
  reset(infile);
  assign(outfile, 'file_2');
  rewrite(outfile);

  while not eof(infile) do
  begin
    read(infile, ch);
    write(outfile, ch);
  end;

  close(infile);
  close(outfile);
end.

(d) Algorithm vs Pseudo code:

  • Algorithm: A step-by-step procedure for solving a problem.
  • Pseudo code: An informal, high-level description of an algorithm that uses natural language and programming constructs.

Question 5

(a) Outline two advantages of black box testing as used in programming.

\(Y=((x+y) \% z+2) / k-x+z \% x\)

Given that \(x=4, z=x / y, y=2\) and \(k=14\)

Answer

(a) Advantages of black box testing:

  • Can be performed by non-programmers.
  • Tests the functionality of the program from the user's perspective.

(b) Circular-linked-list:

(Since I can't draw a diagram, I will describe it. A circular linked list is a linked list in which the last node points back to the first node, forming a circle.)

(c) (i) Call-by-Value vs Call-by-Reference:

  • Call-by-Value: The value of the argument is copied to the parameter. Changes made to the parameter do not affect the argument.
  • Call-by-Reference: The address of the argument is passed to the parameter. Changes made to the parameter do affect the argument.

(ii) C++ program:


#include 

int main() {
    int scores[5];
    int highestScore = 0;

    std::cout << "Enter the scores of five students:" << std::endl;
    for (int i = 0; i < 5; i++) {
        std::cin >> scores[i];
        if (scores[i] > highestScore) {
            highestScore = scores[i];
        }
    }

    std::cout << "Deviations from the highest score:" << std::endl;
    for (int i = 0; i < 5; i++) {
        std::cout << "Student " << i + 1 << ": " << highestScore - scores[i] << std::endl;
    }

    return 0;
}

(d) Evaluation of C++ statement:

Given \(x=4, z=x / y, y=2\) and \(k=14\)

\(z = 4 / 2 = 2\)

\(Y=((x+y) \% z+2) / k-x+z \% x\)

\(Y=((4+2) \% 2+2) / 14-4+2 \% 4\)

\(Y=((6) \% 2+2) / 14-4+2 \% 4\)

\(Y=(0+2) / 14-4+2\)

\(Y=2 / 14-4+2\)

\(Y=0.142857-4+2\)

\(Y=-1.857143\)

Question 6

(a) Explain each of the following terms as used in programming:

  • (i) test development;
  • (ii) test execution.

(b) Outline two characteristics of stack data types as used in programming.

(c) (i) Explain the most appropriate file access method that an online booking company would use justifying your answer.

(ii) Josses prefer to use functions while developing his programs. Explain three reasons that could have led to his preference.

(d) Distinguish between logical and run time errors as used in programming. (4 marks)

Answer

(a) Terms:

  • (i) Test development: The process of creating test cases, test data, and test procedures to verify that a software product meets its requirements.
  • (ii) Test execution: The process of running the test cases and comparing the actual results with the expected results.

(b) Characteristics of stack data types:

  • LIFO (Last-In, First-Out): The last element added to the stack is the first element removed.
  • Limited access: Elements can only be added or removed from the top of the stack.

(c) (i) File access method for an online booking company:

Random access is the most appropriate file access method. This allows the company to quickly access and update specific records in the file, such as booking information, without having to read through the entire file.

(ii) Reasons for preferring functions:

  • Modularity: Functions allow you to break down a program into smaller, more manageable units.
  • Reusability: Functions can be called from multiple parts of the program, reducing code duplication.
  • Readability: Functions improve code readability by encapsulating complex logic into reusable units.

(d) Logical vs Run time errors:

  • Logical errors: Errors in the program's logic that cause it to produce incorrect results.
  • Run time errors: Errors that occur during the execution of the program, such as division by zero or accessing an invalid memory location.

Question 8

(a) State the function of each of the following file commands as used in Pascal programs:

  • (i) truncate;
  • (ii) reset.

(b) Explain each of the following terms as used in programming:

  • (i) function declaration;
  • (ii) function definition.
  • (iii) type casting

(c) Table 2 represents the data collected by an analyst from 10 respondents within Narvon area. Use it to answer the questions that follow:

(i) Using the binary search technique, write a C++ program that would be used to scarch clement 56 from the list.

(ii) Outline the steps that could be used to sort the data in the array in descending order using the bubble sorting algorithm.

(d) Write \(\mathrm{C}++\) expressions that would be used to represent each of the following:

  • (i) declare marks to hold 90.345.
  • (ii) declare \(pi\) as a constant to hold 3.14.

Answer

(a) Pascal file commands:

  • (i) truncate: Deletes all data from a file, leaving it empty.
  • (ii) reset: Opens an existing file for reading, positioning the file pointer at the beginning of the file.

(b) Terms:

  • (i) Function declaration: Declares the name, return type, and parameters of a function.
  • (ii) Function definition: Provides the actual code that implements the function.
  • (iii) Type casting: Converting a value from one data type to another.

(c) (i) C++ program for binary search:


#include 

int main() {
    int data[] = {12, 32, 7, 95, 56, 21, 12, 13, 13, 90};
    int n = sizeof(data) / sizeof(data[0]);
    int target = 56;
    int left = 0, right = n - 1;
    int mid;
    bool found = false;

    while (left <= right) {
        mid = left + (right - left) / 2;
        if (data[mid] == target) {
            std::cout << "Element found at index " << mid << std::endl;
            found = true;
            break;
        } else if (data[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    if (!found) {
        std::cout << "Element not found" << std::endl;
    }

    return 0;
}

(ii) Steps for bubble sort in descending order:

  • Compare the first two elements of the array. If the first element is smaller than the second element, swap them.
  • Compare the second and third elements. If the second element is smaller than the third element, swap them.
  • Continue this process for all adjacent pairs of elements in the array.
  • Repeat steps 1-3 until no more swaps are needed.

(d) C++ expressions:

  • (i) float marks = 90.345;
  • (ii) const float pi = 3.14;
