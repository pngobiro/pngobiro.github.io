Question 1

(a) Outline the function of each of the following textbox control properties as used in Visual Basic program:

  • (i) AutoSize;
  • (ii) Locked;
  • (iii) MultiLine;
  • (iv) ReadOnly.

(b) Explain three hardware considerations to be made when installing a Visual Basic program.

(c) Explain a circumstance that would necessitate the use of each of the following data types in a Visual Basic program:

  • (i) boolean;
  • (ii) Single;
  • (iii) variant.

(d) Distinguish between Active $X$ Data Objects (ADO) and Open Database Connectivity (ODBC) as used in a Visual Basic program:

Answer

(a) Textbox control properties:

  • (i) AutoSize: Determines whether the textbox automatically adjusts its size to fit the text it contains.
  • (ii) Locked: Determines whether the text in the textbox can be edited by the user.
  • (iii) MultiLine: Determines whether the textbox can display multiple lines of text.
  • (iv) ReadOnly: Determines whether the text in the textbox can be modified by the user.

(b) Hardware considerations:

  • Processor speed: Visual Basic programs may require a certain processor speed to run efficiently.
  • Memory: Visual Basic programs require sufficient memory to load and execute.
  • Disk space: Visual Basic programs require sufficient disk space for installation and data storage.

(c) Data types:

  • (i) Boolean: Used to store true/false values.
  • (ii) Single: Used to store single-precision floating-point numbers.
  • (iii) Variant: Used to store any type of data.

(d) ADO vs ODBC:

  • ADO (ActiveX Data Objects): A high-level, object-oriented interface for accessing data from a variety of sources.
  • ODBC (Open Database Connectivity): A low-level, standard API for accessing databases.

Question 2

(a) State the function of each of the following interfaces in a Visual Basic Program:

  • (i) data view window;
  • (ii) watch window;
  • (iii) project explorer window.

(b) Differentiate between form.hide and unload me statements as used in a Visual Basic program.

(c) Explain a circumstance under which each of the following compile error messages may be displayed during Visual Basic program execution:

  • (i) Expected: end of statement;
  • (ii) Syntax error;
  • (iii) Type mismatch.

(d) Figure 1 shows a Visual Basic program form with ADO control having its connection string and record source properties set to connect to a database.

Answer

(a) Interfaces:

  • (i) Data View Window: Allows you to view and manage database connections, tables, and stored procedures.
  • (ii) Watch Window: Allows you to monitor the values of variables and expressions during debugging.
  • (iii) Project Explorer Window: Displays a hierarchical view of all the files and components in your project.

(b) form.hide vs unload me:

  • form.hide: Hides the form but keeps it loaded in memory.
  • unload me: Unloads the form from memory, releasing the resources it was using.

(c) Compile error messages:

  • (i) Expected: end of statement: Occurs when a statement is incomplete or missing a required terminator.
  • (ii) Syntax error: Occurs when the code contains an invalid syntax.
  • (iii) Type mismatch: Occurs when you try to assign a value of one data type to a variable of another data type.

Question 3

(a) Outline the procedure followed in shell sort algorithm in programming.

(b) Explain each of the following interfaces created in a Visual Basic program:

  • (i) multiple document interface;
  • (ii) tree view interface.

(c) Distinguish between refresh and updaterecord methods of data control in a Visual Basic program.

(d) A company pays commission to sale representatives based on the criteria shown in Table1. Use it to answer the question that follows.

Sales Commission Rate
From 0 To 9999 2%
From 10000 To 19999 5%
From 20000 To 29999 10%
From 30000 and above 15%

Table 1

Write a Visual Basic program that willaccept the sales amount through the use of an input box and compute the commission earned. The program should then output the commission in a message box. Use the select case control structure and attach the code to a form load event.

Answer

(a) Shell sort algorithm:

  • Divide the list into sublists of smaller intervals.
  • Sort each sublist using insertion sort.
  • Reduce the interval between sublists and repeat the sorting process.
  • Continue until the interval is 1, at which point the entire list is sorted using insertion sort.

(b) Interfaces:

  • (i) Multiple Document Interface (MDI): An application that allows you to open multiple documents or windows within a single parent window.
  • (ii) Tree View Interface: A hierarchical display of data, where items are organized in a tree-like structure.

(c) refresh vs updaterecord:

  • refresh: Refreshes the data in the data control from the underlying database.
  • updaterecord: Saves any changes made to the current record in the data control to the underlying database.

(d) Program to compute commission:


Private Sub Form_Load()
    Dim sales As Single, commission As Single
    
    sales = InputBox("Enter sales amount:")
    
    Select Case sales
        Case 0 To 9999
            commission = sales * 0.02
        Case 10000 To 19999
            commission = sales * 0.05
        Case 20000 To 29999
            commission = sales * 0.1
        Case 30000 To Single.MaxValue
            commission = sales * 0.15
        Case Else
            MsgBox "Invalid sales amount."
            Exit Sub
    End Select
    
    MsgBox "Commission: " & commission
End Sub

Question 6

(a) Explain three options that a programmer ean specify using on-error statement to manage errors in Visual Basic programming.

(b) A programmer intends to use ADO control to manage records in the database while creating a Visual Basic program. Outline four types of recordset in ADO that the programmer is likely to use.

(c) State two differences between standard modules and form modules as used in a Visual Basic program.

(d) With the aid of an example, outline the function of each of the following operators as used in Visual Basic program:

  • (i) caret (^)
  • (ii) Mod
  • (iii) backslash (\)
  • (iv) AND

Answer

(a) On-error statement options:

  • On Error GoTo label: Transfers control to a specified label when an error occurs.
  • On Error Resume Next: Continues execution with the statement immediately following the statement that caused the error.
  • On Error GoTo 0: Disables any enabled error handler in the current procedure.

(b) ADO recordset types:

  • Table: Represents a table in the database.
  • Dynaset: A dynamic set of records that can be updated.
  • Snapshot: A static copy of a set of records.
  • Forward-only: A recordset that can only be traversed in a forward direction.

(c) Standard modules vs form modules:

  • Standard modules: Contain global variables and procedures that can be accessed from anywhere in the project.
  • Form modules: Contain code specific to a particular form, including event handlers and control declarations.

(d) Operators:

  • (i) caret (^): Exponentiation operator. Example: 2 ^ 3 = 8
  • (ii) Mod: Modulus operator. Returns the remainder of a division. Example: 7 Mod 2 = 1
  • (iii) backslash (\): Integer division operator. Returns the integer part of a division. Example: 7 \ 2 = 3
  • (iv) AND: Logical AND operator. Returns True if both operands are True. Example: True AND False = False

Answer

(a) Program environments:

  • (i) Design environment: The environment where you create and design the user interface and write the code for your application.
  • (ii) Run-time environment: The environment where the compiled application is executed.

(b) Local vs global variables:

  • Scope: Local variables are only accessible within the procedure in which they are declared, while global variables are accessible from anywhere in the project.
  • Lifetime: Local variables exist only while the procedure in which they are declared is executing, while global variables exist for the lifetime of the application.
  • Declaration: Local variables are declared within a procedure, while global variables are declared in a standard module.
  • Memory allocation: Local variables are allocated memory on the stack, while global variables are allocated memory in the data segment.
