Question 5

(a) Evaluate each of the following Visual Basic program statements:

  • (i) \(1+2 * 4 / 2\)
  • (ii) \((1+2) * 4 / 2\)
  • (iii) \(1+2 *(4 / 2)\)
  • (iv) \((1+(10-(2+2)))\)

(b) (i) Distinguish between a linker and a debugger as used in programming.

(ii) Differentiate between source code and object code.

(c) State the function of each of the following operators as used in Visual Basic program.

  • (i) And
  • (ii) Xor
  • (iii) Not
  • (iv) \&

(d) Distinguish between the floor and ceil functions as used in Visual FoxPro programming language.

Answer

(a) Evaluation of Visual Basic program statements:

  • (i) \(1+2 * 4 / 2 = 1 + 8 / 2 = 1 + 4 = 5\)
  • (ii) \((1+2) * 4 / 2 = 3 * 4 / 2 = 12 / 2 = 6\)
  • (iii) \(1+2 *(4 / 2) = 1 + 2 * 2 = 1 + 4 = 5\)
  • (iv) \((1+(10-(2+2))) = (1+(10-4)) = (1+6) = 7\)

(b) (i) Linker vs Debugger:

  • Linker: Combines object code with library routines to create an executable file.
  • Debugger: A tool used to identify and remove errors from a program.

(ii) Source code vs Object code:

  • Source code: The human-readable code written by the programmer.
  • Object code: The machine-readable code generated by the compiler.

(c) Functions of operators:

  • (i) And: Logical AND operator. Returns True if both operands are True.
  • (ii) Xor: Logical Exclusive OR operator. Returns True if only one of the operands is True.
  • (iii) Not: Logical NOT operator. Returns the opposite of the operand.
  • (iv) &: String concatenation operator.

(d) Floor vs Ceil functions:

  • Floor: Returns the largest integer less than or equal to a given number.
  • Ceil: Returns the smallest integer greater than or equal to a given number.

Question 6

(a) Explain each of the following terms as used in programming:

  • (i) test development;
  • (ii) test execution.

(b) Outline two characteristics of stack data types as used in programming.

(c) (i) Explain the most appropriate file access method that an online booking company would use justifying your answer.

(ii) Josses prefer to use functions while developing his programs. Explain three reasons that could have led to his preference.

(d) Distinguish between logical and run time errors as used in programming. (4 marks)

Answer

(a) Terms:

  • (i) Test development: The process of creating test cases, test data, and test procedures to verify that a software product meets its requirements.
  • (ii) Test execution: The process of running the test cases and comparing the actual results with the expected results.

(b) Characteristics of stack data types:

  • LIFO (Last-In, First-Out): The last element added to the stack is the first element removed.
  • Limited access: Elements can only be added or removed from the top of the stack.

(c) (i) File access method for an online booking company:

Random access is the most appropriate file access method. This allows the company to quickly access and update specific records in the file, such as booking information, without having to read through the entire file.

(ii) Reasons for preferring functions:

  • Modularity: Functions allow you to break down a program into smaller, more manageable units.
  • Reusability: Functions can be called from multiple parts of the program, reducing code duplication.
  • Readability: Functions improve code readability by encapsulating complex logic into reusable units.

(d) Logical vs Run time errors:

  • Logical errors: Errors in the program's logic that cause it to produce incorrect results.
  • Run time errors: Errors that occur during the execution of the program, such as division by zero or accessing an invalid memory location.
