Question 2

(a) (i) Describe the term variable life time as used in Visual Basic program. (2 marks)

(ii) Ali wrote a program and declared each of the following as variable names. Explain, giving a reason, why each could be incorrect.

  • I. 123name;
  • II. Number .one;
  • III. End Sub.

(b) Figure 1 shows the start up window in a Visual Basic programming envirenment. Use it to answer the question that follows.

Explain the function of each of the options labelled (i), (ii) and (iii).

(c) Andrew has developed a system and would like to test the system for the errors. Describe three ways in which he can trap the errors during runtime in a Visual Basic program.

Answer

(a) (i) Variable life time:

Refers to the period during which a variable exists in memory. Local variables exist only while their procedure is executing, while module-level or global variables exist for the duration of the application.

(ii) Incorrect variable names:

  • I. 123name: Incorrect because variable names cannot start with a number.
  • II. Number .one: Incorrect because variable names cannot contain spaces or periods.
  • III. End Sub: Incorrect because it is a reserved keyword in Visual Basic.

(b) Function of options:

  • (i) New Project: Creates a new Visual Basic project based on the selected template (e.g., Standard EXE).
  • (ii) Existing Project: Opens an existing Visual Basic project file.
  • (iii) Recent Project: Opens a recently accessed Visual Basic project file.

(c) Ways to trap errors during runtime:

  • On Error GoTo [Label]: Transfers control to a specific error-handling routine when an error occurs.
  • On Error Resume Next: Ignores the error and continues execution with the next statement.
  • Using Try...Catch...Finally blocks (in VB.NET): Provides structured error handling.

Question 3

(a) State six properties of the label control as used in Visual Basic programming. (3 marks)

(b) Distinguish between a variable and an array as used in programming.

(c) Explain two advantages of using debug object to display an error message as opposed to a msgbox(0 in a Visual basic program .

(d) Given the following numbers \(5,2,4,6,1\). Use insertion sort algorithm to arrange the numbers in ascending order. Show each step of the operation.

Answer

(a) Properties of the label control:

  • Caption
  • Name
  • Font
  • ForeColor
  • BackColor
  • AutoSize

(b) Variable vs Array:

  • Variable: Stores a single data value.
  • Array: Stores a collection of data values of the same type, accessed using an index.

(c) Advantages of using Debug object:

  • Does not interrupt program execution like MsgBox.
  • Output is displayed in the Immediate window, which is useful during development and debugging without disturbing the user interface.

(d) Insertion sort steps (ascending order for 5, 2, 4, 6, 1):

  1. Start with the second element (2). Compare with the first (5). Since 2 < 5, shift 5 right and insert 2: [2, 5, 4, 6, 1]
  2. Consider the third element (4). Compare with 5. Since 4 < 5, shift 5 right. Compare 4 with 2. Since 4 > 2, insert 4: [2, 4, 5, 6, 1]
  3. Consider the fourth element (6). Compare with 5. Since 6 > 5, no shift needed: [2, 4, 5, 6, 1]
  4. Consider the fifth element (1). Compare with 6. Since 1 < 6, shift 6 right. Compare with 5. Since 1 < 5, shift 5 right. Compare with 4. Since 1 < 4, shift 4 right. Compare with 2. Since 1 < 2, shift 2 right. Insert 1: [1, 2, 4, 5, 6]

Sorted list: [1, 2, 4, 5, 6]

Question 4

(a) Distinguish between local window and immediate window as used in a Visual Basic programming language.

(b) Explain a circumstance under which each of the following events of Ado Data Control may be used:

  • (i) WillChangeRecordset;
  • (ii) RecordChangeComplete;
  • (iii) FieldChangeComplete,

(c) Distinguish between a command object and recordset object as used in a Visual Basic program.

(d) Write a Visual Basic program that would display the series \(1,1,2,3,5,8,13,21 \ldots \mathrm{~N}\) in a picture box. The program should prompt the user to input the value of N. Attach the code to a command click event. Use a for loop.

Answer

(a) Local window vs Immediate window:

  • Local window: Displays the values of variables that are currently in scope during debugging.
  • Immediate window: Allows executing code statements and evaluating expressions during debugging.

(b) Ado Data Control events:

  • (i) WillChangeRecordset: Occurs before the recordset changes (e.g., due to filtering or sorting). Useful for saving pending changes or cancelling the operation.
  • (ii) RecordChangeComplete: Occurs after one or more records change. Useful for updating related controls or performing actions based on the changed data.
  • (iii) FieldChangeComplete: Occurs after the value of a field changes. Useful for validating the new value or updating dependent fields.

(c) Command object vs Recordset object:

  • Command object: Represents a command (like an SQL query or stored procedure) to be executed against a data source.
  • Recordset object: Represents the set of records returned by executing a command or opening a table.

(d) Visual Basic program (Fibonacci series):


Private Sub Command1_Click()
    Dim N As Integer
    Dim a As Long, b As Long, c As Long
    Dim i As Integer

    N = Val(InputBox("Enter the value of N:"))

    If N <= 0 Then Exit Sub

    a = 0
    b = 1
    Picture1.Print b ' Print the first term

    For i = 2 To N
        c = a + b
        Picture1.Print c
        a = b
        b = c
    Next i
End Sub

Question 5

(a) Outline two ways in which arguments can be passed to a parameter in a Visual Basic program.

(b) Distinguish between procedure-level and module-level variables as used in a Visual Basic program.

(c) Write a Visual Basic program that would accept the first date and the second date through the use of an Inputbox() and display the number of days between the two dates in on a form. Attach the code to a command click event.

(d) (i) Describe a linear data structure as used in programming.

(ii) With the aid of a diagram in each case, describe three types of data structures other than arrays.

Answer

(a) Ways to pass arguments:

  • By Value (ByVal): A copy of the argument's value is passed. Changes to the parameter inside the procedure do not affect the original argument.
  • By Reference (ByRef): The memory address of the argument is passed. Changes to the parameter inside the procedure affect the original argument.

(b) Procedure-level vs Module-level variables:

  • Procedure-level (Local): Declared within a Sub or Function procedure. Only accessible within that procedure. Lifetime is limited to the execution of the procedure.
  • Module-level: Declared in the Declarations section of a module (form, standard, or class). Accessible by all procedures within that module. Lifetime is the duration the module is loaded.

(c) Visual Basic program (Date difference):


Private Sub Command1_Click()
    Dim date1 As Date, date2 As Date
    Dim daysDiff As Long

    On Error GoTo InvalidDate
    date1 = CDate(InputBox("Enter the first date (e.g., MM/DD/YYYY):"))
    date2 = CDate(InputBox("Enter the second date (e.g., MM/DD/YYYY):"))
    On Error GoTo 0 ' Disable error handler

    daysDiff = DateDiff("d", date1, date2)

    Form1.Print "Number of days between the two dates: " & Abs(daysDiff)
    Exit Sub

InvalidDate:
    MsgBox "Invalid date format entered. Please use MM/DD/YYYY.", vbCritical
End Sub

(d) (i) Linear data structure:

A data structure where elements are arranged sequentially or linearly, such that each element is attached to its previous and next adjacent elements. Examples include arrays, linked lists, stacks, and queues.

(ii) Types of data structures (other than arrays):

  • Linked List: Elements (nodes) are linked using pointers. Each node contains data and a pointer to the next node. (Diagram: Boxes connected by arrows)
  • Stack: A LIFO (Last-In, First-Out) structure. Elements are added (pushed) and removed (popped) from the top. (Diagram: Vertical stack of boxes)
  • Queue: A FIFO (First-In, First-Out) structure. Elements are added (enqueued) at the rear and removed (dequeued) from the front. (Diagram: Horizontal line of boxes with entry/exit points)

Question 6

(a) The following are statements in a Visual Basic program. State the function of each statement.

  • (i) recordset.AddNewFieldList, Values;
  • (ii) myRecSet.Open "select * from myTable", MyConnObj, adopenKeyset.

(b) Distinguish between a cls method and a refresh method as used in a Visual Basic program.

(c) Explain three sources of connection of a Ado Data Control to the database. ( 6 marks)

(d) Write a Visual Basic program that would accept the values of \(a, b\), and \(c\) thivough the fise of text boxes. The program should then solve the quadratic equation;

\(x=\frac{-b \pm \sqrt{b^{2}-4 a c}}{2 a}\)

then displays the results on a picture box. Attach the code to a command click event. Use an if statement.

Answer

(a) Function of statements:

  • (i) recordset.AddNew FieldList, Values: Adds a new record to the recordset with specified field values. (Note: Syntax might be slightly different depending on context, often just recordset.AddNew followed by setting field values and recordset.Update).
  • (ii) myRecSet.Open "select * from myTable", MyConnObj, adOpenKeyset: Opens a recordset named myRecSet by executing the SQL query "select * from myTable" using the connection object MyConnObj. `adOpenKeyset` specifies a keyset-type cursor.

(b) cls method vs refresh method:

  • Cls method: Clears graphics and text printed directly onto a Form or PictureBox at runtime.
  • Refresh method: Forces a complete repaint of a form or control, ensuring it reflects the latest state of its properties. For a Data control, it re-queries the database.

(c) Sources of connection for ADO Data Control:

  • Connection String: A string containing parameters like provider, data source location, user ID, and password needed to connect to the database. Set via the ConnectionString property.
  • ODBC Data Source Name (DSN): A pre-configured name on the system that points to a specific database driver and database. Set via the ConnectionString property (e.g., "DSN=MyDataSourceName").
  • Data Link File (.udl): A file containing connection information, created using the Microsoft Data Link tool. Set via the ConnectionString property (e.g., "File Name=C:\MyConnection.udl").

(d) Visual Basic program (Quadratic equation):


Private Sub Command1_Click()
    Dim a As Double, b As Double, c As Double
    Dim discriminant As Double, x1 As Double, x2 As Double

    a = Val(txtA.Text)
    b = Val(txtB.Text)
    c = Val(txtC.Text)

    If a = 0 Then
        Picture1.Print "This is not a quadratic equation (a cannot be 0)."
        Exit Sub
    End If

    discriminant = b ^ 2 - 4 * a * c

    If discriminant < 0 Then
        Picture1.Print "No real roots (discriminant is negative)."
    ElseIf discriminant = 0 Then
        x1 = -b / (2 * a)
        Picture1.Print "One real root: x = " & x1
    Else ' discriminant > 0
        x1 = (-b + Sqr(discriminant)) / (2 * a)
        x2 = (-b - Sqr(discriminant)) / (2 * a)
        Picture1.Print "Two real roots:"
        Picture1.Print "x1 = " & x1
        Picture1.Print "x2 = " & x2
    End If
End Sub

Question 7

(a) Explain two characteristics of control arrays as used in a Visual Basic program.

(b) (i) Maurice, a module two student, developed an application using a Visual Basic program user defined function. Explain two reasons that could justify the use of these functions.

(ii) Write a Visual Basic program that would initialise two values 20 and 30 to two variables and then swap the values using a function and display the results on a form. Attach the code to a command click event. Use fiunctions.

(c) Describe each of the following types of loops giving the general syntax in each case.

  • (i) while....wend
  • (ii) do... while loop;
  • (iii) do... until loop.

Answer

(a) Characteristics of control arrays:

  • Shared Name: All controls in the array share the same Name property.
  • Index Property: Each control in the array is uniquely identified by its Index property (a number).
  • Shared Event Procedures: Controls in an array typically share the same event procedures, with the Index property used to identify which specific control triggered the event.

(b) (i) Reasons for using user-defined functions:

  • Code Reusability: Functions allow writing a block of code once and calling it multiple times from different parts of the program.
  • Modularity: Breaking down complex tasks into smaller, manageable functions improves code organization and readability.

(ii) Visual Basic program (Swap values):


Private Sub Command1_Click()
    Dim val1 As Integer, val2 As Integer
    val1 = 20
    val2 = 30

    Form1.Print "Before swap: val1 = " & val1 & ", val2 = " & val2
    Call SwapValues(val1, val2)
    Form1.Print "After swap: val1 = " & val1 & ", val2 = " & val2
End Sub

Sub SwapValues(ByRef num1 As Integer, ByRef num2 As Integer)
    Dim temp As Integer
    temp = num1
    num1 = num2
    num2 = temp
End Sub

(c) Types of loops:

  • (i) While...Wend: Executes a block of statements as long as a condition is True. (Less common in modern VB, Do While...Loop is preferred).
    Syntax: While condition ... Wend
  • (ii) Do...While Loop: Executes a block of statements as long as a condition is True. Condition can be checked at the beginning (Do While condition ... Loop) or end (Do ... Loop While condition).
  • (iii) Do...Until Loop: Executes a block of statements until a condition becomes True. Condition can be checked at the beginning (Do Until condition ... Loop) or end (Do ... Loop Until condition).

(d) Visual Basic program (Display 2x2 array - Note: Accepts 4 values for a 2x2 array):


Private Sub Command1_Click()
    Dim myArray(1, 1) As String ' Declare a 2x2 array (0-based index)
    Dim i As Integer, j As Integer
    Dim displayStr As String

    ' Accept 4 values for the 2x2 array
    For i = 0 To 1
        For j = 0 To 1
            myArray(i, j) = InputBox("Enter value for position (" & i & ", " & j & "):")
        Next j
    Next i

    ' Display the values in the picture box
    Picture1.Cls ' Clear the picture box
    For i = 0 To 1
        displayStr = "" ' Reset display string for each row
        For j = 0 To 1
            displayStr = displayStr & "Pos(" & i & "," & j & "):" & myArray(i, j) & vbTab
        Next j
        Picture1.Print displayStr
    Next i
End Sub
