Question 1

(a) (i) Outline two reasons for using Visual Basic programming language when developing a Hospital Management System.

(ii) Describe event driven process as used in Visual programming.

(b) Outline the function of each of the following Multiple Document Interface (MDI) in a Visual Programming Language:

  • (i) menu bar
  • (ii) toolbox
  • (iii) code form.

(c) With the aid of an example, distinguish between setting properties at design time and setting the properties at run time.

(d) Figure 1 shows a Visual Basic application used to compute multiplication and division of numbers.

Write a Visual Basic program that uses a function for each computation. The program should display the phrase 'illegal division operation' on a message box when the denominator is zero.

Answer

(a) (i) Reasons for using Visual Basic for Hospital Management System:

  • Rapid Application Development (RAD): Visual Basic allows for quick creation of user interfaces and connecting them to code.
  • Ease of Use: Visual Basic is relatively easy to learn and use, making it suitable for developers with varying levels of experience.

(ii) Event-driven process:

Event-driven programming is a programming paradigm where the flow of the program is determined by events such as user actions (e.g., button clicks, mouse movements), sensor outputs, or messages from other programs or threads. The program waits for events and then executes the appropriate event handler.

(b) Function of MDI elements:

  • (i) Menu bar: Provides a list of commands and options for the application.
  • (ii) Toolbox: Contains a collection of controls that can be added to forms.
  • (iii) Code form: The window where the Visual Basic code is written and edited.

(c) Setting properties at design time vs run time:

  • Design time: Properties are set in the Properties window of the Visual Basic IDE before the program is run. Example: Setting the Caption property of a Label control to "Hello".
  • Run time: Properties are set using code while the program is running. Example: Label1.Caption = "Hello"

(d) Visual Basic program:


Private Sub Command1_Click()
    Dim num1 As Double, num2 As Double, result As Double

    num1 = Val(Text1.Text)
    num2 = Val(Text2.Text)

    If num2 = 0 Then
        MsgBox "Illegal division operation", vbCritical
    Else
        result = Divide(num1, num2)
        Picture1.Print "Division result: " & result
    End If

    result = Multiply(num1, num2)
    Picture1.Print "Multiplication result: " & result
End Sub

Function Multiply(a As Double, b As Double) As Double
    Multiply = a * b
End Function

Function Divide(a As Double, b As Double) As Double
    Divide = a / b
End Function

Question 2

(a) With the aid of an example, explain the function of the underscore symbol in Visual Basic programming

(b) Convert the following statement into its equivalent in Visual Basic programming language.

If any of \(x\) and \(y\) is at least 100 and their product is at most 1000, store their product into \(z\). Otherwise store 1000 into \(z\).

(c) Interpret the following Visual Basic code.


Private Sub Button1_Click()
Dim x As Integer =6
Dim y As Integer =8
Label1.caption =x & y
temp =x
x=y
y= temp
Label2. caption =x & y
End Sub

(d) Write a Visual Basic program that assigns numeric values; \(80,60,70,30,40\) to an array. The program should then computes the average of the values and display the values and average on a list box.

Answer

(a) Function of the underscore symbol:

The underscore symbol (_) is used as a line-continuation character in Visual Basic. It allows you to break a long line of code into multiple lines for readability.

Example:


Dim longVariableName As String = "This is a very long string " _
                                 & "that needs to be broken into " _
                                 & "multiple lines for readability."

(b) Equivalent Visual Basic statement:


Dim x As Integer, y As Integer, z As Integer

If (x >= 100 Or y >= 100) And (x * y <= 1000) Then
    z = x * y
Else
    z = 1000
End If

(c) Interpretation of the Visual Basic code:

The code initializes two integer variables, x and y, to 6 and 8 respectively. It then concatenates the values of x and y and displays the result in Label1. Next, it swaps the values of x and y using a temporary variable. Finally, it concatenates the new values of x and y and displays the result in Label2.

Output:

  • Label1.Caption: 68
  • Label2.Caption: 86

(d) Visual Basic program:


Private Sub Command1_Click()
    Dim scores(4) As Integer ' Array to hold 5 scores
    Dim i As Integer, total As Integer, average As Single

    ' Assign values to the array
    scores(0) = 80
    scores(1) = 60
    scores(2) = 70
    scores(3) = 30
    scores(4) = 40

    ' Calculate the total
    total = 0
    For i = 0 To 4
        total = total + scores(i)
    Next i

    ' Calculate the average
    average = total / 5

    ' Display the values and average in a list box
    List1.Clear
    For i = 0 To 4
        List1.AddItem "Score " & (i + 1) & ": " & scores(i)
    Next i
    List1.AddItem "Average: " & average
End Sub

Question 3

(a) (i) List four objects that may be accessed using project explore window of a Visual Basic program during the development an application.

(ii) Outline two events and two methods associated with a picture box control.

(b) State two differences between sub procedures and event procedures as used in Visual programming

(c) With the aid of an example in each case, distinguish between fwo-dimensional array and parallel array as used in programming.

(d) Write a visual basic program that prompts a user to enter a name of an animal through the use of input box function. The program should then assign the name entered is then assigned to a dynamic array. A user prompted to select Ok or cancel from a message box when required to add more animals or terminate the program. Attach the code to a command button.

Answer

(a) (i) Objects accessed using Project Explorer window:

  • Forms
  • Modules
  • Class Modules
  • User Controls

(ii) Events and methods associated with a picture box control:

  • Events: Click, MouseMove
  • Methods: Print, Cls

(b) Differences between sub procedures and event procedures:

  • Sub procedures are general-purpose code blocks that can be called from anywhere in the program, while event procedures are specifically designed to respond to events triggered by user actions or system events.
  • Event procedures have a predefined naming convention (e.g., Command1_Click), while sub procedures can have any valid name.

(c) Two-dimensional array vs Parallel array:

  • Two-dimensional array: A single array with rows and columns, where all elements are of the same data type. Example: Dim matrix(2, 2) As Integer
  • Parallel arrays: Multiple arrays of the same size, where elements at the same index are related. Example: Dim names(2) As String, ages(2) As Integer (names(0) and ages(0) are related)

(d) Visual Basic program:


Private Sub Command1_Click()
    Dim animals() As String
    Dim animalName As String
    Dim i As Integer
    Dim addMore As Integer

    i = 0
    Do
        animalName = InputBox("Enter the name of an animal:")
        ReDim Preserve animals(i)
        animals(i) = animalName
        i = i + 1

        addMore = MsgBox("Add more animals?", vbOKCancel)
    Loop While addMore = vbOK

    ' Display the animals (optional)
    Dim output As String
    For i = 0 To UBound(animals)
        output = output & animals(i) & vbCrLf
    Next i
    MsgBox output
End Sub

Question 4

(a) Outline each of the following events of a command button as used in Visual programming.

  • (i) MouseHover
  • (ii) KeyPress
  • (iii) MouseDown.

(b) Outline the steps to be followed when creating a report using a data environment given that a database is already created.

(c) (i) Describe the shell sort algorithm used in programming to sort elements as used in programming.

(ii) Distinguish between Visual programming and object-oriented programming.

(d) The formula \(F=\frac{9}{5} C+32\) is used to convert temperature value from Celsius to Fahrenheit where F is temperature in Fahrenheit and C is temperature in Celsius.

Write a Visual Basic program that prompts a user to enter temperature in Celsius using an input function. The program should use a function to convert the temperature to its equivalent in Fahrenheit and display the output on a label.

Answer

(a) Events of a command button:

  • (i) MouseHover: Occurs when the mouse pointer hovers over the command button.
  • (ii) KeyPress: Occurs when a key is pressed while the command button has focus.
  • (iii) MouseDown: Occurs when a mouse button is pressed down while the mouse pointer is over the command button.

(b) Steps to create a report using Data Environment:

  • Add a DataEnvironment object to the project.
  • Create a Connection object to connect to the database.
  • Create a Command object to define the data source for the report (e.g., an SQL query).
  • Add a DataReport object to the project.
  • Set the DataSource and DataMember properties of the DataReport to the Command object.
  • Add controls (e.g., labels, text boxes) to the DataReport and bind them to the fields in the data source.
  • Set the properties of the controls to format the report.

(c) (i) Shell sort algorithm:

Shellsort is a generalization of insertion sort that allows exchange of items that are far apart. The idea is to arrange the list of elements so that, starting anywhere, considering every hth element gives a sorted list. Such a list is said to be h-sorted. It can be implemented with the following steps:

  • Choose an increment sequence.
  • For each increment h in the sequence, h-sort the list.

(ii) Visual programming vs Object-oriented programming:

  • Visual programming: A programming paradigm that uses visual elements (e.g., drag-and-drop interfaces) to create programs.
  • Object-oriented programming: A programming paradigm that organizes software design around data, or objects, rather than functions and logic.

(d) Visual Basic program (Celsius to Fahrenheit):


Private Sub Command1_Click()
    Dim celsius As Single, fahrenheit As Single

    celsius = Val(InputBox("Enter temperature in Celsius:"))

    fahrenheit = ConvertToFahrenheit(celsius)

    Label1.Caption = "Temperature in Fahrenheit: " & fahrenheit
End Sub

Function ConvertToFahrenheit(c As Single) As Single
    ConvertToFahrenheit = (9 / 5) * c + 32
End Function

Question 5

(a) In a Visual Basic program, the variables x and y stores values 10 and 4 respectively. State the output generated from each of the following logical statements when executed:

  • (i) \((x+y)<>15\) And \(y>3\)
  • (ii) \((x+y)<>15\) or \(y>3\)
  • (iii) \(x>=10\) Xor \(y<>0\)
  • (iv) \(\operatorname{Not}(x+y)<>15\)

(b) Distinguish between listbox and FlexGrid controls as used in a Visual Basic program.

(c) Describe each of the following sections of a report design as used in a Visual Basic program.

  • (i) Detail
  • (ii) Page Footer
  • (iii) Report Footer.

(d) Table 1 shows the points and their corresponding grades that can be attained in an examination.

Write a Visual Basic program that allows a user to enter an integer value point through the use of a text box. The program should then display appropriate grade on a label. Use select case control structure and attach the code to a command button.

Answer

(a) Output of logical statements:

  • (i) \((x+y)<>15\) And \(y>3\): (10+4)<>15 And 4>3 => True And True => True
  • (ii) \((x+y)<>15\) or \(y>3\): (10+4)<>15 Or 4>3 => True Or True => True
  • (iii) \(x>=10\) Xor \(y<>0\): 10>=10 Xor 4<>0 => True Xor True => False
  • (iv) \(\operatorname{Not}(x+y)<>15\): Not(10+4)<>15 => Not(True) => False

(b) Listbox vs FlexGrid controls:

  • Listbox: Displays a list of items from which the user can select one or more items.
  • FlexGrid: Displays data in a tabular format, similar to a spreadsheet.

(c) Sections of a report design:

  • (i) Detail: Contains the main body of the report, which is repeated for each record in the data source.
  • (ii) Page Footer: Appears at the bottom of each page of the report.
  • (iii) Report Footer: Appears at the end of the report, typically used for grand totals or summary information.

(d) Visual Basic program:


Private Sub Command1_Click()
    Dim points As Integer, grade As String

    points = Val(Text1.Text)

    Select Case points
        Case Is <= 0
            grade = "Invalid value"
        Case 1, 2
            grade = "Distinction"
        Case 3, 4
            grade = "Credit"
        Case 5, 6
            grade = "Pass"
        Case 7, 8
            grade = "Fail"
        Case Is > 8
            grade = "Invalid value"
    End Select

    Label1.Caption = "Grade: " & grade
End Sub

Question 6

(a) (i) Outline the stage in programing when each of the following errors can be detected.

  • I. syntax errors
  • III. logical errors.

(ii) Explain the error in each of the following Visual Basic statements.

  • I. \(a / x=y / \theta\)
  • II. for ( \(x=0 ; x<10 ; x++\) ) \{ msgbox("Hello word") \}

(b) With the aid of an example, explain a way in which a public procedure can be accessed from a form in Visual Basic programming.

(c) State the Visual Basic control represented by each of the following names:

  • (i) tmrTask
  • (ii) shpTask
  • (iii) fraTask
  • (iv) drvTask.

(d) Every month, a salesperson is paid a fixed amount of Ksh 10,000 . The salesperson is paid an additional commission at the rate of \(5 \%\) of the individual sales greater or equal to Ksh 30.000 or suffers a pay cut of Ksh 1000 if the sales fall below Ksh. 30,000 . The total earning is taxed at a rate of \(15 \%\).

Write a Visual Basic program that allows a user to enter the amount of sales through the use of a text box. The program then computes the commission, the tax deducted and the net pay that the salesperson will receive. The net pay for the month should then be displayed on a label.

Answer

(a) (i) Stage when errors can be detected:

  • I. Syntax errors: Detected during compilation.
  • III. Logical errors: Detected during testing or runtime.

(ii) Errors in Visual Basic statements:

  • I. a / x=y / \theta: Invalid syntax. Division by zero is possible if x or theta is zero. Also, theta is not a valid variable name.
  • II. for ( x=0 ; x<10 ; x++ ) \{ msgbox("Hello word") \}: Invalid syntax. This is C++ syntax, not Visual Basic.

(b) Accessing a public procedure from a form:

Declare the procedure as Public in a module. Then, call the procedure from the form using the module name and procedure name. Example:


'In Module1.bas
Public Sub MyPublicProcedure()
    MsgBox "Hello from Module1!"
End Sub

'In Form1.frm
Private Sub Command1_Click()
    Module1.MyPublicProcedure ' Call the public procedure
End Sub

(c) Visual Basic controls represented by names:

  • (i) tmrTask: Timer control
  • (ii) shpTask: Shape control
  • (iii) fraTask: Frame control
  • (iv) drvTask: DriveListBox control

(d) Visual Basic program:


Private Sub Command1_Click()
    Dim sales As Double, commission As Double, tax As Double, netPay As Double, totalEarning As Double

    sales = Val(Text1.Text)

    If sales >= 30000 Then
        commission = sales * 0.05
    Else
        commission = -1000 ' Pay cut
    End If

    totalEarning = 10000 + commission
    tax = totalEarning * 0.15
    netPay = totalEarning - tax

    Label1.Caption = "Net Pay: Ksh " & netPay
End Sub

(c) Visual Basic program (Prime number check):


Private Sub Command1_Click()
    Dim num As Integer, i As Integer, isPrime As Boolean

    num = Val(InputBox("Enter an integer value:"))

    If num <= 1 Then
        MsgBox "Not a prime number"
        Exit Sub
    End If

    isPrime = True
    For i = 2 To num - 1
        If num Mod i = 0 Then
            isPrime = False
            Exit For
        End If
    Next i

    If isPrime Then
        MsgBox num & " is a prime number"
    Else
        MsgBox num & " is not a prime number"
    End If
End Sub
