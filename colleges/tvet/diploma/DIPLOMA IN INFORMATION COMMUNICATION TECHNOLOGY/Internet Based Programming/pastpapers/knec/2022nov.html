    3. (a) Distinguish between a scheme and a host as used in Uniform Resource locator (URL) structure. (4 marks)

    Answer:

    [Sample Answer]

    • Scheme: The scheme is the first part of a URL and specifies the protocol used to access the resource. It indicates how the browser should communicate with the server. Common schemes include 'http' (Hypertext Transfer Protocol) for standard web pages, 'https' (HTTP Secure) for secure web pages, and 'ftp' (File Transfer Protocol) for file transfer. Example: In 'https://www.example.com', 'https' is the scheme.
    • Host: The host, also known as the domain name, identifies the server where the resource is located. It typically follows the scheme and is the address of the web server. It's the part that a user usually types into the browser's address bar to reach a website. Example: In 'https://www.example.com', 'www.example.com' is the host.

    7. (a) Outline four characteristics of Get request method as used in HTML forms. (4 marks)

    Answer:

    [Sample Answer]

    1. Data Appended to URL: In a GET request, form data is appended to the URL in the query string. This means the data is visible in the browser's address bar.
    2. Limited Data Size: GET requests have limitations on the amount of data that can be sent, as URLs have a maximum length. This makes GET unsuitable for sending large amounts of form data.
    3. Bookmarkable and Shareable: Because the data is part of the URL, GET requests are bookmarkable and shareable. The same request can be repeated by simply accessing the bookmarked or shared URL.
    4. Non-Sensitive Data: GET requests are generally used for non-sensitive data because the data is visible in the URL and may be stored in browser history and server logs. It is not recommended for transmitting sensitive information like passwords or personal details.

    7. (b) The load time is the time it takes to download and display the entire content of a web page. Explain three considerations that a web developer should consider to improve the load time. (6 marks)

    Answer:

    [Sample Answer]

    1. Optimize Images: Large image files significantly increase page load time. Developers should optimize images by:
      • Compression: Use image compression techniques to reduce file size without significant loss of quality.
      • Appropriate Format: Choose the right image format (e.g., WebP, JPEG, PNG) based on the image type and requirements.
      • Responsive Images: Serve appropriately sized images for different devices and screen sizes using responsive image techniques ( or srcset attribute).
    2. Minimize HTTP Requests: Each resource (images, stylesheets, scripts) on a webpage requires an HTTP request. Reducing the number of requests can significantly improve load time. Strategies include:
      • CSS Sprites: Combine multiple small images into a single image sprite and use CSS to display individual parts.
      • Combine Files: Concatenate CSS and JavaScript files to reduce the number of files to download.
      • Inline Critical Resources: Inline critical CSS and JavaScript directly into the HTML to avoid extra requests for initial rendering.
    3. Leverage Browser Caching: Browser caching allows browsers to store static resources locally and reuse them on subsequent visits, reducing server load and improving load time. Developers should:
      • Set Proper Cache Headers: Configure server-side cache headers (e.g., Cache-Control, Expires) to instruct browsers on how long to cache resources.
      • Use a Content Delivery Network (CDN): CDNs distribute website content across multiple servers geographically closer to users, improving content delivery speed and leveraging browser caching effectively.

    8. (c) Explain three methods that can be used to convert string variables to numbers in a JavaScript program (6 marks)

    Answer:

    [Sample Answer]

    1. parseInt(): The 'parseInt()' function parses a string and returns an integer. It converts the string to an integer up to the point where it encounters a character that is not a digit. It can also take a second argument to specify the radix (base) of the number. Example: 'parseInt("123")' returns 123, 'parseInt("123.45")' returns 123, 'parseInt("10", 2)' returns 2 (binary to decimal).
    2. parseFloat(): The 'parseFloat()' function parses a string and returns a floating-point number. It converts the string to a floating-point number up to the point where it encounters a character that is not a digit or a decimal point. Example: 'parseFloat("123.45")' returns 123.45, 'parseFloat("123")' returns 123, 'parseFloat("3.14e2")' returns 314.
    3. Number(): The 'Number()' constructor attempts to convert a string to a number. It can handle both integers and floating-point numbers. If the string cannot be converted to a number, it returns 'NaN' (Not-a-Number). Example: 'Number("123")' returns 123, 'Number("123.45")' returns 123.45, 'Number("abc")' returns NaN.
