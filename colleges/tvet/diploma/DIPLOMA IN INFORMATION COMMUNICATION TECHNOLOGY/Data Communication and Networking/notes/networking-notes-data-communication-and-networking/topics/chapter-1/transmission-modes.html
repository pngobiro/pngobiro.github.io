Data Transmission Modes

Communication between two devices can be simplex, half-duplex, or full-duplex. Each mode determines how data flows between the communicating devices.

📡 Types of Transmission Modes

Simplex Mode

In simplex mode, the communication is unidirectional, as on a one-way street. Only one of the two devices on a link can transmit; the other can only receive.

  • 💻
    Examples: Keyboards and traditional monitors are examples of simplex devices. The keyboard can only introduce input; the monitor can only accept output.
  • ⚡
    Channel Usage: The simplex mode can use the entire capacity of the channel to send data in one direction.

Half-Duplex Mode

In half-duplex mode, each station can both transmit and receive, but not at the same time. When one device is sending, the other can only receive, and vice versa.

  • 📻
    Examples: Walkie-talkies and CB (citizens band) radios are both half-duplex systems.
  • 🔄
    Channel Usage: The entire capacity of a channel is taken over by whichever of the two devices is transmitting at the time.
  • 📊
    Application: Used in cases where there is no need for communication in both directions at the same time.

Full-Duplex Mode

In full-duplex mode, both stations can transmit and receive simultaneously. The full-duplex mode is like a two-way street with traffic flowing in both directions at the same time.

  • 📱
    Example: The telephone network is a common example of full-duplex communication. When two people are communicating by telephone, both can talk and listen at the same time.
  • ↔️
    Channel Usage: Signals going in one direction share the capacity of the link with signals going in the other direction.
  • ⚖️
    Capacity Division: The capacity of the channel must be divided between the two directions.
📝

Key Points to Remember

  • 🔑

    Simplex mode allows communication in only one direction with no possibility for response.

  • 🔑

    Half-duplex mode allows two-way communication but only one direction at a time.

  • 🔑

    Full-duplex mode enables simultaneous two-way communication but requires channel capacity to be shared.
