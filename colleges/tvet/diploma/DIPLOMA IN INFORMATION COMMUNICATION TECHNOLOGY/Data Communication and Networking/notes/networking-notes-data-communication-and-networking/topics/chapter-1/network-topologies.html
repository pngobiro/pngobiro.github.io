Physical Network Topologies

The term physical topology refers to the way in which a network is laid out physically. One or more devices connect to a link; two or more links form a topology. The topology of a network is the geometric representation of the relationship of all the links and linking devices (usually called nodes) to one another.

There are four basic topologies possible: mesh, star, bus, and ring.

üï∏Ô∏è Mesh Topology

In a mesh topology, every device has a dedicated point-to-point link to every other device. The term dedicated means that the link carries traffic only between the two devices it connects.

üìê Mathematical Note

To find the number of physical links in a fully connected mesh network with n nodes:

  • Number of links = n(n-1)/2 for duplex-mode links
  • Each device must have n-1 input/output ports

Advantages

  • ‚úÖ
    Traffic Isolation: Each connection carries its own data load, eliminating traffic problems.
  • ‚úÖ
    Robustness: If one link becomes unusable, it does not incapacitate the entire system.
  • ‚úÖ
    Privacy and Security: Messages travel along dedicated lines, ensuring only intended recipients can access them.
  • ‚úÖ
    Easy Fault Identification: Traffic can be routed to avoid problem areas, making fault isolation straightforward.

Disadvantages

  • ‚ùå
    Excessive Cabling: Requires every device to be connected to every other device.
  • ‚ùå
    Installation Complexity: Installation and reconnection are difficult.
  • ‚ùå
    Space Limitations: The bulk of wiring may exceed available space.
  • ‚ùå
    Cost: Hardware requirements (I/O ports and cables) can be prohibitively expensive.

‚≠ê Star Topology

In a star topology, each device has a dedicated point-to-point link only to a central controller, usually called a hub. The devices are not directly linked to one another.

‚ÑπÔ∏è Important Note

Unlike a mesh topology, a star topology does not allow direct traffic between devices. The controller acts as an exchange: If one device wants to send data to another, it sends the data to the controller, which then relays the data to the other connected device.

Advantages

  • ‚úÖ
    Cost-Effective: Less expensive than mesh topology as each device needs only one link and one I/O port.
  • ‚úÖ
    Easy Management: Easy to install and reconfigure.
  • ‚úÖ
    Robustness: If one link fails, only that link is affected while all other links remain active.

Disadvantages

  • ‚ùå
    Single Point of Failure: Dependency on the hub - if it fails, the whole system is down.
  • ‚ùå
    Cable Requirements: May require more cable length than some other topologies.

üìä Bus Topology

A bus topology is multipoint. One long cable acts as a backbone to link all the devices in a network. Nodes are connected to the bus cable by drop lines and taps.

‚ö†Ô∏è Signal Consideration

As a signal travels along the backbone, some of its energy is transformed into heat. Therefore, it becomes weaker as it travels farther. This limits both the number of taps a bus can support and the distance between those taps.

Advantages

  • ‚úÖ
    Easy Installation: Backbone cable can be laid along the most efficient path.
  • ‚úÖ
    Less Cable Required: Uses less cable than mesh or star topologies.

Disadvantages

  • ‚ùå
    Difficult Modification: Hard to add new devices and reconfigure.
  • ‚ùå
    Signal Degradation: Signal reflection at taps can cause quality issues.
  • ‚ùå
    Single Point of Failure: A break in the bus cable stops all transmission.

‚≠ï Ring Topology

In a ring topology, each device has a dedicated point-to-point connection with only the two devices on either side of it. A signal is passed along the ring in one direction, from device to device, until it reaches its destination.

üí° Signal Processing

Each device in the ring incorporates a repeater. When a device receives a signal intended for another device, its repeater regenerates the bits and passes them along.

Advantages

  • ‚úÖ
    Simple Installation: Easy to install and reconfigure as each device connects only to its immediate neighbors.
  • ‚úÖ
    Simple Fault Isolation: Each device acts as a repeater, making it easy to identify transmission problems.

Disadvantages

  • ‚ùå
    Unidirectional Traffic: Traffic can only flow in one direction.
  • ‚ùå
    Single Point of Failure: A break in the ring can disable the entire network.

Key Points to Remember

  • üîë

    Mesh topology provides the most reliable but most expensive connection method.

  • üîë

    Star topology is cost-effective but depends entirely on the central hub.

  • üîë

    Bus topology is easy to install but difficult to modify and maintain.

  • üîë

    Ring topology provides simple installation but is vulnerable to single-point failures.
