Question 1

(a) (i) Outline three advantages of hybrid topology networks.

(ii) Distinguish between hubs and bridges as used in networking.

(b) (i) Outline three factors that should be considered when selecting a transmission media, other than cost.

(ii) Network administrators use several documents when troubleshooting a network. Describe two documents commonly used.

(c) Network design is a very important phase in networking. Explain three objectives to be attained by the phase.

Answer

(a) (i) Advantages of hybrid topology networks:

A hybrid topology combines two or more different network topologies to create a more efficient network structure. Its advantages include:

  • Flexibility in network design and expansion:
    • Can combine benefits of different topologies (e.g., star and bus)
    • Easier to modify and expand network sections independently
    • Allows customization based on specific departmental needs
  • Better fault tolerance and reliability:
    • If one segment fails, other segments can continue operating
    • Multiple paths for data transmission
    • Redundancy in critical network sections
  • Cost-effective combination of different topologies:
    • Can use expensive technologies only where needed
    • Optimize cost vs. performance for each network segment
    • Better resource utilization

(ii) Distinction between hubs and bridges:

Feature Hubs Bridges
OSI Layer Physical Layer (Layer 1) Data Link Layer (Layer 2)
Operation Simply repeats signals to all ports Forwards frames based on MAC addresses
Intelligence No intelligence in signal handling Can learn and filter traffic
Network Segmentation No segmentation (single collision domain) Creates separate collision domains

(b) (i) Factors for selecting transmission media:

When selecting transmission media, consider these key factors:

  • Bandwidth capacity and speed requirements:
    • Data transfer rates needed for applications
    • Future bandwidth requirements
    • Support for high-speed protocols
  • Maximum distance requirements:
    • Signal degradation over distance
    • Need for repeaters or amplifiers
    • Building or campus layout considerations
  • Environmental factors and interference:
    • Electromagnetic interference (EMI) sources
    • Temperature and humidity effects
    • Physical protection requirements

(ii) Network troubleshooting documents:

Two essential documents for network troubleshooting are:

  • Network topology diagrams:
    • Shows physical connections between devices
    • Illustrates logical network structure
    • Includes details like IP addressing schemes
    • Helps identify potential points of failure
  • Network configuration documentation:
    • Contains device configurations and settings
    • Records IP address assignments and VLANs
    • Lists security policies and access controls
    • Documents software versions and updates

(c) Network design objectives:

A well-planned network design should achieve these key objectives:

  • Scalability for future growth:
    • Ability to add new users and devices
    • Support for new applications and services
    • Flexible architecture for expansion
  • High availability and reliability:
    • Redundant components and paths
    • Fault-tolerant configurations
    • Quick recovery from failures
  • Security and data protection:
    • Access control mechanisms
    • Data encryption capabilities
    • Security policy implementation

Question 2

(a) Explain each of the following terms as used in the Internet:

(i) cookies;

(ii) bookmark.

(b) John was required to install communication software for his company. Outline four factors that he should consider before installing the software, other than cost.

(c) Use of passwords promotes network security. Outline four precautions that should be considered when assigning passwords.

(d) (i) With the aid of examples, describe two categories of wireless networks that could be found in organizations.

(ii) Explain two characteristics of a local area network.

Answer

(a) Terms used in the Internet:

(i) Cookies:

  • Definition: Small text files stored on a user's computer by websites
  • Purpose:
    • Remember user preferences and settings
    • Store login information and session data
    • Track user behavior for analytics
    • Enable shopping cart functionality
  • Types:
    • Session cookies (temporary, deleted when browser closes)
    • Persistent cookies (remain until expiration date)
    • First-party cookies (from visited site)
    • Third-party cookies (from other domains)

(ii) Bookmark:

  • Definition: Stored reference to a specific webpage for quick future access
  • Features:
    • Customizable names for easy identification
    • Hierarchical folder organization
    • Cross-device synchronization
    • Import/export capabilities
  • Benefits:
    • Quick access to frequently visited sites
    • Efficient organization of web resources
    • Time-saving navigation
    • Offline access to saved URLs

(b) Factors to consider before installing communication software:

  • System requirements and compatibility:
    • Hardware specifications (CPU, memory, storage)
    • Operating system version and compatibility
    • Network infrastructure requirements
    • Existing software compatibility
  • Security features and vulnerabilities:
    • Data encryption capabilities
    • Authentication mechanisms
    • Known security issues and patches
    • Compliance with security standards
  • User training requirements:
    • Learning curve complexity
    • Available documentation quality
    • Training resources and support
    • User acceptance considerations
  • Integration with existing systems:
    • API availability and compatibility
    • Data migration capabilities
    • Existing workflow integration
    • Legacy system support

(c) Password assignment precautions:

  • Minimum length and complexity requirements:
    • Minimum 8-12 characters
    • Mix of uppercase and lowercase letters
    • Numbers and special characters
    • Avoid dictionary words and common patterns
  • Regular password changes:
    • Enforce periodic updates (60-90 days)
    • Immediate change if compromise suspected
    • Monitor for unauthorized changes
    • Password change notifications
  • No reuse of previous passwords:
    • Maintain password history
    • Prevent last 5-10 password reuse
    • Check for similar patterns
    • Enforce unique password policy
  • Secure storage of password data:
    • Use strong hashing algorithms
    • Implement salting techniques
    • Encrypted password databases
    • Regular security audits

(d) Categories of wireless networks with examples:

(i) Wireless network categories:

  • WLAN (Wireless Local Area Network):
    • Coverage: Building/campus (30-50m indoors)
    • Speed: Up to 1 Gbps (802.11ac)
    • Common uses: Office networks, public hotspots
    • Example: Office Wi-Fi network connecting computers, printers
  • WPAN (Wireless Personal Area Network):
    • Coverage: Personal space (1-10m)
    • Speed: 1-3 Mbps (Bluetooth)
    • Common uses: Device-to-device connections
    • Example: Connecting phone to wireless headphones

(ii) Characteristics of a local area network:

  • Limited geographical coverage:
    • Typically within a single building/campus
    • Range usually up to few kilometers
    • Single administrative domain
    • Direct connection between devices
  • High data transfer rates:
    • Modern LANs: 1/10/100 Gbps
    • Low latency communication
    • Support for real-time applications
    • High bandwidth availability

Question 3

(a) (i) Outline two advantages of point to point protocol (PPP).

(ii) Distinguish between broadcast and host IP addresses.

(b) (i) With the aid of a diagram, represent the data 10110001 in the form of a digital encoded wave form.

(ii) Convert the digital encoded signal in (i) into on-return-to-zero (NRZ) line code.

(c) Copper cables are prone to several impairments when used in a network. Explain two types of impairments that could be experienced while using the cables.

(d) With the aid of a diagram, describe the User Datagram Protocol (UDP) datagram.

Answer

(a) Point-to-Point Protocol (PPP) and IP Addressing:

(i) Advantages of PPP:

  • Authentication and security features:
    • PAP (Password Authentication Protocol)
    • CHAP (Challenge Handshake Authentication Protocol)
    • Encrypted password exchange
    • Access control and authorization
  • Support for multiple network protocols:
    • IP, IPX, AppleTalk compatibility
    • Protocol multiplexing capabilities
    • Network layer protocol independence
    • Dynamic protocol configuration

(ii) Broadcast vs Host IP Addresses:

Feature Broadcast Address Host Address
Purpose Send packets to all devices Target specific device
Format Host bits all 1s Unique host portion
Example 192.168.1.255 192.168.1.10
Usage Network announcements Direct communication

(b) Digital Signal Representation:

(i) Digital encoded waveform for 10110001:

Voltage
High  ┌──┐    ┌──┐┌──┐        ┌──┐
     │  │    │  ││  │        │  │
Low   └──┘────┘  └┘  └────────┘  └──
     1  0  1  1  0  0  0  1
     |  |  |  |  |  |  |  |
Time  ───────────────────────────▶

(ii) NRZ (Non-Return-to-Zero) line code:

Voltage
High  ┌──┐    ┌─────┐        ┌──┐
     │  │    │     │        │  │
Low   └──┘────┘     └────────┘  └──
     1  0  1  1  0  0  0  1
     |  |  |  |  |  |  |  |
Time  ───────────────────────────▶

Note: In NRZ encoding, the signal level stays constant during the bit period without returning to zero between bits.

(c) Copper Cable Impairments:

  • Attenuation:
    • Definition: Progressive weakening of signal strength over distance
    • Causes: Resistance in cable, signal absorption
    • Effects: Reduced signal amplitude, potential data loss
    • Solutions: Signal amplifiers, repeaters, proper cable selection
  • Electromagnetic Interference (EMI):
    • Definition: Signal distortion from external electromagnetic sources
    • Sources: Power lines, motors, other cables
    • Effects: Signal noise, data corruption
    • Solutions: Shielding, proper cable routing, twisted pair design

(d) UDP Datagram Structure:

UDP Header Format (8 bytes)
Source Port (16 bits) Destination Port (16 bits)
Length (16 bits) Checksum (16 bits)
Data (Variable length)
  • Header Fields:
    • Source Port (16 bits): Identifies sending application port
    • Destination Port (16 bits): Identifies receiving application port
    • Length (16 bits): Total datagram length (header + data)
    • Checksum (16 bits): Error detection mechanism
  • Key Characteristics:
    • Connectionless protocol
    • No guaranteed delivery
    • No sequencing or flow control
    • Minimal overhead

Question 4

(a) Explain three Internet services that could be utilized by students in technical institutions.

(b) A good network design should produce a scalable network. Outline four ways of achieving this network objective.

(c) Differentiate between symmetric and asymmetric keys as used in network security.

(d) Routing protocols are critical in data transmission. Explain three functions that likely to be performed by these protocols.

Answer

(a) Internet Services for Technical Institution Students:

  • Email for Academic Communication:
    • Student-teacher communication for queries and feedback
    • Digital assignment submission and tracking
    • Group collaboration and project coordination
    • Department announcements and notifications
  • Web-based Research and Online Libraries:
    • Access to academic journals and databases
    • Digital textbooks and reference materials
    • Research tools and citation managers
    • Academic video libraries and tutorials
  • Online Learning Management Systems (LMS):
    • Course material access and management
    • Online assessments and grading
    • Interactive discussion forums
    • Progress tracking and analytics

(b) Achieving Network Scalability:

  • Modular Network Design:
    • Hierarchical three-layer architecture (Core, Distribution, Access)
    • Independent module upgrades and maintenance
    • Simplified troubleshooting and management
    • Easy addition of new network segments
  • Standardized Protocols:
    • Industry-standard protocols (TCP/IP, OSPF, BGP)
    • Vendor-neutral implementations
    • Cross-platform compatibility
    • Future-proof technology adoption
  • Proper IP Addressing Scheme:
    • Hierarchical addressing structure
    • VLSM implementation for efficient use
    • Room for future expansion
    • Logical network segmentation
  • Scalable Network Equipment:
    • Modular hardware platforms
    • Stackable switch configurations
    • Upgradeable capacity and features
    • Redundant component support

(c) Symmetric vs. Asymmetric Key Encryption:

Feature Symmetric Encryption Asymmetric Encryption
Key Usage Same key for encryption and decryption Different keys (public/private) for encryption/decryption
Speed Faster processing Slower processing
Security Key must be shared securely Public key can be freely distributed
Common Uses Bulk data encryption Digital signatures, key exchange
Examples AES, DES, 3DES RSA, DSA, ECC

(d) Routing Protocol Functions:

  • Path Determination between Networks:
    • Best path calculation based on metrics
    • Alternative path identification
    • Load balancing capabilities
    • Policy-based routing decisions
  • Route Updates and Maintenance:
    • Dynamic routing table updates
    • Topology change detection
    • Route convergence management
    • Dead route removal
  • Loop Prevention:
    • Split horizon implementation
    • Route poisoning techniques
    • Maximum hop count limits
    • Loop detection algorithms

Note: Different routing protocols implement these functions using various algorithms and mechanisms. For example, OSPF uses Dijkstra's algorithm for path determination, while EIGRP uses the DUAL algorithm.

Question 5

(a) Differentiate between half duplex and full duplex transmission modes.

(b) Explain three functions of data link layer.

(c) TCP/IP is known for its capability to provide reliable transportation of data packets from the source to destination. Explain three features of the protocol that promotes its reliable transport.

(d) Network operating system is very critical in corporate networks. Explain two functions of the software.

Answer

(a) Half Duplex vs Full Duplex Transmission:

Feature Half Duplex Full Duplex
Direction One direction at a time Both directions simultaneously
Example Walkie-talkie communication Telephone conversation
Efficiency Lower bandwidth utilization Higher bandwidth utilization
Channel Usage Single shared channel Two separate channels
Common Uses Radio systems, CSMA/CD networks Modern Ethernet, fiber optics

(b) Data Link Layer Functions:

  • Framing:
    • Encapsulates data into frames with headers/trailers
    • Adds frame delimiters and synchronization
    • Manages frame size and structure
    • Provides addressing at MAC level
  • Error Control:
    • Detects errors using CRC or checksums
    • Implements error correction mechanisms
    • Manages frame acknowledgments
    • Handles retransmission of corrupted frames
  • Flow Control:
    • Prevents buffer overflow at receiver
    • Implements sliding window protocols
    • Manages transmission rates
    • Handles congestion avoidance

(c) TCP/IP Reliable Transport Features:

  • Connection-oriented Protocol:
    • Establishes connection via three-way handshake
    • Maintains connection state information
    • Ensures ordered delivery of packets
    • Provides graceful connection termination
  • Error Detection and Retransmission:
    • Uses sequence numbers for packet tracking
    • Implements acknowledgment mechanism
    • Retransmits lost or corrupted packets
    • Employs timeout-based recovery
  • Flow Control Mechanisms:
    • Uses sliding window protocol
    • Implements congestion control
    • Adjusts transmission rate dynamically
    • Prevents network overload

(d) Network Operating System Functions:

  • Resource Sharing and Management:
    • File and printer sharing services
    • Network device management and configuration
    • Bandwidth allocation and QoS
    • Storage and backup management
  • User Authentication and Access Control:
    • User account management and permissions
    • Security policy enforcement
    • Directory services (e.g., Active Directory)
    • Audit logging and monitoring

Note: Modern network operating systems like Windows Server and Linux integrate these functions with advanced features such as virtualization, cloud services, and automated management tools to enhance network efficiency and security.

(b) (i) Outline two applications of a CSMA/CD technology.

(ii) Distinguish between channel B and channel D as used in integrated services digital network (ISDN).

(c) Cloud computing is widely used in business firms. Explain two benefits that could be influencing the trend.

(d) Explain three circumstances that would necessitate the installation of a point-to-point protocol in corporate networks.

Answer

(a) Fiber Optic Cable Classifications:

Feature Single-mode Fiber Multi-mode Fiber
Core Size 8-10 microns 50-100 microns
Distance Range Up to 100+ kilometers Up to 2 kilometers
Light Source Laser LED
Bandwidth Higher (100+ Gbps) Lower (10 Gbps typical)
Cost More expensive Less expensive

(b) CSMA/CD and ISDN:

(i) CSMA/CD Technology Applications:

  • Ethernet Networks:
    • Traditional 10BASE-T/100BASE-TX networks
    • Shared media environments
    • Collision detection and management
    • Network access control
  • Local Area Networks:
    • Bus topology implementations
    • Half-duplex media sharing
    • Bandwidth optimization
    • Network traffic management

(ii) ISDN Channel Comparison:

Feature Channel B Channel D
Purpose User data transmission Signaling and control
Speed 64 kbps 16 kbps (BRI)
Connection Type Circuit-switched Packet-switched
Usage Voice, data, video Call setup, teardown

(c) Cloud Computing Benefits:

  • Cost Reduction through Shared Resources:
    • Pay-per-use pricing model reduces capital expenses
    • Elimination of hardware maintenance costs
    • Reduced IT staffing requirements
    • Optimized resource utilization
  • Scalability and Flexibility:
    • On-demand resource allocation
    • Elastic computing capabilities
    • Geographic distribution options
    • Quick deployment and provisioning

(d) Point-to-Point Protocol Installation Requirements:

  • Direct Connection Requirements:
    • Branch office connections
    • Data center links
    • Backup circuits
    • Remote site access
  • Dedicated Bandwidth Needs:
    • Guaranteed throughput requirements
    • Quality of Service (QoS) demands
    • Low-latency applications
    • Consistent performance needs
  • Security Requirements:
    • Private communication channels
    • Data isolation requirements
    • Regulatory compliance needs
    • Encryption capabilities

Note: Point-to-Point Protocol is particularly valuable in enterprise environments where reliable, secure, and dedicated connections are essential for mission-critical operations.

Question 7

(a) Outline two challenges of installing a peer to peer network in organizations.

(b) Figure 1 shows an Ethernet frame. Outline the functions of the parts labelled (i) and (ii).

(c) Susan, an intern student in a certain company discovered that the company was using a token ring network. Explain two advantages of the token ring network that could have influenced the company.

(d) (i) Describe two modes of licensing for network software.

(ii) Assume you are hired to troubleshoot a client's network. Explain three troubleshooting methods that you could consider to use.

Answer

(a) Peer-to-Peer Network Installation Challenges:

  • Lack of Centralized Security and Control:
    • Difficulty implementing uniform security policies
    • No centralized authentication mechanism
    • Distributed access control management
    • Security vulnerabilities from individual machine configurations
  • Resource Management Difficulties:
    • Inconsistent resource availability
    • Complex backup and recovery procedures
    • Scattered data distribution
    • Limited administrative oversight

(b) Ethernet Frame Components:

Component Function Details
(i) Preamble Synchronization
  • 7 bytes of alternating 1s and 0s
  • Enables receiver clock synchronization
  • Marks frame beginning
  • Ensures reliable frame detection
(ii) Source Address Sender's MAC Address
  • 48-bit unique hardware identifier
  • Identifies sending network interface
  • Used for frame routing
  • Enables response addressing

(c) Token Ring Network Advantages:

  • Deterministic Access Method:
    • Guaranteed maximum wait time for network access
    • Predictable network performance under load
    • No collision problems like in Ethernet
    • Fair access for all nodes
  • Priority Mechanism:
    • 8 levels of transmission priority
    • Critical data gets preferential treatment
    • Better support for time-sensitive applications
    • Configurable QoS parameters

(d) Network Management:

(i) Software Licensing Modes:

  • Per-user Licensing:
    • Based on number of concurrent users
    • Individual user authentication required
    • Flexible for varying team sizes
    • Usage tracking capabilities
  • Site Licensing:
    • Covers entire organization or location
    • Unlimited users within premises
    • Simpler administration
    • Cost-effective for large deployments

(ii) Network Troubleshooting Methods:

  • Ping (ICMP Echo):
    • Tests basic network connectivity
    • Measures round-trip time
    • Identifies packet loss
    • Verifies IP configuration
  • Traceroute:
    • Maps network path to destination
    • Identifies routing problems
    • Shows hop-by-hop latency
    • Reveals network bottlenecks
  • Network Analyzer:
    • Deep packet inspection
    • Protocol analysis
    • Bandwidth utilization monitoring
    • Security threat detection

Note: Effective network troubleshooting typically requires a combination of these methods, along with systematic problem isolation and documentation of findings. Each method provides unique insights into different aspects of network performance and reliability.

Question 8

(a) (i) Outline four roles of user agent as used in email software.

(ii) Figure 2 shows a typical tool bar of a browser program. Outline the functions of each of the tools labelled (i), (ii), (iii) and (iv).

(b) (i) Define the term node as applied in data transmission.

(ii) Data communication system should exhibit distinctive features. Explain two desirable features that could be exhibited by the system.

(c) (i) Outline two factors that should be considered when selecting an Internet bandwidth, other than cost.

(ii) Explain two circumstances under which unshielded twisted pair (UTP) cables are most applicable in network installation.

Answer

(a) Email and Browser Components:

(i) User Agent Roles in Email Software:

  • Composing Email Messages:
    • Rich text editor with formatting options
    • File attachment handling
    • Address book integration
    • Template management
  • Sending Email Messages:
    • SMTP protocol implementation
    • Mail server communication
    • Delivery status tracking
    • Message queuing and retry
  • Receiving Email Messages:
    • POP3/IMAP protocol support
    • Message filtering and sorting
    • Spam detection
    • Folder organization
  • Displaying Email Messages:
    • HTML/Plain text rendering
    • Attachment preview
    • Thread visualization
    • Search functionality

(ii) Browser Toolbar Functions:

Button Function Details
(i) Back Previous Page Returns to the last visited page in browser history
(ii) Forward Next Page Advances to pages visited after current page
(iii) Refresh Page Reload Updates current page content from server
(iv) Home Homepage Returns to configured start page

(b) Data Communication Components:

(i) Node Definition:

  • Key Characteristics:
    • Connection point in network topology
    • Data processing and forwarding capability
    • Network address assignment
    • Protocol implementation
  • Common Types:
    • End devices (computers, printers)
    • Network devices (routers, switches)
    • Servers and workstations
    • IoT devices

(ii) Data Communication System Features:

  • Reliability:
    • Error detection and correction
    • Data integrity verification
    • Delivery confirmation
    • Fault tolerance mechanisms
  • Security:
    • Data encryption and privacy
    • Access control systems
    • Authentication protocols
    • Intrusion detection

(c) Network Infrastructure Considerations:

(i) Internet Bandwidth Selection Factors:

  • Number of Users:
    • Concurrent user estimates
    • Peak usage patterns
    • Growth projections
    • User behavior analysis
  • Application Requirements:
    • Video conferencing needs
    • Cloud service usage
    • File sharing volume
    • Real-time applications

(ii) UTP Cable Applications:

  • Short-distance Connections:
    • Office workstation connections
    • Horizontal cabling (≤100m)
    • Patch panel to device links
    • Desktop peripherals
  • Indoor Environments:
    • Office building installations
    • Structured cabling systems
    • Low EMI environments
    • Cost-effective deployments

Note: UTP cables are the most common choice for indoor network installations due to their cost-effectiveness, ease of installation, and sufficient performance for most office applications.
