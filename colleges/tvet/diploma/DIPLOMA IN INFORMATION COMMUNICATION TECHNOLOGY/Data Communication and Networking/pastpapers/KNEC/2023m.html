Question 1 (20 marks)

(a) Explain two factors that should be considered when selecting an Internet Service Provider (ISP), other than cost.

(b) (i) Outline four limitations of e-learning platforms.

(ii) Paul was required to design a computer network for a client. Outline four guidelines that he should follow during the design stage.

(c) (i) Differentiate between 2.4 GHZ and 5.0 GHZ Wi-Fi frequency bands.

(ii) Network authentication is used to keep off unauthorized users from accessing network resources. Describe two types of authentications that are commonly used.

Answer

(a) Two factors to consider when selecting an ISP (other than cost):

  1. Bandwidth and Speed: Assess the available bandwidth options (upload/download speeds) and check for consistency, especially during peak usage times. Ensure the offered speed meets the user's requirements for activities like streaming, gaming, or remote work.
  2. Service Reliability and Support: Investigate the ISP's network uptime guarantees (often specified in Service Level Agreements - SLAs). Check customer reviews regarding reliability and the availability and quality of technical support (e.g., 24/7 support, response times).

(b) (i) Four limitations of e-learning platforms:

  1. Technical Requirements & Issues: Requires learners to have reliable internet access and suitable devices. Technical glitches, software incompatibility, or server downtime can disrupt learning.
  2. Lack of Face-to-Face Interaction: Reduced opportunities for spontaneous discussions, immediate clarification from instructors, and building personal connections with peers and teachers.
  3. Need for Self-Discipline & Motivation: Learners require strong self-discipline, time management skills, and motivation to stay engaged without the structure of a physical classroom.
  4. Digital Divide & Accessibility: Not all learners have equal access to the necessary technology or the digital literacy skills required to navigate online platforms effectively.

(ii) Four guidelines for network design:

  1. Scalability: Design the network with future growth in mind. It should be easy to add more users, devices, or services without a major overhaul. Use modular designs and appropriate hardware.
  2. Reliability and Availability: Implement redundancy for critical components (like routers, switches, links) to minimize downtime. Plan for fault tolerance and quick recovery.
  3. Security: Integrate security measures from the beginning. This includes firewalls, access control lists (ACLs), intrusion detection/prevention systems (IDPS), VPNs for remote access, and strong authentication mechanisms.
  4. Performance: Ensure the network can handle the expected traffic load with acceptable speed and latency. Consider Quality of Service (QoS) mechanisms to prioritize critical traffic.

(c) (i) Differences between 2.4 GHz and 5.0 GHz Wi-Fi bands:

Feature 2.4 GHz Band 5.0 GHz Band
Range Longer range, signals travel further. Shorter range, signals don't travel as far.
Speed Generally lower maximum speeds. Generally higher maximum speeds.
Penetration Better at penetrating walls and solid objects. Worse at penetrating walls and solid objects.
Interference More prone to interference from microwaves, Bluetooth devices, cordless phones, etc. More crowded. Less prone to interference as fewer devices use this band. Less crowded.

(ii) Two types of network authentication:

  1. Password-Based Authentication: The most common method where users provide a username and a secret password. Often enhanced with policies like password complexity, expiry, and sometimes Multi-Factor Authentication (MFA) for added security.
  2. Certificate-Based Authentication: Uses digital certificates issued by a trusted Certificate Authority (CA). The client presents a certificate to the server (and vice-versa) for verification. It's generally considered more secure than passwords alone, often used in VPNs and secure Wi-Fi (WPA2/3-Enterprise).

Question 2 (20 marks)

(a) (i) Outline three characteristics of client-server computing.

(ii) Distinguish between managed and unmanaged network switches.

(b) (i) Outline three challenges of Bluetooth communication.

(ii) Patrick was required to troubleshoot a network interface card that was not showing physical connectivity. Outline four possible causes of the problem.

(c) During the installation a computer star topology network, several components should be considered. Describe three such components.

Answer

(a) (i) Three characteristics of client-server computing:

  1. Centralized Resources: Server(s) host and manage shared resources (files, printers, databases, applications). Clients request access to these resources.
  2. Role Specialization: Clients (typically user workstations) initiate requests, while servers (powerful machines) respond to these requests and provide services.
  3. Scalability: The system can be scaled by adding more clients or upgrading/adding servers independently to handle increased load or offer new services.

(ii) Differences between managed and unmanaged network switches:

Feature Managed Switch Unmanaged Switch
Configuration Allows configuration via CLI, web interface, or SNMP. Offers features like VLANs, QoS, port mirroring, STP, etc. Plug-and-play, no configuration options available or needed. Operates with default settings.
Monitoring Provides network monitoring capabilities (traffic statistics, error logs). Typically offers no monitoring features beyond basic LED indicators.
Security Offers security features like port security, ACLs, 802.1X authentication. Provides minimal security, relies on physical security.
Cost Generally more expensive. Generally less expensive.
Use Case Enterprise networks, networks requiring control, segmentation, and monitoring. Small home or office networks (SOHO), simple network expansion.

(b) (i) Three challenges of Bluetooth communication:

  1. Limited Range: Bluetooth typically has a short operational range (Class 2: ~10 meters, Class 1: up to 100 meters), limiting its use for long-distance communication.
  2. Interference: Operates in the crowded 2.4 GHz ISM band, making it susceptible to interference from Wi-Fi, microwaves, and other devices, potentially affecting performance and reliability.
  3. Security Vulnerabilities: While improving, Bluetooth has known vulnerabilities (e.g., Bluejacking, Bluesnarfing) that could allow unauthorized access or data interception if not properly secured.

(ii) Four possible causes of NIC not showing physical connectivity (Link Light Off):

  1. Cable Issue: The network cable might be faulty, damaged, not plugged in securely at one or both ends, or the wrong type (e.g., crossover instead of straight-through).
  2. Port Issue (Switch/Router): The port on the switch, router, or hub that the NIC is connected to might be faulty, administratively disabled, or configured incorrectly (e.g., speed/duplex mismatch).
  3. NIC Hardware Failure: The Network Interface Card itself might have failed or be malfunctioning.
  4. NIC Driver Issue / Disabled NIC: The NIC driver might be corrupted, missing, or incorrect. The NIC might also be disabled in the operating system's device manager or in the computer's BIOS/UEFI settings.

(c) Three components considered during star topology installation:

  1. Central Device (Hub/Switch/Router): A central connectivity device where all nodes connect. The choice (hub vs. switch vs. router) depends on network size, performance needs, and required features (e.g., collision handling, routing).
  2. Network Cabling (e.g., UTP): Individual cables (commonly Unshielded Twisted Pair like Cat 5e or Cat 6) run from each node directly to a port on the central device. Cable quality and length limitations must be considered.
  3. Network Interface Cards (NICs): Each device (computer, printer) needs a NIC to connect to the network cable and communicate with the central device and other nodes.

Question 3 (20 marks)

(a) Explain two disadvantages of fibre optic cables.

(b) (i) Outline two problems associated with token ring topology network.

(ii) Distinguish between switched virtual circuits (SVC) and permanent virtual connection (PVC) as used in Frame Relay configurations.

(c) Open-end Company Ltd. intends to install a data security device that has just emerged in the market. Explain two strategies that the company would use in order to enable its staff cope with the proposed installation.

(d) Explain three circumstances that would necessitate the use of Challenge-Handshake Authentication Protocol (CHAP) in a WAN connection.

Answer

(a) Two disadvantages of fibre optic cables:

  1. Cost: Fiber optic cables, associated equipment (transceivers, connectors), and installation are generally more expensive than copper cabling (like UTP). Specialized skills and tools are needed for installation and repair, adding to the cost.
  2. Fragility and Installation Complexity: The glass core is fragile and can be easily damaged by excessive bending or physical stress during installation or handling. Splicing or terminating fiber requires precision and specialized equipment.

(b) (i) Two problems associated with token ring topology network:

  1. Single Point of Failure: Although designed with some fault tolerance (like bypassing faulty stations), a break in the ring or a failure in the central MAU (Multistation Access Unit) can bring down the entire segment.
  2. Complexity and Cost: Token Ring hardware (NICs, MAUs) was historically more expensive and complex to configure and manage compared to Ethernet technologies that eventually became dominant. Troubleshooting could also be more difficult.

(ii) Differences between SVC and PVC in Frame Relay:

Feature Switched Virtual Circuit (SVC) Permanent Virtual Connection (PVC)
Connection Setup Established dynamically on demand, like a phone call. Requires call setup and teardown phases. Pre-configured by the network provider. The connection path is always available.
Duration Temporary, exists only for the duration of the data transfer session. Permanent or semi-permanent, remains active regardless of data transfer activity.
Use Case Suitable for infrequent or sporadic connections where a dedicated path isn't needed constantly. Suitable for frequent, consistent communication between specific sites (e.g., main office to branch). Most common type in Frame Relay.
Overhead Higher overhead due to call setup/teardown signalling. Lower overhead as the path is pre-established.

(c) Two strategies for staff coping with a new security device:

  1. Comprehensive Training and Communication: Provide thorough training on how the new device works, why it's necessary (benefits to security), and how it impacts their daily workflow. Communicate changes clearly and proactively address concerns through Q&A sessions or dedicated support channels.
  2. Phased Rollout and User Support: Implement the device gradually, perhaps starting with a pilot group. This allows for identifying and resolving issues before a full deployment. Provide readily accessible support (helpdesk, documentation, champions) during and after the rollout to assist users adapting to the new system.

(d) Three circumstances necessitating CHAP use in a WAN connection:

  1. Need for Secure Authentication: When connecting over potentially insecure public networks (like the internet for VPNs or dial-up), CHAP provides a more secure authentication method than PAP because it doesn't transmit the password directly across the link. It uses a challenge-response mechanism with hashing.
  2. Periodic Re-authentication Requirement: CHAP periodically re-challenges the remote end during the session to verify its identity continuously, preventing session hijacking if credentials were somehow compromised after the initial connection.
  3. Interoperability with Systems Requiring CHAP: Connection to legacy systems or specific provider equipment that mandates the use of CHAP for authentication over protocols like PPP (Point-to-Point Protocol).

Question 4 (20 marks)

(a) (i) Outline two typical characteristics of a Thin Ethernet standard.

(ii) Ethernet uses data encapsulation technique to facilitate data transmission from sender to receiver. Explain two benefits of using this technique.

(b) Mercy was required to justify the decision to installing a computer network in her company. Outline four justifications she could have provided.

(c) (i) Explain two advantages of getting software assurance when procuring a communication software.

(ii) Paul was required to troubleshoot an email server problem within his company's network. Outline six causes of the problem that he could have identified.

Answer

(a) (i) Two typical characteristics of Thin Ethernet (10BASE2):

  1. Coaxial Cable (RG-58): Uses a specific type of thin coaxial cable (RG-58 A/U) with a 50-ohm impedance.
  2. Bus Topology & T-Connectors: Typically implemented in a bus topology where devices are daisy-chained using BNC T-connectors attached directly to the NICs. Terminators (50-ohm) are required at both ends of the bus segment. (Max segment length 185m, max 30 devices/segment).

(ii) Two benefits of Ethernet data encapsulation:

  1. Layered Communication & Interoperability: Encapsulation allows higher-layer protocols (like IP, TCP) to be carried over the Ethernet network without needing to know the specifics of the physical transmission. The Ethernet frame adds headers (like MAC addresses) and trailers (like FCS for error checking) needed for Layer 2 transmission, enabling different layers to work together.
  2. Addressing and Error Detection: The Ethernet header contains source and destination MAC addresses, ensuring the frame reaches the correct device on the local network segment. The Frame Check Sequence (FCS) in the trailer allows the receiving NIC to detect if the frame was corrupted during transmission.

(b) Four justifications for installing a computer network:

  1. Resource Sharing: Allows multiple users to share expensive hardware (like printers, scanners, servers) and software licenses, reducing costs.
  2. Data Sharing and Centralized Storage: Enables easy sharing of files and information among employees. Centralized data storage simplifies backups, enhances data security, and ensures consistency.
  3. Improved Communication & Collaboration: Facilitates internal communication through email, instant messaging, and collaborative software, improving teamwork and efficiency.
  4. Centralized Management and Security: Simplifies administration tasks like software updates, user management, and security policy enforcement from a central point, enhancing overall network security and manageability.

(c) (i) Two advantages of software assurance:

  1. Access to New Versions & Updates: Provides rights to upgrade to the latest versions of the software released during the assurance period at no additional licensing cost, ensuring access to new features, performance improvements, and security enhancements.
  2. Support & Training Resources: Often includes access to technical support services, online training materials, deployment planning assistance, and sometimes other benefits like disaster recovery rights or home use programs, helping maximize the value of the software investment.

(ii) Six causes of an email server problem:

  1. Network Connectivity Issues: Problems with the server's network connection, DNS resolution failures (cannot find domains or MX records), or firewall blocking necessary ports (e.g., 25, 110, 143, 587, 993, 995).
  2. Service Failure: The core email services (like SMTP, POP3, IMAP services) on the server may have stopped running or crashed.
  3. Storage Issues: The server running out of disk space for storing emails or logs, or corruption in the mail store database.
  4. Authentication/Security Problems: Incorrect user credentials, expired SSL/TLS certificates preventing secure connections, or misconfigured authentication mechanisms.
  5. Blacklisting: The server's IP address might be listed on Real-time Blackhole Lists (RBLs) due to spam activity, preventing other servers from accepting its mail.
  6. Resource Exhaustion: The server lacking sufficient CPU, RAM, or handling too many connections, leading to slow performance or service unavailability.

(a) (i) Outline two reasons that could influence the use of ISDN in WAN connections.

(ii) Distinguish between simplex communication and half duplex data communication.

(b) Figure 1 shows a diagram of a fiber cable. Use it to answer the question that follows.

Describe the parts labelled (i) and (ii)

(c) Maria was investigating several characteristics of a demand priority media access method. Outline four characteristics that she could have established.

(d) OSPF creates a database that stores available link-state information of a network. Explain three tables that are likely to be found in the database.

Answer

(a) (i) Two reasons for using ISDN in WAN connections:

  1. Higher Speed than Dial-up: ISDN (Basic Rate Interface - BRI typically offered 128 kbps) provided significantly faster data rates than traditional analog modem dial-up connections, making it suitable for faster internet access or remote office connectivity before broadband became widespread.
  2. Reliable Digital Connection & Multiple Channels: It offered an end-to-end digital connection, generally more reliable than analog lines. BRI provided two 'B' channels (64 kbps each) for data/voice and one 'D' channel for signaling, allowing simultaneous voice and data calls or channel bonding for higher speed.

(ii) Differences between simplex and half-duplex communication:

Feature Simplex Half-Duplex
Direction of Flow Data flows in only one direction. Data flows in both directions, but only one direction at a time.
Transmission/Reception One device is always the transmitter, the other is always the receiver. Devices can transmit and receive, but not simultaneously. They must take turns.
Example Traditional radio/TV broadcasting, keyboard input to a computer. Walkie-talkies, CB radio.

(b) Description of fiber cable parts (assuming standard structure):

  1. Part (i) - Core: The central part of the fiber, typically made of highly pure glass (or sometimes plastic), through which the light signals travel. Its diameter is very small (e.g., 9 µm for single-mode, 50 or 62.5 µm for multi-mode).
  2. Part (ii) - Cladding: The layer of glass (or plastic) immediately surrounding the core. It has a lower refractive index than the core, causing light to be reflected back into the core via total internal reflection, thus guiding the light along the fiber.
*(Note: The figure itself is missing, but this describes the typical core and cladding)*

(c) Four characteristics of Demand Priority media access method (100VG-AnyLAN):

  1. Centralized Control (Hub-based): Unlike CSMA/CD, access is controlled by an intelligent hub. Nodes request permission to transmit from the hub.
  2. Priority Levels: Supports two priority levels (high and normal). High-priority requests (e.g., for time-sensitive data like video) are serviced before normal-priority requests.
  3. Round-Robin Polling: The hub polls nodes in a round-robin fashion within each priority level to grant transmission opportunities, preventing collisions and ensuring predictable access.
  4. Support for Ethernet and Token Ring Frames: Designed to carry both standard Ethernet frames and Token Ring frames, aiming for interoperability. (Operated at 100 Mbps).

(d) Three tables found in the OSPF database (Link-State Database - LSDB):

  1. Adjacency Table (Neighbor Table): Contains information about the router's directly connected OSPF neighbors with whom it has formed an adjacency. It lists neighbor router IDs and their state (e.g., Full, Down).
  2. Link-State Database Table (Topology Table): Stores all Link-State Advertisements (LSAs) received from routers within the same OSPF area. This table represents the complete topology map of the area from the perspective of the router.
  3. Routing Table (Forwarding Database): While technically derived *from* the LSDB after running the SPF algorithm, the resulting best paths are stored here. It contains destination networks, the cost (metric) to reach them, and the next-hop router/interface to forward packets towards that destination.

Question 6 (20 marks)

(a) Explain three challenges facing both service providers and users when connecting Internet in rural areas

(b) Ann intends to design an institutional network with several subnets. Outline four reasons that would influence her decision to use subnets.

(c) Differentiate between zero-day and patch as used in computer software security.

(d) UDP protocol lacks connection requirements and data verification causing issues when transmitting packets. Explain three possible issues.

Answer

(a) Three challenges for Internet connectivity in rural areas:

  1. Infrastructure Deployment Cost & ROI (Provider Challenge): The cost to deploy broadband infrastructure (like fiber or fixed wireless) over large distances with low population density is very high. Providers face difficulty achieving a return on investment compared to densely populated urban areas.
  2. Geographic & Environmental Obstacles (Provider/User Challenge): Difficult terrain (mountains, forests), long distances, and lack of existing infrastructure (like utility poles) make deployment physically challenging and expensive. Weather can also impact wireless signal reliability.
  3. Affordability & Digital Literacy (User Challenge): Even where service is available, the cost might be prohibitive for lower-income rural residents. Additionally, users may lack the necessary digital skills or devices to effectively utilize the internet connection.

(b) Four reasons for using subnets in an institutional network:

  1. Improved Performance / Reduced Congestion: Subnetting breaks a large network into smaller broadcast domains. This reduces broadcast traffic within each subnet, leading to less congestion and better overall network performance.
  2. Enhanced Security: Subnets allow for the logical segmentation of the network (e.g., separating departments like Finance, HR, Student Labs). Access Control Lists (ACLs) can be applied between subnets to control traffic flow and restrict access to sensitive resources.
  3. Simplified Administration & Troubleshooting: Managing smaller network segments is easier. Network problems can often be isolated to a specific subnet, simplifying troubleshooting and reducing the impact of issues on the entire institution.
  4. Logical Organization & Addressing Efficiency: Allows the network to be organized logically based on departments, buildings, or function. It also enables more efficient use of the assigned IP address space.

(c) Difference between zero-day and patch:

Feature Zero-day Patch
Definition A vulnerability in software that is unknown to the vendor or public, and for which no official fix (patch) exists. A piece of software code designed to be applied over existing software to fix bugs, improve performance, or, commonly, fix security vulnerabilities.
Vendor Awareness Vendor is unaware of the vulnerability. Vendor is aware of the issue and has developed a fix.
Exploitation Attackers can exploit it before a defense is available ("zero days" of protection). Applying the patch mitigates the vulnerability it addresses.

(d) Three possible issues due to UDP's lack of connection/verification:

  1. Packet Loss (Unreliability): UDP does not guarantee delivery. Packets can be dropped by routers due to congestion or errors, and UDP itself provides no mechanism to detect or retransmit lost packets. The application layer must handle this if reliability is needed.
  2. Out-of-Order Delivery: Packets may take different paths through the network and arrive at the destination in a different order than they were sent. UDP does not sequence packets, so the receiving application might receive data out of order.
  3. Data Corruption (Limited Error Checking): While UDP has an optional checksum for header/data integrity, it's not mandatory (especially in IPv4) and doesn't guarantee detection of all errors. There's no mechanism for acknowledging correct receipt or requesting retransmission of corrupted data.

Question 7 (20 marks)

(a) (i) Outline four steps used by Secure Copy Protocol (SCP) during file transfer.

(ii) Figure 2 shows typical tools in an Internet browser. Use it to answer the question that follows.

State the function for each of the tools labelled (i), (ii), (iii) and (iv).

(b) (i) Outline two roles of network interface card.

(ii) With the aid of a diagram, outline a data communication system.

(c) (i) Outline two advantages of a Secure Hypertext Transfer Protocol (HTTPS).

(ii) Explain two circumstances under which coaxial cable would be the most appropriate when installing a computer network.

Answer

(a) (i) Four steps used by SCP during file transfer:

  1. Establish SSH Connection: SCP relies on SSH (Secure Shell) for security. The first step is to establish an authenticated and encrypted SSH connection between the client and the remote server.
  2. Initiate SCP Command: The client sends the SCP command over the established SSH connection, specifying the source file(s), destination path, and transfer options (like preserving timestamps or recursive copy).
  3. Data Transfer: The file data is transferred securely over the encrypted SSH channel. SCP handles the file transfer protocol details within this secure tunnel.
  4. Connection Closure: Once the transfer is complete, the SCP process finishes, and the underlying SSH connection may be closed (or kept open if used for other commands).

(ii) Functions of browser tools (assuming standard icons):

Label Typical Tool Function
(i) Back Arrow Navigates to the previously visited page in the current tab's history.
(ii) Refresh/Reload Icon Reloads the current web page, fetching the latest version from the server or cache.
(iii) Home Icon Navigates to the user's configured home page.
(iv) Bookmark/Favorites Icon (Star) Adds the current page to the user's bookmarks or favorites list for easy access later.
*(Note: The figure itself is missing, functions are based on common browser UI)*

(b) (i) Two roles of a network interface card (NIC):

  1. Physical Connection Interface: Provides the physical port (e.g., RJ-45 for Ethernet, antenna for Wi-Fi) to connect the computer to the network medium (cable or airwaves). It handles the transmission and reception of electrical or radio signals.
  2. Data Link Layer Functions: Formats outgoing data into Ethernet frames (or other appropriate frame types), adds the unique MAC address of the NIC, and performs error checking (like calculating FCS). It also processes incoming frames, checking the destination MAC address and FCS.

(ii) Diagram of a data communication system:

Components of a Data Communication System

  • Source: Creates the data/message to be sent (e.g., a user typing).
  • Transmitter: Converts the data into signals suitable for transmission (e.g., modem, NIC).
  • Transmission System: The physical path or medium over which signals travel (e.g., cable, air).
  • Receiver: Converts the received signals back into data (e.g., modem, NIC).
  • Destination: The device or user that receives the data/message.

(c) (i) Two advantages of HTTPS:

  1. Confidentiality (Encryption): Encrypts the data exchanged between the user's browser and the web server using SSL/TLS. This prevents eavesdroppers from reading sensitive information like login credentials, credit card numbers, or personal data.
  2. Integrity and Authentication: Ensures that the data hasn't been tampered with during transmission. The SSL/TLS certificate also helps authenticate the web server, verifying that the user is connecting to the legitimate website and not an imposter site (helps prevent phishing).

(ii) Two circumstances for using coaxial cable:

  1. Cable Television (CATV) Distribution: Coaxial cable is still widely used for distributing television signals from the service provider to homes due to its good bandwidth capacity over long distances and effective shielding against interference.
  2. Broadband Cable Internet: Used in Hybrid Fiber-Coax (HFC) networks where fiber runs to a neighborhood node, and coaxial cable provides the final connection ("last mile") to individual homes for high-speed internet access (e.g., DOCSIS technology). Its shielding is better than UTP for carrying signals over longer residential drops.

Question 8 (20 marks)

(a) Differentiate between loopback and automatic private IP addresses.

(b) Explain three functions of session layer of the OSI Model.

(c) Mark, a network support assistant, was required to test TCP/IP connection for a newly installed network in their company. Describe three tools that he could use to complete the task.

(d) Nadia was advised to carry out a clean installation for a network operating software for her company. Outline four advantages of this type of installation.

Answer

(a) Differences between loopback and Automatic Private IP Addressing (APIPA):

Feature Loopback Address APIPA Address
Address Range Typically 127.0.0.1 (part of the 127.0.0.0/8 range). 169.254.0.0 to 169.254.255.255 (169.254.0.0/16 range).
Purpose Used for testing the TCP/IP stack on the local machine. Packets sent to a loopback address are processed locally and do not leave the host. Self-assigned by a device when it's configured for DHCP but cannot reach a DHCP server. Allows basic communication on the local subnet only.
Communication Scope Strictly internal to the host machine. Local network segment only. Cannot be routed.
Assignment Pre-defined standard, configured automatically on the loopback interface. Automatically assigned by the OS under specific conditions (DHCP failure).

(b) Three functions of the Session Layer (Layer 5) of the OSI Model:

  1. Dialog Control (Session Management): Establishes, manages, and terminates communication sessions between applications. It determines whose turn it is to transmit (using tokens) and controls the flow of dialogue (e.g., half-duplex or full-duplex).
  2. Synchronization: Inserts checkpoints into the data stream. If a communication failure occurs, the session can resume from the last acknowledged checkpoint, avoiding the need to retransmit the entire data stream.
  3. Session Establishment & Termination: Handles the orderly setup and teardown of sessions, ensuring that both ends are ready to communicate and that the connection is closed gracefully when finished.

(c) Three tools for testing TCP/IP connection:

  1. Ping: Sends ICMP Echo Request packets to a target IP address or hostname and waits for Echo Replies. It verifies basic reachability (Layer 3 connectivity) and measures round-trip time (latency). Command: `ping `
  2. Traceroute (tracert on Windows): Shows the path (sequence of routers) that packets take to reach a destination host. It helps identify where connectivity breaks down or where high latency occurs along the path. Command: `traceroute ` or `tracert `
  3. Ipconfig (Windows) / ifconfig or ip addr (Linux/macOS): Displays the TCP/IP configuration details of the local machine's network interfaces, including IP address, subnet mask, default gateway, and DNS servers. Useful for verifying the local host's configuration. Command: `ipconfig /all` or `ip addr show`
    4. *(Other tools like netstat, nslookup, telnet could also be mentioned)*

(d) Four advantages of a clean installation for network operating software:

  1. Removes Legacy Issues: Eliminates potential problems, conflicts, misconfigurations, or malware inherited from the previous installation or upgrade process. Starts with a clean slate.
  2. Improved Performance & Stability: Often results in better system performance and stability as there is no accumulated clutter (old drivers, registry entries, fragmented files) from previous versions.
  3. Ensures Compatibility: Guarantees that only compatible drivers and system files for the specific hardware and OS version are installed, reducing potential conflicts.
  4. Opportunity for Reconfiguration: Provides a chance to re-evaluate and implement optimal system configurations, partitioning schemes, and security settings from scratch, rather than carrying over potentially outdated ones.
