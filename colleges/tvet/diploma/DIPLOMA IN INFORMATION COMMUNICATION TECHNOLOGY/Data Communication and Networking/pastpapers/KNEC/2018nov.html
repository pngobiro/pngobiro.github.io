Question 1 (20 marks)

(a) Differentiate between collision domain and broadcast domain as used in data communication.

(b) Explain three functions of the data link layer as applied in OSI model.

(c) With the aid of a diagram, describe the TCP/IP model.

(d) Command line network operating systems are rarely used by end users in organizations. Explain two disadvantages that could be affecting their popularity.

Answer

(a) Collision Domain vs. Broadcast Domain:

  • Collision Domain: A network segment where data packets can collide if multiple devices transmit simultaneously. Devices within the same collision domain compete for media access (e.g., devices connected to a hub). Switches segment collision domains.
  • Broadcast Domain: A network segment where a broadcast frame (sent to all devices) will be propagated. All devices in the same broadcast domain receive broadcast traffic (e.g., devices connected to the same switch or within the same VLAN). Routers segment broadcast domains.

(b) Three functions of the Data Link Layer (OSI Model):

  1. Framing: Encapsulates network layer packets into frames, adding headers and trailers with control information (like MAC addresses).
  2. Physical Addressing: Adds source and destination MAC addresses to the frame header for local delivery on the network segment.
  3. Error Control: Detects (and sometimes corrects) errors that occur during physical transmission using techniques like CRC (Cyclic Redundancy Check) included in the frame trailer.
  4. (Also Flow Control, Media Access Control)

(c) TCP/IP Model:

The TCP/IP model is a conceptual framework for network protocols, consisting of four layers:

  • Application Layer: Provides network services directly to user applications (e.g., web browsing, email).
  • Transport Layer: Handles end-to-end communication, segmentation, and reliability (TCP) or speed (UDP).
  • Internet Layer: Responsible for logical addressing (IP addresses) and routing packets across networks.
  • Network Access Layer (or Link Layer): Deals with physical transmission of data over the network medium and physical addressing (MAC addresses).

(d) Two disadvantages of Command Line Network Operating Systems for end users:

  1. Steep Learning Curve: Requires users to memorize commands and syntax, making it less intuitive and harder to learn compared to GUIs.
  2. Less User-Friendly: Lacks visual aids and direct manipulation, making tasks more abstract and potentially error-prone for non-technical users.

Question 2 (20 marks)

(a) Outline four categories of communication software that could be found in computers (4 marks)

(b) For each of the following scenario, identify the most appropriate Internet service:

  1. real-time exchange of text messaging;
  2. personal website for sharing personal opinions on a particular subject;
  3. making digital recordings of a broadcast available for downloading;
  4. enable people located in different geographical region to hold meetings.

(c)

  1. Define the term link aggregation as used in computer network design.
  2. Sarah was required to identify parts of a data communication system during a job interview. Outline four parts that she could have identified.

(d)

  1. Outline two requirements for using file transfer protocol (FTP) in the Internet.
  2. Explain two circumstances under which coaxial cables are most applicable in local area network installations.

Answer

(a) Four categories of communication software:

  1. Email Clients: Software for sending, receiving, and managing emails (e.g., Microsoft Outlook, Mozilla Thunderbird).
  2. Web Browsers: Software for accessing and displaying web pages (e.g., Chrome, Firefox, Edge).
  3. Instant Messaging (IM) Clients: Software for real-time text-based communication (e.g., WhatsApp Desktop, Slack, Microsoft Teams).
  4. Collaboration Software: Platforms integrating various communication tools like chat, video conferencing, and file sharing (e.g., Microsoft Teams, Zoom, Google Workspace).

(b) Appropriate Internet service for scenarios:

  1. Real-time text messaging: Instant Messaging (IM) or Chat services.
  2. Personal website for opinions: Blog platform or personal website hosting.
  3. Downloading broadcast recordings: Podcasting service or file hosting/sharing service.
  4. Meetings for geographically dispersed people: Video Conferencing or Web Conferencing services (e.g., Zoom, Teams, Google Meet).

(c)(i) Link Aggregation:

Link aggregation (also known as port trunking, bonding, or teaming) is the practice of combining multiple physical network connections (ports) into a single logical link. This increases bandwidth beyond what a single connection could provide and offers redundancy, as traffic can failover to remaining links if one connection fails.

(c)(ii) Four parts of a data communication system:

  1. Message: The information (data) to be communicated.
  2. Sender (Source): The device that creates and sends the message.
  3. Medium (Channel): The physical path over which the message travels (e.g., cable, airwaves).
  4. Receiver (Destination): The device that receives the message.
  5. (Also Protocol: Rules governing communication)

(d)(i) Two requirements for using FTP:

  1. FTP Client Software: Software installed on the user's computer to initiate connections and manage file transfers (e.g., FileZilla, WinSCP).
  2. FTP Server Access: A running FTP server to connect to, often requiring a server address, username, and password for authentication (though anonymous access is sometimes allowed).

(d)(ii) Two circumstances for using coaxial cables in LANs:

  1. Legacy Installations: Older networks (like 10BASE2 or 10BASE5 Ethernet) were built using coaxial cable, and it might still be in place.
  2. High Interference Environments: Coaxial cable offers better shielding against electromagnetic interference (EMI) compared to early unshielded twisted pair (UTP) cables, making it suitable in electrically noisy areas (though modern shielded UTP or fiber is now preferred).

Question 3 (20 marks)

(a) Explain three challenges associated with installing WIFI hot spots for students use in technical institutions in Kenya.

(b) Lucas was required to select a network design for a client. Outline four factors that he could consider, other than cost.

(c) Differentiate between packet filter and stateful inspection firewalls as used in network security.

(d) Border Gateway Protocol (BGP) is a phenomenon protocol in transport layer. Explain three reasons that could be influencing its use.

Answer

(a) Three challenges of installing Wi-Fi hotspots in Kenyan technical institutions:

  1. Infrastructure Costs: High initial investment for access points, cabling, controllers, and potential upgrades to the existing network backbone.
  2. Coverage and Interference: Ensuring adequate signal strength across large campuses with diverse building materials can be difficult, and interference from other devices or networks can degrade performance.
  3. Security and Management: Implementing robust authentication, preventing unauthorized access, managing bandwidth allocation, and protecting against cyber threats require significant technical expertise and ongoing effort.
  4. (Also: Power reliability, ongoing maintenance costs, user support)

(b) Four factors (other than cost) for selecting a network design:

  1. Scalability: The design should accommodate future growth in terms of users, devices, and traffic volume without requiring a complete overhaul.
  2. Reliability/Availability: The network must be dependable, with minimal downtime. This involves considering redundancy for critical components.
  3. Performance: The network must provide adequate bandwidth and low latency to support the applications and services used by the client.
  4. Security: The design must incorporate measures to protect data and resources from unauthorized access, modification, or denial of service.
  5. (Also: Manageability, existing infrastructure compatibility)

(c) Packet Filter vs. Stateful Inspection Firewalls:

  • Packet Filtering Firewall: Operates mainly at the Network layer. It examines each packet header individually based on static rules (IP addresses, ports, protocol type). It doesn't track the state of connections. Fast but less secure.
  • Stateful Inspection Firewall: Operates at Network and Transport layers. It examines packet headers *and* maintains a state table to track active connections. Decisions are based on rules and the context of the connection state (e.g., allowing return traffic for an established connection). More secure and context-aware than packet filtering.

(d) Three reasons for Border Gateway Protocol (BGP) use:

Note: BGP operates at the Application layer but functions as the core routing protocol of the Internet (between Autonomous Systems), not typically considered a Transport layer protocol itself. The question might be slightly misphrased.

  1. Inter-Autonomous System Routing: BGP is the standard protocol used to exchange routing information between different Autonomous Systems (ASes) - large networks or groups of networks under single administrative control (like ISPs). It's essential for global Internet routing.
  2. Policy-Based Routing: BGP allows network administrators to implement complex routing policies based on business relationships, traffic engineering needs, and security requirements, rather than just technical metrics like hop count.
  3. Scalability: BGP is designed to handle the massive scale of the Internet routing table, efficiently managing hundreds of thousands of network prefixes.

Question 4 (20 marks)

(a) Explain each of the following terms as used in the Internet:

  1. registrar;
  2. domain name.

(b) Juliet was advised by her friends to use genuine licensed communication software in her company. Explain two reasons that could inform her friend's advice.

(c) Hackers are known to exploit email vulnerabilities to attack private networks. Outline four examples of such vulnerabilities that they could exploit. (4 marks)

(d)

  1. With the aid of examples, describe two wireless network topologies. (4 marks)
  2. Explain two advantages of using token passing as used in local area network.

Answer

(a) Internet Terms Explained:

  1. Registrar: An organization accredited by ICANN (Internet Corporation for Assigned Names and Numbers) or a national ccTLD authority to manage the reservation of Internet domain names. Customers register domain names through registrars (e.g., GoDaddy, Namecheap).
  2. Domain Name: A human-readable label used to identify network resources, such as websites, on the Internet (e.g., `google.com`, `knec.ac.ke`). It corresponds to an underlying IP address through the Domain Name System (DNS).

(b) Two reasons for using genuine licensed communication software:

  1. Security Updates and Patches: Licensed software typically receives regular security updates from the vendor to protect against newly discovered vulnerabilities. Pirated software often lacks this support, leaving systems exposed.
  2. Legality and Compliance: Using unlicensed software is illegal and violates copyright laws. Companies can face significant fines and legal action if audited and found to be non-compliant.
  3. (Also: Access to support, full features, ethical considerations)

(c) Four email vulnerabilities exploited by hackers:

  1. Phishing Attachments/Links: Malicious attachments (e.g., infected documents) or links to fake websites designed to steal credentials or install malware.
  2. Spoofing: Forging the sender's email address to make the email appear legitimate, often used in phishing or business email compromise (BEC) attacks.
  3. Lack of Encryption: Sending sensitive information over unencrypted email connections (not using TLS/SSL) allows eavesdropping.
  4. Weak Authentication: Insufficient protection for email accounts (e.g., weak passwords, no multi-factor authentication) allows unauthorized access.

(d)(i) Two wireless network topologies (with examples):

  1. Infrastructure Mode: Wireless devices (clients) connect to a central Access Point (AP), which typically bridges the wireless network to a wired network. All communication goes through the AP. Example: Typical home or office Wi-Fi setup where laptops and phones connect to a wireless router.
  2. Ad Hoc Mode (IBSS - Independent Basic Service Set): Wireless devices connect directly to each other in a peer-to-peer manner without a central AP. Example: Two laptops connecting directly via Wi-Fi to share files without a router.
  3. (Also: Wireless Mesh Network)

(d)(ii) Two advantages of token passing:

  1. Deterministic Access: Provides predictable access times as each node gets a turn to transmit when it holds the token. This is suitable for time-sensitive applications.
  2. Collision Elimination: Since only the node holding the token can transmit, collisions are avoided, leading to more efficient use of bandwidth under high load compared to contention-based methods like CSMA/CD.

Question 5 (20 marks)

(a)

  1. Outline three limitations of a mesh network.
  2. Distinguish between managed and unmanaged network switches.

(b)

  1. Outline three factors that determines the throughput of a transmission media. (3 marks)
  2. A lecturer described various network troubleshooting techniques in a data communication and networking class. Outline four guidelines that he could have mentioned to the class.

(c) The network design phase is prone to several constraints. Explain three typical constraints that are likely to affect organizations during this phase.

Answer

(a)(i) Three limitations of a mesh network:

  1. Complexity: Configuration and management can be complex due to the multiple interconnected paths and routing decisions.
  2. Cost: Can be expensive to implement, especially in wired mesh networks, due to the amount of cabling required for full redundancy. Wireless mesh can also require more sophisticated nodes.
  3. Scalability Issues (Potential): While offering redundancy, managing routing and potential latency in very large mesh networks can become challenging.

(a)(ii) Managed vs. Unmanaged Network Switches:

  • Unmanaged Switch: Plug-and-play device with no configuration options. Operates purely at Layer 2, forwarding traffic based on MAC addresses. Simple and inexpensive.
  • Managed Switch: Offers configuration options via CLI, web interface, or SNMP. Supports features like VLANs, QoS, link aggregation, port mirroring, and security settings, providing greater control, flexibility, and monitoring capabilities. More expensive and complex.

(b)(i) Three factors determining transmission media throughput:

  1. Bandwidth: The theoretical maximum data carrying capacity of the medium, often measured in Hz or bits per second.
  2. Signal-to-Noise Ratio (SNR): The ratio of signal power to noise power. Higher SNR allows for higher data rates with fewer errors.
  3. Physical Characteristics: Factors like the type of medium (copper, fiber, wireless), cable length/distance, shielding, and quality of connectors influence signal propagation and susceptibility to impairments.

(b)(ii) Four guidelines for network troubleshooting:

  1. Follow a Systematic Approach: Use a structured methodology (e.g., OSI model top-down or bottom-up) to isolate the problem logically.
  2. Gather Information: Collect detailed symptoms from users, check logs, use diagnostic tools (ping, traceroute, ipconfig/ifconfig), and understand the network baseline.
  3. Test One Change at a Time: When implementing potential solutions, change only one variable at a time and test thoroughly to confirm if it resolved the issue.
  4. Document Everything: Record the problem, steps taken, changes made, and the final resolution. This helps with future troubleshooting and knowledge sharing.

(c) Three typical constraints during network design phase:

  1. Budget: Financial limitations often dictate the choice of hardware, software, cabling, and implementation scope.
  2. Technical Expertise: Lack of skilled personnel within the organization to design, implement, or manage the proposed network can be a constraint.
  3. Existing Infrastructure: The need to integrate with or upgrade existing legacy systems, cabling, or equipment can impose limitations on the design choices.
  4. (Also: Time constraints, physical site limitations, organizational policies)

Question 6 (20 marks)

(a) Outline two roles of computer networks in organizations.

(b) Figure 1 shows an Ethernet frame. Outline the function of each of the parts labelled (i) and (ii). (4 marks)

(c) CSMA/CD is not commonly used in network interface card configurations. Explain two limitations that could be hampering its use.

(d)

  1. Outline four reasons for updating a network operating system (NOS).
  2. Liz discovered that her company network was having a problem that would require attention of their network administrator. Explain three typical signs that she could have discovered.

Answer

(a) Two roles of computer networks in organizations:

  1. Resource Sharing: Networks enable sharing of hardware (printers, storage devices) and software resources among users, reducing costs and improving efficiency.
  2. Communication and Collaboration: Networks facilitate communication through email, messaging, and collaborative tools, enabling teamwork regardless of physical location.

(b) Functions of Ethernet frame parts:

  1. Part (i) - Start Frame Delimiter (SFD): Marks the beginning of the frame and helps synchronize the receiver with the incoming data stream. It follows the preamble and contains a specific bit pattern (10101011).
  2. Part (ii) - CRC (Cyclic Redundancy Check): A 32-bit error-detecting code calculated over the entire frame (except preamble and SFD). The receiver recalculates the CRC and compares it with the received value to detect transmission errors.

(c) Two limitations of CSMA/CD:

  1. Distance Limitation: CSMA/CD effectiveness is limited by the maximum network diameter, as collision detection must occur within a specific time window. This restricts the physical size of the network.
  2. Performance Degradation: Under heavy network load, frequent collisions can significantly reduce network efficiency as devices spend more time backing off and retransmitting.

(d)(i) Four reasons for updating a network operating system:

  1. Security Patches: Protect against newly discovered vulnerabilities and threats.
  2. Bug Fixes: Resolve known software issues and improve system stability.
  3. New Features: Access to new functionality and improvements in system capabilities.
  4. Hardware Support: Enable compatibility with newer hardware devices and technologies.

(d)(ii) Three typical signs of network problems:

  1. Slow Network Performance: Unusually slow response times, delayed file transfers, or laggy applications.
  2. Intermittent Connectivity: Random disconnections or inability to access network resources.
  3. Error Messages: Frequent network-related error messages when trying to access resources or services.

Question 7 (20 marks)

(a). Describe two disadvantages of using fibre optic cables.

(b)

  1. List four examples of signal impairment that could be experienced in a transmission medium.
  2. Distinguish between Link Control Protocol (LCP) and Network Control Protocol (NCP) as used in WAN.

(c) Cloud computing is a technology that is being adapted by business firms. Explain two types of cloud computing services that could be used.

(d) Explain three circumstances that would necessitate the installation of a frame relay WAN connection in organizations.

Answer

(a) Two disadvantages of using fiber optic cables:

  1. High Initial Cost: Fiber optic cables and associated equipment (transceivers, connectors) are more expensive than copper alternatives. Installation also requires specialized tools and skilled technicians.
  2. Physical Vulnerability: Fiber optic cables are more fragile than copper cables and can be damaged by sharp bends or physical stress. Repairs require specialized equipment and expertise.

(b)(i) Four examples of signal impairment in transmission media:

  1. Attenuation: Weakening of signal strength over distance
  2. Noise: Unwanted electrical or electromagnetic interference that distorts the signal
  3. Distortion: Changes in signal shape or timing due to varying propagation speeds at different frequencies
  4. Cross-talk: Signal interference between adjacent cables or channels

(b)(ii) LCP vs. NCP in WAN:

  • Link Control Protocol (LCP): Establishes, configures, tests, and terminates the data link connection. Handles basic link setup and maintenance functions like authentication and compression.
  • Network Control Protocol (NCP): Configures and manages different network layer protocols that run over the WAN link. Each network protocol (IP, IPX, etc.) has its own NCP to handle protocol-specific configuration.

(c) Two types of cloud computing services:

  1. Software as a Service (SaaS): Delivers applications over the internet on a subscription basis. Users access software through web browsers without local installation (e.g., Gmail, Microsoft 365).
  2. Infrastructure as a Service (IaaS): Provides virtualized computing resources over the internet (e.g., virtual machines, storage, networking). Users manage their own applications and data while the provider manages the infrastructure.

(d) Three circumstances for frame relay WAN implementation:

  1. Cost-Effective Multi-Site Connectivity: Organizations with multiple locations needing reliable, cost-effective WAN connectivity with flexible bandwidth allocation.
  2. Variable Bandwidth Requirements: When traffic patterns vary significantly, frame relay's Committed Information Rate (CIR) and burst capabilities provide efficient bandwidth utilization.
  3. Data-Centric Applications: Organizations primarily running data applications that can tolerate some delay but require reliable delivery with minimal overhead.

Question 8 (20 marks)

(a)

  1. Outline two benefits of installing ISDN connections in organizations.
  2. Distinguish between host-ID and network-ID as used in IP addressing

(b) Cyclic Redundancy Check (CRC) is an error-detecting scheme commonly used in computer networks. Outline four of its strengths that could be driving its popularity.

(c) Satellite links are not commonly used in local area networks (LAN). Outline four reasons that discourage their use.

(d) With the aid of a diagram, describe the User Datagram Protocol (UDP) datagram.

Answer

(a)(i) Two benefits of installing ISDN connections:

  1. Digital End-to-End Connection: Provides fully digital transmission, resulting in higher quality and more reliable data communication compared to analog connections.
  2. Multiple Channels: Supports simultaneous voice and data transmission through multiple B (Bearer) channels, allowing voice calls while maintaining data connectivity.

(a)(ii) Host-ID vs. Network-ID in IP addressing:

  • Network-ID: The portion of an IP address that identifies the network segment. All devices on the same network share the same network-ID (determined by subnet mask).
  • Host-ID: The portion of an IP address that uniquely identifies a specific device within the network segment. Must be unique within that network.

(b) Four strengths of CRC error detection:

  1. High Error Detection Rate: Very effective at detecting burst errors and common transmission errors.
  2. Computational Efficiency: Can be implemented efficiently in hardware using simple shift registers.
  3. Low Overhead: Adds relatively small overhead (typically 32 bits) to the frame.
  4. Mathematical Reliability: Based on polynomial division, providing strong mathematical guarantees for error detection.

(c) Four reasons satellite links are not common in LANs:

  1. High Latency: Satellite communication involves significant propagation delay (typically 500+ ms round trip) unsuitable for LAN applications requiring quick response times.
  2. Cost: Satellite equipment and service subscriptions are much more expensive than traditional LAN technologies.
  3. Weather Sensitivity: Signal quality can be affected by atmospheric conditions, reducing reliability.
  4. Unnecessary Range: LANs typically cover small geographic areas where terrestrial technologies are more practical and cost-effective.

(d) UDP Datagram Structure:

UDP (User Datagram Protocol) provides a simple, connectionless transport service:

  • Source Port: Identifies sending application
  • Destination Port: Identifies receiving application
  • Length: Total length of UDP datagram (header + data)
  • Checksum: Optional error checking of header and data
  • Data: Application data (optional)
