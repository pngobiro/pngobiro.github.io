Information System Security and Controls

🔒 Security Threats and Controls

⚠️

Classes of Threats

  • 👤 Human Error: Wrong input data, programming errors, operator errors
  • 🌋 Natural Disasters: Earthquakes, typhoons, tornados, hurricanes
  • ⚡ Hardware/Software Failure: Network failures, power failures, software maintenance issues
  • 🔨 Computer Abuse: Theft, sabotage, unauthorized access, hacking
🔓 Common Attack Methods
  • 🐎 Trojan Horse: Unauthorized actions hidden in legitimate programs
  • 💣 Logic Bomb: Malicious code triggered by specific conditions
  • 🦠 Computer Viruses: Deliberately introduced malicious code
  • 🎯 Data Diddling: Unauthorized data modification during input

🛡️ Control Measures

🔐

General Controls

  • 👥 Administrative: Separation of duties, supervision, screening
  • 📝 System Development: Secure and auditable systems, documentation
  • ⚙️ Operational: System access, maintenance, storage
  • 🏢 Physical Protection: Environmental controls, security measures
🎯

Application Controls

  • 📥 Input Control: Accuracy and completeness validation
  • ⚙️ Processing Control: Cross-checking computations
  • 💾 Database Control: Backups, recovery, authorization
  • 🔄 Communication Control: Encryption, digital signatures
  • 📤 Output Control: Distribution and handling

🔄 Disaster Recovery

📋 Recovery Planning

A disaster recovery plan specifies how a company will maintain and restore critical information systems in the event of a disaster. It is essential for business continuity and may be required by regulatory agencies or insurance providers.

🎯

Key Components

  • 🚨 Emergency Plan: Defines disaster scenarios and required actions
  • 💾 Backup Plan: Details data backup and system recovery procedures
  • 🔄 Test Plan: Ensures recovery procedures work as intended
  • 🛠️ Recovery Plan: Outlines steps to restore normal operations
