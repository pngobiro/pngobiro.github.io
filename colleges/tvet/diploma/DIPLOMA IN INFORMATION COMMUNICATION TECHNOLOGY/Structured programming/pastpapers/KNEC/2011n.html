(b) Quick Sort Procedure

Quick Sort is a divide-and-conquer sorting algorithm. The general procedure is:

  1. Pick a Pivot: Choose an element from the array to be the pivot (e.g., first element, last element, median, or random element).
  2. Partition: Rearrange the array elements so that all elements smaller than the pivot come before it, and all elements greater than the pivot come after it. Equal elements can go on either side. After partitioning, the pivot is in its final sorted position.
  3. Recursively Sort Sub-arrays: Recursively apply the Quick Sort algorithm to the sub-array of elements smaller than the pivot and the sub-array of elements greater than the pivot.
  4. Base Case: The recursion stops when a sub-array has zero or one element, as it is already considered sorted.
(c) Modular vs. Monolithic Design
  • Modular Design: Involves breaking down a large program into smaller, independent, self-contained modules (like functions, procedures, classes). Each module focuses on a specific task or functionality. This approach promotes reusability, maintainability, easier testing, and team collaboration. Changes in one module are less likely to affect others.
  • Monolithic Design: Refers to designing a program as a single, large, tightly integrated unit where all components are part of one large codebase. Changes often require understanding and modifying large parts of the system, making it harder to maintain, debug, and update, especially for complex applications. It lacks clear separation of concerns.

(b) (i) Parameter Passing

Parameter passing is the mechanism by which data (arguments or actual parameters) is transferred from the calling part of a program (e.g., main program, another function) to a subprogram (function or procedure) when it is invoked. This allows subprograms to operate on different data values each time they are called.

Common methods include:

  • Pass by Value: A copy of the argument's value is sent to the subprogram. Changes made to the parameter inside the subprogram do not affect the original argument in the caller.
  • Pass by Reference (or Address): The memory address of the argument is sent to the subprogram. Changes made to the parameter inside the subprogram directly modify the original argument in the caller. (Implemented using pointers in C, VAR parameters in Pascal).
(b) (ii) Random vs. Sequential File Organization
  • Sequential File Organization: Records are stored and accessed one after another in a fixed physical order, usually the order they were written. To access a specific record, you typically need to read through all preceding records. Simple, suitable for batch processing.
  • Random (or Direct) File Organization: Records are stored based on a key or address, allowing them to be accessed directly without reading through other records. Requires a mechanism (like hashing or indexing) to map the key to a physical location. Faster for accessing specific records, suitable for interactive systems.

(d) C Program: Print Number Pattern using `while` loop

#include <stdio.h>

int main() {
    int row = 1; // Current row number
    int numInRow; // Counter for numbers within a row
    int startNum; // Starting number for the current row

    // Outer loop controls the rows (1 to 4)
    while (row <= 4) {
        startNum = row; // The first number in the row is the row number
        numInRow = 1;   // Reset the counter for numbers in this row

        // Inner loop prints the numbers for the current row
        // It should print 'row' number of items
        while (numInRow <= row) {
            printf("%d ", startNum);
            startNum++; // Increment the number to print
            numInRow++; // Increment the count of numbers printed in this row
        }
        printf("\n"); // Move to the next line after finishing a row
        row++; // Move to the next row
    }

    return 0;
}

(d) Pascal Program: Average Height using Procedure

program AverageHeight;

const
  NumStudents = 5;

type
  HeightArray = array[1..NumStudents] of Real; // Use Real for heights

var
  heights: HeightArray;
  i: Integer;
  avgHeight: Real;

// Procedure to calculate and output the average height
procedure CalculateAverageHeight(const h: HeightArray; var average: Real);
var
  j: Integer;
  sum: Real;
begin
  sum := 0.0;
  for j := 1 to NumStudents do
  begin
    sum := sum + h[j];
  end;

  if NumStudents > 0 then
    average := sum / NumStudents
  else
    average := 0.0; // Avoid division by zero

  Writeln;
  Writeln('Average height of the ', NumStudents, ' students is: ', average:0:2);
end;

begin
  // Accept heights
  Writeln('Enter the heights of ', NumStudents, ' students:');
  for i := 1 to NumStudents do
  begin
    Write('Height of student ', i, ': ');
    Readln(heights[i]);
  end;

  // Call procedure to calculate and display average
  CalculateAverageHeight(heights, avgHeight); // Pass array and variable for result

  Readln; // Pause
end.

Answer

(a) Definitions
  • (i) Record: A composite data structure that groups together related data items (fields or members) of potentially different data types under a single name. Allows treating related data as a single unit (e.g., a student record containing name, age, ID).
  • (ii) Linked List: A linear, dynamic data structure where elements (nodes) are not stored in contiguous memory locations. Each node contains data and a pointer (or link) to the next node in the sequence. Allows efficient insertion and deletion compared to arrays but lacks direct random access.
(b) Programming Terms
  • (i) Debugging: The systematic process of finding, analyzing, and removing errors (bugs) from computer programs. It involves running the program, observing its behavior, identifying the source of errors (syntax, runtime, or logical), and correcting the code.
  • (ii) Dry Running (Desk Checking): A manual process of tracing the execution of a program or algorithm step-by-step on paper, without using a computer. The programmer simulates the computer's actions, tracking variable values and program flow to understand the logic and detect potential errors before or during coding.
(c) (i) Selection Sort Logic (Sorting 4 1 3 2)

Selection sort works by repeatedly finding the minimum element from the unsorted part of the list and putting it at the beginning (or end) of the sorted part.

Logic Description:

  1. Pass 1:
    • Start with the first element (4) as the current minimum's position.
    • Scan the rest of the list (1, 3, 2) to find the actual minimum element (1).
    • Swap the minimum element (1) with the element at the first position (4).
    • List becomes: `1 4 3 2`. The first element (1) is now sorted.
  2. Pass 2:
    • Consider the unsorted part starting from the second element (4). Assume 4 is the minimum.
    • Scan the rest of the unsorted list (3, 2) to find the minimum (2).
    • Swap the minimum element (2) with the element at the second position (4).
    • List becomes: `1 2 3 4`. The first two elements (1, 2) are sorted.
  3. Pass 3:
    • Consider the unsorted part starting from the third element (3). Assume 3 is the minimum.
    • Scan the rest of the unsorted list (4). 3 is still the minimum.
    • No swap needed.
    • List remains: `1 2 3 4`. The first three elements (1, 2, 3) are sorted.
  4. The last element (4) is automatically in its correct place. The list is sorted.
(c) (ii) Global vs. Local Variable
  • Local Variable: Declared inside a specific function, procedure, or block of code. Its scope is limited to that block; it can only be accessed from within where it's declared. It is created when the block is entered and destroyed when the block is exited.
  • Global Variable: Declared outside of any function or procedure, typically at the beginning of the program file. Its scope is the entire program; it can be accessed (and potentially modified) from any function or procedure within the same file (or program, depending on linkage). It exists for the entire duration of the program's execution.

  • Abnormal/Invalid Data: Input data that falls outside the expected range or format (e.g., negative numbers where only positive are expected, letters where numbers are expected, out-of-range values). Used to test the program's error handling capabilities and robustness.

(c) `while` vs. `do while`
  • `while` loop: A pre-test loop. The condition is checked *before* the loop body is executed. If the condition is initially false, the loop body will never execute.
    Syntax (C): `while (condition) { /* statements */ }`
    Syntax (Pascal): `while condition do begin (* statements *) end;`
  • `do while` loop: A post-test loop. The loop body is executed *at least once*, and then the condition is checked *after* the execution. The loop continues as long as the condition is true.
    Syntax (C): `do { /* statements */ } while (condition);`
    Syntax (Pascal): `repeat (* statements *) until condition;` (Note: Pascal's `repeat..until` loops until the condition is TRUE, the opposite of C's `do..while`).

Answer

(a) Advantages of Binary Search
  • Efficiency: Significantly faster than sequential search for large datasets, with a time complexity of O(log n).
  • Reduced Comparisons: Requires fewer comparisons on average to find an element compared to sequential search, especially in large lists.

(Note: Requires the list to be sorted beforehand.)

(b) Tree Data Structure

A tree is a non-linear, hierarchical data structure consisting of nodes connected by edges. Key characteristics include:

  • A designated root node (the topmost node).
  • Each node (except the root) has exactly one parent node.
  • Nodes can have zero or more child nodes.
  • Nodes with no children are called leaf nodes.
  • There are no cycles (paths starting and ending at the same node without traversing the same edge twice).

Trees are used to represent hierarchical relationships, such as file systems, organization charts, or for efficient searching and sorting (e.g., Binary Search Trees, B-Trees).

(c) (i) Merge Sort vs. Insertion Sort
  • Merge Sort: A divide-and-conquer algorithm. It recursively divides the list into halves, sorts the halves, and then merges the sorted halves. Guaranteed O(n log n) time complexity but requires O(n) auxiliary space. Stable sort.
  • Insertion Sort: Builds the final sorted list one item at a time. It iterates through the input list and for each element, finds the correct position within the already sorted portion and inserts it there. Efficient for small lists or nearly sorted lists (O(n) best case), but O(n²) average/worst case. In-place sort (O(1) space). Stable sort.
