Question 1 (20 marks)

(a) Outline three advantages of structured programming languages. (3 marks)

(b) Differentiate between low level and high level programming languages. (4 marks)

(c) Table 1 shows a five-day schedule of trips for students in various departments. Use it to answer the questions that follow.

(i) Draw a flowchart to represent the logic of a program that could accept the day number and output the destination. (6 marks)

(ii) Using a switch statement, write a C program to implement the program logic. (7 marks)

Answer

(a) Advantages of Structured Programming Languages:

  • Readability/Understandability: Use of control structures (sequence, selection, iteration) and modularity (functions/procedures) makes code easier to read and follow.
  • Maintainability: Well-structured code is easier to debug, modify, and enhance because changes are often localized within specific modules.
  • Reduced Complexity: Breaking down problems into smaller modules reduces overall complexity compared to monolithic approaches.
  • Increased Productivity: Modularity allows for code reuse and easier team collaboration.
(Any three)

(b) Low-Level vs. High-Level Languages:

  • Low-Level Languages: Provide little or no abstraction from the computer's hardware architecture. They deal directly with processor instructions and memory addresses. Examples include machine code (binary) and assembly language. They offer high performance and control but are difficult to write, read, and port to different hardware.
  • High-Level Languages: Provide strong abstraction from hardware details. They use syntax closer to human language and offer features like variables, control structures, functions, and often object-orientation. Examples include C, Pascal, Python, Java. They are easier to write, read, maintain, and are generally more portable, but may have slightly lower performance than highly optimized low-level code.

(c) (ii) C Program (Trip Destination using Switch):

#include <stdio.h>

int main() {
    int dayNumber;
    char *destination; // Pointer to hold the destination string

    // Prompt user for the day number
    printf("Enter the day number (1-5): ");
    if (scanf("%d", &dayNumber) != 1) {
        printf("Invalid input. Please enter an integer.\n");
        return 1; // Indicate error
    }

    // Use switch statement to determine the destination
    switch (dayNumber) {
        case 1:
            destination = "Museum";
            break; // Exit switch
        case 2:
            destination = "National Park";
            break;
        case 3:
            destination = "Parliament";
            break;
        case 4:
            destination = "Cocacola Plant";
            break;
        case 5:
            destination = "Airport";
            break;
        default: // Handles any other number
            destination = "Invalid Day Entered";
            break;
    }

    // Output the destination
    printf("Day %d Destination: %s\n", dayNumber, destination);

    return 0;
}

Question 2 (20 marks)

(a) Explain each of the following statements as used in C programming;

(i) sentinel; (2 marks)

(ii) break; (2 marks)

(b) Explain two disadvantages of monolithic programming. (4 marks)

(c) Distinguish between source code and object code as used in programming. (4 marks)

(d) The area A of a triangle is found using the formula A = sqrt(S(s-a)(S-b)(S-c)) where a, b and c are the dimensions of a triangle and s = (a+b+c)/2.

Write a C program that would prompt for the three dimensions of a triangle, computes the area and display the results to the nearest 3 decimal places. (8 marks)

Answer

(a) C Programming Statements:

  • (i) sentinel: A sentinel value (or flag value) is a special value used in the context of loops or data processing to indicate the termination of input or a sequence of data. It's a value that would not normally occur as a valid data item, signaling that there is no more data to process. For example, using -1 to signal the end of a list of positive scores.
  • (ii) break: A control flow statement that, when executed within a loop (for, while, do...while) or a switch statement, causes the immediate termination of that innermost loop or switch. Program execution resumes at the statement immediately following the terminated construct.
(Similar to 2021j Q2d(i))

(b) Disadvantages of Monolithic Programming:

  • Low Maintainability: Changes in one part can have unexpected side effects elsewhere, making updates difficult and risky.
  • Poor Reusability: Tightly coupled code makes it hard to reuse specific functionalities in other contexts.
  • Difficult Debugging: Isolating errors is challenging due to the interconnected nature of the codebase.
  • Limited Scalability/Flexibility: Hard to modify or scale individual features; often requires recompiling/redeploying the entire application.
(Any two - similar to 2021n Q4a(i))

(c) Source Code vs. Object Code:

  • Source Code: Human-readable instructions written by a programmer in a high-level or assembly language.
  • Object Code: Machine-readable code (often machine code or bytecode) produced by a compiler or assembler by translating the source code. It's not easily understood by humans but can be executed by the computer (directly or via an interpreter/virtual machine).
(Similar to 2021n Q1b)

(d) C Program (Triangle Area - Heron's Formula):

#include <stdio.h>
#include <math.h> // Required for sqrt() function

int main() {
    double a, b, c; // Dimensions of the triangle sides
    double s;       // Semi-perimeter
    double area;

    // Prompt user for the three dimensions
    printf("Enter the length of side a: ");
    if (scanf("%lf", &a) != 1 || a <= 0) {
        printf("Invalid input for side a.\n");
        return 1;
    }

    printf("Enter the length of side b: ");
    if (scanf("%lf", &b) != 1 || b <= 0) {
        printf("Invalid input for side b.\n");
        return 1;
    }

    printf("Enter the length of side c: ");
    if (scanf("%lf", &c) != 1 || c <= 0) {
        printf("Invalid input for side c.\n");
        return 1;
    }

    // Validate if the sides can form a triangle (Triangle Inequality Theorem)
    if ((a + b <= c) || (a + c <= b) || (b + c <= a)) {
        printf("Error: The given dimensions do not form a valid triangle.\n");
        return 1;
    }

    // Calculate the semi-perimeter
    s = (a + b + c) / 2.0;

    // Calculate the area using Heron's formula
    // Ensure the argument to sqrt is non-negative (should be if triangle is valid)
    double term = s * (s - a) * (s - b) * (s - c);
    if (term < 0) {
        // This might happen due to floating-point inaccuracies for near-degenerate triangles
        term = 0;
    }
    area = sqrt(term);

    // Display the results
    printf("\nTriangle Dimensions: a=%.3lf, b=%.3lf, c=%.3lf\n", a, b, c);
    printf("Semi-perimeter (s): %.3lf\n", s);
    printf("Calculated Area: %.3lf\n", area); // Display to 3 decimal places

    return 0;
}

Question 4 (20 marks)

(a) (i) Outline two ways of checking the correctness of a program. (2 marks)

(ii) Describe two categories of data that could be used to test a program. (4 marks)

(b) Explain a circumstance under which each of the following parameter passing methods are most appropriate:

(i) pass by value; (2 marks)

(ii) pass by reference. (2 marks)

(c) Distinguish between technical and user documentation of a program. (4 marks)

(d) Write a Pascal program that would accept a positive integer. The program should then determine the square of the number and display the number and its square through the use of a procedure. (6 marks)

Answer

(a) (i) Ways of Checking Program Correctness:

  • Testing: Running the program with various sets of input data (test cases) and comparing the actual output against the expected output.
  • Code Reviews/Walkthroughs: Having peers review the source code to identify logical errors, style issues, or potential bugs.
  • Formal Verification: Using mathematical methods to prove that the program meets its specification (less common for general applications, used in critical systems).
  • Debugging: Using debugging tools to step through code execution, inspect variables, and identify the root cause of errors when testing reveals problems.
(Any two)

(a) (ii) Categories of Test Data:

  • Normal/Valid Data: Typical, expected input values for which the program is designed to work correctly.
  • Boundary Data: Input values at the edges or limits of valid ranges (e.g., minimum value, maximum value, empty input, largest possible input). Errors often occur at boundaries.
  • Invalid/Erroneous Data: Input values outside the valid range or of the wrong type (e.g., negative numbers where positive are expected, text where numbers are expected). Used to test the program's error handling capabilities.
  • Extreme Data: Very large or very small valid values, or large volumes of data, to test performance and handling of potential overflows or resource limits.
(Any two distinct categories with description)

(b) Parameter Passing Appropriateness:

  • (i) Pass by Value: Appropriate when the function/procedure needs to use the value of an argument but should *not* modify the original variable in the calling code. It protects the caller's data from unintended changes within the subprogram. Also suitable for passing small amounts of data where the copying overhead is negligible.
  • (ii) Pass by Reference: Appropriate when the function/procedure *needs* to modify the original argument variable in the calling code (e.g., to return multiple results or update a shared state). Also appropriate for passing large data structures (like arrays or records) to avoid the performance overhead of copying the entire structure.
(Similar to 2021j Q3c)

(c) Technical vs. User Documentation:

  • Technical Documentation: Written for other developers, system administrators, or maintainers. It describes the program's internal structure, design, algorithms, APIs, data structures, installation, and maintenance procedures. It focuses on *how* the system works internally.
  • User Documentation: Written for the end-users of the software. It explains how to install, configure, and use the program's features to accomplish tasks, often including tutorials, examples, and troubleshooting guides. It focuses on *what* the user can do with the software and *how* to do it from their perspective.
(Similar to 2021n Q3a)

Question 5 (20 marks)

(a) (i) State two operations that could be carried out in a queue data structure. (2 marks)

(ii) Describe a linked list as applied in programming. (2 marks)

(b) (i) State the meaning of each of the following file handling modes:

I. write (1 mark)

II. w (1 mark)

III. r (1 mark)

(ii) Given that x is a variable in C program that stores a numeric value, distinguish between x++ and ++x as used in the program operations. (4 marks)

(c) (i) Outline the steps involved in swapping two elements in an array during sorting. (4 marks)

(ii) With the aid of an illustration sort the following numbers in ascending order using a selection sort algorithm. (5 marks)

8, 4, 6, 12, 3, 2, 5

Answer

(a) (i) Queue Operations:

  • Enqueue (or Add, Insert): Adding an element to the rear of the queue.
  • Dequeue (or Remove, Delete): Removing the element from the front of the queue.
  • Peek (or Front): Viewing the element at the front of the queue without removing it.
  • IsEmpty: Checking if the queue contains any elements.
  • IsFull: Checking if the queue has reached its maximum capacity (for array-based implementations).
(Any two)

(a) (ii) Linked List Description:

A linked list is a linear dynamic data structure where elements (nodes) are not necessarily stored in contiguous memory locations. Each node contains the data element itself and at least one pointer (or link) that points to the next node in the sequence. This allows the list to grow or shrink dynamically during runtime.

(b) (i) C File Handling Modes Meaning:

  • I. `write` (Not a standard C mode): This is likely referring to the concept of writing, but standard C modes are strings like `"w"`, `"wb"`, `"a"`, `"ab"`, `"r+"`, `"w+"`, `"a+"`.
  • II. `"w"` (Write): Opens a file for writing. If the file exists, its contents are discarded (truncated). If the file does not exist, it is created.
  • III. `"r"` (Read): Opens an existing file for reading only. If the file does not exist, `fopen()` fails.

(b) (ii) `x++` vs. `++x` (C):

  • x++ (Postfix Increment): The expression evaluates to the *current* value of `x` *before* it is incremented. After the value is used (or the statement/expression completes), `x` is then incremented by 1.
  • ++x (Prefix Increment): `x` is incremented by 1 *first*, and then the expression evaluates to the *new*, incremented value of `x`.

Example: If `x = 5;`, then `y = x++;` results in `y=5` and `x=6`. If `x = 5;`, then `y = ++x;` results in `y=6` and `x=6`.

(c) (i) Steps for Swapping Two Array Elements:

To swap the values of `array[i]` and `array[j]`:

  1. Declare a temporary variable (`temp`) of the same data type as the array elements.
  2. Store the value of the first element (`array[i]`) in the temporary variable: `temp = array[i];`
  3. Assign the value of the second element (`array[j]`) to the first element's position: `array[i] = array[j];`
  4. Assign the value stored in the temporary variable to the second element's position: `array[j] = temp;`

(c) (ii) Selection Sort Illustration:

Array: [8, 4, 6, 12, 3, 2, 5]

Pass 1:

  • Find smallest in [8, 4, 6, 12, 3, 2, 5] -> 2 (at index 5)

Pass 3:

  • Find smallest in [6, 12, 4, 8, 5] -> 4 (at index 4)
  • Swap 4 with element at index 2 (6)
  • Array: [2, 3, 4, 12, 6, 8, 5]

Pass 4:

  • Find smallest in [12, 6, 8, 5] -> 5 (at index 6)
  • Swap 5 with element at index 3 (12)
  • Array: [2, 3, 4, 5, 6, 8, 12]

Pass 5:

  • Find smallest in [6, 8, 12] -> 6 (at index 4)
  • Swap 6 with element at index 4 (itself)
  • Array: [2, 3, 4, 5, 6, 8, 12]

Answer

(a) Characteristics of Reusable Code/Programs:

  • Modularity: Code is organized into independent modules (functions, procedures, classes) with well-defined interfaces and specific responsibilities.
  • Low Coupling: Modules have minimal dependencies on each other, meaning changes in one module are less likely to require changes in others.
  • High Cohesion: Elements within a single module are strongly related and focused on a single task.
  • Generality/Parameterization: Modules are designed to solve a general problem rather than a specific instance, often using parameters to adapt their behavior.
  • Good Documentation: Clear documentation explains what the code does, how to use it (API), and its limitations.

(b) (i) Random Access (File Organization):

Random access (or direct access) is a file organization method where records can be read or written directly at any location within the file, without needing to process preceding records. This is typically achieved using record addresses or keys that map to specific locations, allowing for fast retrieval or update of individual records. Disk drives inherently support random access.

(b) (ii) `gets()` vs. `puts()` (C):

  • gets(char *str): Reads a line of text from standard input (`stdin`) and stores it into the string pointed to by `str`. It stops reading when a newline character (`\n`) is encountered or end-of-file occurs. **Crucially, `gets()` is extremely unsafe because it doesn't check buffer boundaries, leading to potential buffer overflows if the input is longer than the buffer.** It should *never* be used; `fgets()` is the safe alternative. The newline character is read but not stored in the string; a null terminator (`\0`) is added instead.
  • puts(const char *str): Writes the string pointed to by `str` to standard output (`stdout`), followed by a newline character (`\n`). It stops writing when it encounters the null terminator (`\0`) in the string (the null terminator itself is not written).

(d) Pascal Program (Read and Display File):

Note: Accessing drive "D:" directly is specific to Windows and might not work on other systems. Standard Pascal usually works with files in the current directory or specified relative/absolute paths according to the OS. This example assumes the file is accessible.

program ReadStudentFile;

var
  studentFile: Text; // Declare a file variable of type Text
  fileName: string;
  lineContent: string;

begin
  // Specify the file name (adjust path as needed for your system)
  // fileName := 'D:\students.txt'; // Windows specific path
  fileName := 'students.txt'; // Assumes file is in the same directory as the program

  // Associate the file variable with the physical file
  Assign(studentFile, fileName);

  // Open the file for reading
  {$I-} // Disable I/O error checking temporarily
  Reset(studentFile);
  {$I+} // Re-enable I/O error checking

  // Check if the file was opened successfully
  if IOResult <> 0 then
  begin
    Writeln('Error: Could not open file "', fileName, '". Make sure it exists.');
  end
  else
  begin
    Writeln('--- Contents of ', fileName, ' ---');
    // Read and display each line until the end of the file
    while not Eof(studentFile) do
    begin
      Readln(studentFile, lineContent); // Read one line
      Writeln(lineContent);             // Display the line on the screen
    end;
    Writeln('--- End of File ---');

    // Close the file
    Close(studentFile);
  end;

  Readln; // Keep console window open
end.

Note: Added basic error handling for file opening using `IOResult`. The file path might need adjustment depending on the operating system and file location.

Question 7 (20 marks)

(a) Explain the term in-built function as used in C programming. (2 marks)

(b) Distinguish between a record and an array data structure as used in programming. (4 marks)

(c) (i) Describe sequential file organization as used in programming. (2 marks)

(ii) Using binary search algorithm, state the steps used to search for a value 45 in the following list of numbers: (6 marks)

12 15 18 20 25 30 45 50 75

(d) Write a Pascal program that would display all the odd numbers from 1 to 29 alongside their squares. Use the for loop. (6 marks)

Answer

(a) In-built Function (C):

An in-built function (also called a standard library function or built-in function) is a pre-defined function provided as part of the C language standard library. These functions perform common tasks (like input/output, mathematical calculations, string manipulation, memory allocation) and are available for programmers to use without having to write the code for them from scratch. Examples include `printf()`, `scanf()`, `sqrt()`, `strcpy()`, `malloc()`.

(b) Record vs. Array:

  • Array: A collection of elements of the *same* data type, stored in contiguous memory locations and accessed using an integer index (or subscript). Used for storing lists or sequences of homogeneous data.
  • Record (or Struct in C): A collection of elements (called fields or members) that can be of *different* data types, grouped together under a single name. Used for representing an entity with multiple related attributes (e.g., a student record with name, age, ID). Fields are accessed using their names (e.g., `student.name`).

(c) (i) Sequential File Organization:

Records are stored one after another in a predetermined order, usually based on a key field. To access a specific record, one typically has to read through all the preceding records from the beginning of the file. It's simple and efficient for applications that process most or all records in sequence (batch processing).

(Similar to 2021j Q5b)

(c) (ii) Binary Search Steps (Search for 45):

List: [12, 15, 18, 20, 25, 30, 45, 50, 75] (Sorted)

Indices: 0, 1, 2, 3, 4, 5, 6, 7, 8

  1. Step 1:
    • Set `low = 0`, `high = 8`.
    • Calculate `mid = floor((0 + 8) / 2) = 4`.
    • Compare `searchValue` (45) with `list[mid]` (list[4] = 25).
    • 45 > 25, so search the upper half. Set `low = mid + 1 = 5`.
  2. Step 2:
    • `low = 5`, `high = 8`.
    • Calculate `mid = floor((5 + 8) / 2) = 6`.
    • Compare `searchValue` (45) with `list[mid]` (list[6] = 45).
    • 45 == 45. Value found at index 6.
  3. Step 3: Terminate search.

Result: Value 45 found at index 6.

Answer

(a) Function of C Escape Characters:

  • (i) \t: Horizontal Tab - Moves the cursor to the next horizontal tab stop position on the current line.
  • (ii) \n: Newline - Moves the cursor to the beginning of the next line.

(b) (i) General Format for Declaring a Structure (C):

struct StructureTagName {
    DataType1 memberName1;
    DataType2 memberName2;
    // ... more members ...
    DataTypeN memberNameN;
}; // Don't forget the semicolon at the end

// Optional: Using typedef for easier variable declaration
typedef struct {
    DataType1 memberName1;
    DataType2 memberName2;
    // ... more members ...
    DataTypeN memberNameN;
} TypeAliasName; // Now you can declare variables as: TypeAliasName myVariable;

Explanation:

  • struct: Keyword to declare a structure.
  • StructureTagName: An optional name (tag) for the structure type.
  • DataTypeX: The data type of a member (e.g., `int`, `float`, `char`, another struct).
  • memberNameX: The name of a member variable within the structure.
  • {}: Braces enclose the list of members.
  • ;: A semicolon is required after the closing brace.
  • typedef (optional): Creates an alias for the structure type, making variable declarations simpler.

(b) (ii) Reasons for Using Functions:

  • Modularity/Organization: Break down complex problems into smaller, manageable, logical units, improving code structure and readability.
  • Reusability: Write a piece of code once (within a function) and call it multiple times from different parts of the program, reducing redundancy.
  • Abstraction: Hide implementation details. Callers only need to know what the function does (its interface), not how it does it.
  • Maintainability/Debugging: Easier to test, debug, and modify smaller, independent functions than a large block of code.

(c) Flowchart for Repeat-Until Loop (Pascal):

+-------------------+
|       ...         | // Code before loop
+-------------------+
    |
    V
+-------------------+  <--+ // Loop entry point
|                   |     |
|   Execute Loop    |     |
|       Body        |     |
|                   |     |
+-------------------+     |
    |                     |
    V                     |
+-------------------+ NO  |
| Is Condition True?|-----+
+-------------------+
    | YES
    V
+-------------------+
|       ...         | // Code after loop
+-------------------+

Description:

  1. The statements within the loop body are executed *at least once*.
  2. After executing the body, the `UNTIL` condition is evaluated.
  3. If the condition is `FALSE`, control jumps back to the beginning of the loop body, and it executes again.
  4. If the condition is `TRUE`, the loop terminates, and execution continues with the statement following the loop.
