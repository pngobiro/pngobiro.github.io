Question 1 (20 marks)

(a) (i) Outline three types of errors that can occur during program development. (3 marks)

(ii) Describe two hardware requirements necessary for installation of programming software in a computer. (4 marks)

(b) Differentiate between a robust program and a legible program as used in program design. (4 marks)

(c) With the aid of an example, describe the term escape sequence as used in C programming language. (3 marks)

(d) Write a Pascal program that allows the user to key in an amount of money in Kenya shillings. The program should then calculate the equivalent of the amount in either dollar or pound through the use of functions and output the amount keyed in and dollar or pound equivalent. Use the rates: 1 dollar = Ksh 80 and 1 pound = Ksh 120. (6 marks)

Answer

(a) (i) Types of Program Errors
  • Syntax Errors: Violations of the programming language's grammar rules (e.g., missing semicolons, misspelled keywords). Detected by the compiler/interpreter.
  • Runtime Errors: Errors that occur during program execution (e.g., division by zero, accessing invalid memory, file not found). Cause program crashes or abnormal termination if not handled.
  • Logical Errors: Errors in the program's logic that cause it to produce incorrect or unexpected results, even if it runs without crashing (e.g., using the wrong formula, incorrect conditional logic).
(a) (ii) Hardware Requirements for Programming Software
  • Processor (CPU): Needs sufficient processing power (speed, cores) to handle compilation, execution, and potentially IDE operations smoothly. Minimum requirements vary by software.
  • Memory (RAM): Requires enough RAM to load the operating system, the programming environment (IDE), the compiler/interpreter, and the program being developed/run without excessive swapping to disk.
  • Storage (Hard Drive/SSD): Needs sufficient disk space to install the programming tools (compiler, IDE, libraries) and store source code files, project files, and compiled executables.
(b) Robust vs. Legible Program
  • Robust Program: Designed to handle unexpected inputs or error conditions gracefully without crashing. It anticipates potential problems (e.g., invalid user input, file errors) and includes error handling mechanisms.
  • Legible Program: Written in a clear, understandable, and well-formatted way. Uses meaningful variable names, consistent indentation, comments, and modular structure to make the code easy for humans to read, understand, and maintain.
(c) Escape Sequence in C

An escape sequence is a sequence of characters starting with a backslash (`\`) used within string or character literals to represent special characters that are difficult or impossible to type directly or have special meaning.

Example: The sequence `\n` represents the newline character. When used in `printf`, it moves the cursor to the beginning of the next line.


printf("Hello\nWorld!");

Output:

Hello
World!

Other examples include `\t` (tab), `\\` (backslash), `\"` (double quote).

Answer

(a) Identifier Definition

An identifier is a user-defined name given to entities like variables, constants, functions, procedures, types, etc., in a program. It allows the programmer to refer to these entities symbolically in the code. Identifiers must follow specific naming rules defined by the programming language (e.g., starting with a letter, containing only letters, numbers, and underscores).

(b) (i) Structure Chart

A structure chart is a top-down diagram used in program design that shows the overall architecture of a program. It illustrates how different modules (functions or procedures) are organized and how they relate to each other, including the flow of data and control between them. It helps visualize the program's hierarchy and decomposition into smaller, manageable parts.

(b) (ii) Flowchart Advantages and Disadvantages

Advantages:

  • Visual Clarity: Provides a clear graphical representation of program logic and flow, making it easy to understand, especially for simple algorithms.
  • Effective Communication: Serves as a good communication tool between programmers and non-technical stakeholders to explain program logic.
  • Debugging Aid: Can help in tracing the logic flow and identifying errors during the design or debugging phase.

Disadvantages:

  • Time-Consuming & Complex: Can become very large, complex, and difficult to draw/maintain for complex programs. Modifications can be tedious.
  • Lack of Detail: May not capture all the nuances or details of the implementation compared to pseudocode or actual code.
  • Non-Standard Symbols (Historically): While standards exist, variations in symbol usage could sometimes lead to ambiguity.
(c) (i) Control Structures for Reading Text File Records

To read all records from a text file until the end is reached, the following control structures are commonly used:

  • While Loop: A `while not EOF(fileVariable) do ...` loop continues reading records as long as the end-of-file marker has not been reached.
  • Repeat...Until Loop: A `repeat ... until EOF(fileVariable)` loop can also be used, although it processes the loop body at least once, which might require careful handling if the file could be empty.

(d) C Program: Calculate Book Pages

#include <stdio.h>

// Function to calculate the number of pages
int calculatePages(int totalWords, int wordsPerPage) {
    if (wordsPerPage <= 0) {
        return -1; // Indicate error for invalid wordsPerPage
    }
    // Use ceiling division: (numerator + denominator - 1) / denominator
    // Or cast to float for division and use ceil() from math.h
    // Simple integer division might truncate pages
    int pages = (totalWords + wordsPerPage - 1) / wordsPerPage;
    return pages;
}

int main() {
    int totalWordsInBook;
    int wordsPerPageCount;
    int numberOfPages;

    // Prompt user for input
    printf("Enter the total number of words in the book: ");
    scanf("%d", &totalWordsInBook);

    printf("Enter the number of words per page: ");
    scanf("%d", &wordsPerPageCount);

    // Validate input
    if (totalWordsInBook < 0 || wordsPerPageCount <= 0) {
        printf("\nError: Invalid input. Words and words per page must be positive.\n");
        return 1; // Indicate error
    }

    // Calculate pages using the function
    numberOfPages = calculatePages(totalWordsInBook, wordsPerPageCount);

    // Output the results
    printf("\n--- Book Details ---\n");
    printf("Total words in book: %d\n", totalWordsInBook);
    printf("Words per page: %d\n", wordsPerPageCount);
    printf("Calculated number of pages: %d\n", numberOfPages);

    return 0;
}

Question 3 (20 marks)

(a) (i) Define the term symbolic constant as used in C programming. (1 mark)

(ii) Describe each of the following programming design concepts:

  • I. modular; (2 marks)
  • II. bottom up. (2 marks)

(b) Differentiate between a procedure and a function as used in Pascal programming. (4 marks)

(c) With the aid of a flowchart, describe the flow of instruction in a repeat...until loop as used in programming. (4 marks)

(d) Write a C program that accepts 10 integers from the keyboard, stores them in an array and then accepts an integer search element, searches through the array to check whether the search element exists. The program should then output the 10 numbers and a comment on whether the element is found or not. (7 marks)

Answer

(a) (i) Symbolic Constant (C)

A symbolic constant in C is a name given to a constant value, typically defined using the `#define` preprocessor directive. The preprocessor replaces every occurrence of the symbolic constant's name in the code with its defined value *before* compilation begins. It's a form of text substitution.

Example: `#define PI 3.14159`

(a) (ii) Programming Design Concepts
  • I. Modular Design: Breaking down a large program into smaller, independent, and reusable modules (like functions or procedures). Each module performs a specific task, making the program easier to design, code, test, debug, and maintain. It promotes code organization and reusability.
  • II. Bottom-Up Design: A design approach where individual low-level modules or components are designed and implemented first. These components are then combined and integrated to build larger modules and eventually the complete system. It focuses on building blocks and then assembling them.
(b) Pascal Procedure vs. Function
  • Procedure: A block of code designed to perform a specific task or action. It does *not* return a value directly to the point where it was called. It can modify parameters passed by reference (VAR parameters). Declared using the `procedure` keyword.
  • Function: A block of code designed to perform a calculation or task and *return a single value* of a specific data type to the point where it was called. The function name itself is used within the function body to assign the return value. Declared using the `function` keyword, followed by the parameters and the return type (e.g., `: integer`).
(c) Repeat...Until Loop Flowchart
+---------------------+
│ START LOOP          │
+----------+----------+
           │
           ▼
+----------+----------+
│ Execute Loop Body   │
│ (Statements)        │
+----------+----------+
           │
           ▼
+----------+----------+ Yes
│ Condition Met?      │◄----┐
│ (UNTIL condition)   │    │ No
+---------------------+    │
           │                 │
           └-----------------┘
           │ Yes
           ▼
+----------+----------+
│ END LOOP            │
+---------------------+

Description: The `REPEAT...UNTIL` loop first executes the block of statements within its body *at least once*. After executing the body, it evaluates the specified `UNTIL` condition. If the condition is `FALSE`, the loop repeats, executing the body again. If the condition is `TRUE`, the loop terminates, and program execution continues with the statement following the `UNTIL` keyword.

(d) C Program: Array Search

#include <stdio.h>

#define SIZE 10 // Define the size of the array

int main() {
    int numbers[SIZE];
    int i, searchElement, found = 0; // found flag initialized to 0 (false)

    // Prompt user to enter 10 integers
    printf("Enter %d integers:\n", SIZE);
    for (i = 0; i < SIZE; i++) {
        printf("Enter integer %d: ", i + 1);
        scanf("%d", &numbers[i]);
    }

    // Prompt user for the search element
    printf("\nEnter the integer to search for: ");
    scanf("%d", &searchElement);

    // Search through the array
    for (i = 0; i < SIZE; i++) {
        if (numbers[i] == searchElement) {
            found = 1; // Set found flag to 1 (true)
            break;     // Exit loop once element is found
        }
    }

    // Output the 10 numbers entered
    printf("\nThe numbers entered were: ");
    for (i = 0; i < SIZE; i++) {
        printf("%d ", numbers[i]);
    }
    printf("\n");

    // Output the search result
    if (found == 1) { // Check the flag
        printf("%d was found in the array.\n", searchElement);
    } else {
        printf("%d was not found in the array.\n", searchElement);
    }

    return 0;
}

(c) (i) Binary Search vs. Sequential Search
  • Sequential Search: Checks each element in a list one by one, starting from the beginning, until the target element is found or the end of the list is reached. Works on both sorted and unsorted lists. Time complexity is O(n) in the worst/average case.
  • Binary Search: Requires the list to be **sorted**. It repeatedly divides the search interval in half. It compares the target value to the middle element; if they don't match, the half in which the target cannot lie is eliminated, and the search continues on the remaining half. Much faster for large, sorted lists. Time complexity is O(log n).

(d) Pascal Program: Read Student Scores from File

program StudentScoreCalculator;

uses SysUtils; // For StrToIntDef or other conversions if needed, and file handling

type
  TStudentRec = record
    StudentName: string[50];
    Maths, CompAppl, OS: Integer;
    Total: Integer;
    Average: Real;
  end;

var
  examFile: TextFile;
  student: TStudentRec;
  lineText: string;
  scoreStr: string;
  i, startPos: Integer;

// Helper function to safely parse integer from substring
function GetScore(const s: string; var pos: Integer): Integer;
var
  valStr: string;
begin
  valStr := '';
  // Skip leading spaces
  while (pos <= Length(s)) and (s[pos] = ' ') do Inc(pos);
  // Read digits
  while (pos <= Length(s)) and (s[pos] in ['0'..'9']) do
  begin
    valStr := valStr + s[pos];
    Inc(pos);
  end;
  // Convert string to integer, default to 0 on error
  Result := StrToIntDef(valStr, 0);
end;


begin
  Assign(examFile, 'C:\exam.txt'); // Assuming file is named exam.txt

  // Check if file exists before trying to open
  if not FileExists('C:\exam.txt') then
  begin

    Readln;
    Halt(1); // Exit program
  end;

  {$I-} // Disable I/O error checking
  Reset(examFile);
  {$I+} // Re-enable I/O error checking

  if IOResult <> 0 then
  begin
    Writeln('Error opening file C:\exam.txt for reading.');
    Readln;
    Halt(1);
  end;

  // Read and discard the header line
  if not Eof(examFile) then
    Readln(examFile, lineText);

  // Process each student record line
  while not Eof(examFile) do
  begin
    Readln(examFile, lineText);

    // Basic parsing assuming somewhat fixed-width or space-separated
    // This is fragile and depends heavily on exact file format
    startPos := 1;

    // Extract Name (assuming it ends before first score) - Adjust width as needed
    student.StudentName := Trim(Copy(lineText, startPos, 12)); // Adjust 12 based on expected name width
    startPos := 13; // Move past name area

    // Extract Scores
    student.Maths := GetScore(lineText, startPos);
    student.CompAppl := GetScore(lineText, startPos);
    student.OS := GetScore(lineText, startPos);

    // Calculate Total and Average
    student.Total := student.Maths + student.CompAppl + student.OS;
    if (student.Maths >= 0) and (student.CompAppl >= 0) and (student.OS >= 0) then // Basic check
       student.Average := student.Total / 3.0

(a) (i) Loop Terms
  • I. Endless Loop (Infinite Loop): A loop whose condition for termination is never met, causing it to execute indefinitely until manually interrupted or the program runs out of resources. Often caused by logical errors in the loop's condition or update statement.
  • II. Nested Loop: A loop structure where one loop is placed inside the body of another loop. The inner loop executes completely for each single iteration of the outer loop. Used for tasks involving multi-dimensional data or repetitive actions within repetitions.
  • III. Branching Loop: This is not standard terminology. It likely refers to a loop that contains branching statements like `if-else` or `switch` within its body, allowing different actions based on conditions during each iteration. Alternatively, it might loosely refer to using `break` or `continue` statements to alter the normal flow of the loop.
(a) (ii) C `switch` Statement Syntax

switch (expression) {
    case constant_expression_1:
        // statements for case 1
        break; // Optional: exits the switch
    case constant_expression_2:
        // statements for case 2
        break;
    // ... more cases
    default: // Optional: executes if no other case matches
        // statements for default case
        break;
}

The `expression` must evaluate to an integral type (int, char). Each `case` label must be a unique constant integral value. Execution falls through to the next case if `break` is omitted.

(b) Formal Parameter

A formal parameter is a variable declared in the header (signature) of a function or procedure. It acts as a placeholder that receives a value (or a reference/address) passed from the calling part of the program when the function/procedure is invoked. The actual value/variable passed during the call is called an argument (or actual parameter).

Example (Pascal): In `procedure PrintSum(a, b: integer);`, `a` and `b` are formal parameters.

(c) Use of Extreme Data in Testing

Extreme data (also known as boundary value data) is used during program testing to check the program's behavior at the edges or limits of valid input ranges. This is crucial because errors often occur at these boundaries.

Circumstances:

  • Testing conditions right at the minimum or maximum allowed values (e.g., if input must be 1-100, test with 1 and 100).
  • Testing values just inside and just outside the valid range (e.g., test with 0, 1, 100, 101 for the 1-100 range).
  • Testing with empty or null inputs if applicable (e.g., empty string, zero value).

This helps ensure the program handles edge cases correctly and doesn't fail unexpectedly at the limits.

(a) (i) Index (Program Documentation)

In program documentation (like a user manual or technical reference), an index is an alphabetical list of keywords, topics, function names, or concepts mentioned in the document, along with the page numbers where they can be found. It helps users quickly locate specific information within the documentation.

(a) (ii) Signed vs. Unsigned Integers (C)
  • Signed Integers (`int`, `short`, `long`, `long long`): Can represent both positive and negative whole numbers, plus zero. A portion of the bits (usually the most significant bit) is used to store the sign (0 for positive, 1 for negative). This limits the maximum positive value compared to unsigned integers of the same size.
  • Unsigned Integers (`unsigned int`, `unsigned short`, etc.): Can only represent non-negative whole numbers (zero and positive values). All bits are used to represent the magnitude of the number, allowing them to store larger positive values than their signed counterparts of the same size.
(b) Reasons for Dry Running Procedures with Stack Overflow

A "Stack Overflow" error typically occurs due to excessive function calls, often from infinite or very deep recursion, consuming all available space on the call stack.

  1. Detect Infinite Recursion: Dry running (manually tracing the execution step-by-step) the procedures suspected of causing the overflow can help identify if a recursive function fails to reach its base case, leading to endless calls that fill the stack.
  2. Identify Deep Call Chains: Tracing the sequence of procedure/function calls can reveal unexpectedly deep nesting or long chains of calls (even without recursion) that might exhaust stack memory, especially if large local variables are declared within those functions.
(c) Pseudocode: Dividend Calculation

BEGIN
  DECLARE memberName AS String
  DECLARE memberNumber AS String // Or Integer
  DECLARE monthsInSociety AS Integer
  DECLARE sharesAmount AS Real
  DECLARE dividendRate AS Real
  DECLARE dividendAmount AS Real

  INPUT memberName
  INPUT memberNumber
  INPUT monthsInSociety
  INPUT sharesAmount

  // Determine dividend rate based on conditions
  IF monthsInSociety >= 6 THEN
    IF sharesAmount > 50000 THEN
      dividendRate = 0.06 // 6%
    ELSE
      dividendRate = 0.04 // 4%
    END IF
  ELSE
    dividendRate = 0.0 // No dividend if member for less than 6 months
  END IF

  // Calculate dividend amount
  dividendAmount = sharesAmount * dividendRate

  // Output results
  DISPLAY "Member Name: ", memberName
  DISPLAY "Member Number: ", memberNumber
  DISPLAY "Shares: Kshs ", sharesAmount
  DISPLAY "Months in Society: ", monthsInSociety
  DISPLAY "Dividend Rate: ", (dividendRate * 100), "%"
  DISPLAY "Dividend Amount: Kshs ", dividendAmount
END
(d) C Program: Print Number Pattern

#include <stdio.h>

int main() {
    int i, j;

    // Outer loop controls the number of rows (from 1 down to 5 implicitly)
    // Or more clearly, control the end number for the inner loop
    for (i = 1; i <= 5; i++) { // Controls which number the inner loop prints down to

        for (j = 5; j >= i; j--) {

        }

    }

(a) (i) Advantages of Quick Sort
  • Fast Average Performance: Generally performs very well on average, with a time complexity of O(n log n), making it one of the fastest comparison-based sorts for typical data.
  • In-Place Sorting (Mostly): It typically requires only a small, logarithmic amount of auxiliary memory for the recursion stack (O(log n) space), making it memory-efficient compared to sorts like Merge Sort which require O(n) auxiliary space.
(a) (ii) Modulation vs. Demodulation

(Note: This topic relates more to data communications than typical structured programming)

  • Modulation: The process of varying one or more properties (amplitude, frequency, or phase) of a periodic waveform (the carrier signal) with a modulating signal (typically containing information) to transmit that information. Essentially, encoding information onto a carrier wave suitable for transmission over a medium.
  • Demodulation: The reverse process of extracting the original information-bearing signal (the modulating signal) from the modulated carrier wave received at the destination.
(b) Justification for Using Data Files
  • Persistence: Data stored in files remains available even after the program terminates, unlike data stored only in RAM (variables), which is lost. This allows data to be saved and reused across multiple program runs.
  • Large Data Volumes: Files can store much larger amounts of data than can typically fit into the computer's main memory (RAM) at one time.
  • Data Sharing: Files provide a standard way for different programs or different users to share or exchange data.

(c) (i) Advantages of Using Subprograms
  • Modularity: Breaks down a complex program into smaller, manageable, and logical units, making the code easier to understand, develop, and organize.
  • Reusability: Subprograms written for a specific task can be called multiple times from different parts of the program (or even used in other programs), reducing code duplication.

(c) (ii) Comma Operator in C

The comma operator (`,`) links two or more expressions into a single expression, where the value of the entire comma-separated expression is the value of the rightmost expression. The expressions are evaluated from left to right.

It is most commonly used in `for` loops to include multiple initialization or update steps.

Example:


#include <stdio.h>

int main() {
    int i, j;
    // Using comma operator in for loop update section
    for (i = 0, j = 10; i < j; i++, j--) {
        printf("i=%d, j=%d\n", i, j);
    }

    // Comma operator in a regular expression
    int x = (printf("Assigning... "), 5); // Prints "Assigning... " and assigns 5 to x
    printf("x = %d\n", x); // Prints x = 5

    return 0;
}

In the `for` loop, `i++, j--` are evaluated sequentially in each iteration. In the assignment to `x`, `printf` is executed first, and then `5` is evaluated and becomes the result of the whole expression, which is assigned to `x`.
