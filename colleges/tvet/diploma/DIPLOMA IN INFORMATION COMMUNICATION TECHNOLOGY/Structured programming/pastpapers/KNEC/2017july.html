(a) (i) Explain one disadvantage of using goto statement in a program. (2 marks)

(ii) Distinguish between procedural and visual programming. (4 marks)

(b) A student would like to write a program that could compute and display the average of 10 integers entered through the keyboard one at a time. Use a flow chart to design the program. (4 marks)

(c) Assuming Pascal programming language, evaluate the expression;

Answer

(a) (i) Disadvantage of `goto` Statement:

The primary disadvantage is that it makes program logic difficult to follow and understand. Unconditional jumps can lead to unstructured "spaghetti code," making debugging, maintenance, and modification significantly harder. It breaks the clear flow of control encouraged by structured programming constructs (sequence, selection, iteration).

(a) (ii) Procedural vs. Visual Programming:

  • Procedural Programming: Focuses on defining a sequence of steps or procedures (functions, subroutines) that the computer executes to accomplish a task. The emphasis is on the 'how' – the step-by-step instructions. Examples: C, Pascal, Fortran.
  • Visual Programming: Allows developers to create programs by manipulating graphical elements (icons, blocks, forms) in an Integrated Development Environment (IDE) rather than primarily writing textual code. It often involves designing user interfaces visually and connecting predefined components or event handlers. Examples: Visual Basic (form design), Scratch, LabVIEW.

(c) Pascal Expression Evaluation:

Expression: Z = a + b mod c * (d * 2)

Given: a=10, b=23, c=7, d=35

Order of Operations (Pascal): Parentheses first, then `*`, `/`, `div`, `mod` (left-to-right), then `+`, `-` (left-to-right).

  1. Parentheses: (d * 2) -> (35 * 2) = 70
  2. Expression becomes: Z = 10 + 23 mod 7 * 70
  3. `mod` and `*` (Left-to-Right):
    • 23 mod 7 -> 2 (Remainder of 23 / 7)
    • 2 * 70 -> 140
  4. Expression becomes: Z = 10 + 140
  5. Addition: 10 + 140 = 150

Result: Z = 150

(d) (i) Identifier Validity (C):

  • myval: Valid. Reason: Starts with a letter and contains only letters and digits.
  • const: Invalid. Reason: It is a reserved keyword in C used for declaring constants.
  • integer: Valid. Reason: Starts with a letter and contains only letters. (Although potentially confusing as it mimics a type name, it's not a keyword).
  • switch: Invalid. Reason: It is a reserved keyword in C used for the switch control structure.

(d) (ii) Function of Preprocessor Directives (C):

Preprocessor directives are instructions processed by the C preprocessor *before* the actual compilation begins. Their main function is to modify the source code text based on the directives given. Common functions include: including header files (`#include`), defining macros and symbolic constants (`#define`), and conditional compilation (`#if`, `#ifdef`, `#ifndef`, `#else`, `#elif`, `#endif`).

(c) (ii) C Program (Random Numbers 0 to 1):

#include <stdio.h>
#include <stdlib.h> // Required for rand() and srand()
#include <time.h>   // Required for time()

int main() {
    int i;
    int count = 10; // Number of random numbers to generate

    // Seed the random number generator using the current time
    // This ensures different sequences each time the program runs
    srand(time(NULL));

    printf("Generating %d random numbers between 0.0 and 1.0:\n", count);

    for (i = 0; i < count; ++i) {
        // rand() generates an integer between 0 and RAND_MAX
        // Divide by RAND_MAX (as a double) to get a value between 0.0 and 1.0
        double random_value = (double)rand() / RAND_MAX;
        printf("%.4f ", random_value); // Print with 4 decimal places
    }
    printf("\n");

    return 0;
}

Note: `rand()` produces pseudo-random integers. Dividing by `RAND_MAX` scales the result to the [0.0, 1.0] range. `srand(time(NULL))` is crucial for seeding to get different results on different runs.

Answer

(a) (i) Pointer (Pascal):

A pointer is a special type of variable that holds the memory address of another variable. Instead of storing data directly, it stores the location where data can be found. Pointers are used for dynamic memory allocation, creating linked data structures (like lists and trees), and passing parameters by reference indirectly.

(a) (ii) `register` vs. `static` Storage Classes (C):

  • register: A storage class specifier that suggests to the compiler that the variable should be stored in a CPU register (if possible) for faster access. It can only be applied to local variables (including function parameters). You cannot take the address (`&`) of a register variable. The compiler may ignore the suggestion if no registers are available or if the variable type is unsuitable.
  • static:
    • When used with a *local* variable (inside a function): It gives the variable static duration, meaning it retains its value between function calls, unlike automatic local variables which are recreated each time. Its scope remains local to the function.
    • When used with a *global* variable or function: It limits the variable's or function's linkage to the current source file (internal linkage), meaning it cannot be accessed directly from other source files using `extern`.

(b) C Program (Prime Number Check):

#include <stdio.h>

(b) (ii) C Program Interpretation:

Corrected Code (as interpreted in question setup):

#include <stdio.h>

int main () {
    int a = 0; int b = 15; int c ; // c is uninitialized
    if (a && b ) { // Logical AND: a is 0 (false), b is 15 (true)
        printf("Line 1 - Condition is true\n" );
    }
    // Assuming 'all b' meant 'a || b'
    if (a || b) { // Logical OR: a is 0 (false), b is 15 (true)
        printf("Line 2 - Condition is true\n" );
    }
    return 0;
}

Interpretation:

  1. Variables `a`, `b`, `c` are declared. `a` is 0, `b` is 15, `c` is uninitialized.
  2. First `if` condition: `a && b`.
    • In C, 0 is treated as false, and non-zero values (like 15) are treated as true.
    • The expression becomes `false && true`.
    • Logical AND (`&&`) requires both operands to be true to result in true. Since the left operand (`a`) is false, the result is false.
    • Due to short-circuiting, `b` is not even evaluated.
    • The body of the first `if` statement is skipped.
  3. Second `if` condition: `a || b`.
    • The expression becomes `false || true`.
    • Logical OR (`||`) requires at least one operand to be true to result in true. Since the right operand (`b`) is true, the result is true.
    • The body of the second `if` statement is executed.
  4. `printf("Line 2 - Condition is true\n" );` is executed.
  5. The program returns 0.

Output:

Line 2 - Condition is true

(b) Algorithm to Remove Item from Queue (Dequeue):

Assumptions: Array-based circular queue with `front`, `rear` indices and `count`.

PROCEDURE Dequeue(queueArray, front, rear, count, maxSize)
  IF count = 0 THEN // Check if queue is empty
    PRINT "Error: Queue Underflow"
    RETURN ErrorIndicator // Or handle error appropriately
  ELSE
    // Get the item from the front
    SET removedItem = queueArray[front]

    // Move the front index forward (circularly)
    SET front = (front + 1) MOD maxSize

    // Decrement the count of items
    DECREMENT count

    RETURN removedItem
  END IF
END PROCEDURE
(Note: Error handling and return value mechanism may vary based on implementation details).

Answer

(a) Advantages of Modular Programming:

  • Easier Development/Management: Breaks down a large, complex program into smaller, more manageable units (modules), making development and understanding easier.
  • Reusability: Modules designed for specific tasks can often be reused in different parts of the same program or in other programs, saving development time.
  • Improved Maintainability: Changes or bug fixes are often localized to specific modules, reducing the risk of introducing errors elsewhere in the system.
  • Team Collaboration: Different developers or teams can work on separate modules concurrently.
(Any two)

(b) (i) I. Binary Tree Construction:

Data items: 56, 64, 40, 59, 34, 72, 47

      56
     /  \
    40   64
   / \   / \
  34 47 59 72

Insertion Steps: 56(root), 64(R), 40(L), 59(R-L), 34(L-L), 72(R-R), 47(L-R)

(b) (i) II. Degree of the Binary Tree:

The degree of a tree is the maximum degree of any node in the tree. The degree of a node is the number of children it has. In this tree:

  • Nodes 56, 40, 64 have degree 2.
  • Nodes 34, 47, 59, 72 have degree 0.

The maximum degree is 2. Therefore, the degree of the binary tree is 2.

(b) (ii) Tree Data Structure Traversals:

  • Pre-order Traversal: Visit the Root node first, then recursively traverse the Left subtree, then recursively traverse the Right subtree (Root-Left-Right).
  • In-order Traversal: Recursively traverse the Left subtree first, then visit the Root node, then recursively traverse the Right subtree (Left-Root-Right). For a Binary Search Tree, this yields nodes in sorted order.
  • Post-order Traversal: Recursively traverse the Left subtree first, then recursively traverse the Right subtree, then visit the Root node (Left-Right-Root).

            printf("%.2lf KSH is equivalent to %.2lf USD\n", amount, convertedAmount);
        } else {
             printf("Invalid KSH amount.\n");
        }
    } else if (strcmp(currencyType, "USD") == 0) {
        convertedAmount = usdToKsh(amount);
         if (convertedAmount >= 0) {
            printf("%.2lf USD is equivalent to %.2lf KSH\n", amount, convertedAmount);
         } else {
             printf("Invalid USD amount.\n");
         }
    } else {
        printf("Invalid currency type entered. Please enter KSH or USD.\n");
    }

    return 0;
}

Answer

(a) Approaches for Stepwise Refinement:

  • Top-Down Design: Starts with the overall problem/function and breaks it down into smaller, more manageable sub-problems or modules. Each sub-problem is then further refined until the steps are simple enough to be implemented directly. Focuses on the high-level structure first.
  • Bottom-Up Design: Starts by identifying and implementing the lowest-level, basic functions or utilities first. These are then combined to build larger, more complex modules, eventually forming the complete program. Focuses on building blocks first.

(b) Pre-increment vs. Post-increment (C):

  • ++x (Prefix Increment): The variable `x` is incremented by 1 *first*, and then the *new*, incremented value of `x` is used as the result of the expression.
  • x++ (Postfix Increment): The *original* value of `x` (before incrementing) is used as the result of the expression, and *then* the variable `x` is incremented by 1.
(Similar to 2018j Q5b(ii))

(c) (i) Quicksort Definition:

Quicksort is an efficient, comparison-based sorting algorithm that uses a divide-and-conquer strategy. It works by selecting a 'pivot' element from the array and partitioning the other elements into two sub-arrays, according to whether they are less than or greater than the pivot. The sub-arrays are then sorted recursively.

Answer

(a) Techniques to Find Item in Ordered List:

  • Linear Search (Sequential Search): Starts from the beginning of the list and checks each element one by one until the target item is found or the end of the list is reached. Although simple, it doesn't take advantage of the list being ordered and is inefficient (O(n)) for large lists.
  • Binary Search: Efficiently finds an item by repeatedly dividing the search interval in half. It compares the target value with the middle element; if they don't match, the half in which the target cannot lie is eliminated, and the search continues on the remaining half. Requires the list to be sorted. Much faster (O(log n)) for large lists.
  • Interpolation Search: An improvement over binary search for uniformly distributed sorted data. It estimates the position of the target value based on its value relative to the first and last elements, rather than just dividing the interval in half. Can be faster than binary search (approaching O(log log n)) for uniform data but degrades to O(n) for non-uniform data.
(Any two)

(b) Actual vs. Formal Parameters:

  • Formal Parameters: The variables declared in the header (signature) of a function or procedure definition. They act as placeholders that receive values when the function/procedure is called. Their scope is local to the function/procedure.
  • Actual Parameters (Arguments): The actual values or variables passed to a function or procedure when it is called from another part of the program (the calling code). These values are assigned to (for pass-by-value) or referenced by (for pass-by-reference) the corresponding formal parameters.

(c) (i) Cause of Stack Underflow Error:

A stack underflow error occurs when an attempt is made to Pop (remove) or Peek (view) an element from a stack that is currently empty. There is no element on the stack to perform the operation on.
