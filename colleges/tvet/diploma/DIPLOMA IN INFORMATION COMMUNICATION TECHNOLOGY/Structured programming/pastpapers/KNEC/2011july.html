Answer

(a) (i) Advantages of Structured Programming
  • Readability & Understandability: Code is easier to read and understand due to logical structure (sequence, selection, iteration) and avoidance of excessive `goto` statements.
  • Maintainability: Easier to modify and debug because changes in one part are less likely to have unintended consequences elsewhere. Modularity (using functions/procedures) helps isolate components.
  • Reduced Complexity: Breaks down complex problems into smaller, manageable units, making the overall program logic simpler to follow and manage.
  • Increased Productivity: Promotes code reuse through functions/procedures and makes development more systematic.
*(Any three)*
(a) (ii) 1st vs. 4th Generation Languages
  • 1st Generation Languages (1GL - Machine Language):
    • Consists of binary code (0s and 1s) directly understood by the computer's CPU.
    • Machine-dependent (specific to a processor architecture).
    • Very difficult and tedious for humans to write, read, and debug.
    • No translation needed for execution.
  • 4th Generation Languages (4GLs):
    • High-level languages designed to be closer to human language than 3GLs, often domain-specific (e.g., databases, report generation).
    • Focus on *what* needs to be done rather than *how* (declarative).
    • Designed for rapid application development and increased programmer productivity for specific tasks.
    • Examples: SQL, Focus, report generators.
(b) Top-Down vs. Bottom-Up Design
  • Top-Down Design: Starts with the overall problem or main function and breaks it down into smaller, more manageable sub-problems or modules. The process continues recursively, refining each module until the lowest level of detail is reached. Focuses on the overall structure first.
  • Bottom-Up Design: Starts by designing and implementing the lowest-level, fundamental components or modules first. These modules are then combined and integrated to build larger modules and eventually the complete system. Focuses on building blocks and then assembling them.

(d) Types of Errors
  • (i) Endless loop: This is a **Logical Error**. The program runs, but the logic controlling the loop termination is flawed, causing it to repeat indefinitely.
  • (ii) Mismatched quotes: This is a **Syntax Error**. Violates the grammatical rules of the programming language regarding string literals. The compiler/interpreter will detect this before execution.

(a) C Keywords: `continue` and `break`
  • (i) `continue`:** Used inside loops (`for`, `while`, `do-while`). It immediately skips the rest of the current iteration of the loop body and proceeds to the next iteration (checking the loop condition again or performing the loop update).
  • (ii) `break`:** Used inside loops (`for`, `while`, `do-while`) or `switch` statements. Inside a `switch`, it terminates the `switch` statement, preventing fall-through to subsequent cases.

(a) Data Structures
  • (i) Queue: A linear data structure following the First-In, First-Out (FIFO) principle. Elements are added (enqueued) at one end (rear) and removed (dequeued) from the other end (front). Like a waiting line.
  • (ii) Tree: A non-linear, hierarchical data structure composed of nodes connected by edges. It has a root node, parent-child relationships, and no cycles. Used for representing hierarchies and efficient searching/sorting.
  • (iii) Linked List: A linear, dynamic data structure where elements (nodes) contain data and a pointer to the next node. Nodes are not necessarily stored contiguously in memory. Allows efficient insertions/deletions.

(c) C Program: Selection Sort (Descending)

#include <stdio.h>

#define SIZE 7 // Size of the array given: [ 12 | 50 | 2 | 105 | 23 | 8 | 77 ]

// Function to perform selection sort in descending order
void selectionSortDescending(int arr[], int n) {
    int i, j, maxIndex, temp;

    // One by one move boundary of unsorted subarray
    for (i = 0; i < n - 1; i++) {
        // Find the maximum element in unsorted array
        maxIndex = i;
        for (j = i + 1; j < n; j++) {
            if (arr[j] > arr[maxIndex]) { // Compare for greater value
                maxIndex = j;
            }
        }

        // Swap the found maximum element with the first element
        if (maxIndex != i) {
            temp = arr[maxIndex];
            arr[maxIndex] = arr[i];
            arr[i] = temp;
        }
    }
}

// Function to print an array
void printArray(int arr[], int size) {
    int i;
    for (i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main() {
    int arr[SIZE] = {12, 50, 2, 105, 23, 8, 77};

    printf("Original array: \n");
    printArray(arr, SIZE);

    selectionSortDescending(arr, SIZE);

    printf("\nSorted array in descending order: \n");
    printArray(arr, SIZE);

    return 0;
}

Answer

(a) Insertion Sort vs. Merge Sort
  • Insertion Sort: Builds the final sorted array one item at a time. It iterates through the input array and for each element, finds the correct position within the already sorted portion of the array and inserts it there, shifting larger elements up. It's simple, efficient for small or nearly sorted arrays (O(n) best case), but has an average/worst-case time complexity of O(n²). It's an in-place sort (O(1) space).
  • Merge Sort: A divide-and-conquer algorithm. It recursively divides the array into two halves, sorts each half, and then merges the two sorted halves back together. It has a guaranteed time complexity of O(n log n) for all cases, making it efficient for large datasets. However, it requires additional memory (O(n) auxiliary space) for the merging process, so it's not in-place.
(b) Global vs. Local Variables
  • (i) Global Variable: Declared outside any function or block. It is accessible from any part of the program (within its scope, usually the entire file or program). Its lifetime persists for the entire duration of the program's execution. Should be used sparingly to avoid potential side effects and naming conflicts.
  • (ii) Local Variable: Declared inside a function, procedure, or a specific block of code (e.g., inside loop braces). It is only accessible within that specific scope. It is created when the function/block is entered and destroyed when the function/block is exited. Preferred for limiting scope and better memory management.
(c) C Program: Linear Search

#include <stdio.h>

#define SIZE 5

int main() {

        }

    // Output the result
    printf("\nArray elements: ");
    for (i = 0; i < SIZE; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");

    if (found == 1) {
        // The element 'i' will be the index where it was found + 1 if break was used
        // Or simply state found
        printf("%d was found in the array.\n", searchElement);
    } else {
        printf("%d was not found in the array.\n", searchElement);
    }

    return 0;
}

(a) (i) Dereferencing a Pointer

Dereferencing a pointer means accessing the value stored at the memory address that the pointer points to. In C, this is typically done using the indirection operator (`*`). For example, if `p` is a pointer to an integer, `*p` gives the integer value stored at the address held by `p`.

(a) (ii) Pascal Functions `abs()` and `sqr()`
  • `abs(x)`:** Returns the absolute value of the argument `x`. If `x` is positive or zero, it returns `x`; if `x` is negative, it returns `-x`. Works for integer and real types.
  • `sqr(x)`:** Returns the square of the argument `x` (i.e., `x * x`). Works for integer and real types.
(b) Advantages of Using Pointers
  • Dynamic Memory Allocation: Pointers are essential for allocating and deallocating memory dynamically during program execution (e.g., using `malloc`/`free` in C, `New`/`Dispose` in Pascal), allowing data structures to grow or shrink as needed.
  • Efficient Array/String Handling: Pointers provide an efficient way to access and manipulate elements of arrays and strings, often leading to more concise and faster code compared to array indexing in certain operations.
  • Pass by Reference Simulation (in C): Allow functions to modify the original variables passed as arguments by passing their addresses (pointers), effectively simulating pass-by-reference.
  • Implementing Complex Data Structures: Pointers are fundamental for building dynamic data structures like linked lists, trees, and graphs, where nodes need to link to each other.

(c) C Program: Output Array Elements Vertically using Pointers

#include <stdio.h>

#define SIZE 5

int main() {
    int arr[SIZE] = {8, 6, 7, 5, 2};

        printf("%d\n", *ptr); // Print the value pointed to by ptr

    }

}
(d) C File Modes
  • (i) `w` (write): Opens a file for writing. If the file exists, its contents are discarded (truncated to zero length). If the file does not exist, it is created.
  • (ii) `a` (append): Opens a file for appending (writing at the end of the file). If the file does not exist, it is created. Data is always added to the end, existing content is preserved.
  • (iii) `r+` (read/update): Opens an existing file for both reading and writing. The file must exist. Allows modifying existing content.
(e) C Pointer Segment Output

(d) C Program: Write Characters to File

#include <stdio.h>
#include <stdlib.h> // For exit()

int main() {
    FILE *filePtr; // File pointer

    // Attempt to open the file in write mode ("w") on drive C
    // Note: Writing directly to C:\ might require administrator privileges
    // and is generally not recommended. Using a path in Documents or Desktop is safer.
    // Example using a relative path instead: filePtr = fopen("module1.txt", "w");
    filePtr = fopen("C:\\module1.txt", "w"); // Use double backslashes for path in C string

    // Check if the file was opened successfully
    if (filePtr == NULL) {
        perror("Error opening file C:\\module1.txt"); // Print system error message
        return 1; // Indicate error
        // Or use: exit(1);
    }

    // Write characters 'm', 'n', 'p' to the file
    if (putc('m', filePtr) == EOF) {
        perror("Error writing 'm' to file");
        fclose(filePtr); // Close file before exiting
        return 1;
    }
    if (putc('n', filePtr) == EOF) {
        perror("Error writing 'n' to file");
        fclose(filePtr);
        return 1;
    }
    if (putc('p', filePtr) == EOF) {
        perror("Error writing 'p' to file");
        fclose(filePtr);
        return 1;
    }

    printf("Successfully wrote 'm', 'n', 'p' to C:\\module1.txt\n");

    // Close the file
    if (fclose(filePtr) == EOF) {
         perror("Error closing file");
         return 1;
    }

    return 0; // Indicate success
}

Note: Writing directly to the root of C: drive often fails due to permissions. It's better practice to write to user-specific directories. Error handling for file operations is included.

(c) (i) Pascal Value vs. Variable Parameters
  • Value Parameters (Default): Declared without the `VAR` keyword (e.g., `procedure DoSomething(x: integer)`). A *copy* of the argument's value is passed to the subprogram. Any modifications made to the parameter *inside* the subprogram do **not** affect the original argument variable in the calling code.
  • Variable Parameters (`VAR`): Declared with the `VAR` keyword before the parameter name (e.g., `procedure ChangeValue(VAR y: integer)`). The *memory address* (reference) of the argument variable is passed to the subprogram. Any modifications made to the parameter *inside* the subprogram **directly affect** the original argument variable in the calling code.
(c) (ii) Pascal Program: Sum 1 to 10 using `while`

  counter: Integer;
  total: Integer;

begin
  total := 0;
  counter := 1; // Initialize counter before the loop

  // While loop to add integers from 1 to 10
  while counter <= 10 do
  begin
    total := total + counter;
    counter := counter + 1; // Increment counter inside the loop
  end;

  Writeln('The sum of integers from 1 to 10 is: ', total);

  Readln; // Pause
end.

  INPUT taxableIncome

  taxReliefRate = 0.0 // Default to 0 relief

  // Determine relief rate based on conditions
  // Note: Ambiguity in table for Permanent category needs clarification.
  // Assuming Category 3 is Perm >= 2M, Category 4 is Perm >= 1M
  CASE category OF

      IF amountInsured >= 1000000 THEN
        taxReliefRate = 0.08 // 8%
      END IF
    2: // Contract
      IF amountInsured >= 2000000 THEN
        taxReliefRate = 0.10 // 10%
      END IF
    3: // Permanent (Assuming >= 2M)
      IF amountInsured >= 2000000 THEN
        taxReliefRate = 0.12 // 12%
      END IF
    4: // Permanent (Assuming >= 1M and not covered by Cat 3)

         END IF

    5: // Other
