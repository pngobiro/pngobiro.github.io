Question 1 (20 marks)

(a) State two phases that constitute program maintenance. (2 marks)

(b) (i) Outline four functions of a computer compiler. (4 marks)

(ii) State two advantages of bubble sort algorithm as used in computer programming. (2 marks)

(c) Differentiate between technical documentation and user documentation. (4 marks)

(d) Write a C program that could be used to generate the following output. Use for loop control structure. (8 marks)

1 2 3 4 5
2 4 6 8 10
3 6 9 12 15
4 8 12 16 20
5 10 15 20 25

Answer

(a) Phases of Program Maintenance:

  1. Corrective Maintenance: Fixing bugs and errors discovered after the program has been deployed.
  2. Adaptive Maintenance: Modifying the program to work in a new or changed environment (e.g., new operating system, hardware, or external system interfaces).
  3. Perfective Maintenance: Enhancing the program by adding new features, improving performance, or increasing usability based on user feedback or new requirements.
  4. Preventive Maintenance: Making changes to improve future maintainability or reliability and reduce the potential for future problems (e.g., code refactoring, updating documentation).
(Any two distinct phases)

(b) (i) Functions of a Compiler:

  • Lexical Analysis: Breaks the source code into basic units called tokens (keywords, identifiers, operators, etc.).
  • Syntax Analysis (Parsing): Checks if the sequence of tokens conforms to the grammatical rules of the programming language, often building a parse tree.
  • Semantic Analysis: Checks for meaning and logical consistency (e.g., type checking, ensuring variables are declared before use).
  • Code Generation: Translates the intermediate representation of the program into the target language (usually machine code or assembly).
  • Code Optimization: Improves the generated code to make it run faster or use less memory.
  • Error Reporting: Detects and reports syntax and static semantic errors found during the analysis phases.
(Any four - similar to 2017nov Q3a)

(b) (ii) Advantages of Bubble Sort:

  • Simplicity: Easy to understand and implement, especially for beginners.
  • Space Efficiency: It's an in-place sorting algorithm, requiring minimal extra memory (O(1) auxiliary space).
  • Detects Sorted Data: An optimized version can detect if the list is already sorted and terminate early (if no swaps occur in a pass).
(Any two)

(c) Technical vs. User Documentation:

  • Technical Documentation: Aimed at developers/maintainers, describes internal system design, architecture, algorithms, APIs, code structure, installation, and maintenance procedures. Focuses on *how* the system works.
  • User Documentation: Aimed at end-users, explains how to install, configure, and use the software's features to accomplish tasks. Includes tutorials, user manuals, FAQs. Focuses on *what* the user can do.
(Similar to 2018j Q4c)

Answer

(a) List (Programming):

A list is an abstract data type (ADT) representing a finite, ordered sequence of values (elements), where the same value may occur more than once. Common operations include adding elements, removing elements, accessing elements by position, and finding the length of the list. Lists can be implemented using various data structures, such as arrays or linked lists.

(b) (i) Web Programming vs. Web Scripting:

  • Web Programming: A broader term encompassing the development of web applications, including both server-side logic (handling requests, database interaction, business logic - often using languages like Java, Python, PHP, Ruby, C#) and client-side logic (user interface interactions - often using JavaScript).
  • Web Scripting: Typically refers specifically to writing scripts (often using languages like JavaScript, PHP, Perl, Python) that automate tasks or add dynamic behavior to web pages. It's often focused on either client-side scripting (running in the browser, like JavaScript for UI manipulation) or server-side scripting (running on the server to generate dynamic content, like PHP). It's a subset of web programming.

(b) (ii) Circumstance for Using 5th Generation Language (5GL):

A programmer might choose a 5GL when the primary goal is to solve problems by specifying *constraints* and *goals*, rather than explicitly coding an algorithm. These languages are often used in artificial intelligence, expert systems, and constraint programming domains where the programmer defines *what* needs to be achieved, and the language/system figures out *how* to achieve it.

(d) Pascal Program (Password Strength Check):

program PasswordChecker;
var
  password: string[4]; // Fixed length string of 4 characters
  i: Integer;
  digitCount: Integer;
  isStrong: Boolean;

begin
  Write('Enter a 4-character password: ');
  Readln(password);

  // Validate length (optional but good practice)
  if Length(password) <> 4 then
  begin
    Writeln('Error: Password must be exactly 4 characters long.');
  end
  else
  begin
    // Count digits in the password
    digitCount := 0;
    for i := 1 to 4 do // Pascal strings are 1-indexed
    begin
      // Check if the character is a digit ('0' through '9')
      if (password[i] >= '0') and (password[i] <= '9') then
      begin
        digitCount := digitCount + 1;
      end;
    end;

    // Determine strength based on digit count
    if digitCount = 4 then // Check if *all four* are numbers
      isStrong := True
    else
      isStrong := False;

    // Output result
    if isStrong then
      Writeln('Strong password')
    else
      Writeln('Weak password');
  end;

  Readln; // Keep console window open
end.

Note: The interpretation of "four of the characters entered are numbers" is taken to mean *exactly* four (i.e., all characters must be digits) for it to be strong.

(a) Function of Pascal Memory Management Commands:

  • (i) dispose(): Releases memory previously allocated dynamically using `new()`. It takes a pointer variable as an argument and returns the memory block pointed to by that pointer back to the heap, making it available for reuse. Failure to dispose allocated memory leads to memory leaks.
  • (ii) new(): Dynamically allocates a block of memory from the heap large enough to hold a variable of the type the pointer points to. It takes a pointer variable as an argument and sets that pointer to the address of the newly allocated memory block.

(b) Data Structures Description:

  • (i) Queue: A linear data structure that follows the First-In, First-Out (FIFO) principle. Elements are added (enqueued) at one end (rear) and removed (dequeued) from the other end (front).
  • (ii) Stack: A linear data structure that follows the Last-In, First-Out (LIFO) principle. Elements are added (pushed) and removed (popped) from the same end, called the top.
(Similar to 2017july Q5c)

(c) (i) Monolithic vs. Procedural Programming:

  • Monolithic: The entire program logic is contained within a single, large block or module (often the main program). There is little or no separation into independent subprograms. Control flow often relies heavily on jumps (`goto`). Difficult to read, maintain, and reuse code.
  • Procedural: The program is broken down into smaller, named procedures or functions, each performing a specific task. The main program coordinates calls to these procedures. Improves organization, readability, maintainability, and reusability compared to monolithic programming.
(Similar to 2018j Q2b)

(c) (ii) Pascal String Functions:

  • I. `Pos(substring, mainString)`: Searches for the first occurrence of `substring` within `mainString`. Returns an integer representing the starting position (index) of `substring` if found, or 0 if not found.
  • II. `Insert(sourceString, destinationString, position)`: Inserts the `sourceString` into the `destinationString` starting at the specified integer `position`. Modifies the `destinationString`.

(d) Pascal Program (Triangle Hypotenuse):

program TriangleHypotenuse;
uses
  Math; // Required for Sqrt and Power/Sqr functions

var
  sideA, sideB: Real;
  hypotenuseY: Real;

// Function to calculate hypotenuse
function CalculateY(A, B: Real): Real;
begin
  // Basic validation for non-negative sides
  if (A >= 0) and (B >= 0) then
    // Calculate Y = Sqrt(A^2 + B^2)
    // Using Sqr function is standard in Pascal
    CalculateY := Sqrt(Sqr(A) + Sqr(B))
    // Alternatively using Power: CalculateY := Power(Power(A, 2) + Power(B, 2), 0.5);
  else
  begin
    Writeln('Error: Sides A and B must be non-negative.');
    CalculateY := -1.0; // Indicate error
  end;
end;

begin // Main program
  Write('Enter the length of side A: ');
  Readln(sideA);

  Write('Enter the length of side B: ');
  Readln(sideB);

  // Call the function to calculate hypotenuse
  hypotenuseY := CalculateY(sideA, sideB);

  // Display the result if calculation was successful
  if hypotenuseY >= 0 then
  begin
    Writeln;
    Writeln('Side A: ', sideA:0:2);
    Writeln('Side B: ', sideB:0:2);
    Writeln('Calculated Hypotenuse (Y): ', hypotenuseY:0:2);
  end;

  Readln; // Keep console window open

(a) (ii) Source Code vs. Object Code:

  • Source Code: Human-readable instructions written by a programmer in a high-level or assembly language.
  • Object Code: Machine-readable code (often machine code or bytecode) produced by a compiler or assembler by translating the source code.
(Similar to 2017nov Q7c(ii))

(b) Importance of a Procedure (Pascal):

Procedures are important because they promote modularity and code reuse. By encapsulating a specific task or sequence of operations within a named procedure, the main program logic becomes simpler and easier to understand. The same procedure can be called multiple times from different parts of the program, avoiding code duplication and making maintenance easier (a change only needs to be made in one place).

Justification: Modularity improves organization and maintainability; reusability reduces code size and effort.

(c) (i) Advantages of Quick Sort:

  • Fast Average Performance: Typically has an average time complexity of O(n log n), which is very efficient for large datasets.
  • In-Place Sorting: Usually performs the sort directly within the original array, requiring only a small amount of auxiliary memory (O(log n) on average for the recursion stack).
  • Widely Used and Studied: A well-understood algorithm with many available implementations and optimizations.
(Any three)

(c) (ii) Subprogram for Factorial Calculation:

The most appropriate subprogram is a Function.

Justification: The factorial calculation produces a single, well-defined result (the factorial value) based on its input number. Functions are specifically designed to compute and return a single value, making the code clear and allowing the result to be directly used in expressions or assignments (e.g., `result := Factorial(5);`). A procedure could achieve this using a `VAR` parameter, but it's less idiomatic for calculations that naturally produce one result.

Answer

(a) Circumstance for Using `repeat...until`:

A `repeat...until` loop is used when the loop body must execute *at least once* before the terminating condition is checked. It's suitable for situations where an action needs to be performed first, and then a decision is made whether to repeat it, such as validating user input where you need to get the input at least once before checking if it's valid.

(b) Extreme Data vs. Abnormal Data:

  • Extreme Data: Input values that are at the *limits* of the valid range or represent the largest/smallest possible valid inputs. Used to test boundary conditions and handling of potential overflows or large volumes. Example: Testing a percentage calculation with 0% and 100%; testing a list processing function with a very large list.
  • Abnormal Data (Invalid/Erroneous Data): Input values that fall *outside* the valid range or are of the wrong data type. Used to test the program's error detection and handling capabilities. Example: Entering negative numbers for age, text when a number is expected, values exceeding defined limits.

(d) Pseudocode (Sum of Multiples of 5):

START
  DECLARE sum AS INTEGER
  DECLARE number AS INTEGER

  SET sum = 0
  SET number = 1

  WHILE number <= 100 DO
    // Check if number is a multiple of 5
    IF number MOD 5 = 0 THEN
      // Add number to sum
      SET sum = sum + number
    END IF

    // Move to the next number
    INCREMENT number
  END WHILE

  DISPLAY "Sum of multiples of 5 from 1 to 100 is: ", sum
STOP

Alternative using FOR loop:

START
  DECLARE sum AS INTEGER
  DECLARE number AS INTEGER

  SET sum = 0

  FOR number = 1 TO 100 DO
    IF number MOD 5 = 0 THEN
      SET sum = sum + number
    END IF
  END FOR

  DISPLAY "Sum of multiples of 5 from 1 to 100 is: ", sum
STOP
