Question 1 (20 marks)

(a) Describe compilation as used in programming. (2 marks)

(b) Differentiate between iteration and recursion as used in programming. (4 marks)

(c) Peter, a student in a college, was presented with the following ten numeric values: 56, 49, 70, 83, 23, 45, 67, 76, 55 and 37. He was asked to create a data structure that would allow him to have access to each value directly.

  • (i) Identify the most appropriate data structure that he would create. (1 mark)
  • (ii) Using the data structure in (i), write a C program that would display the data. (5 marks)

(d) (i) Describe two items that could be added to a user manual in order to assist in the access of details and information. (4 marks)

(ii) Explain two importance of documenting all the stages during program development. (4 marks)

Answer

(a) Compilation:

Compilation is the process of translating source code written in a high-level programming language (like C or Pascal) into a lower-level language, typically machine code or bytecode, that can be understood and executed by a computer's processor or a virtual machine. This translation is performed by a program called a compiler.

(Similar to 2017nov Q3a)

(b) Iteration vs. Recursion:

  • Iteration: Uses loop constructs (like `for`, `while`, `do...while`, `repeat...until`) to repeatedly execute a block of code until a specific condition is met. It maintains state using loop control variables.
  • Recursion: A function calls itself, either directly or indirectly, to solve a smaller instance of the same problem. It relies on a base case to stop the recursive calls and uses the function call stack to manage state.

(c) Data Structure for Direct Access:

  • (i) Most Appropriate Data Structure: An Array. Arrays provide direct access to elements using an integer index.
  • (ii) C Program to Display Array Data: 
    #include <stdio.h>

int main() {
    // Initialize the array with the given values
    int values[] = {56, 49, 70, 83, 23, 45, 67, 76, 55, 37};
    int size = sizeof(values) / sizeof(values[0]); // Calculate array size
    int i;

    printf("The numeric values are:\n");
    for (i = 0; i < size; ++i) {
        // Access each element directly using its index
        printf("%d ", values[i]);
    }
    printf("\n");

    return 0;
}

(d) User Manual Assistance Items:

  • (i) Table of Contents: Provides an overview of the manual's structure and allows users to quickly navigate to specific sections or topics.
  • (ii) Index: An alphabetical listing of keywords, topics, and features with corresponding page numbers, enabling users to find specific information quickly.
  • (iii) Glossary: Defines technical terms or jargon used within the manual that might be unfamiliar to the user.
  • (iv) Search Function (for online manuals): Allows users to search for specific keywords or phrases within the entire document.
(Any two)

(d) (ii) Importance of Documenting Stages:

  • Maintainability: Documentation (like design specs, code comments, test plans) makes it easier for developers (including future ones) to understand, debug, modify, and enhance the software later.
  • Communication & Collaboration: Provides a common understanding for team members, stakeholders, and users, facilitating better communication and reducing misunderstandings during development and use.
  • Knowledge Transfer/Training: Helps new team members get up to speed quickly and serves as a basis for creating user training materials.
  • Requirement Verification: Ensures the final product aligns with the initial requirements and design decisions.
(Any two - similar to 2017nov Q3b(ii))

Question 2 (20 marks)

(a) Write operators order of precedence for evaluating mathematical expressions as used in Pascal programming. (3 marks)

(b) (i) David wrote a program with the seven days of a week declared as enumerated type in Pascal programming. Write two statements that would be used to generate a message “IT’S A WEEKDAY” if the day is not Sunday or Saturday in the program. (4 marks)

(ii) Differentiate between PRED and SUCC predefined functions as applied in Pascal programming. (4 marks)

(c) Write a C program that could print all numbers that are divisible by 9 from 1 to 99. Use for loop. (5 marks)

(d) A student was given a computer program code to study. Outline four characteristics that he may use to ascertain that the program is written using structured programming language. (4 marks)

Answer

(a) Pascal Operator Precedence (Mathematical):

  1. Parentheses `()` - Highest
  2. Unary operators (`+`, `-`, `NOT`)
  3. Multiplying operators (`*`, `/`, `DIV`, `MOD`, `AND`) - Evaluated Left-to-Right
  4. Adding operators (`+`, `-`, `OR`, `XOR`) - Evaluated Left-to-Right
  5. Relational operators (`=`, `<>`, `<`, `>`, `<=`, `>=`, `IN`) - Lowest (within mathematical/logical context)

(b) (i) Pascal Statements for Weekday Check (Enumerated Type):

type
  DayOfWeek = (Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday);
var
  today: DayOfWeek;
begin
  // Assume 'today' has been assigned a value (e.g., today := Wednesday;)

  // Statement 1: Using IF and OR
  if (today = Saturday) or (today = Sunday) then
    { Do nothing or handle weekend }
  else
    Writeln('IT''S A WEEKDAY'); // Note: Double apostrophe for literal '

  // Statement 2: Using IF and NOT IN (Set constructor)
  if not (today in [Saturday, Sunday]) then
    Writeln('IT''S A WEEKDAY');

  // Statement 3: Using IF and AND (Less direct)
  if (today >= Monday) and (today <= Friday) then
     Writeln('IT''S A WEEKDAY');

end.
(Any two distinct valid statements/approaches)

(b) (ii) `PRED` vs. `SUCC` Functions (Pascal):

These functions operate on ordinal types (integers, characters, enumerated types, booleans, subranges).

  • `PRED(X)` (Predecessor): Returns the value that comes immediately *before* `X` in the defined order of its ordinal type. It causes an error if `X` is the first value in its type (e.g., `PRED(Monday)` if Monday is the first day).
  • `SUCC(X)` (Successor): Returns the value that comes immediately *after* `X` in the defined order of its ordinal type. It causes an error if `X` is the last value in its type (e.g., `SUCC(Sunday)` if Sunday is the last day).

(c) C Program (Numbers Divisible by 9):

#include <stdio.h>

int main() {
    int i;

    printf("Numbers divisible by 9 between 1 and 99:\n");

    // Loop from 1 to 99
    for (i = 1; i <= 99; ++i) {
        // Check if the number is divisible by 9
        if (i % 9 == 0) {
            printf("%d ", i);
        }
    }
    printf("\n"); // Print a newline at the end

    return 0;
}

(d) Characteristics of Structured Programming Language Code:

  • Modularity: Code is broken down into smaller, reusable units like functions or procedures.
  • Use of Control Structures: Relies primarily on sequence, selection (if/else, switch/case), and iteration (for, while, repeat/do-while) constructs for control flow.
  • Limited/No `GOTO` Statements: Avoids or strictly limits the use of unconditional jumps (`goto`) which lead to unstructured code.
  • Clear Structure/Readability: Code is typically well-formatted with indentation and meaningful names, making it easier to read and understand the logic.
  • Top-Down Design Approach: Often developed using a top-down methodology, breaking the main problem into sub-problems.
(Any four)

Answer

(a) C Data Conversion Specifiers:

Used in functions like `printf` and `scanf`:

  • %d or %i: Signed decimal integer.
  • %u: Unsigned decimal integer.
  • %f: Decimal floating-point (float/double).
  • %c: Single character.
  • %s: String of characters.
  • %p: Pointer address.

  • %o: Octal integer.

(b) Uses of `RESET` Function (Pascal):

  1. Open Existing File for Reading: Associates a file variable with an existing external file and prepares it for reading data from the beginning. If the file doesn't exist, it typically causes an I/O error (unless error checking is disabled).
  2. Position File Pointer to Beginning: For a file that is already open, `Reset` repositions the file pointer back to the start of the file, allowing reading to begin again from the first record or character.

(c) Factors for Choosing Programming Language:

  • Platform Constraints: Target OS, hardware, or environment compatibility.
  • Performance Requirements: Need for speed or memory efficiency.
  • Ecosystem (Libraries/Tools): Availability of relevant libraries, frameworks, and development tools.
  • Maintainability/Scalability: Language features supporting code organization and large projects.

(d) Pascal Program (Customer Discount Logic):

program CustomerDiscount;
uses
  SysUtils; // For StrToIntDef, AnsiUpperCase (UpCase is standard)

var
  category: Char; // A or B
  amountSpent: Real;
  yearsOfTrade: Integer;
  discountRate: Real;
  discountAmount: Real;

begin
  discountRate := 0.0; // Default no discount

  // Get Customer Category
  Write('Enter Customer Category (A or B): ');
  Readln(category);
  category := UpCase(category); // Case-insensitive

  // Get Amount Spent
  Write('Enter Amount Spent: ');
  Readln(amountSpent);

  // Apply logic based on category
  if category = 'A' then
  begin
    if amountSpent >= 10000 then
      discountRate := 0.15 // 15%
    else
      discountRate := 0.0; // No discount
  end
  else if category = 'B' then
  begin
    // Get Years of Trade only if Category B
    Write('Enter Years of Trade: ');
    Readln(yearsOfTrade);

    if yearsOfTrade >= 10 then // Corrected logic >= 10
      discountRate := 0.10 // 10%
    else
      discountRate := 0.05; // 5%
  end
  else
  begin
    Writeln('Invalid Category Entered.');
    discountRate := 0.0; // Ensure no discount for invalid category
  end;

  // Calculate discount amount
  discountAmount := amountSpent * discountRate;

  // Display the result
  Writeln;
  Writeln('Customer Category: ', category);
  Writeln('Amount Spent: ', amountSpent:0:2);
  if category = 'B' then // Only show years if relevant
     Writeln('Years of Trade: ', yearsOfTrade);
  Writeln('Discount Offered: ', discountAmount:0:2, ' (', (discountRate * 100):0:0, '%)');

  Readln; // Keep console window open
end.

Question 4 (20 marks)

(a) Write the output of each of the following logical operators when executed using C programming language.

  • (i) !false (1 mark)
  • (ii) true && false (1 mark)
  • (iii) true || false (1 mark)

(b) Given that a=10, b=5 and c=2 determine the values of x in each of the following statements as used in Pascal programming. Show your working:

  • (i) x = 2 * a mod (b - 3) / sqrt(c + 2) (2 marks)
  • (ii) x = 2 * c + sqr(a + 1) * c (2 marks)

(c) Tom would like to use random file in his program:

  • (i) Outline three advantages of this file; (3 marks)
  • (ii) Explain two limitations of this approach to file organization. (4 marks)

(d) Jane, a programmer in a hospital was tasked to develop a program using Pascal programming language. The following data items are considered as input: Patient name, patient age, gender F/M and amount paid.

  • (i) Identify the most appropriate data structure for this data. Justify your answer. (2 marks)
  • (ii) Write segment codes in Pascal programming language to read data in the structure identified in (i). (4 marks)

Answer

(a) C Logical Operator Output:

  • (i) `!false`: The `!` (NOT) operator inverts the boolean value. Since `false` is 0, `!false` evaluates to 1 (true).
  • (ii) `true && false`: The `&&` (AND) operator requires both operands to be true. Since one is false, the result is 0 (false).
  • (iii) `true || false`: The `||` (OR) operator requires at least one operand to be true. Since one is true, the result is 1 (true).

(b) Pascal Expression Evaluation:

Given: a=10, b=5, c=2

  • (i) `x = 2 * a mod (b - 3) / sqrt(c + 2)`
    1. Parentheses: `(b - 3)` -> `(5 - 3)` = 2
    2. Parentheses/Function: `(c + 2)` -> `(2 + 2)` = 4, then `sqrt(4)` = 2.0
    3. Expression: `x = 2 * 10 mod 2 / 2.0`
    4. `*`, `mod`, `/` (Left-to-Right):
      • `2 * 10` = 20
      • `20 mod 2` = 0 (Remainder of 20 / 2)
      • `0 / 2.0` = 0.0
    5. Result: x = 0.0 (or 0 if x is Integer, though intermediate division suggests Real)
  • (ii) `x = 2 * c + sqr(a + 1) * c`
    1. Parentheses: `(a + 1)` -> `(10 + 1)` = 11
    2. Function/Square: `sqr(11)` = 121
    3. Expression: `x = 2 * 2 + 121 * 2`
    4. Multiplication (Left-to-Right):
      • `2 * 2` = 4
      • `121 * 2` = 242
    5. Expression: `x = 4 + 242`
    6. Addition: `4 + 242` = 246
    7. Result: x = 246

(c) Random Access File:

  • (i) Advantages:
    1. Fast Access: Allows direct reading or writing of any record without processing preceding ones, leading to quick retrieval/updates.
    2. Efficient Updates: Records can be updated in place without rewriting the entire file.
    3. Suitable for Interactive Systems: Ideal for applications requiring immediate access to specific data (e.g., databases, transaction processing).
  • (ii) Limitations:
    1. Fixed-Length Records (Often): Many implementations require fixed-length records for easy address calculation, which can lead to wasted space if data lengths vary significantly.
    2. Complexity: Managing keys, addresses, and potential collisions (in hashing) can be more complex than sequential files.
    3. Less Efficient for Sequential Processing: Reading all records in order might be slower than with a purely sequential file due to the overhead of calculating/looking up addresses.
    4. Potential Wasted Space: If keys are not densely packed, there might be unused gaps in the file.
    (Any two limitations)

Answer

(a) Function of `default` Statement (C `switch`):

The `default` statement within a `switch` block provides a case that executes if none of the other `case` labels match the value of the switch expression. It acts as a catch-all for values not explicitly handled by specific `case` statements. It is optional, but good practice to include for handling unexpected values.

(b) Flowchart for `REPEAT...UNTIL` Loop (Pascal):

+-------------------+
|       ...         | // Code before loop
+-------------------+
    |
    V
+-------------------+  <--+ // Loop entry point
|                   |     |
|   Execute Loop    |     |
|       Body        |     |
|                   |     |
+-------------------+     |
    |                     |
    V                     |
+-------------------+ NO  |
| Is Condition True?|-----+ // Condition checked AFTER body
+-------------------+
    | YES
    V
+-------------------+
|       ...         | // Code after loop
+-------------------+

Description: The loop body executes first, *at least once*. Then, the `UNTIL` condition is evaluated. If the condition is `FALSE`, the loop repeats. If the condition is `TRUE`, the loop terminates.

Question 6 (20 marks)

(a) Outline two similarities between a procedure and a function as used in programming. (2 marks)

(b) Explain three typical errors that are likely to occur during file I/O operations in a program. (6 marks)

(c) Differentiate between merge sort and quick sort techniques as used in programming. (4 marks)

(d) (i) The following are names of students in a programming class: Leonard, Bancy, Faith, Olive, Quinter, Alice, Patrick, Grace, Helen and Mercy.

Represent them in a binary tree. (5 mark)

(ii) State the result generated when the tree is traversed using the post order strategy. (3 marks)

Answer

(a) Similarities between Procedure and Function:

  • Subprograms: Both are named blocks of code designed to perform a specific task, separate from the main program block.
  • Parameter Passing: Both can accept input parameters (passed by value or by reference) to operate on.
  • Modularity & Reusability: Both promote modular programming by encapsulating logic and allow code to be called multiple times from different locations.
  • Local Variables: Both can declare their own local variables with scope limited to the subprogram itself.
(Any two)

(b) Typical File I/O Errors:

  1. File Not Found: Attempting to open a file for reading (`Reset` in Pascal, `fopen` with "r" mode in C) that does not exist at the specified path.
  2. Permission Denied: Trying to open a file for reading or writing without having the necessary operating system permissions (e.g., trying to write to a read-only file or a protected directory).
  3. Disk Full / Quota Exceeded: Attempting to write data to a file when there is no available space left on the storage device or the user's quota has been reached.
  4. End-of-File (EOF) Reached Unexpectedly: Trying to read data from a file after the end-of-file marker has already been encountered, often leading to errors or invalid data being read.
  5. Invalid File Format/Data: Reading data from a file that is not in the expected format (e.g., trying to read an integer when the file contains text), leading to conversion errors.
(Any three explained)

(c) Merge Sort vs. Quick Sort:

  • Merge Sort:
    • Method: Divide and conquer. Divides the array into halves, recursively sorts them, and then merges the sorted halves.
    • Time Complexity: O(n log n) in all cases (worst, average, best). Very stable performance.
    • Space Complexity: O(n) auxiliary space required for the merging step (not in-place).
    • Stability: Stable (preserves relative order of equal elements).
  • Quick Sort:
    • Method: Divide and conquer. Selects a pivot, partitions the array around the pivot, and recursively sorts the sub-arrays.
    • Time Complexity: Average case O(n log n), Worst case O(n²). Performance depends on pivot selection.
    • Space Complexity: O(log n) average (recursion stack), O(n) worst case. (In-place partitioning).
    • Stability: Not stable by default.
(Key differences: Merge Sort has guaranteed O(n log n) but needs extra space; Quick Sort is often faster in practice but has O(n²) worst case and is typically in-place). (Similar to 2017nov Q6c)

Question 7 (20 marks)

(a) Patrick, a programmer, developed a program for his client. Outline four ways that he could make the program easy to read and understandable. (4 marks)

(b) Maria decided to use a compiler rather than an interpreter during program translation. Outline four reasons that could have led her to make this decision. (4 marks)

Write the pseudocode that the student would use to design this program. (5 marks)

(a) Ways to Make Program Readable/Understandable:

  • Meaningful Naming: Use clear, descriptive names for variables, functions, constants.
  • Consistent Formatting/Indentation: Apply a standard visual layout to the code to highlight structure.
  • Modularity: Break the program into smaller, focused functions/procedures.
  • Comments: Add explanations for complex or non-obvious code sections.
  • Avoid "Magic Numbers": Use named constants instead of unexplained literal values.
  • Keep Functions Short: Functions should ideally perform one specific task.

(b) Reasons for Using Compiler over Interpreter:

  • Performance: Compiled code (native machine code) generally runs significantly faster than interpreted code.
  • Standalone Executable: Compilation produces an executable file that can run without needing the source code or an interpreter installed on the user's machine.
  • Early Error Detection: Compilers perform thorough syntax and static semantic checks, catching many errors before runtime.
  • Code Protection: Compiled machine code is harder to reverse-engineer than interpreted source code.
  • Optimization: Compilers often perform complex optimizations to improve code speed and size.

(d) (i) Order of Declarations in Pascal:

While not strictly enforced by all compilers for all sections, the conventional and recommended order within a program, procedure, or function block is:

  1. LABEL
  2. CONST
  3. TYPE
  4. VAR
  5. PROCEDURE / FUNCTION (Subprogram definitions)

(d) (ii) Rules for Composing Identifiers (Pascal):

  • Must Start with a Letter: The first character must be an alphabet character (A-Z, a-z). (Some Pascals allow underscore, but standard is letter).
  • Subsequent Characters: Can be letters, digits (0-9), or underscores (_).
  • No Spaces or Special Symbols: Cannot contain spaces or symbols like `+`, `-`, `*`, `/`, `@`, `#`, etc.
  • Not a Reserved Word: Cannot be the same as a Pascal keyword (e.g., `BEGIN`, `END`, `IF`, `WHILE`, `VAR`, `TYPE`).
  • Case Insensitivity (Typically): Standard Pascal is case-insensitive, meaning `MyVariable`, `myvariable`, and `MYVARIABLE` usually refer to the same identifier (though specific compilers might differ).
  • Length Limit (Compiler Specific): While standard Pascal didn't specify a strict limit, practical limits exist depending on the compiler (often very long, e.g., 63 or 127 characters significant).

Question 8 (20 marks)

(a) Cynthia tested a program and she encountered an error when she entered a zero value as input for a mathematical expression.

  • (i) Describe the type of error that occurred. (2 marks)
  • (ii) State the possible consequence when the error occurs. (1 mark)

(b) Outline two functions of technical documentation in programming. (4 marks)

(c) Outline the procedure that would be followed before performing each of the following in data structures:

  • (i) Adding an element in a stack; (3 marks)
  • (ii) Removing an element from a queue. (3 marks)

(d) The interest accrued I for a principal amount P after period of time T at the rate R is given by the formula I = P × R × T.

Write a Pascal program that prompts a user to enter the principal amount in the main program and then pass this value to a procedure named compute. The procedure calculates the interest at a rate of 12% per annum for a period of 5 years. The procedure then returns the interest to main program for output. (7 marks)

Answer

(a) (i) Error Type (Division by Zero):

This is a Runtime Error, specifically an arithmetic error (division by zero). It occurs during program execution when the input value causes an invalid mathematical operation.

(a) (ii) Consequence of Division by Zero Error:

The most likely consequence is the abrupt termination (crash) of the program. The operating system or runtime environment usually intercepts this fatal error and stops the program's execution, often displaying an error message.

(b) Functions of Technical Documentation:

  • System Understanding: Helps developers understand the program's architecture, design, algorithms, data structures, and internal workings.
  • Maintenance & Modification: Provides essential information for debugging, updating, or extending the software by current or future developers.
  • Integration: Describes APIs and interfaces, enabling the program to be integrated with other systems.
  • Installation & Configuration: May include details for system administrators on how to install, configure, and manage the software environment.

(c) Procedure Before Stack/Queue Operations:

  • (i) Adding to Stack (Push): Before pushing an element, check if the stack is full (Stack Overflow condition). If it is full, report an error or handle the overflow; otherwise, proceed with adding the element to the top and updating the stack pointer/count.
  • (ii) Removing from Queue (Dequeue): Before dequeuing an element, check if the queue is empty (Queue Underflow condition). If it is empty, report an error or handle the underflow; otherwise, proceed with removing the element from the front and updating the front pointer/count.

(d) Pascal Program (Interest Calculation using Procedure):

program CalculateInterest;

const
  RATE = 0.12; // 12% per annum
  TIME = 5;    // 5 years

var
  principal: Real;
  interestResult: Real;

// Procedure to compute interest
// Takes Principal by value, returns Interest via VAR parameter
procedure ComputeInterest(P: Real; var I: Real);
const
  R = RATE; // Use constants defined outside if preferred
  T = TIME;
begin
  if P >= 0 then // Basic validation
    I := P * R * T
  else
  begin
    Writeln('Error: Principal cannot be negative.');
    I := -1.0; // Indicate error
  end;
end;

begin // Main program
  Write('Enter the principal amount: ');
  Readln(principal);

  // Call the procedure to calculate interest
  ComputeInterest(principal, interestResult);

  // Display the result if calculation was successful
  if interestResult >= 0 then
  begin
    Writeln;
    Writeln('Principal Amount: ', principal:0:2);
    Writeln('Interest Rate: ', RATE * 100:0:0, '% per annum');
    Writeln('Time Period: ', TIME, ' years');
    Writeln('Calculated Interest: ', interestResult:0:2);
  end;

  Readln; // Keep console window open
end.
