Answer

(a) (i) Function of Pivot Element:

In sorting algorithms like Quicksort, the pivot element is used as a reference point during the partitioning step. The array (or sub-array) is rearranged such that all elements smaller than the pivot come before it, and all elements greater than the pivot come after it. The final position of the pivot divides the array into two partitions that are then sorted recursively.

(a) (ii) Function of `break` Statement:

The `break` statement immediately terminates the execution of the innermost enclosing loop (`for`, `while`, `do...while`) or `switch` statement. Control passes to the statement immediately following the terminated loop or switch.

(Similar to 2018j Q2a(ii))

(a) (iii) Circumstances for Using Iteration:

  • When a block of code needs to be executed a fixed number of times (e.g., using a `for` loop with a counter).
  • When a block of code needs to be executed repeatedly as long as a certain condition remains true (e.g., using a `while` loop).
  • When processing elements in a collection (like an array or list) one by one.
  • When the repetition logic is straightforward and doesn't naturally lend itself to a recursive definition.
(Any two)

(b) (i) Binary Tree:

A hierarchical data structure where each node has at most two children, referred to as the left child and the right child. It starts from a single root node, and each child node itself can be the root of another binary tree (subtree). They are often used for efficient searching and sorting (Binary Search Trees) or representing hierarchical relationships.

(b) (ii) Program Designing vs. Program Coding:

  • Program Designing: The stage where the logical solution (algorithm) and overall structure of the program are planned. It involves breaking down the problem, choosing data structures, defining modules/functions, and specifying interfaces, often using tools like flowcharts or pseudocode. Focus is on *how* the problem will be solved conceptually.
  • Program Coding: The stage where the designed algorithm and structure are translated into actual source code using a specific programming language according to its syntax and rules. Focus is on *implementing* the designed solution.

(c) C Programming Keywords:

From the list: `if, Const, define, write, double, Char, float`

C Keywords: `if`, `double`, `char`, `float`

(Note: `const` is a keyword, but often written lowercase. `define` is a preprocessor directive, not strictly a keyword. `write` is a system call/library function name, not a keyword. `Const` and `Char` with initial caps are not standard C keywords).

Question 3 (20 marks)

(a) (i) Define the term modular programming. (2 marks)

(ii) Outline two methods of passing parameters to a subprogram. (2 marks)

(b) (i) Describe the general syntax of a case control structure as used in Pascal programming. (2 marks)

(ii) Explain a circumstance under which an endless loop may occur in a program. (2 marks)

(c) Given that a=6, b=4, and c=10. Compute the output from each of the following C statements.

  • (i) (a > 6) && ((a * c) < b) (2 marks)
  • (ii) (a <= b) || ((a * c) > (a * b)) (3 marks)
  • (iii) (a * b) % c + (c / 2 * b) (1 marks)

(d) Ann, a computer student, intends to write a program that computes the total and average of all the even numbers from 20 to 50. Draw a flowchart to represent the logic of the program. (6 marks)

Answer

(a) (i) Modular Programming:

An approach to software design that emphasizes separating the functionality of a program into independent, interchangeable modules (like functions or procedures), such that each module contains everything necessary to execute only one aspect of the desired functionality. It aims to reduce complexity, improve maintainability, and enable code reuse.

(a) (ii) Methods of Passing Parameters:

  • Pass by Value: A copy of the argument's value is passed. Changes inside the subprogram do not affect the original argument.
  • Pass by Reference: The memory address of the argument is passed. Changes inside the subprogram *do* affect the original argument.
(Similar to 2016nov Q2a)

(b) (i) Pascal `case` Statement Syntax:

CASE selectorExpression OF
  constantList1: statement1;
  constantList2: statement2;
  ...
  constantListN: statementN
[ ELSE
  elseStatement(s); ]
END;
(Similar to 2017july Q5c)

(b) (ii) Circumstance for Endless Loop:

An endless (or infinite) loop occurs when the loop's terminating condition is never met. This can happen if:

  • The condition is always true (e.g., `while TRUE do ...`).
  • The variable(s) controlling the loop condition are never updated within the loop body in a way that will eventually make the condition false.
  • A logical error prevents the condition from ever becoming false.

(c) C Statement Evaluation:

Given: a=6, b=4, c=10

  • (i) `(a > 6) && ((a * c) < b)`
    1. `(a > 6)` -> `(6 > 6)` -> false (0)
    2. Since the left operand of `&&` is false, short-circuiting occurs. The right side is not evaluated.
    3. Result: 0 (false)
  • (ii) `(a <= b) || ((a * c) > (a * b))`
    1. `(a <= b)` -> `(6 <= 4)` -> false (0)
    2. Right side of `||`: `((a * c) > (a * b))`
      • `(a * c)` -> `(6 * 10)` = 60
      • `(a * b)` -> `(6 * 4)` = 24
      • `(60 > 24)` -> true (1)
    3. Expression becomes: `false || true`
    4. Result: 1 (true)
  • (iii) `(a * b) % c + (c / 2 * b)`
    1. `(a * b)` -> `(6 * 4)` = 24
    2. `24 % c` -> `24 % 10` = 4 (Remainder)
    3. `(c / 2 * b)` (Integer division and multiplication, left-to-right):
      • `c / 2` -> `10 / 2` = 5
      • `5 * b` -> `5 * 4` = 20
    4. Expression becomes: `4 + 20`
    5. Result: 24

Question 4 (20 marks)

(a) Outline two disadvantages of monolithic programming. (2 marks)

(b) (i) Describe two features of third generation programming languages. (4 marks)

(ii) Differentiate between a text file and a record as used in programming. (4 marks)

(c) (i) Differentiate between for loop and switch control structures. (4 marks)

(ii) Under what circumstance would a continue command be used in a C program. (2 marks)

(d) With the aid of an example, describe divide and conquer algorithms as used in data structures. (4 marks)

Answer

(a) Disadvantages of Monolithic Programming:

  • Low Maintainability: Changes in one part can have unexpected side effects elsewhere.
  • Poor Reusability: Tightly coupled code is hard to reuse.
  • Difficult Debugging: Isolating errors is challenging.
  • Limited Scalability/Flexibility: Hard to modify or scale individual features.
(Any two - similar to 2018j Q2b)

(b) (i) Features of Third Generation Languages (3GLs):

  • Higher Abstraction: More abstracted from hardware details compared to assembly language, using more English-like keywords and structures.
  • Procedural or Object-Oriented: Support structured programming concepts like procedures/functions, loops, selection, and potentially object-oriented features (classes, inheritance).
  • Increased Portability: Code can often be compiled and run on different machine architectures with fewer modifications than assembly.
  • Easier to Learn and Use: Syntax is closer to human language, making them generally easier to learn, write, and debug than low-level languages.
(Any two)

(b) (ii) Text File vs. Record:

  • Text File: A sequence of characters stored on secondary storage, often organized into lines. Can store various kinds of data but typically read/written sequentially as characters or lines. Data interpretation depends on the program reading it.
  • Record: A structured data type (in memory or within a file) that groups related data items (fields) of potentially different types under a single name. Represents a single entity. In files (like binary files or databases), records provide a structured way to store and retrieve related data elements together.

(c) (i) `for` Loop vs. `switch` Statement:

  • `for` Loop: An iteration (looping) control structure used to execute a block of code repeatedly, typically a predetermined number of times based on an initialization, condition, and update expression. Controls repetition.
  • `switch` Statement: A selection control structure used to choose one block of code to execute from multiple options based on the value of a single expression (usually an integer or character). Controls selection based on discrete values.

(c) (ii) Circumstance for Using `continue` (C):

The `continue` statement is used inside a loop (`for`, `while`, `do...while`) when you want to skip the rest of the current iteration's code block and immediately proceed to the next iteration's condition check/update. It's useful for bypassing processing for certain conditions within the loop without exiting the loop entirely.

(d) Divide and Conquer Algorithm Description:

Divide and Conquer is an algorithmic paradigm that solves a problem by:

  1. Divide: Breaking the original problem down into smaller, independent sub-problems of the same type.
  2. Conquer: Solving the sub-problems recursively. If the sub-problems are small enough (base case), solve them directly.
  3. Combine: Combining the solutions of the sub-problems to form the solution to the original problem.

Example: Merge Sort

  • Divide: Split the array [8, 3, 1, 7, 0, 10, 2] into [8, 3, 1, 7] and [0, 10, 2].
  • Conquer: Recursively sort the halves: [1, 3, 7, 8] and [0, 2, 10].
  • Combine: Merge the sorted halves: [0, 1, 2, 3, 7, 8, 10].

(a) Function of `goto` Command (C):

The `goto` statement provides an unconditional jump to a labeled statement within the same function. It transfers program control directly to the specified label. Its use is heavily discouraged in structured programming as it can make code difficult to read, follow, and maintain, leading to "spaghetti code".

(b) (i) C Debugging Techniques:

  • I. Tracing: Following the execution path of the program step-by-step, often by inserting print statements (`printf`) to display the values of variables or indicate which parts of the code are being executed. Debuggers also provide automated tracing features.
  • II. Stepping: Using a debugger tool to execute the program one line or one instruction at a time. This allows the programmer to observe the program's state (variable values, call stack) at each step and pinpoint where errors occur. Debuggers typically offer "step over" (execute function call without entering), "step into" (enter function call), and "step out" (finish current function and return).

(b) (ii) Control Structures for Reading Records from Text File:

  • `while` loop with EOF check: This is the most common and appropriate structure. A `while` loop continues as long as the end-of-file (EOF) has not been reached. Inside the loop, one record (or line) is read and processed.
    • Justification: Handles files with an unknown number of records correctly, stopping precisely when all data has been read.
    // Example C concept
while (fscanf(filePtr, "%s %d", name, &age) == 2) { // Check if read was successful
    // Process record
}
  • `for` loop (if number of records is known): If the exact number of records in the file is known beforehand, a `for` loop iterating that many times could be used, reading one record per iteration.
    • Justification: Simpler structure if the record count is fixed and guaranteed. Less flexible if the count might vary.
    // Example C concept
for (i = 0; i < NUM_RECORDS; ++i) {
    fscanf(filePtr, "%s %d", name, &age);
    // Process record
}

(d) C Program (Bubble Sort Array):

#include <stdio.h>
#include <stdlib.h> // For malloc and free

// Function to perform bubble sort
void bubbleSort(int arr[], int n) {
    int i, j, temp;
    int swapped;
    for (i = 0; i < n - 1; i++) {
        swapped = 0; // Flag to optimize if already sorted
        for (j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // Swap elements
                temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = 1;
            }
        }
        // If no two elements were swapped by inner loop, then break
        if (swapped == 0)
            break;
    }
}

// Function to print an array
void printArray(int arr[], int size) {
    int i;
    for (i = 0; i < size; i++)
        printf("%d ", arr[i]);
    printf("\n");
}

int main() {
    int n, i;
    int *list; // Pointer for dynamic array

    // Get the number of elements
    printf("Enter number of elements: ");
    if (scanf("%d", &n) != 1 || n <= 0) {
        printf("Invalid number of elements.\n");
        return 1;
    }

    // Allocate memory for the array dynamically
    list = (int *)malloc(n * sizeof(int));
    if (list == NULL) {
        printf("Memory allocation failed!\n");
        return 1;
    }

    // Get elements from user
    printf("Enter %d integers:\n", n);
    for (i = 0; i < n; ++i) {
        printf("Element %d: ", i + 1);
        if (scanf("%d", &list[i]) != 1) {
             printf("Invalid input.\n");
             free(list); // Free allocated memory before exiting
             return 1;
        }
    }

    // Sort the array
    bubbleSort(list, n);

    // Output the sorted list
    printf("\nSorted list in ascending order:\n");
    printArray(list, n);

    // Free the allocated memory
    free(list);

    return 0;
}

(a) (i) Function of `stdio.h` (C):

The `stdio.h` (Standard Input/Output) header file in C provides declarations for standard input/output functions (like `printf`, `scanf`, `fopen`, `fclose`, `getchar`, `putchar`), related types (like `FILE`), and macros (like `EOF`, `NULL`). (a) (ii) Circumstance for Using Extreme Test Data:

Extreme test data (values at the very limits of valid ranges, or very large/small valid inputs) is used to test boundary conditions and the program's ability to handle potentially problematic values, such as those that might cause numerical overflows, underflows, or performance issues with large datasets.

(c) (ii) Reason for Choosing `switch` over `if`:

A programmer might choose a `switch` statement over a series of nested `if-else if` statements when the selection logic is based on comparing a single variable or expression against multiple *discrete constant values* (usually integers or characters).

Justification: In such cases, a `switch` statement can often be clearer, more readable, and potentially more efficient than a long chain of `if-else if` conditions, as it directly maps the expression's value to the corresponding code block.

(d) C Program (Geometric Calculations using Functions):

#include <stdio.h>

#define PI 3.1415926535 // Define PI if M_PI is not available

// Function prototypes
double calculateCircleArea(double radius);
double calculateCubeVolume(double side);
double calculateSphereVolume(double radius);

int main() {
    int choice;

    printf("2. Volume of Cube\n");

            }
            result = calculateCircleArea(dimension);
            printf("Area of the circle is: %.2f\n", result);
            break;

        case 2: // Cube Volume
            printf("Enter side length of the cube: ");
             if (scanf("%lf", &dimension) != 1 || dimension < 0) {
                printf("Invalid side length.\n"); return 1;
            }
            result = calculateCubeVolume(dimension);
            printf("Volume of the cube is: %.2f\n", result);
            break;

        case 3: // Sphere Volume
            printf("Enter radius of the sphere: ");
             if (scanf("%lf", &dimension) != 1 || dimension < 0) {
                printf("Invalid radius.\n"); return 1;
            }
            result = calculateSphereVolume(dimension);
            printf("Volume of the sphere is: %.2f\n", result);
            break;

        default:

    return PI * radius * radius;
    // return M_PI * pow(radius, 2); // Alternative using math.h constants/functions

    return side * side * side;
    // return pow(side, 3); // Alternative using math.h
}

double calculateSphereVolume(double radius) {
    return (4.0 / 3.0) * PI * radius * radius * radius;
    // return (4.0/3.0) * M_PI * pow(radius, 3); // Alternative
}

Answer

(a) (i) Pascal Declarations:

  • I. `type`: The `TYPE` keyword introduces a section where custom data types (like records, arrays, enumerated types, pointers, subranges) are defined by the programmer.
  • II. `label`: The `LABEL` keyword introduces a section where labels (identifiers followed by a colon, e.g., `MyLabel:`) are declared. These labels serve as targets for `GOTO` statements.
(Similar to 2016nov Q3a(i) for type)

(a) (ii) Effect of Logical Error:

The primary effect of a logical error is that the program produces incorrect or unexpected results, even though it might compile and run without crashing. The program doesn't behave as intended because the underlying algorithm or its implementation is flawed.

(b) Pointer vs. Linked List:

  • Pointer: A variable that stores the memory address of another variable. It's a fundamental mechanism for indirect memory access and dynamic memory management.
  • Linked List: A dynamic data structure composed of nodes, where each node contains data and one or more pointers linking it to other nodes (typically the next node). Pointers are *used* to implement the links between nodes in a linked list.
(Key difference: Pointer is a variable type, Linked List is a data structure built using pointers).

(c) Array vs. Queue:

  • Array: A data structure storing a fixed-size collection of elements of the *same* type in *contiguous* memory, accessed by an index. Allows efficient random access (O(1)).
  • Queue: An abstract data type (ADT) representing a sequence where elements are added at one end (rear) and removed from the other (front), following the First-In, First-Out (FIFO) principle. Can be implemented using arrays or linked lists. Access is restricted to the front/rear.

(d) Pascal Program (Factorial using Procedure):

program FactorialCalculator;

var
  num: Integer;
  factResult: LongInt; // Use LongInt for factorial result

// Procedure to calculate factorial
// Returns result via VAR parameter
procedure CalculateFactorial(n: Integer; var result: LongInt);
var
  i: Integer;
  tempFact: LongInt;
begin
  // Validate input range (greater than 1 AND less than or equal to 10)
  if (n < 2) or (n > 10) then
  begin
    Writeln('Error: Number must be between 2 and 10 inclusive.');
    result := -1; // Indicate error
  end
  else
  begin
    // Calculate factorial iteratively
    tempFact := 1;
    for i := 1 to n do
    begin
      // Check for potential overflow before multiplication (optional but safer)
      // if MaxLongInt div i < tempFact then ... handle overflow ...
      tempFact := tempFact * i;
    end;
    result := tempFact;
  end;
end;

begin // Main program
  Write('Enter a number between 2 and 10: ');
  Readln(num);

  // Call the procedure
  CalculateFactorial(num, factResult);

  // Output the result if valid
  if factResult <> -1 then
  begin
    Writeln('The factorial of ', num, ' is ', factResult);
  end;

  Readln; // Keep console window open
end.

Note: The procedure calculates the factorial and returns it via a `VAR` parameter as requested. Input validation for the range 2-10 is included. A function would also be suitable here.
