Question 1 (20 marks)

(a) Outline four advantages of high-level programming languages. (4 marks)

(b) (i) Explain the term bug as used in programming. (2 marks)

(ii) Differentiate between function call and function definition as used in C language. (4 marks)

(c) The following was executed in a C language program.

(7%2) * 3 + 4 < 35

Showing the working, state the Boolean output generated. (4 marks)

(d) A student is graded based on exam performance and class attendance. When the performance is above 50% and class attendance greater than 75%, the student is awarded “Pass”. When the class attendance is less than 75%, the student retakes the course. Otherwise, the student sits for a supplementary exam.

Draw a program flowchart to represent this logic. (6 marks)

Answer

(a) Advantages of High-Level Programming Languages:

  • Readability: Code is closer to human language, making it easier to read, understand, and maintain.
  • Productivity: Programmers can write code faster as they deal with higher-level abstractions rather than low-level hardware details.
  • Portability: Programs written in high-level languages can often be run on different types of computers with minimal or no modification (requires a compiler/interpreter for the target platform).
  • Easier Debugging: Errors are often easier to identify and fix due to more understandable syntax and better error messages.
  • Abstraction: Hides complex hardware details, allowing programmers to focus on problem-solving logic.

(b) (i) Bug:

A bug is an error, flaw, or fault in a computer program or system that causes it to produce an incorrect or unexpected result, or to behave in unintended ways.

(b) (ii) Function Call vs. Function Definition (C):

  • Function Definition: This is where the actual code for the function is written. It includes the function's return type, name, parameters (if any), and the block of statements that perform the function's task. It defines *what* the function does and *how* it does it.
  • Function Call: This is an instruction that tells the program to execute a function that has been defined elsewhere. It involves using the function's name followed by parentheses, potentially including arguments that are passed to the function's parameters. It *uses* the function.

(a) (i) Pascal Comment Symbols:

(a) (ii) Purpose of Statements:

  • goto: Unconditionally transfers program control to a specified labeled statement within the same function/procedure. Its use is generally discouraged in modern structured programming as it can lead to complex and hard-to-read code ("spaghetti code").
  • continue: Skips the rest of the current iteration of a loop (for, while, do-while in C; for, while, repeat in Pascal) and proceeds to the next iteration.

(b) Errors with Global Variables:

  • Name Collisions/Shadowing: A local variable within a function/procedure might accidentally be given the same name as a global variable. Depending on the language rules, this can lead to the local variable "hiding" the global one within that scope, causing unexpected behavior if the programmer intended to modify the global variable.
  • Unintended Side Effects/Modification: Since global variables can be accessed and modified from anywhere in the program (including different functions or modules), it becomes difficult to track where and when their values are changed. One function might unintentionally alter a global variable in a way that breaks the logic of another function relying on that variable. This makes debugging harder.

Answer

(a) (i) Modular Design:

Modular design is a programming approach where a software system is broken down into smaller, independent, self-contained units called modules. Each module typically performs a specific task or function and has a well-defined interface for interacting with other modules. This promotes code reusability, maintainability, and easier debugging, as changes in one module are less likely to affect others.

(a) (ii) Importance of Pointers:

  • Dynamic Memory Allocation: Pointers are essential for allocating and deallocating memory during program execution (e.g., using `malloc`/`free` in C, `New`/`Dispose` in Pascal). This allows programs to handle data whose size is not known at compile time.
  • Efficient Data Structure Implementation: Pointers are fundamental for creating complex and efficient data structures like linked lists, trees, and graphs, where elements need to link to one another in memory.
  • Passing Arguments by Reference: Pointers allow functions to modify the original variables passed to them (pass-by-reference semantics), which is useful for returning multiple values or modifying large data structures without the overhead of copying.
  • Direct Memory Access/Manipulation: Pointers provide low-level access to memory addresses, enabling direct manipulation of memory contents, which can be necessary for system programming or performance optimization (though it requires caution).

(b) Sequence vs. Iteration Control Structures:

  • Sequence: This is the default control structure where program statements are executed one after another in the order they appear in the code. There are no jumps or repetitions, just a linear flow from one instruction to the next.
  • Iteration (or Repetition/Looping): This structure allows a block of code (the loop body) to be executed repeatedly as long as a certain condition is met (e.g., while, for loops) or until a condition becomes true (e.g., repeat..until loops). It's used for tasks that need to be performed multiple times.

Answer

(a) (i) Disadvantages of Monolithic Programming:

  • Low Maintainability: Changes in one part of the program can have unintended consequences in other parts, making maintenance difficult and risky.
  • Poor Reusability: Code is tightly coupled, making it hard to reuse specific functionalities in other programs or parts of the same program without significant modification.
  • Difficult Debugging: Isolating errors can be challenging as the entire codebase is interconnected. An error in one section might manifest symptoms elsewhere.
  • Limited Scalability: Modifying or scaling specific features is hard; often, the entire application needs to be recompiled and redeployed even for small changes.
  • Collaboration Challenges: Difficult for multiple developers to work on different parts simultaneously without conflicts due to the lack of clear boundaries.

(a) (ii) Program Documentation Types:

  • Internal Documentation (Code Comments): Explanations written directly within the source code (using comment symbols like //, /* */, {}, (* *)) to clarify complex logic, variable purposes, or function behaviors for other programmers (or the future self).
  • External Documentation: Documents separate from the source code that provide broader information about the program. Examples include:
    • User Manuals: Guide end-users on how to install, configure, and use the software.
    • System Documentation: Describes the overall system architecture, design choices, data structures, algorithms, APIs, and installation/maintenance procedures for developers and system administrators.
    • Requirements Specification: Details what the software is supposed to do.

(b) Doubly Linked List:

A doubly linked list is a linear data structure where each element (node) contains not only the data but also two pointers (or links): one pointing to the *previous* node in the sequence and another pointing to the *next* node. This allows for traversal in both forward and backward directions.

  Head -->+------+     +------+     +------+<-- Tail
          | Data |---->| Data |---->| Data |
Null <----| Prev |<----| Prev |<----| Prev |
          | Next |     | Next |     | Next |----> Null
          +------+     +------+     +------+
            Node 1       Node 2       Node 3

Diagram of a Doubly Linked List

  • The `Head` pointer points to the first node.
  • The `Tail` pointer (optional but common) points to the last node.
  • Each node's `Next` pointer points to the subsequent node (or Null if it's the last node).
  • Each node's `Prev` pointer points to the preceding node (or Null if it's the first node).

(a) (i) Stages in Program Development Process:

  1. Problem Definition/Analysis: Understanding the problem clearly, defining requirements, inputs, outputs, and constraints.
  2. Program Design: Planning the solution using tools like flowcharts, pseudocode, or algorithms. Breaking down the problem into smaller modules.
  3. Coding/Implementation: Writing the program code in a chosen programming language based on the design.
  4. Testing and Debugging: Running the program with test data to find and fix errors (bugs). Includes unit testing, integration testing, etc.
  5. Documentation: Writing explanations for users (user manual) and developers (technical/system documentation).
  6. Deployment and Maintenance: Installing the program for users and making necessary updates, corrections, or enhancements over time.

(a) (ii) Term Explanations:

  • Dry Run: Manually tracing the execution of a program or algorithm on paper, step-by-step, using sample data. It helps verify the logic and identify potential errors before actual coding or execution on a computer.
  • Compilation: The process of translating source code written in a high-level programming language (like C or Pascal) into a low-level language (like machine code or bytecode) that the computer can directly execute or interpret. This is done by a program called a compiler.

(b) Fixed vs. Dynamic Data Structures:

  • Fixed Data Structures: These have a predetermined size and memory allocation that is set at compile time and cannot be changed during program execution. Examples include static arrays. Their size is fixed regardless of how much data is actually stored.
  • Dynamic Data Structures: These can grow or shrink in size during program execution as needed. Memory is allocated or deallocated at runtime. Examples include linked lists, trees, and dynamically allocated arrays (using pointers and functions like `malloc`/`New`). They offer flexibility in handling varying amounts of data.

  • (ii) Convert a character to its ASCII code: Ord(char)
  • (iii) Count the number of characters in a string: Length(string)
  • (iv) Display the previous number in an enumerated list: Pred(enumValue)

Answer

(a) Items in Program Documentation for Ease of Access:

  • Table of Contents: Provides an overview of the document structure and allows users to quickly navigate to specific sections.
  • Index: An alphabetical listing of keywords, topics, function names, etc., with page numbers or links, enabling quick lookup of specific information.
  • Glossary: Defines technical terms, acronyms, or domain-specific jargon used within the documentation or program.
  • Search Functionality (for online docs): Allows users to search for keywords across the entire documentation set.
  • Cross-References/Hyperlinks: Links related sections or concepts together, allowing users to easily jump between relevant parts of the documentation.
  • Clear Headings and Subheadings: Organizes information logically and makes it easy to scan for relevant topics.

(b) Event-Driven vs. Object-Oriented Programming:

  • Event-Driven Programming (EDP): Focuses on the flow of program execution being determined by events (e.g., user actions like mouse clicks or key presses, sensor outputs, messages from other programs). The program typically has a main loop that waits for events and then calls specific event handler functions to respond to them. Often used for graphical user interfaces (GUIs).
  • Object-Oriented Programming (OOP): Focuses on organizing code around "objects," which bundle data (attributes) and the functions (methods) that operate on that data. Key principles include encapsulation, inheritance, and polymorphism. It models real-world entities as objects and emphasizes data hiding and modularity through classes.

Note: These paradigms are not mutually exclusive. Many modern applications, especially those with GUIs, use an object-oriented approach to structure the code, where the objects themselves respond to events (e.g., a Button object handles its own Click event).

(d) C Program (Larger Number using Compound Operator):

The question likely refers to the ternary conditional operator (? :), which acts like a compact if-else expression.

#include <stdio.h>

int main() {
    int num1, num2, larger;

    // Prompt user for input
    printf("Enter the first number: ");
    scanf("%d", &num1);

    printf("Enter the second number: ");
    scanf("%d", &num2);

  • \" : Double Quote - Represents a literal double quote character within a string literal.

  • \0 : Null Character - Represents the null terminator, often used to mark the end of strings.

(a) (ii) Portability:

Portability in programming refers to the ability of software (source code or compiled program) to be run on different computer systems (different hardware architectures, operating systems, or environments) with minimal or no modification. High-level languages generally offer better portability than low-level languages because compilers and interpreters handle the platform-specific details.

(b) Pascal File Operations:

Question 8 (20 marks)

(a) (i) Outline two advantages of binary search algorithm. (2 marks)

(ii) Explain two reasons for passing parameters by reference in a subprogram. (4 marks)

(b) Differentiate between serial and sequential file organization methods. (4 marks)

(c) Write a program in C language that prompts a user to enter a number. The program then computes and displays the square root of the number. (4 marks)

(d) Write a program in Pascal language that prompts a user to enter weight in kilograms and height in meters. The program then computes Body Mass Index (BMI) using a function. The program then displays underweight when the BMI is less than 18, Normal when the BMI range is between 18 and 25, otherwise overweight.

(Hint: BMI=Weight/(Height)^2) (6 marks)

Answer

(a) (i) Advantages of Binary Search Algorithm:

  • Efficiency: Very efficient for searching large, sorted datasets. It has a logarithmic time complexity (O(log n)), meaning the time taken increases very slowly as the dataset size grows.
  • Speed: Significantly faster than linear search for large arrays because it eliminates half of the remaining search space in each step.

Note: The main disadvantage is that it requires the data to be sorted beforehand.

(a) (ii) Reasons for Passing Parameters by Reference:

  • Modifying Original Arguments: Allows the called subprogram (function/procedure) to directly modify the value of the original variable passed from the calling code. This is necessary when the subprogram needs to change the state of the caller's data.
  • Efficiency with Large Data: Avoids the overhead of copying large data structures (like arrays or records/structs) when passing them to a subprogram. Passing by reference only passes the memory address, which is much faster and uses less memory than copying the entire structure.

(b) Serial vs. Sequential File Organization:

  • Serial File Organization: Records are stored one after another in the order they are added, typically appended to the end of the file. There is no inherent ordering based on a key field. Accessing a specific record usually requires reading through all preceding records (like a cassette tape). Simple to implement but inefficient for searching.
  • Sequential File Organization: Records are stored physically ordered based on the value of a specific key field (e.g., student ID, account number). This allows for efficient processing of records in key order. While records are physically sequential, accessing a specific record might still involve reading from the beginning, but techniques like indexed sequential access (ISAM) can improve random access speed by using an index.

(c) C Program (Square Root):

#include <stdio.h>
#include <math.h> // Required for the sqrt() function

int main() {
    double number, squareRoot; // Use double for potentially non-integer results

    // Prompt user for input
    printf("Enter a non-negative number: ");
    scanf("%lf", &number); // %lf format specifier for double

    // Check if the number is non-negative
    if (number >= 0) {
        // Calculate the square root using the sqrt() function from math.h
        squareRoot = sqrt(number);

        // Display the result

        printf("Error: Cannot calculate the square root of a negative number.\n");
    }

    return 0;
}

Note: Remember to compile C code using `math.h` with the `-lm` flag on Linux/macOS (e.g., `gcc program.c -o program -lm`) to link the math library.

(d) Pascal Program (BMI Calculator):

program BMICalculator;

var
  weightKg: Real;
  heightM: Real;
  bmiValue: Real;
  category: string;

// Function to calculate BMI
function CalculateBMI(weight: Real; height: Real): Real;
begin
  if height > 0 then // Avoid division by zero
    CalculateBMI := weight / (height * height) // BMI = Weight / (Height^2)
  else
    CalculateBMI := 0; // Return 0 or handle error appropriately
end;

// Function or Procedure to determine category (optional, can be in main block)
function GetBMICategory(bmi: Real): string;
begin
  if bmi <= 0 then
    GetBMICategory := 'Invalid Data'
  else if bmi < 18.5 then // Common threshold is 18.5
    GetBMICategory := 'Underweight'
  else if bmi < 25 then // Common range 18.5 to 24.9
    GetBMICategory := 'Normal'
  else
    GetBMICategory := 'Overweight';
end;

begin
  // Get input from user
  Write('Enter weight in kilograms: ');
  Readln(weightKg);

  Write('Enter height in meters: ');
  Readln(heightM);

  // Calculate BMI using the function
  bmiValue := CalculateBMI(weightKg, heightM);

  // Determine the category
  category := GetBMICategory(bmiValue);

  // Display the results
  Writeln; // Blank line
  Writeln('Weight: ', weightKg:0:2, ' kg');
  Writeln('Height: ', heightM:0:2, ' m');
  if bmiValue > 0 then
    Writeln('Calculated BMI: ', bmiValue:0:2)
  else
    Writeln('Could not calculate BMI (invalid height?)');

  Writeln('Category: ', category);

  Readln; // Keep console window open
end.

Note: The BMI thresholds used (18, 25) are based on the question. Standard medical thresholds often use 18.5 and 24.9. The code uses 18.5 and 25 for clarity based on standard practice but adheres to the question's logic for the final category output.
