Question 1 (20 marks)

(a) State two programming paradigms giving an example of a programming language in each case. (4 marks)

(b) Differentiate between structured and user defined data types as used in C programming language. (4 marks)

(c) Explain each of the following stages used when solving a problem through computer programming: (6 marks)

  • (i) Problem definition;
  • (ii) Writing the algorithm;
  • (iii) Testing the solution.

(d) Write a C program that prompts a user to enter three integers and displays the largest of them. Use if ... else control structure. (6 marks)

Answer

(a) Programming Paradigms:

  1. Procedural/Imperative Programming: Programs are composed of procedures/functions that operate on data. Example: C, Pascal, FORTRAN.
  2. Object-Oriented Programming: Programs are organized as collections of objects that contain both data and code. Example: Java, C++, Python.
  3. Functional Programming: Programs are constructed by applying and composing functions. Example: Haskell, Lisp, ML.
  4. Logic Programming: Programs express facts and rules about problems. Example: Prolog.
(Any two with examples)

(b) Structured vs. User Defined Data Types (C):

  • Structured Data Types:
    • Built into the language and provided by default.
    • Examples include arrays, structures (struct), and unions.
    • Have predefined organization and behavior.
    • Can be used directly without additional definition.
  • User Defined Data Types:
    • Created by the programmer using `typedef` keyword.
    • Can be based on existing types or combinations thereof.
    • Custom designed for specific program needs.
    • Must be defined before use.

(c) Stages in Problem Solving through Programming:

  • (i) Problem Definition:
    • Clear understanding and statement of what needs to be solved.
    • Identify inputs, required outputs, and processing requirements.
    • Define constraints and limitations.
    • Establish success criteria.
  • (ii) Writing the Algorithm:
    • Design step-by-step solution to the problem.
    • Break down complex operations into simpler steps.
    • Use pseudocode or flowcharts to represent the logic.
    • Ensure completeness and logical flow.
  • (iii) Testing the Solution:
    • Verify correctness of implementation.
    • Use test data to check program behavior.
    • Debug and fix any errors found.
    • Validate against requirements.

Question 2 (20 marks)

(a) Explain two ways through which parameters can be passed to a function. (4 marks)

(b) With the aid of an example in each case, distinguish between relational and logical operators as used in structured programming. (4 marks)

(c) Describe each of the following data structures as used in programming: (4 marks)

(d) Table 1 shows a criteria used by a supermarket to award discounts to the customers based on the amount spent.

Amount Spent (Kshs) Discount Rate
10000 - 1499910%
15000 - 2499915%
25000 - 2999920%
30000 and over25%

Write a C program that prompts a user for the amount spent. The program should use a function to calculate and display the net amount paid by the customer. (8 marks)

Hint: Net amount = Amount spent — (Discount Rate × Amount spent)

Answer

(a) Parameter Passing Methods:

  1. Pass by Value:
    • A copy of the argument's value is passed to the function
    • Changes to the parameter inside the function do not affect the original argument
    • Example in C: void square(int x) { x = x * x; }
  2. Pass by Reference:
    • The memory address (reference) of the argument is passed
    • Changes to the parameter affect the original argument
    • Example in C: void swap(int *x, int *y) { int temp = *x; *x = *y; *y = temp; }

(b) Relational vs. Logical Operators:

  • Relational Operators:
    • Compare two values and return true (1) or false (0)
    • Examples: 
      int x = 5, y = 10;
x < y    // Returns 1 (true)
x == y   // Returns 0 (false)
x >= 5   // Returns 1 (true)
  • Logical Operators:
    • Combine multiple conditions and return true (1) or false (0)
    • Examples: 
      int a = 1, b = 0;
a && b   // AND: Returns 0 (false)
a || b   // OR: Returns 1 (true)
!a       // NOT: Returns 0 (false)

(c) Data Structures:

  • (i) Linked List:
    • A sequential collection of elements, where each element points to the next element
    • Dynamic data structure that can grow or shrink during execution
    • Elements (nodes) contain data and a pointer/reference to the next node
    • Non-contiguous memory allocation
    • Good for frequent insertions and deletions
  • (ii) Array:
    • Collection of elements of the same data type stored in contiguous memory locations
    • Fixed size (in most implementations)
    • Elements accessed using index numbers
    • Good for random access and when size is known
    • Memory efficient for sequential data

(d) C Program (Discount Calculator):

#include <stdio.h>

// Function to calculate discount rate based on amount
double getDiscountRate(double amount) {
    if (amount >= 30000) {
        return 0.25;  // 25% discount
    } else if (amount >= 25000) {
        return 0.20;  // 20% discount
    } else if (amount >= 15000) {
        return 0.15;  // 15% discount
    } else if (amount >= 10000) {
        return 0.10;  // 10% discount
    } else {
        return 0.0;   // No discount
    }
}

// Function to calculate and display net amount
void calculateNetAmount(double amount) {
    double discountRate = getDiscountRate(amount);
    double discount = amount * discountRate;
    double netAmount = amount - discount;
    
    printf("\nAmount Spent: %.2f\n", amount);
    printf("Discount Rate: %.1f%%\n", discountRate * 100);
    printf("Discount Amount: %.2f\n", discount);
    printf("Net Amount to Pay: %.2f\n", netAmount);
}

int main() {
    double amountSpent;
    
    printf("Enter amount spent: ");
    if (scanf("%lf", &amountSpent) != 1 || amountSpent < 0) {
        printf("Invalid amount entered.\n");
        return 1;
    }
    
    calculateNetAmount(amountSpent);
    return 0;
}

The program includes input validation and uses functions for modularity. The discount calculation is separated from the main logic for better organization and reusability.

(c) Pseudocode (Product of Numbers 1-15):

START
   DECLARE product AS INTEGER
   DECLARE counter AS INTEGER
   
   SET product = 1
   SET counter = 1
   
   WHILE counter <= 15 DO
      SET product = product * counter
      INCREMENT counter
   END WHILE
   
   DISPLAY "The product of numbers 1 to 15 is: ", product
END

Question 7 (20 marks)

(a) Explain two ways in which records can be organized in a computer file. (4 marks)

(b) Describe each of the following types of files generated during program development and compilation: (4 marks)

  • (i) Source File;
  • (ii) Object File.

(c) Write a Pascal program that prompts a user to enter two integer values and divides the first by the second. Provide an error handler to display the text "Do not enter the value zero as a second value" if the second value is zero. Use goto statement. (6 marks)

(d) Write a Pascal program that prompts a user to enter the value of radius and uses it to calculate the area of a circle. Initialize constant pie to 3.142 (6 marks)

Answer

(a) Record Organization in Computer Files:

  1. Sequential Organization:
    • Records are stored one after another in physical order
    • New records are added at the end of the file
    • Must read through all previous records to access a specific record
    • Good for batch processing and when all records need to be processed
  2. Random/Direct Organization:
    • Records can be accessed directly using a key/address
    • Each record has a unique address for direct access
    • Faster access to individual records
    • Good for interactive applications requiring quick access

(b) Types of Files in Program Development:

  • (i) Source File:
    • Contains human-readable program code
    • Written in a programming language (e.g., .c, .pas files)
    • Can be edited and modified by programmers
    • Needs to be compiled before execution
  • (ii) Object File:
    • Contains machine code produced by compiler
    • Binary format, not human-readable
    • Created as an intermediate step before linking
    • Combined with other object files to create executable

        else
        {
            printf("%d", i);
            printf(" is an odd number \n");
        }
    }
    return 0;
}

Answer

(a) Rules for Choosing Variable Names:

  1. Must Begin with Letter or Underscore:
    • First character must be a letter (a-z, A-Z) or underscore (_)
    • Cannot start with numbers or special characters
  2. Use Only Valid Characters:
    • Can contain letters, numbers, and underscores
    • No spaces or special characters allowed
  3. Be Descriptive and Meaningful:
    • Name should indicate the purpose or content
    • Avoid single-letter names (except for simple counters)
  4. Follow Language Conventions:
    • Observe case sensitivity rules
    • Use appropriate naming conventions (camelCase, snake_case)

(b) Top-down Program Design Recommendations:

  1. Complex Systems:
    • When dealing with large, complex problems
    • Helps break down into manageable sub-problems
    • Makes the development process more organized
  2. Team Development:
    • When multiple developers work on the same project
    • Allows clear division of tasks
    • Facilitates parallel development

(c) Declared vs. Defined Constants:

Declared Constants (const) Defined Constants (#define)
Processed by compiler Processed by preprocessor
Memory allocated during execution Simple text substitution before compilation
Type checking performed No type checking
Example: const int MAX = 100; Example: #define MAX 100
