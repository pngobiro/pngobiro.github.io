Introduction to Relational Databases

A Relational Database Management System (RDBMS) is a database management system based on the relational model introduced by E. F. Codd. The data in an RDBMS is stored in database objects called tables, with data organized in rows and columns.

Core Components

  • Relational data structure
  • Relational integrity constraints
  • Relational algebra or equivalent SQL

Characteristics of Relational Databases

  • Table-Based Structure - Data must be represented in tables
  • Column Access - Data within columns must be accessible by specifying table name, column name, and primary key
  • Null Value Support - Systematic handling of missing and inapplicable information
  • Active Catalog - Online catalog support for database information
  • Language Support - Support for data definition, manipulation, and transaction management
  • View Updates - Views must be updatable by the system
  • Set Operations - Support for insert, update, and delete operations on sets
  • Data Independence - Support for both logical and physical data independence
  • Integrity Rules - Constraints must be stored in the catalog
  • Distribution Independence - Applications should run unchanged when data is redistributed

Relational Calculus

Relational calculus is a non-procedural query language that describes what data is to be retrieved rather than how to retrieve it. There are two types:

Tuple Relational Calculus

  • Uses tuple variables that range over tuples
  • Expressions define the retrieved tuples
  • Uses logical operators and quantifiers
  • Results in a new relation

Domain Relational Calculus

  • Uses domain variables that range over domains
  • Expressions specify attribute values
  • More similar to first-order logic
  • Equivalent in power to tuple calculus

Common Relational Operators

  • Selection - Filters rows based on a condition
  • Projection - Selects specific columns
  • Join - Combines tables based on related columns
  • Union - Combines rows from similar tables
  • Intersection - Finds common rows between tables
  • Division - Finds rows that match all values in another table

Practical Applications

  • Data retrieval and querying
  • Data manipulation and updates
  • Database design and normalization
  • Business intelligence and reporting
  • Data integration and migration
  • Application development
