Transaction Processing

A transaction is a unit of work that should be treated as a whole. It either completes entirely or not at all.

ACID Properties

  • Atomicity - Transaction is all or nothing
  • Consistency - Database remains in a consistent state
  • Isolation - Transactions execute independently
  • Durability - Changes are permanent once committed

Transaction States

  • Active - Initial state
  • Partially Committed - After last operation
  • Committed - After successful completion
  • Failed - After unsuccessful execution
  • Aborted - After rollback
  • Terminated - Final state

Concurrency Control

Mechanisms to handle multiple users accessing the database simultaneously.

Concurrency Problems

  • Lost Update - One transaction overwrites another's changes
  • Dirty Read - Reading uncommitted changes
  • Non-repeatable Read - Different results from same query
  • Phantom Read - New rows appear in repeated query

Control Mechanisms

  • Locking
    • Shared (Read) Locks
    • Exclusive (Write) Locks
    • Two-Phase Locking
  • Timestamp Ordering
  • Optimistic Control
  • Multiversion Control

Database Recovery

Techniques to restore database to a consistent state after failure.

Types of Failures

  • Transaction failures
  • System crashes
  • Media failures
  • Natural disasters

Recovery Techniques

  • Backup
    • Full backup
    • Incremental backup
    • Differential backup
  • Transaction Log
  • Checkpoint Recovery
  • Shadow Paging

Database Security

Security Measures

  • Authentication - Verifying user identity
  • Authorization - Controlling access rights
  • Access Control - Managing permissions
  • Data Encryption - Protecting sensitive data
  • Auditing - Tracking database activities

Types of Controls

  • Administrative controls
  • Physical controls
  • Technical controls
  • Encryption mechanisms

Database Administration

DBA Responsibilities

  • Installation and configuration
  • Schema definition
  • Storage management and monitoring
  • Security and authorization
  • Backup and recovery
  • Performance tuning

Monitoring Tasks

  • Space utilization
  • User activity
  • System performance
  • Error logs
  • Backup status

Management Best Practices

  • Regular backups and testing
  • Implementing security policies
  • Performance monitoring and tuning
  • Documentation maintenance
  • Regular updates and patches
  • Disaster recovery planning
  • User training and support
