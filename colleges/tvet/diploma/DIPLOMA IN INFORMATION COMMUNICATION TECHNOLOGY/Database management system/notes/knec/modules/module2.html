Introduction to Database Organization

Data organization refers to the method of classifying and organizing data sets to make them more useful. The organization of a database is mainly based on:

  • The size of the organization
  • The structure of the organization
  • Required level of security

Centralized Database

A centralized database is a database that is located, stored, and maintained in a single location. This location is often a central computer or database system.

Advantages of Centralized Databases

  • Data integrity is maximized and redundancy is minimized
  • Better data security due to single storage location
  • Better data preservation through fault-tolerant setup
  • Easier end-user access due to simple database design
  • More cost-effective than other types
  • Easier data maintenance and updates

Disadvantages of Centralized Databases

  • High dependency on network connectivity
  • Potential bottlenecks due to high traffic
  • Limited concurrent access
  • Risk of complete data loss in case of hardware failure

Client-Server Architecture

The data processing is split into distinct parts: requester (client) and provider (server). The client sends requests to servers to perform specified tasks.

Types of Client-Server Architecture

  • Two-tier Architecture - Direct communication between client and server
  • Three-tier Architecture - Includes presentation, application, and data tiers

Advantages

  • Centralized control and administration
  • Easier resource management
  • Better backup and recovery options
  • Improved scalability
  • Remote accessibility

Disadvantages

  • Network congestion risks
  • Higher costs for setup and maintenance
  • Requires professional IT management
  • Server failure affects all clients

Distributed Database Systems

A distributed database is a collection of multiple interconnected databases spread physically across various locations that communicate via a computer network.

Features of Distributed Databases

  • Databases are logically interrelated
  • Data is physically stored across multiple sites
  • Sites are connected via network
  • Each site can manage data independently

Types of Distributed Databases

  • Homogeneous - All sites use identical DBMS
  • Heterogeneous - Sites may use different DBMS

Advantages

  • Better reliability and availability
  • Improved performance
  • Easier expansion
  • Local data control and access

Disadvantages

  • Complex design and maintenance
  • Increased storage requirements
  • Potential security risks
  • Complex backup and recovery procedures
