1.1 Meaning of DBMS

A database is a collection of information that is organized so that it can be easily accessed, updated and managed.

A Database Management System (DBMS) is software that allows creation and manipulation of databases, enabling users to store, process and analyze data easily. It provides:

  • An interface for database operations
  • Data protection and security
  • Data consistency maintenance for multiple users

Examples of Popular DBMS:

  • MySQL - Open source relational database management system
  • MS Access - Desktop database application
  • Oracle Database - Enterprise-level RDBMS
  • Microsoft SQL Server - Windows-based RDBMS
  • PostgreSQL - Open source object-relational DBMS

Characteristics of DBMS

  • Data stored in Tables
  • Reduced Redundancy
  • Data Consistency
  • Support for Multiple Users
  • Query Language Support
  • Security Features
  • Transaction Support

Database Schema

A database schema is the skeleton structure that represents the logical view of the entire database. It defines:

  • How data is organized
  • Relationships between data elements
  • Formal description of data elements

Types of Schema:

  • Physical Database Schema - Describes actual storage of data
  • Logical Database Schema - Defines logical constraints and data structures

Key Players in Database Design and Development

  • Database Administrator (DBA) - Responsible for installation, configuration, upgrading, administration, monitoring, maintenance, and security of databases
  • Database Designer - Defines the database structure including tables, indexes, views, constraints, triggers, and stored procedures
  • Database Developer - Implements database designs and writes database-specific code
  • End Users - Individuals who interact with the database through applications
