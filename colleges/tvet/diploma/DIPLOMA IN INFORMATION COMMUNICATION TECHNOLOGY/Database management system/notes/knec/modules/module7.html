Introduction to SQL

SQL (Structured Query Language) is the standard language for managing and manipulating relational databases.

Categories of SQL Commands

  • DDL (Data Definition Language) - CREATE, ALTER, DROP, TRUNCATE
  • DML (Data Manipulation Language) - SELECT, INSERT, UPDATE, DELETE
  • DCL (Data Control Language) - GRANT, REVOKE
  • TCL (Transaction Control Language) - COMMIT, ROLLBACK, SAVEPOINT

Basic SQL Queries

SELECT Statement

  • Basic syntax: SELECT column FROM table
  • Selecting multiple columns
  • Selecting all columns with *
  • Using WHERE clause for filtering
  • ORDER BY for sorting results

Common Operators

  • Comparison operators: =, <>, >, <, >=, <=
  • Logical operators: AND, OR, NOT
  • LIKE for pattern matching
  • IN for multiple value matching
  • BETWEEN for range queries

Working with Joins

Types of Joins

  • INNER JOIN - Returns matching records from both tables
  • LEFT JOIN - Returns all records from left table and matching from right
  • RIGHT JOIN - Returns all records from right table and matching from left
  • FULL JOIN - Returns all records when there's a match in either table
  • CROSS JOIN - Returns Cartesian product of both tables

Aggregate Functions

  • COUNT() - Counts number of rows
  • SUM() - Calculates sum of values
  • AVG() - Calculates average of values
  • MAX() - Finds maximum value
  • MIN() - Finds minimum value

Grouping

  • GROUP BY clause
  • HAVING clause for filtering groups
  • Combining multiple aggregate functions

Subqueries

  • Single Row Subqueries - Return one row
  • Multiple Row Subqueries - Return multiple rows
  • Correlated Subqueries - Reference outer query
  • Nested Subqueries - Multiple levels of subqueries

Query Optimization Techniques

  • Using appropriate indexes
  • Avoiding SELECT *
  • Using WHERE clauses effectively
  • Minimizing the use of OR conditions
  • Using EXPLAIN PLAN
  • Proper join techniques
  • Limiting result sets

Advanced SQL Features

  • Views - Virtual tables based on query results
  • Stored Procedures - Precompiled SQL statements
  • Triggers - Automatic actions on data changes
  • Transactions - Ensuring data integrity
  • Window Functions - Calculations across row sets

SQL Best Practices

  • Write clear, readable queries
  • Use meaningful table and column aliases
  • Comment complex queries
  • Follow consistent naming conventions
  • Use parameters for dynamic values
  • Consider performance impact
  • Regular testing and optimization
  • Proper error handling
