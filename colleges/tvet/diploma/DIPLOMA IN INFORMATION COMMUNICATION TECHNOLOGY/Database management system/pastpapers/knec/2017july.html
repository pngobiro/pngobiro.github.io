Database Management Systems

July 2017 Past Paper

Paper Code: 2920/206

Duration: 3 hours

Module II

INSTRUCTIONS TO CANDIDATES

  • You should have an answer booklet for this examination.
  • This paper consists of EIGHT questions.
  • Answer any FIVE in the answer booklet provided.
  • All questions carry equal marks.
  • Candidates should answer the questions in English.
  • This paper consists of 6 printed pages.

Question 1

(a) Define each of the following terms as used in databases: (4 marks)

  • (i) Normalization;
  • (ii) Compound key.

(b) Explain the term database organization as used in databases. (2 marks)

(c) Distinguish between the role of a database end user and database developer. (4 marks)

(d) Table 1 is a structure of a table named Department in a database. Use it to answer the questions that follow.

Write an SQL statement to perform each of the following: (8 marks)

  • (i) Rename the field named DeptNo to departmentName;
  • (ii) Remove the field named Location;
  • (iii) Add a field named dateStarted having date type;
  • (iv) Remove the table.

(e) Explain the data type varchar as used in the structure of the table. (2 marks)

Answers

(a) Database terms:

  • (i) Normalization: Process of organizing data to minimize redundancy and dependency
  • (ii) Compound key: Key composed of multiple columns that are all foreign keys

(b) Database organization:

Logical structuring of data into related tables and relationships to ensure efficient storage and retrieval

(c) End user vs Developer:

  • End user: Interacts with database through applications, queries data
  • Developer: Creates and maintains database structure, writes applications

(d) SQL statements:

ALTER TABLE Department
RENAME COLUMN DeptNo TO departmentName;

ALTER TABLE Department
DROP COLUMN Location;

ALTER TABLE Department
ADD COLUMN dateStarted DATE;

DROP TABLE Department;

(e) VARCHAR data type:

Variable-length character string that stores only the characters used, up to specified maximum length

Question 2

(a) Explain two reasons that may lead an organisation not to use a database management system. (4 marks)

(b) Outline two roles of each of the following in a database management system environment: (4 marks)

  • (i) Hardware;
  • (ii) Data.

(c) Interpret the following SQL statement: (3 marks)

SELECT deptNo, count(*)
FROM Employees
group by deptNo;

(d) Table 2 shows unnormalised table. State a step by step process of normalising to 3NF. (9 marks)

Answers

(a) Reasons not to use DBMS:

  • High initial investment cost
  • Simple data requirements with few users
  • Limited hardware resources
  • Real-time requirements that DBMS overhead can't meet

(b) Roles:

(i) Hardware roles:

  • Data storage and retrieval
  • Processing database operations

(ii) Data roles:

  • Information storage and representation
  • Support for decision making

(c) SQL interpretation:

Counts the number of employees in each department, grouping results by department number

(d) Normalization steps to 3NF:

  1. First Normal Form (1NF):
    • Remove repeating groups
    • Create separate rows for each value
  2. Second Normal Form (2NF):
    • Meet 1NF requirements
    • Remove partial dependencies
  3. Third Normal Form (3NF):
    • Meet 2NF requirements
    • Remove transitive dependencies

Question 3

(a) Outline two reasons for considering database security. (2 marks)

(b) Explain two recovery techniques that can be used in case of a database failure. (4 marks)

(c) Explain each of the following terms as used during the creation of tables in a database: (6 marks)

  • (i) cascade;
  • (ii) restrict;
  • (iii) set null.

(d) Study the following narrative and use it to answer the question that follows:

A student must be enrolled on only one course and a course must have students enrolled on it. Each course is identified by a deptId, courseId and course name. Each course must have a number of modules and a module must be part of at least one course. A module is identified by moduleId, module name and must have a lecturer as a subject leader but a lecturer may not necessarily be a subject leader or may lead more than one module.

Draw an entity relationship diagram showing cardinality and optionality. (8 marks)

Answers

(a) Database security reasons:

  • Protect sensitive data from unauthorized access
  • Maintain data integrity and prevent corruption

(b) Recovery techniques:

  • Backup and restore:
    • Full database backup
    • Point-in-time recovery
  • Transaction logs:
    • Logging all changes
    • Rolling back incomplete transactions

(c) Table creation terms:

  • CASCADE: Changes to parent propagate to child records
  • RESTRICT: Prevents changes if dependent records exist
  • SET NULL: Sets child record values to NULL when parent is modified

(d) ER Diagram components:

  • Entities:
    • Student
    • Course (deptId, courseId, courseName)
    • Module (moduleId, moduleName)
    • Lecturer
  • Relationships:
    • Student to Course (M:1, mandatory both sides)
    • Course to Module (M:M, mandatory both sides)
    • Module to Lecturer (M:1, mandatory for module, optional for lecturer)

Question 4

(a) Distinguish between the Grant and Revoke commands as used in databases. (4 marks)

(b) With the aid of a diagram distinguish between the output FROM the union and the intersection operators. (6 marks)

(c) Explain two characteristics of the conceptual model of a database design. (4 marks)

(d)

  • (i) Write the algebraic expressions equivalent to the following SQL statement: (3 marks)
    • SELECT Distinct Gender, Salary
FROM Employee;
  • (ii) Write the SQL statement equivalent to the following algebraic expression: (3 marks)
    • σassetcategory=furniture (ASSET)

Answers

(a) GRANT vs REVOKE:

  • GRANT: Gives permissions/privileges to users
  • REVOKE: Removes previously granted permissions

(b) Union vs Intersection:

  • Union (A ∪ B): All records from both sets
  • Intersection (A ∩ B): Only records common to both sets

(c) Conceptual model characteristics:

  • Technology independent
  • Represents business view of data
  • Shows entities and relationships
  • Abstracts implementation details

(d) Expressions:

(i) SQL to Algebra:

π DISTINCT Gender, Salary (Employee)

(ii) Algebra to SQL:

SELECT * 
FROM ASSET 
WHERE assetcategory = 'furniture';

Question 5

(a) Define the term data abstraction as used in database management systems. (2 marks)

(b) Explain two disadvantages of database approach. (4 marks)

(c) With the aid of an example describe two categories of SQL statements as used in databases. (5 marks)

(d) Table 3 shows a table named Employee in a database used by employer to manage employees' salaries. Write an SQL statement that would: (9 marks)

  • (i) display the salary of all employees whose Grade value is null;
  • (ii) change Alex's salary to 30000;
  • (iii) display all employees with grade A and salaries with values between 10000 and 15000;
  • (iv) display the minimum and maximum salary.

Answers

(a) Data abstraction:

Hiding implementation details while showing only relevant features to users

(b) Database approach disadvantages:

  • High initial cost
  • Complex backup and recovery
  • Requires specialized staff
  • Performance overhead

(c) SQL statement categories:

  • DDL (Data Definition Language): 
    CREATE TABLE, ALTER TABLE, DROP TABLE
  • DML (Data Manipulation Language): 
    SELECT, INSERT, UPDATE, DELETE

(d) SQL queries:

SELECT salary 
FROM Employee 
WHERE Grade IS NULL;

UPDATE Employee 
SET salary = 30000 
WHERE name = 'Alex';

SELECT * 
FROM Employee 
WHERE grade = 'A' 
AND salary BETWEEN 10000 AND 15000;

SELECT MIN(salary), MAX(salary) 
FROM Employee;

Question 6

(a) Outline three types of end users in a database environment. (3 marks)

(b) Differentiate between the prime attribute and a non-prime attribute as used in databases. (4 marks)

(c) Describe two ways of classifying database management system. (4 marks)

(d) Table 4 shows a database table named students. Write a relational calculus statement that would:

  • (i) display the records for the fields SID, SNAME and LNAME of all students. (3 marks)
  • (ii) display the records in the field SNAME, LNAME and MARKS for all students with MARKS values more than 60. (3 marks)
  • (iii) display the MARKS values for a student with SNAME is "Catherine" and LNAME "Joseph". (3 marks)

Answers

(a) Types of end users:

  • Naive users (application users)
  • Casual users (query users)
  • Sophisticated users (analysts)

(b) Prime vs Non-prime attributes:

  • Prime attribute: Part of any candidate key
  • Non-prime attribute: Not part of any candidate key

(c) DBMS classification:

  • By data model:
    • Relational
    • Object-oriented
    • Network
  • By number of users:
    • Single-user
    • Multi-user

(d) Relational calculus:

{t | Student(t) ∧ t[SID, SNAME, LNAME]}

{t | Student(t) ∧ t[MARKS] > 60 ∧ t[SNAME, LNAME, MARKS]}

{t | Student(t) ∧ t[SNAME]="Catherine" ∧ t[LNAME]="Joseph" ∧ t[MARKS]}

Question 7

(a) Explain each of the following terms as used in databases: (4 marks)

  • (i) Two-tier client server architecture;
  • (ii) Candidate key.

(b) Susan, a database administrator is working on a table named Employee with the following attributes:

Employee (#EmployeeNumber, DepartmentNumber, hours, EmployeeNumber, DepartmentName, DepartmentLocation)

  • (i) State three functional dependencies that can be derived from the table. (3 marks)
  • (ii) Outline three challenges she would encounter when working with data on the table. (3 marks)

(c) Explain two reasons that may lead to the migration from the file based to a database centered approach. (4 marks)

(d) With the aid of a diagram outline the stages of the database design cycle. (6 marks)

Answers

(a) Database terms:

  • (i) Two-tier client server: Architecture where client directly communicates with database server
  • (ii) Candidate key: Minimal set of attributes that can uniquely identify a tuple

(b) Employee table analysis:

(i) Functional dependencies:

  • EmployeeNumber → DepartmentNumber, hours
  • DepartmentNumber → DepartmentName, DepartmentLocation
  • EmployeeNumber → DepartmentName

(ii) Challenges:

  • Data redundancy (EmployeeNumber repeated)
  • Update anomalies
  • Data inconsistency

(c) Migration reasons:

  • Need for data sharing and integration
  • Reduce data redundancy
  • Better data consistency
  • Improved data security

(d) Database design cycle stages:

  • Requirements analysis
  • Conceptual design
  • Logical design
  • Physical design
  • Implementation
  • Testing and maintenance

Question 8

(a) Explain two circumstances under which an organization would implement a centralized database system. (4 marks)

(b) With the aid of a diagram outline two paradigms the early models of databases were based on. (4 marks)

(c) Distinguish between the entity relationship diagrams in figure 1 and in figure 2. (4 marks)

(d) Write SQL statements that would perform each of the following operations on the table named Department: (8 marks)

  • (i) Make the field Dnumber a primary key;
  • (ii) Make the field Dname VARCHAR(50) not to accept a null value;
  • (iii) Make the field named Dnumber to hold values greater than 10.

Answers

(a) Centralized database circumstances:

  • Small organization with single location
  • Need for tight data control
  • Limited resources for distributed system
  • High security requirements

(b) Early database paradigms:

  • Hierarchical model (tree structure)
  • Network model (graph structure)

(c) ER diagram differences depend on figures provided in exam

(d) SQL statements:

ALTER TABLE Department
ADD PRIMARY KEY (Dnumber);

ALTER TABLE Department
MODIFY Dname VARCHAR(50) NOT NULL;

ALTER TABLE Department
ADD CONSTRAINT chk_Dnumber 
CHECK (Dnumber > 10);
