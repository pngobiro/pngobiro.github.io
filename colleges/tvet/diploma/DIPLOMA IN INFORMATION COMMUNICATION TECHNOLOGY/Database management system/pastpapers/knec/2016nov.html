Database Management Systems

November 2016 Past Paper

Paper Code: 2920/206

Duration: 3 hours

Module II

INSTRUCTIONS TO CANDIDATES

  • Answer any FIVE of the following EIGHT questions in this paper in the answer booklet provided.
  • All questions carry equal marks.
  • Candidates to answer the questions in English.
  • This paper consists of 7 printed pages.
  • Candidates should check the question paper to ascertain that all pages are printed.

Question 1

(a) Define the following terms:

  • (i) Database Schema
  • (ii) Database Management System (4 marks)

(b) Table 1 shows a student's results slip. Normalize to 3NF. (9 marks)

(c) The following SQL statement has errors:

SELECT Name location
FROM EMPLOYEE
Where location=Nairobi;

Identify the errors and correct the statements. (3 marks)

(d) Differentiate between fully functional and transitive dependency as used in databases. (4 marks)

Answers

(a) Definitions:

  • (i) Database Schema: Formal description of the structure of a database including tables, fields, relationships, and constraints
  • (ii) Database Management System: Software system that enables users to create, maintain, and access databases

(c) SQL Statement Errors:

  • No comma between Name and location
  • location=Nairobi needs quotes around string value
  • Where should be WHERE (case sensitive)

Corrected statement:

SELECT Name, location
FROM EMPLOYEE
WHERE location = 'Nairobi';

(d) Dependencies:

  • Fully functional: Non-key attributes depend on entire primary key
  • Transitive: Attribute depends on another through an intermediate attribute

Question 2

(a) Explain the function of the following in a database management systems:

  • (i) Data Manipulations Language;
  • (ii) Data Definition Language. (4 marks)

(b) With the aid of an example in each case distinguish between binary and shared locks in concurrent processing. (4 marks)

(c) Table 2 shows the design details of a table named Asset. Write an SQL statement that would:

  • (i) Create the table; (4 marks)
  • (ii) Add the Asset_Category column to the table; (2 marks)
  • (iii) Add the specified values to the table. (4 marks)

(d) A database administrator has defined the data type of a field as Decimal (6,2). With the aid of an example interpret this data type. (2 marks)

Answers

(a) Database languages:

  • (i) DML: Used to manipulate data (SELECT, INSERT, UPDATE, DELETE)
  • (ii) DDL: Used to define database structure (CREATE, ALTER, DROP)

(b) Lock types:

  • Binary locks:
    • Exclusive access
    • Example: Write lock on a record
  • Shared locks:
    • Multiple readers allowed
    • Example: Multiple users reading same record

(c) SQL Statements:

CREATE TABLE Asset (
    AssetID VARCHAR(10) PRIMARY KEY,
    AssetName VARCHAR(50),
    PurchaseDate DATE,
    Cost DECIMAL(10,2)
);

ALTER TABLE Asset
ADD COLUMN Asset_Category VARCHAR(30);

INSERT INTO Asset 
VALUES ('A001', 'Laptop', '2023-01-15', 1200.00, 'Electronics');

(d) Decimal(6,2):

  • 6 total digits with 2 decimal places
  • Example: 1234.56 (valid), 123456.78 (invalid - too many digits)

Question 3

(a) Define the term weak entity as used in databases. (2 marks)

(b) Explain two circumstances under which a hospital would use a database management system. (4 marks)

(c) Study the scenario below:

A computer company undertakes a number of projects. Either an external client or an internal department handles a project. An external client or internal department may have several projects at one go. A consultant manages each project.

Draw an entity relationship diagram to represent the scenario showing the relationship cardinality. (6 marks)

(d) Table 3 shows details of Students. Write an algebraic expression that would:

  • (i) Display all records of female students who attained more than 70 marks; (2 marks)
  • (ii) Display all the course_Id of the students; (2 marks)
  • (iii) Rename the relation Students to 'Student Marks' and the attribute Student_Id to S_id. (4 marks)

Answers

(a) Weak entity:

An entity that cannot exist without another entity and doesn't have its own primary key

(b) Hospital DBMS use cases:

  • Patient records management
  • Appointment scheduling
  • Inventory management
  • Staff scheduling

(c) ER Diagram components:

  • Entities:
    • Project
    • External Client
    • Internal Department
    • Consultant
  • Relationships:
    • Client/Department to Project (1:M)
    • Consultant to Project (1:M)

(d) Relational algebra:

(i) Female students with marks > 70:

σgender='F' ∧ marks>70 (Students)

(ii) Course IDs:

πcourse_Id (Students)

(iii) Rename operation:

ρStudent_Marks(S_id←Student_Id) (Students)

Question 4

(a) Outline two functions of a system analyst during database development. (2 marks)

(b) Explain two advantages of a distributed database systems. (4 marks)

(c)

  • (i) Explain two benefits of normalization in databases. (4 marks)
  • (ii) Explain a scenario in the table that may lead to occurrence of each of the following anomalies:
    • update anomaly; (2 marks)
    • insertion anomaly; (2 marks)
    • deletion anomaly. (2 marks)

(d) Distinguish between the output of SQL Statement A and B: (4 marks)

A.                          B.
SELECT Distinct (Grade)    SELECT Grade
FROM employees;           FROM employees;

Answers

(a) System analyst functions:

  • Requirements gathering and analysis
  • Database design and modeling

(b) Distributed database advantages:

  • Improved reliability and availability
  • Better performance through local access
  • Scalability and growth flexibility
  • Location independence

(c) Normalization:

(i) Benefits:

  • Reduces data redundancy
  • Ensures data consistency
  • Simplifies data maintenance
  • Improves data integrity

(ii) Anomalies:

  • Update: Same information stored in multiple places needs update
  • Insertion: Cannot add new record without related data
  • Deletion: Deleting one fact unintentionally removes other facts

(d) SQL output difference:

  • Statement A: Returns unique grade values only
  • Statement B: Returns all grade values including duplicates

Question 5

(a) Outline four components of a database management system. (4 marks)

(b) Explain two circumstances that would make an organisation to implement a client server database architecture. (4 marks)

(c) Using the given tables differentiate between the output generated from the operations A∪B and A∩B operations. (6 marks)

(d)

  • (i) A database administrator wants to apply the union, intersection and difference operation to a relation. Explain two conditions he must check that the relations fulfil before applying the operations. (4 marks)
  • (ii) State the meaning of ACID in Database Management Systems. (2 marks)

Answers

(a) DBMS components:

  • Query processor
  • Storage manager
  • Security manager
  • Transaction manager

(b) Client-server implementation circumstances:

  • Large number of concurrent users
  • Need for centralized data control
  • Complex data processing requirements
  • Better security requirements

(c) Set operations:

  • A∪B (Union): All records from both tables without duplicates
  • A∩B (Intersection): Only records that exist in both tables

(d) Relation operations:

(i) Conditions for set operations:

  • Same number of attributes
  • Compatible data types for corresponding attributes

(ii) ACID:

  • Atomicity: All or nothing transaction execution
  • Consistency: Database remains in valid state
  • Isolation: Transactions execute independently
  • Durability: Committed changes are permanent

Question 6

(a) Define the roles of each of the following personnel in databases:

  • (i) Specialised end user;
  • (ii) Database designers. (4 marks)

(b) State two differences between the file based approach and the database approach. (4 marks)

(c) Table 5 is a table named Results in a database. Write an SQL statement that would perform each of the following:

  • (i) display all records from the fields FirstName, SirName and IDNO; (2 marks)
  • (ii) display the DOB value for John Wallace; (3 marks)
  • (iii) display all records whose marks range from 50 to 90; (3 marks)
  • (iv) sort all the records from highest to lowest based on marks; (2 marks)
  • (v) display all records whose firstname start with letter J. (2 marks)

Answers

(a) Database roles:

  • Specialized end user: Advanced user who creates custom queries/reports
  • Database designer: Designs database structure, relationships, and constraints

(b) File vs Database approach:

  • Data redundancy: High in file-based, minimized in database
  • Data independence: Absent in file-based, present in database
  • Data sharing: Limited in file-based, efficient in database
  • Data integrity: Harder to maintain in file-based, easier in database

(c) SQL Statements:

SELECT FirstName, SirName, IDNO 
FROM Results;

SELECT DOB 
FROM Results 
WHERE FirstName = 'John' 
AND SirName = 'Wallace';

SELECT * 
FROM Results 
WHERE marks BETWEEN 50 AND 90;

SELECT * 
FROM Results 
ORDER BY marks DESC;

SELECT * 
FROM Results 
WHERE FirstName LIKE 'J%';

Question 7

(a) Outline two differences between homogeneous and heterogeneous distributed database systems. (4 marks)

(b) Explain two criteria that may be used to choose physical design in database. (4 marks)

(c) Interpret the given entity relationship diagram as used in database design. (4 marks)

(d) Use tables C and D to answer the questions that follow:

  • (i) Write an algebraic expression that would join the tables C and D. (2 marks)
  • (ii) Write the output generated from the expression in (i) above. (3 marks)
  • (iii) Write the output of the following expression: C ⋈hours=15 D (3 marks)

Answers

(a) Distributed database differences:

  • Homogeneous:
    • Same DBMS at all locations
    • Uniform data management
  • Heterogeneous:
    • Different DBMSs at different locations
    • Requires additional translation layer

(b) Physical design criteria:

  • Performance requirements
  • Storage capacity
  • Access patterns
  • Hardware constraints

(d) Join operations:

(i) Natural join:

C ⋈ D

(ii) Theta join:

C ⋈hours=15 D

Question 8

(a) Outline four types of database threats a database administrator is likely to encounter. (4 marks)

(b) Describe each of the following phases of the database life cycle:

  • (i) requirements phase; (2 marks)
  • (ii) conceptual phase; (2 marks)
  • (iii) physical design phase. (2 marks)

(c) Write an SQL statement that would:

  • (i) count all the employees in DeptNo 02; (2 marks)
  • (ii) calculate the average salary of all employees; (2 marks)
  • (iii) calculate the total salary for all employees; (2 marks)
  • (iv) display all records for employees whose salary is less or equal to the average salary of all employees. (4 marks)

Answers

(a) Database threats:

  • Unauthorized access
  • SQL injection attacks
  • Hardware/software failures
  • Data corruption

(b) Database lifecycle phases:

  • Requirements phase: Gathering and analyzing user needs
  • Conceptual phase: Creating high-level database design
  • Physical design phase: Implementing design on specific DBMS

(c) SQL Statements:

SELECT COUNT(*) 
FROM employees 
WHERE DeptNo = '02';

SELECT AVG(salary) 
FROM employees;

SELECT SUM(salary) 
FROM employees;

SELECT * 
FROM employees 
WHERE salary <= (
    SELECT AVG(salary) 
    FROM employees
);
