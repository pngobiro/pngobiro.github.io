Question 1

(a) Define the term data model as used in databases. (2 marks)

Answer:

A data model is a conceptual representation of data structures and relationships between data elements that reflects how data will be organized and accessed in a database. It defines the logical structure of a database and determines how data can be stored, organized, and manipulated.

(b) Outline four disadvantages of a hierarchical database. (4 marks)

Answer:

  1. Limited flexibility in representing complex relationships - can only represent one-to-many relationships
  2. Data redundancy - same data may need to be stored multiple times in different branches
  3. Difficult to modify structure - changes to the hierarchy can be complex and time-consuming
  4. Complex query processing - accessing data may require traversing multiple paths through the hierarchy

(c) Describe four steps involved in the design phase of a database system. (8 marks)

Answer:

  1. Conceptual Design:
    • Creating Entity-Relationship diagrams
    • Identifying entities and relationships
    • Defining attributes and constraints
  2. Logical Design:
    • Converting conceptual model to logical model
    • Normalizing relations
    • Defining keys and relationships
  3. Physical Design:
    • Specifying storage structures
    • Defining access methods
    • Planning for performance optimization
  4. Implementation Design:
    • Creating actual database schema
    • Implementing security measures
    • Setting up backup procedures

Question 2

(a) Outline three disadvantages of a database management system. (3 marks)

Answer:

  1. Cost: High initial investment in software, hardware, and training
  2. Complexity: Requires specialized staff and technical expertise to manage and maintain
  3. Vulnerability: Centralized data storage means system failures can affect all users and applications

(b) Explain three benefits of a centralized database server system. (6 marks)

Answer:

  1. Data Consistency:
    • Single source of truth for all data
    • Reduces data redundancy and inconsistencies
  2. Enhanced Security:
    • Centralized control over access rights
    • Easier to implement and monitor security measures
  3. Simplified Maintenance:
    • Updates and backups managed from one location
    • Easier to maintain data integrity and perform system upgrades

(c) Distinguish between distributed database and client/server database. (4 marks)

Answer:

Distributed Database Client/Server Database
Data is stored across multiple physical locations Data is stored centrally on the server
Each location can operate independently Clients depend on server for data access
Requires complex synchronization mechanisms Simpler data management and synchronization
Higher fault tolerance through redundancy Single point of failure at server level

Question 3

(a) Outline four benefits of usability testing during development. (4 marks)

Answer:

  1. Early identification of user interface issues and problems
  2. Reduced development costs by fixing issues before full implementation
  3. Improved user satisfaction and system adoption rates
  4. Better alignment between user needs and system functionality

(b) Describe three components of a database management system. (6 marks)

Answer:

  1. Data Dictionary (System Catalog):
    • Stores metadata about database structure
    • Contains information about tables, columns, and relationships
  2. Query Processor:
    • Parses and optimizes SQL queries
    • Creates efficient execution plans
  3. Storage Manager:
    • Handles physical data storage and retrieval
    • Manages memory buffers and disk space

Question 4

(a) Describe when to use integrity constraints. (2 marks)

Answer:

Integrity constraints are used during database design to:

  • Ensure data accuracy and consistency
  • Enforce business rules and logical relationships between data

(b) Explain two advantages of an online database system. (4 marks)

Answer:

  1. Accessibility:
    • 24/7 access from any location with internet connection
    • Multiple users can access simultaneously
  2. Real-time Updates:
    • Immediate data updates and synchronization
    • Current information available to all users instantly

(c) Distinguish between authentication and authorization. (4 marks)

Answer:

Authentication Authorization
Verifies who the user is Determines what the user can do
Involves username/password checks Involves permission and access rights
Happens before authorization Occurs after successful authentication

(d) Normalization questions:

Answer:

  1. Objectives of normalization:
    • Eliminate data redundancy
    • Ensure data integrity
    • Reduce data anomalies
  2. Table not in 1NF because:
    • Subject and grade could have multiple values per student
    • Student name is not atomic (could contain first, middle, last names)
  3. Advantages of denormalization:
    • Improved query performance
    • Reduced need for complex joins
  4. Dependencies:
    • Functional Dependency: StudentNo → StudentName (student number determines student name)
    • Transitive Dependency: StudentNo → SubjectCode → Grade (grade depends on subject code, which depends on student number)

Question 6

(a) Define database terms: (4 marks)

Answer:

  1. Cross Product: An operation that combines every row from one table with every row from another table, resulting in all possible combinations
  2. Natural Join: A join operation that automatically matches columns with the same name between tables and eliminates duplicate columns
  3. Cartesian Product: Same as cross product - combines each row of one table with every row of another table (results in m × n rows where m and n are the number of rows in each table)

(b) Distinguish between index and unique index. (4 marks)

Answer:

Index Unique Index
Allows duplicate values Enforces unique values
Improves query performance Improves performance and ensures uniqueness
Can be applied to any column Only for columns that must contain unique values

Question 7

(a) Disadvantages of traditional database approach: (6 marks)

Answer:

  1. Data Redundancy:
    • Same data stored in multiple files
    • Wastes storage space
  2. Data Inconsistency:
    • Multiple copies may not be synchronized
    • Updates need to be made in multiple places
  3. Limited Data Sharing:
    • Difficult to share data between applications
    • Each application maintains its own data files

Question 8

(a) Distinguish between tuple and domain: (4 marks)

Answer:

Tuple Domain
A row in a relation/table Set of allowable values for an attribute

(b) Relational algebra notations: (6 marks)

Answer:

  • σ (Sigma) - Selection operator (filters rows)
  • π (Pi) - Projection operator (selects columns)
  • ⋈ (Join) - Natural join of relations
  • ∪ (Union) - Combines two compatible relations
  • - (Minus) - Set difference between relations
  • × (Cross) - Cartesian product of relations

(c) Normalize invoice data to 3NF: (10 marks)

Answer:

To normalize to 3NF, we would typically:

  1. Identify all functional dependencies
  2. Create tables to eliminate partial dependencies (2NF)
  3. Create additional tables to eliminate transitive dependencies (3NF)
  4. Ensure all non-key attributes are dependent only on the primary key

Note: Specific normalization steps would depend on the invoice data provided in the question.
