Database Management Systems

November 2018 Past Paper

Paper Code: 2920206

Duration: 3 hours

Module II

INSTRUCTIONS TO CANDIDATES

  • This paper consists of EIGHT questions.
  • Answer any FIVE questions in the answer booklet provided.
  • ALL questions carry equal marks.
  • Candidates should answer the questions in English.

Question 1

(a) Outline four characteristics of a Database Management System. (4 marks)

(b) Explain the Three-Schema Architecture of a Database Management System. (4 marks)

(c) Outline two threats to data stored in a Database Management System. (2 marks)

(d) A newly established manufacturing company intends to design a database from scratch. State four relational database languages they could use. (4 marks)

(e) With the aid of an appropriate symbol explain each of the following components of an Entity-Relationship Diagram (ERD):

  • (i) entity;
  • (ii) relationship;
  • (iii) attribute (6 marks)

Answers

(a) Characteristics of a DBMS:

  • Data independence - separation of data and programs
  • Data integrity and security - ensures data accuracy and protection
  • Concurrent access - multiple users can access simultaneously
  • Data consistency - maintains data accuracy across the database

(b) Three-Schema Architecture:

  • External Schema (View Level) - how different users see the data
  • Conceptual Schema (Logical Level) - describes structure of whole database
  • Internal Schema (Physical Level) - describes physical storage of data

(c) Threats to stored data:

  • Hardware/software failures
  • Malicious access/attacks

(d) Relational database languages:

  • SQL (Structured Query Language)
  • QBE (Query By Example)
  • QUEL (QUEry Language)
  • DATALOG

(e) ERD Components:

  • (i) Entity: Represented by a rectangle - a person, place, thing, or concept
  • (ii) Relationship: Represented by a diamond - shows how entities are related
  • (iii) Attribute: Represented by an oval - properties/characteristics of an entity

Question 2

(a) Outline three duties of a Database Administrator. (3 marks)

(b) Explain each of the following types of database architectures:

  • (i) Client Server;
  • (ii) Distributed. (4 marks)

(c) With the aid of examples in each case distinguish between Data Definition Language (DDL) and Data Manipulation Language (DML). (6 marks)

(d) During a database maintenance session it was discovered that the Data dictionary had malfunctioned.

  • (i) Explain two functions of this dictionary in a Database. (4 marks)
  • (ii) State three Components of this dictionary. (3 marks)

Answers

(a) Duties of a Database Administrator:

  • Database design and implementation
  • Security administration
  • Performance monitoring and optimization

(b) Database architectures:

  • (i) Client Server: Architecture where database server processes queries and manages data, while clients request services
  • (ii) Distributed: Data is stored across multiple physical locations but managed as one logical database

(c) DDL vs DML:

DDL (Data Definition Language):

  • Used to define database structure
  • Examples: CREATE, ALTER, DROP, TRUNCATE

DML (Data Manipulation Language):

  • Used to manipulate data within database
  • Examples: SELECT, INSERT, UPDATE, DELETE

(d) Data Dictionary:

(i) Functions:

  • Stores metadata about database objects
  • Provides information about data relationships

(ii) Components:

  • Table definitions
  • Column attributes
  • Constraints and indexes

Question 3

(a) Outline three properties of a relational table. (3 marks)

(b) A company intends to integrate its Database Management System. Explain three structural components that may be considered. (6 marks)

(c) Differentiate between superkey and candidate key as used in Database management systems. (4 marks)

(d)

  • (i) Describe the object oriented database model. (4 marks)
  • (ii) An ICT Technician intends to perform database recovery procedures from the Enterprise's backup. State three types of backup that he may use in the recovery process. (3 marks)

Answers

(a) Properties of a relational table:

  • Each cell contains a single value
  • Each column has a unique name
  • All values in a column have the same data type

(b) Structural components for DBMS integration:

  • Data Integration Layer - combines data from different sources
  • Application Integration Layer - connects different applications
  • Process Integration Layer - coordinates business processes

(c) Superkey vs Candidate Key:

  • Superkey: Set of attributes that can uniquely identify a tuple in a relation (can contain extra attributes)
  • Candidate Key: Minimal superkey (no subset of it can be a superkey)

(d)

(i) Object Oriented Database Model:

  • Data stored as objects
  • Supports inheritance and encapsulation
  • Objects contain both data and methods
  • Complex relationships can be represented naturally

(ii) Types of backup:

  • Full backup - complete copy of database
  • Incremental backup - only changes since last backup
  • Differential backup - all changes since last full backup

Question 4

(a) Define each of the following terms as used in the Entity-Relationship Modelling:

  • (i) specialisation;
  • (ii) aggregation. (4 marks)

(b) With the aid of an example in each case distinguish between composite attributes. (4 marks)

(c) Explain each of the following terms as used in relational databases:

  • (i) Cardinality ratio;
  • (ii) Participation constraint. (4 marks)

(d) The following narrative is a representation of information about banks:

  • There are multiple banks and each bank has many branches. Each branch has Multiple customers.
  • Customer has a name address that consists of house number area and city and one or more phone numbers.
  • Customers have various types of accounts.
  • Some Customers also had taken different types of loans from these bank branches.
  • Account has number type and balance.
  • One customer can have multiple accounts and Loans.

(i) Identify four Entities in the narrative. (2 marks)

(ii) Draw an Entity-Relationship Diagram to represent the information. (6 marks)

Answers

(a) ER Modeling terms:

  • (i) Specialization: Process of defining subclasses of an entity type, inheriting characteristics
  • (ii) Aggregation: Representing a relationship between entities as a higher-level entity

(b) Composite attributes example:

  • Address (composite) consists of street, city, state, zip code
  • Name (composite) consists of first name, middle name, last name

(c) Relational database terms:

  • (i) Cardinality ratio: Maximum number of relationships an entity can participate in (1:1, 1:N, M:N)
  • (ii) Participation constraint: Whether entity participation in relationship is mandatory or optional

(d)

(i) Entities:

  • Bank
  • Branch
  • Customer
  • Account

(ii) ER Diagram would show:

  • Bank to Branch: One-to-Many relationship
  • Branch to Customer: One-to-Many relationship
  • Customer to Account: One-to-Many relationship
  • Customer to Loan: One-to-Many relationship
  • Address and Phone as multi-valued attributes of Customer
  • Account attributes: number, type, balance

Question 5

(a) Peter designed a Database Management System using a relational Model

  • (i) Outline three challenges that he may have encountered during the design. (3 marks)
  • (ii) State three database schemas he may have used. (3 marks)

Answers

(i) Challenges in relational design:

  • Identifying proper relationships between tables
  • Ensuring data integrity and normalization
  • Dealing with complex data types and relationships

(ii) Database schemas:

  • External schema (user view)
  • Conceptual schema (logical design)
  • Internal schema (physical storage)

Question 6

(a) Outline four trends in database technology. (4 marks)

(b) Explain the referential Integrity rule. (2 marks)

(c) Peter has been tasked to design a database by following the database design life cycle.

  • (i) Explain the conceptual modelling stage of this life cycle. (2 marks)
  • (ii) Outline four advantages of the model in (i) (4 marks)

(d) Table 2 and table 3 shows fruits and snacks respectively available at different Kiosks.

Compute:

  • (i) πac(FRUIT)
  • (ii) σB=2(SNACK)
  • (iii) Natural join
  • (iv) Outer join (8 marks)

Answers

(a) Current database trends:

  • Cloud databases
  • NoSQL databases
  • Big data analytics
  • In-memory databases

(b) Referential Integrity:

Foreign key must either be null or match a valid primary key in the referenced table

(c)

(i) Conceptual modeling stage:

  • Creating high-level model of data and relationships
  • Independent of physical implementation details

(ii) Advantages:

  • Easy to understand and communicate
  • Independent of implementation details
  • Helps identify entities and relationships early
  • Provides foundation for logical design

(d) Relational algebra operations:

  • (i) πac(FRUIT): Projects columns a and c from FRUIT table
  • (ii) σB=2(SNACK): Selects rows where column B equals 2
  • (iii) Natural join: Combines tables based on common columns
  • (iv) Outer join: Includes all rows from both tables, matching where possible

Question 7

(a) Outline two uses of database systems in hotel industry. (2 marks)

(b) Distinguish between weak and strong entities. (4 marks)

(c) A database has been scheduled to run an automatic update on a daily basis. Explain three types of integrity constraints that must be checked during update operation. (6 marks)

Answers

(a) Database uses in hotel industry:

  • Reservation management system
  • Guest information and history tracking

(b) Weak vs Strong entities:

  • Strong entity: Has its own primary key, exists independently
  • Weak entity: Depends on strong entity for identification, cannot exist alone

(c) Integrity constraints for updates:

  • Domain constraints - values must be within defined domains
  • Entity integrity - primary key cannot be null
  • Referential integrity - foreign keys must match valid primary keys
