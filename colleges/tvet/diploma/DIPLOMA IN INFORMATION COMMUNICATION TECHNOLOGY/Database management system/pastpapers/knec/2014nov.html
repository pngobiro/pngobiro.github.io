November 2014 Past Paper

Paper Code: 2920/206

Duration: 3 hours

Module II

INSTRUCTIONS TO CANDIDATES

  • This paper consists of EIGHT questions.
  • Answer any FIVE questions in the answer booklet provided.
  • All questions carry equal marks.
  • Candidates should answer the questions in English.
  • This paper consists of 14 printed pages.
  • Candidates should check the question paper to ensure that all pages are printed as indicated and no questions are missing.

Question 1

(a) Outline two uses of a data model in a database. (2 marks)

(b) Distinguish between data redundancy and data inconsistency as used in databases. (4 marks)

(c) Describe three components of a three tiered architecture in a client server database system. (6 marks)

(d) An organisation uses a time card system to track hours worked by employees to be used when computing payments. A time card should have hours worked, date submitted, a unique id and status. Each time card is associated with exactly one employee and each employee has a unique id, name, address and payment type. An employee submits a time card every pay period. Each employee is associated with exactly one manager and each manager has a unique id and a name. Each manager is in charge of multiple employees and each manager approves time cards for multiple employees.

Draw an ER diagram to represent the information. (8 marks)

Answers

(a) Uses of data model:

  • Provides conceptual representation of data structure
  • Facilitates communication between designers and users

(b) Data redundancy vs inconsistency:

  • Data redundancy:
    • Same data stored in multiple places
    • Wastes storage space
  • Data inconsistency:
    • Different versions of same data in different places
    • Results from uncontrolled redundancy

(c) Three-tier architecture components:

  • Presentation tier:
    • User interface
    • Handles data presentation and user input
  • Application tier:
    • Business logic
    • Processes client requests
  • Data tier:
    • Database server
    • Manages data storage and retrieval

(d) ER Diagram components:

  • Entities:
    • Employee (emp_id, name, address, payment_type)
    • TimeCard (card_id, hours_worked, date_submitted, status)
    • Manager (manager_id, name)
  • Relationships:
    • Employee to TimeCard (1:M)
    • Employee to Manager (M:1)
    • Manager to TimeCard (1:M, approval)

Question 2

(a) Distinguish between a sophisticated user and an end user as used in databases. (4 marks)

(b) Describe three classifications of Database Management Systems. (6 marks)

(c) Distinguish between non-lossy decomposition and lossy decomposition as used in normalization. (4 marks)

(d) Figure 1 shows a type of database organisation model. Use it to answer the question that follows.

Explain three benefits of using this type of database model. (6 marks)

Answers

(a) User types:

  • Sophisticated user:
    • Understands database structure
    • Can write complex queries
    • Often programmers or analysts
  • End user:
    • Uses pre-built applications
    • Limited technical knowledge
    • Focuses on data entry and retrieval

(b) DBMS classifications:

  • By data model:
    • Relational
    • Object-oriented
    • Network
  • By number of users:
    • Single-user
    • Multi-user
  • By database distribution:
    • Centralized
    • Distributed

(c) Decomposition types:

  • Non-lossy:
    • Original relation can be reconstructed
    • No information loss
  • Lossy:
    • Cannot fully reconstruct original relation
    • Some information may be lost

Question 3

(a) Andrew, a manager would like to purchase a Database Management System for his company. Outline four factors he should consider during the selection. (4 marks)

(b) The following are SQL statements. Use them to answer the question that follows.

Create Table (Staffno varchar(5)lname(varchar(15)salary decimal ((7,2));
INSERT TO STAFF (''ST678''Lenny' "15700");
SELECT staffnolastnamesalary

Identify and correct the errors found in the statements. (4 marks)

(c) Mary developed a system with the following modules; sales, invoicing and payroll all maintaining their own data files. Explain three disadvantages she would experience from this approach. (6 marks)

(d) Describe three database security measures that can be applied on data within a network. (6 marks)

Answers

(a) DBMS selection factors:

  • Cost and budget constraints
  • Hardware/software requirements
  • Technical support availability
  • Scalability and performance

(b) SQL statement corrections:

CREATE TABLE Staff (
    Staffno VARCHAR(5),
    lname VARCHAR(15),
    salary DECIMAL(7,2)
);

SELECT staffno, lastname, salary

(c) Disadvantages of separate modules:

  • Data redundancy across modules
  • Data inconsistency problems
  • Difficult data integration
  • Higher maintenance costs

(d) Database security measures:

  • Authentication:
    • Username/password
    • Multi-factor authentication
  • Authorization:
    • Access control lists
    • Role-based access
  • Encryption:
    • Data-at-rest encryption
    • Data-in-transit encryption

Question 4

(a) Describe a normal form as used in normalization. (2 marks)

(b) Outline four factors to consider when setting a Database Management System. (4 marks)

(c) Describe the following during database designs:

  • (i) Logical; (2 marks)
  • (ii) Physical. (2 marks)

(d) Explain three objectives of a Query builder in a database system. (6 marks)

(e) Eric would like to de-normalise a table. Explain two problems that he may encounter from this approach. (4 marks)

Answers

(a) Normal form:

A set of rules that database relations must follow to minimize redundancy and dependency issues

(b) DBMS setup factors:

  • Hardware configuration
  • Operating system compatibility
  • Network infrastructure
  • Storage requirements

(c) Database design levels:

  • Logical design:
    • Abstract representation of data
    • Independent of physical implementation
  • Physical design:
    • Actual implementation details
    • Storage structures and access methods

(d) Query builder objectives:

  • Simplify query creation
  • Reduce syntax errors
  • Improve query performance

(e) De-normalization problems:

  • Data redundancy and inconsistency
  • Update anomalies
  • Increased storage requirements

Question 5

(a) Distinguish between COUNT DISTINCT and COUNT * as used in structured query language. (4 marks)

(b) Explain three advantages of the contemporary database architecture. (6 marks)

(c) Figure 2 shows a customer's order form. Perform each of the following normal forms: (10 marks)

  • (i) 1st Normal Form
  • (ii) 2nd Normal Form
  • (iii) 3rd Normal Form

Answers

(a) COUNT functions:

  • COUNT DISTINCT:
    • Counts only unique values
    • Ignores duplicates
  • COUNT *:
    • Counts all rows including duplicates
    • Includes NULL values

(b) Contemporary database architecture advantages:

  • Better scalability
  • Improved data security
  • Enhanced performance
  • Greater flexibility

(c) Normalization process:

1NF:

  • Remove repeating groups
  • Create separate rows for each value

2NF:

  • Meet 1NF requirements
  • Remove partial dependencies

3NF:

  • Meet 2NF requirements
  • Remove transitive dependencies

Question 6

(a) Explain two benefits an organisation would gain from using a multidimensional database. (4 marks)

(b) State three differences between a join operator and a union operator as used in relational algebra. (6 marks)

(c) State the output when each of the following operations are applied between the tables:

  • (i) R∪K (2 marks)
  • (ii) π names, grade (R) (2 marks)
  • (iii) R-K (2 marks)
  • (iv) R∩K (2 marks)

Answers

(a) Multidimensional database benefits:

  • Complex analysis capabilities
  • Better performance for analytical queries
  • Improved data visualization
  • Enhanced decision support

(b) Join vs Union operators:

Join Union
Combines columns Combines rows
Requires related columns Requires identical structure
Maintains all columns Maintains unique rows

(c) Relational operations:

  • R∪K: All rows from both tables, no duplicates
  • π names, grade (R): Only names and grade columns from R
  • R-K: Rows in R but not in K
  • R∩K: Rows common to both R and K

Question 7

(a) Using the given Lectures and Course tables, state the output when the following operations are applied:

  • (i) Lecturers ⋈ Course (2 marks)
  • (ii) Lectures ⟕ Course (2 marks)
  • (iii) Lecturers ⟖ Course (3 marks)
  • (iv) department avg(salary) as avgsal (3 marks)

(b) Write SQL statements to:

  • (i) count the number of records with the product name "oil"; (2 marks)
  • (ii) display the fields salesman, age, dailysales for those salesman whose age is more than 30 and daily sales is in the range of 20000 and 40000; (2 marks)
  • (iii) add a column named department that would store data of at most five characters; (2 marks)
  • (iv) sort the records in ascending order by Salesman and dailysales; (2 marks)
  • (v) create a view from products table with the fields salesman and age. (2 marks)

Answers

(a) Join operations:

  • Natural join (⋈): Matching records from both tables
  • Left outer join (⟕): All records from Lectures, matching from Course
  • Right outer join (⟖): All records from Course, matching from Lectures
  • Average salary by department: Groups by department with average salary

(b) SQL statements:

SELECT COUNT(*) 
FROM products 
WHERE product_name = 'oil';

SELECT salesman, age, dailysales 
FROM products 
WHERE age > 30 
AND dailysales BETWEEN 20000 AND 40000;

ALTER TABLE products 
ADD department VARCHAR(5);

SELECT * FROM products 
ORDER BY Salesman, dailysales ASC;

CREATE VIEW sales_view AS 
SELECT salesman, age 
FROM products;

Question 8

(a) Outline the function of each of the following SQL statements:

  • (i) FROM; (1 mark)
  • (ii) HAVING; (1 mark)
  • (iii) ORDER BY; (1 mark)
  • (iv) WITH. (1 mark)

(b) Distinguish between relational algebra and structured query language as used in databases. (4 marks)

(c) State the output when the following SQL statements are applied on the Customer and Order tables: (12 marks)

SELECT ID, NAME, AMOUNT, DATE
FROM CUSTOMERS
INNER JOIN ORDERS
ON CUSTOMERS.ID=ORDERS.CUSTOMERS_ID

SELECT CUSTOMER_ID, NAME, AMOUNT
FROM CUSTOMERS
FULL JOIN ORDERS
ON CUSTOMERS.ID=ORDERS.CUSTOMERS_ID

Answers

(a) SQL statements:

  • FROM: Specifies tables to query from
  • HAVING: Filters grouped results
  • ORDER BY: Sorts result set
  • WITH: Defines temporary named result set

(b) Relational algebra vs SQL:

  • Relational algebra:
    • Procedural language
    • Mathematical notation
    • Theoretical foundation
  • SQL:
    • Non-procedural language
    • English-like syntax
    • Practical implementation

(c) SQL joins:

  • INNER JOIN:
    • Only matching records from both tables
    • Shows customer orders with all details
  • FULL JOIN:
    • All records from both tables
    • Shows all customers and orders, with NULL where no match
