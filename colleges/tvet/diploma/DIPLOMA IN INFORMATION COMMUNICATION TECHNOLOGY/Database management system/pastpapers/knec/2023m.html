Question 1

(a) Outline four transaction control language commands used in a Structured Query Language (SQL). (4 marks)

Answer

  1. BEGIN TRANSACTION - Starts a new transaction
  2. COMMIT - Makes transaction changes permanent
  3. ROLLBACK - Undoes changes made in current transaction
  4. SAVEPOINT - Creates points within transaction to ROLLBACK to

(b) Explain the function of each of the following in a database:

(i) foreign keys;

(ii) stored procedure. (4 marks)

Answer

(i) Foreign Keys:

  • Establish relationships between tables
  • Maintain referential integrity between related tables

(ii) Stored Procedure:

  • Prepared SQL code that can be saved and reused
  • Encapsulates a set of operations or queries to be executed together

(c) Table 1 shows a relation named workers in a database. Use it to answer the questions that follow.

Id Name Age Department Salary
1 Romny 32 IT 24000.00
2 Kahali 28 Support 18000.00
3 Mebe 27 Support 24000.00
4 David 35 IT 78000.00
5 Peter 29 Support 54000.00
6 George 24 Finance 72000.00

Write a Structured Query Language statement:

(i) which was used to create the workers relation having the field Id with unique values. (3 marks)

Answer

CREATE TABLE workers (
    Id INT PRIMARY KEY,
    Name VARCHAR(50),
    Age INT,
    Department VARCHAR(50),
    Salary DECIMAL(10,2)
);

(ii) that would change all the values to read 20000 in the Salary column for workers in support Department. (2 marks)

Answer

UPDATE workers
SET Salary = 20000
WHERE Department = 'Support';

(iii) that would delete the record for the worker named Mebe. (2 marks)

Answer

DELETE FROM workers
WHERE Name = 'Mebe';

(iv) that would display the column Name and Department for workers whose age is between 25 and 30 sorted in descending order by Department. (2 marks)

Answer

SELECT Name, Department
FROM workers
WHERE Age BETWEEN 25 AND 30
ORDER BY Department DESC;

(v) that would display the columns Department and the sum of salary for each department displayed as 'TOTAL SALARY'. (3 marks)

Answer

SELECT Department, SUM(Salary) AS 'TOTAL SALARY'
FROM workers
GROUP BY Department;

(i) AVG()

(ii) COUNT()

(iii) LAST() (3 marks)

Question 3

(a) Explain a circumstance that would necessitate the use of each of the following data types in SQL:

(i) char;

(ii) varchar. (4 marks)

Answer

char:

  • Used for fixed-length character strings
  • When data length is constant (e.g., state codes, ZIP codes)

varchar:

  • Used for variable-length character strings
  • When data length varies (e.g., names, addresses)

Answer

TRUNCATE DROP

Table structure remains intact Table and data both are removed

Question 4

(a)(i) Explain the meaning of subquery as used in SQL. (2 marks)

Answer

A subquery is a query nested inside another query that returns data used by the main query. It can be used in SELECT, INSERT, UPDATE, DELETE statements.

(ii) Explain a circumstance when subqueries are used in SQL. (2 marks)

Answer

  • When you need to use the result of one query as a condition in another query
  • When comparing values against aggregated results from another table

(b) Distinguish between data manipulation language and data query language as used in database management systems. (4 marks)

Answer

Data Manipulation Language (DML) Data Query Language (DQL)
Used to modify database content (INSERT, UPDATE, DELETE) Used to retrieve data (SELECT)
Changes the data values Only reads data, no modifications

Question 5

(a) Outline the function of each of the following logical operators in SQL:

(i) EXISTS;

(ii) AND;

(iii) ISNULL. (3 marks)

Answer

  • EXISTS: Tests for existence of rows in a subquery
  • AND: Combines multiple conditions, all must be true
  • ISNULL: Checks if a value is NULL

(b) Explain the term identity column as used in SQL. (2 marks)

Answer

An identity column automatically generates sequential numeric values when new records are inserted. It's commonly used for primary keys.

(d) The following relational schema is obtained from a database used by a lecturer to manage scores in tests for different units:

Student(sid: integer, sname: string)
Unit(uid: integer, uname: string)
Test(sid: integer, uid: integer, cat: integer, endterm: integer)

Write the SQL code that would:

(i) create the Test table having relevant fields; (2 marks)

Answer

CREATE TABLE Test (
    sid INTEGER,
    uid INTEGER,
    cat INTEGER,
    endterm INTEGER
);

(ii) modify the test table by adding the foreign key constraints and another constraint that would limit the scores entered in cat and endterm columns to be less than or equal to100. (4 marks)

Answer

ALTER TABLE Test
ADD CONSTRAINT fk_student FOREIGN KEY (sid) REFERENCES Student(sid),
ADD CONSTRAINT fk_unit FOREIGN KEY (uid) REFERENCES Unit(uid),
ADD CONSTRAINT chk_scores CHECK (cat <= 100 AND endterm <= 100);

(iii) display the column named sname as 'STUDENT NAME' and column uname displayed as 'UNIT NAME' for each student whose uid equal to 100. (3 marks)

Answer

SELECT s.sname AS 'STUDENT NAME', u.uname AS 'UNIT NAME'
FROM Student s
JOIN Test t ON s.sid = t.sid
JOIN Unit u ON t.uid = u.uid
WHERE t.uid = 100;

Question 6

(a) Explain each of the following terms as used in database management systems:

(i) trigger;

(ii) cardinality. (4 marks)

Answer

Trigger:

  • Special stored procedure that automatically executes when specific events occur
  • Used to maintain data integrity and automate actions

Cardinality:

  • Number of relationships an entity can have with another entity
  • Expressed as one-to-one, one-to-many, or many-to-many

(b)(i) Outline four factors that should be considered while planning and executing data migration. (4 marks)

Answer

  1. Data volume and complexity
  2. System compatibility
  3. Downtime requirements
  4. Data quality and cleanup needs

(ii) Outline four reasons for preparing a plan during the database migration. (4 marks)

Answer

  1. Minimize data loss risks
  2. Ensure proper resource allocation
  3. Establish clear timeline and milestones
  4. Define rollback procedures

(c) Explain the requirement for each of the following conditions to occur in a database:

(i) a table to be in Third Normal Form; (2 marks)

(ii) a referential integrity. (2 marks)

Answer

(i) Third Normal Form (3NF):

  • Table must be in 2NF
  • No transitive dependencies between non-key attributes

(ii) Referential Integrity:

  • Foreign key values must match primary key values in referenced table
  • Ensures data consistency between related tables

(d) Explain two methods that would be used by the designer to restrict some users from accessing some portions of data in a database system. (4 marks)

Answer

  1. Views:
    • Create virtual tables showing only allowed data
    • Hide sensitive columns or rows
  2. Access Control Lists (ACL):
    • Define user permissions and roles
    • Grant/deny specific operations on objects

Question 7

(a) Outline the importance of each of the following transaction properties in databases:

(i) consistency;

(ii) isolation;

(iii) durability. (3 marks)

Answer

  • Consistency: Ensures database remains in valid state after transaction
  • Isolation: Prevents interference between concurrent transactions
  • Durability: Guarantees committed changes are permanent

(b) Outline the function of each of the following logical operators used in SQL:

(i) BETWEEN;

(ii) IN;

(iii) ANY. (3 marks)

Answer

  • BETWEEN: Tests if value falls within a range
  • IN: Tests if value matches any in a list
  • ANY: Compares value with any value in a subquery result

(c) Distinguish between conceptual schema and external schema views in database management systems. (4 marks)

Answer

Conceptual Schema External Schema
Complete logical view of database User-specific view of database
Describes all entities and relationships Shows only relevant portions for specific users

Question 8

(a)(i) Explain the term database management systems. (2 marks)

Answer

A Database Management System (DBMS) is software that manages the storage, retrieval, and updating of data in a database while controlling access and maintaining data integrity.

(ii) Outline four advantages of database management systems. (4 marks)

Answer

  1. Data independence
  2. Reduced data redundancy
  3. Data integrity and consistency
  4. Centralized data management

(b) Distinguish between each of the following pairs of terms as used in database management systems:

(i) backup and recovery; (4 marks)

(ii) Integrity constraints and encryption. (4 marks)

Answer

Backup vs Recovery:

Backup Recovery
Creates copy of data for safekeeping Restores data after failure/loss
Preventive measure Corrective measure

Integrity Constraints vs Encryption:

Integrity Constraints Encryption
Rules to maintain data accuracy Technique to protect data confidentiality
Ensures data validity Ensures data security
