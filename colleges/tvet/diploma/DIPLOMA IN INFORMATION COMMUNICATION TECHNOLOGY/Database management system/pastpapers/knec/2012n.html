Database Management Systems

2012 November Past Paper

Question 1

(a) Define the following terms as used in Database Management:

  1. Candidate key
  2. Data independence
  3. Data redundancy
  4. Entity Relationship Diagram

Answer:

  1. Candidate key: A column or combination of columns that can uniquely identify a row in a table. All candidate keys are potential primary keys, but only one is chosen as the primary key.
  2. Data independence: The ability to modify the schema at one level of the database system without affecting the schema at the next higher level. This allows changes to the physical data storage without affecting the logical structure and vice versa.
  3. Data redundancy: The storage of the same data item in multiple places within a database. This can lead to data inconsistency and waste of storage space if not properly managed.
  4. Entity Relationship Diagram (ERD): A graphical representation of entities and their relationships in a database. It shows the logical structure of databases and helps in designing and understanding database schemas.

(b) Explain THREE types of relationships that can exist between entities in a database.

Answer:

  1. One-to-One (1:1) Relationship:
    • Each entity in the first set is related to at most one entity in the second set
    • Example: One person has one passport number, and one passport number belongs to one person
  2. One-to-Many (1:M) Relationship:
    • An entity in the first set can be related to many entities in the second set
    • Example: One department has many employees, but each employee belongs to only one department
  3. Many-to-Many (M:M) Relationship:
    • Entities in both sets can be related to multiple entities in the other set
    • Example: Students can enroll in multiple courses, and courses can have multiple students

Question 2

Explain FIVE advantages of using a Database Management System (DBMS) over a traditional file system.

Answer:

  1. Data Independence:
    • DBMS provides logical and physical data independence
    • Applications remain unaffected by changes in data storage structures and access methods
  2. Reduced Data Redundancy:
    • DBMS uses normalization to minimize data duplication
    • Saves storage space and reduces data inconsistency risks
  3. Data Consistency and Integrity:
    • DBMS enforces integrity constraints automatically
    • Ensures data accuracy and consistency across the database
  4. Concurrent Access Control:
    • Multiple users can access data simultaneously
    • DBMS manages transactions to prevent conflicts
  5. Data Security:
    • DBMS provides comprehensive security features
    • Controls user access through authentication and authorization

Question 3

(a) With the aid of a diagram, explain THREE components of a database environment.

Answer:

  1. Hardware:
    • Physical devices used to store and access the database
    • Includes servers, storage devices, and network infrastructure
  2. Software:
    • DBMS software itself
    • Application programs and utilities
    • Operating system that manages hardware resources
  3. Users:
    • Database administrators (DBAs)
    • Application programmers
    • End users who interact with the database

(b) State and explain THREE duties of a Database Administrator (DBA).

Answer:

  1. Database Design and Implementation:
    • Creating and modifying database schema
    • Defining data structures and relationships
    • Implementing security measures
  2. Database Security Management:
    • Setting up user accounts and permissions
    • Implementing backup and recovery procedures
    • Monitoring database access and usage
  3. Performance Monitoring and Optimization:
    • Monitoring database performance
    • Tuning queries and database structure for efficiency
    • Managing storage and planning for growth

Question 4

Consider the following customer order database:

CUSTOMER (CustomerID, CustomerName, City)
ORDER (OrderID, CustomerID, OrderDate, Total)
ORDER_DETAIL (OrderID, ProductID, Quantity, UnitPrice)
PRODUCT (ProductID, ProductName, UnitPrice, Category)

Write SQL statements to:

  1. List all customers who have not placed any orders
  2. Find the total value of orders for each customer
  3. List products that have never been ordered
  4. Find customers who have placed orders totaling more than $1000

Answer:

  1. List customers with no orders: 
    SELECT CustomerName, City 
FROM CUSTOMER 
WHERE CustomerID NOT IN (SELECT CustomerID FROM ORDER);
  2. Total value of orders per customer: 
    SELECT c.CustomerName, SUM(o.Total) as TotalOrders
FROM CUSTOMER c
LEFT JOIN ORDER o ON c.CustomerID = o.CustomerID
GROUP BY c.CustomerID, c.CustomerName;
  3. Products never ordered: 
    SELECT ProductName
FROM PRODUCT
WHERE ProductID NOT IN (
    SELECT DISTINCT ProductID 
    FROM ORDER_DETAIL
);
  4. Customers with orders over $1000: 
    SELECT c.CustomerName, SUM(o.Total) as TotalOrders
FROM CUSTOMER c
JOIN ORDER o ON c.CustomerID = o.CustomerID
GROUP BY c.CustomerID, c.CustomerName
HAVING SUM(o.Total) > 1000;

Question 5

Explain the following concepts as used in database normalization:

  1. First Normal Form (1NF)
  2. Second Normal Form (2NF)
  3. Third Normal Form (3NF)

Answer:

  1. First Normal Form (1NF):
    • All attributes must contain atomic (indivisible) values
    • No repeating groups or arrays
    • Each column contains only one value
    • Example: Breaking down an address field into separate fields for street, city, and postal code
  2. Second Normal Form (2NF):
    • Must be in 1NF
    • All non-key attributes must be fully functionally dependent on the primary key
    • No partial dependencies
    • Example: Separating order details into separate tables for orders and products
  3. Third Normal Form (3NF):
    • Must be in 2NF
    • No transitive dependencies
    • Non-key attributes must not depend on other non-key attributes
    • Example: Moving category description to a separate table from products

Question 6

Given a Books table with the following structure:

Books (BookID, Title, Author, ISBN, Price, Publisher, Year)

Write SQL statements to:

  1. Create the Books table with appropriate data types
  2. Insert two records into the Books table
  3. Update the price of all books published before 2010 by increasing it by 10%
  4. Delete all books by a specific author

Answer:

  1. Create Books table: 
    CREATE TABLE Books (
    BookID INT PRIMARY KEY,
    Title VARCHAR(100) NOT NULL,
    Author VARCHAR(50) NOT NULL,
    ISBN VARCHAR(13) UNIQUE,
    Price DECIMAL(10,2),
    Publisher VARCHAR(50),
    Year INT
);
  2. Insert two records: 
    INSERT INTO Books (BookID, Title, Author, ISBN, Price, Publisher, Year)
VALUES (1, 'Database Design', 'John Smith', '9780123456789', 49.99, 'Tech Books', 2012);

INSERT INTO Books (BookID, Title, Author, ISBN, Price, Publisher, Year)
VALUES (2, 'SQL Basics', 'Jane Doe', '9780987654321', 39.99, 'Code Press', 2015);
  3. Update prices for old books: 
    UPDATE Books
SET Price = Price * 1.1
WHERE Year < 2010;
  4. Delete books by author: 
    DELETE FROM Books
WHERE Author = 'John Smith';

Question 7

Explain FIVE database security measures that can be implemented in a DBMS.

Answer:

  1. User Authentication:
    • Username and password requirements
    • Multi-factor authentication
    • Biometric authentication where applicable
  2. Access Control:
    • Role-based access control (RBAC)
    • Granular permissions for tables and views
    • Row-level and column-level security
  3. Data Encryption:
    • Encryption at rest for stored data
    • Encryption in transit for data transfer
    • Secure key management
  4. Audit Trails:
    • Logging of all database access attempts
    • Tracking of data modifications
    • Regular audit log reviews
  5. Backup and Recovery:
    • Regular automated backups
    • Secure off-site backup storage
    • Tested recovery procedures

Question 8

Write short notes on the following database concepts:

  1. Referential integrity
  2. Database views
  3. Transaction management
  4. Data dictionary

Answer:

  1. Referential Integrity:
    • Ensures relationships between tables remain consistent
    • Prevents orphaned records in related tables
    • Implemented through foreign key constraints
    • Defines actions for updates/deletes (CASCADE, SET NULL, etc.)
  2. Database Views:
    • Virtual tables based on SELECT statements
    • Provides data abstraction and security
    • Can simplify complex queries
    • Helps implement data access control
  3. Transaction Management:
    • Ensures database consistency during concurrent access
    • Implements ACID properties (Atomicity, Consistency, Isolation, Durability)
    • Handles commit and rollback operations
    • Manages transaction logs for recovery
  4. Data Dictionary:
    • Contains metadata about the database
    • Stores information about tables, columns, and relationships
    • Includes user and security information
    • Essential for database administration and maintenance
