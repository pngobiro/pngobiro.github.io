Question 1

(a) Outline four advantages of database systems. (4 marks)

Answer

  1. Data Independence: Separation of data and application programs
  2. Data Consistency: Eliminates data redundancy and maintains data integrity
  3. Data Security: Centralized control over data access and protection
  4. Data Sharing: Multiple users can access and share data simultaneously

(b) Explain each of the following phases of database design:

(i) database planning; (2 marks)

Answer

Database Planning:

  • Initial phase that defines project scope and objectives
  • Involves resource allocation, timeline setting, and cost estimation

(ii) requirement analysis; (2 marks)

Answer

Requirement Analysis:

  • Gathering and documenting user needs and system requirements
  • Analyzing business processes and data requirements

(iii) physical design. (2 marks)

Answer

Physical Design:

  • Translating logical design into specific database storage structures
  • Defining file organizations and indexes

(i) entity set; (2 marks)

Answer

Answer

(d) Write an SQL statement for each of the following relational algebra expressions:

Question 2

(a)(i) Outline two soft skills that a database administrator should possess. (2 marks)

Answer

  1. Communication Skills: Ability to effectively communicate with users and team members
  2. Problem-Solving Skills: Ability to analyze and resolve database issues efficiently

(ii) Outline four functions of a server in client-server architecture. (4 marks)

Answer

  1. Data Storage: Manages and stores data centrally
  2. Data Processing: Executes queries and processes data requests
  3. Security Management: Controls access and ensures data security
  4. Resource Management: Manages system resources and concurrent access

(b) Describe each of the following database backup methods:

(i) full; (2 marks)

Answer

Full Backup:

  • Complete backup of all database files
  • Creates a complete copy of the entire database

(ii) incremental. (2 marks)

Answer

Incremental Backup:

  • Only backs up changes made since the last backup
  • Faster and requires less storage than full backup

(c) Distinguish between full and partial functional dependencies as used in database normalization. (4 marks)

Answer

Full Functional Dependency Partial Functional Dependency
Attribute depends on entire primary key Attribute depends on part of primary key
Cannot be reduced further Can be broken down into smaller dependencies

Question 3

(a) Explain two advantages of data warehousing. (4 marks)

Answer

  1. Improved Decision Making:
    • Provides consolidated historical data
    • Enables better business intelligence and analysis
  2. Enhanced Data Quality:
    • Standardizes data from multiple sources
    • Ensures consistency and accuracy

(b) Paul was hired to coordinate a database project which failed before the completion time. Explain three reasons that could have caused this failure. (6 marks)

Answer

  1. Poor Planning:
    • Inadequate requirement analysis
    • Unrealistic timelines and resources
  2. Lack of Stakeholder Involvement:
    • Insufficient user input and feedback
    • Poor communication with stakeholders
  3. Technical Issues:
    • Complex integration problems
    • Inadequate testing and quality assurance

(c) Differentiate between form and report as used in databases. (4 marks)

Answer

Form Report
Used for data entry and modification Used for data presentation and analysis
Interactive interface for users Static presentation of data
Input-oriented Output-oriented

(d) With the aid of a syntax describe each of the following clauses as used in SQL:

(i) IN; (3 marks)

Answer

SELECT column_name
FROM table_name
WHERE column_name IN (value1, value2, ...);

The IN clause allows you to specify multiple values in a WHERE clause.

(ii) BETWEEN. (3 marks)

Answer

SELECT column_name
FROM table_name
WHERE column_name BETWEEN value1 AND value2;

The BETWEEN clause selects values within a given range (inclusive).

Question 4

(a) Explain the term data independence as used in a database. (2 marks)

Answer

Data Independence: The ability to modify the schema at one level without affecting the schema at a higher level. This means:

  • Changes to physical storage don't affect application programs
  • Applications remain unaffected by changes to data structure

(b) Describe three generations of database systems. (6 marks)

Answer

  1. First Generation (1960s):
    • File-based systems
    • Hierarchical and Network databases
  2. Second Generation (1970s-1980s):
    • Relational databases
    • SQL standardization
  3. Third Generation (1990s-Present):
    • Object-oriented databases
    • NoSQL and distributed databases

(c) Distinguish between schema and instance as used in databases. (4 marks)

Answer

Schema Instance
Logical structure/design of database Actual data stored at a given time
Relatively static Changes frequently
Describes database structure Contains actual data values

Answer

To normalize to 2NF, we'll split into two tables:

(ii) Outline three anomalies eliminated by normalization. (3 marks)

Answer

  1. Update Anomaly: Having to update multiple records for the same data change
  2. Insert Anomaly: Unable to insert certain data without other related data
  3. Delete Anomaly: Unintentionally losing data when deleting related information

Question 5

(a) Explain each of the following terms as applied in database systems security:

(i) discretionary access control; (2 marks)

Answer

Discretionary Access Control (DAC):

  • Access control based on user identity and ownership
  • Owners can grant/revoke access permissions to other users

(ii) view. (2 marks)

Answer

View:

  • Virtual table based on result set of an SQL statement
  • Used to restrict access to sensitive data

(c) Write SQL statements to:

(i) create the table named Customer; (3 marks)

Answer

    CustomerNo VARCHAR(10) PRIMARY KEY,
    CustomerName VARCHAR(50),
    Address VARCHAR(100),
    Location VARCHAR(50)
);

(ii) create a foreign key constraint in Plot table to reference Customer table; (2 marks)

Answer

ALTER TABLE Plot
ADD FOREIGN KEY (CustomerNo)
REFERENCES Customer(CustomerNo);

(iii) extract plot details for all plots whose price is greater than 500000; (3 marks)

Answer

SELECT *
FROM Plot
WHERE Price > 500000;

(iv) extract the CustomerName and PlotNo for all customers whose location is Embu. (3 marks)

Answer

SELECT c.CustomerName, p.PlotNo
FROM Customer c
JOIN Plot p ON c.CustomerNo = p.CustomerNo
WHERE p.Location = 'Embu';

Question 8

(a) State two quantifiers used in relational calculus. (2 marks)

Answer

  1. Universal Quantifier (∀): "For all" condition
  2. Existential Quantifier (∃): "There exists" condition

(b)(i) Explain the term closure as used in relational algebra. (2 marks)

Answer

Closure: The property that the result of any relational operation is also a relation that can be used as input to other operations.

(ii) With the aid of a Venn diagram describe union operation as used in relational algebra. (4 marks)

Union Operation (∪):

  • Combines all unique tuples from both relations
  • Relations must be union-compatible
  • Eliminates duplicate records
