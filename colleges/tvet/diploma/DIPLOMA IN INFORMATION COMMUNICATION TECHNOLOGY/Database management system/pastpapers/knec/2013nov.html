Database Management Systems

November 2013 Past Paper

Paper Code: 2920/206

Duration: 3 hours

Module II

INSTRUCTIONS TO CANDIDATES

  • This paper consists of EIGHT questions.
  • Answer any FIVE questions in the answer booklet provided.
  • All questions carry equal marks.
  • Candidates should answer the questions in English.
  • This paper consists of 12 printed pages.
  • Candidates should check the question paper to ensure that all pages are printed as indicated and no questions are missing.

Question 1

(a) Outline four benefits that a college would achieve from using a database in maintaining students records. (4 marks)

(b) Distinguish between a physical database designer and an application developer as used in databases. (4 marks)

(c) Describe each of the following components of a database management system:

  • (i) query processor; (2 marks)
  • (ii) database manager control language interface; (2 marks)
  • (iii) recovery manager. (2 marks)

(d) The manager at Malimoto organization intends to incorporate client/server computing in their operations. Explain three benefits that the organization would achieve from this move. (6 marks)

Answers

(a) Benefits of database for student records:

  • Data integrity and consistency
  • Reduced data redundancy
  • Improved data security
  • Efficient data retrieval and updates

(b) Roles comparison:

  • Physical database designer:
    • Focuses on physical storage structure
    • Handles performance optimization
  • Application developer:
    • Creates user interfaces
    • Implements business logic

(c) DBMS components:

  • Query processor: Analyzes and optimizes database queries
  • Database manager control language interface: Handles DDL and DML commands
  • Recovery manager: Handles system failures and restores database consistency

(d) Client/server benefits:

  • Better resource utilization
  • Improved data security
  • Enhanced scalability
  • Centralized data management
  • Reduced network traffic
  • Better maintenance and upgrades

Question 2

(a) Distinguish between a distributed database and distributed database management system. (4 marks)

(b) Explain three goals of requirement analysis phase in a Database Design Life Cycle. (6 marks)

(c) Distinguish between structured query language and relational algebra as used in databases. (4 marks)

(d) Figure 1 shows an entity relationship diagram representing entity relationship in an organization.

  • (i) Represent the diagram as a relational schema. (5 marks)
  • (ii) Identify a relational set in the figure. (1 mark)

Answers

(a) Distributed systems:

  • Distributed database:
    • Physical data distribution across locations
    • Logical integration of distributed data
  • Distributed DBMS:
    • Software system managing distributed database
    • Handles data distribution transparency

(b) Requirement analysis goals:

  • Identify data requirements
  • Understand business rules
  • Define system scope
  • Document user needs
  • Establish constraints
  • Validate requirements

(c) SQL vs Relational Algebra:

  • SQL:
    • Declarative language
    • English-like syntax
    • Implementation oriented
  • Relational Algebra:
    • Procedural language
    • Mathematical notation
    • Theoretical foundation

Question 3

(a) Outline four functions of an alter command as used in Structured Query Language. (4 marks)

(b) Distinguish between a grant and a revoke command as used in Structured Query Language. (4 marks)

(c) Amos an ICT student created a database with redundant data. Explain three problems that the users of this database would experience. (6 marks)

(d) State three differences between data administration and database administration. (6 marks)

Answers

(a) ALTER command functions:

  • Add new columns
  • Modify existing columns
  • Add or remove constraints
  • Rename table or columns

(b) GRANT vs REVOKE:

  • GRANT: Gives permissions to users
  • REVOKE: Removes previously granted permissions

(c) Problems with redundant data:

  • Update anomalies
  • Insertion anomalies
  • Deletion anomalies
  • Increased storage costs
  • Data inconsistency

(d) Administration differences:

Data Administration Database Administration
Strategic planning Technical implementation
Data standards Database maintenance
Policy making Performance tuning

Question 4

(a)

  • (i) Define the term view as used in Structured Query Language. (2 marks)
  • (ii) State three characteristics of a simple view as used in Structured Query Language. (3 marks)

(b) With the aid of an example describe a spatial database. (3 marks)

(c) State two differences between a unique constraint and a primary key constraint as used in Structured Query Language. (4 marks)

(d) Table 1 shows a table named drivers in a database with details of drivers in a college. Write an SQL statement that would be able to:

  • (i) define a table constraint that would ensure that every value in the salary field is greater than 14000; (3 marks)
  • (ii) extract the two characters starting from the second character of the drivers name; (2 marks)
  • (iii) compute tax at a rate of 8% for those earning over 30000 and display output in a field named tax. (3 marks)

Answers

(a) Views in SQL:

  • (i) View: Virtual table based on result set of SQL query
  • (ii) Simple view characteristics:
    • Based on single table
    • No aggregations
    • Allows updates

(b) Spatial database:

Database optimized for storing and querying geographical data. Example: GIS system storing map coordinates

(c) Constraints comparison:

  • UNIQUE: Allows NULL values, multiple per table
  • PRIMARY KEY: No NULL values, only one per table

(d) SQL statements:

-- i. Salary constraint
ALTER TABLE drivers
ADD CONSTRAINT salary_check 
CHECK (salary > 14000);

-- ii. Extract characters
SELECT SUBSTRING(driver_name, 2, 2) 
FROM drivers;

-- iii. Calculate tax
SELECT *, 
CASE 
    WHEN salary > 30000 THEN salary * 0.08 
END AS tax 
FROM drivers;

Question 5

(a)

  • (i) Outline four factors to be considered when performing first normal form (1NF) in databases. (4 marks)
  • (ii) Outline two characteristics of a table in a third normal form (3NF). (2 marks)

(b) Outline five factors that may lead to the failing of an inventory database during an operation. (5 marks)

(c) Figure 2 shows a dependency diagram for fields c1, c2, c3, c4 and c5. State giving a reason the types of dependencies used in the figure. (9 marks)

Answers

(a) Normalization:

(i) 1NF factors:

  • Eliminate repeating groups
  • Create separate tables for each set of related data
  • Identify primary keys
  • All attributes must be atomic

(ii) 3NF characteristics:

  • Table is in 2NF
  • No transitive dependencies

(b) Inventory database failure factors:

  • Hardware failure
  • Software corruption
  • Network issues
  • Concurrent access conflicts
  • Data integrity violations

(c) Dependencies types:

  • Functional dependency
  • Partial dependency
  • Transitive dependency

Question 6

(a) Explain three factors that may lead an organisation to use an object oriented database approach. (6 marks)

(b) Explain two approaches used in database design. (4 marks)

(c) Figure 3 shows a database model.

  • (i) Identify the type of model; (1 mark)
  • (ii) State three advantages of using the model identified in (i). (3 marks)

(d) With the aid of a labelled diagram in each case outline the type of cardinality represented in each of the following statements:

  • (i) a manager can only manage a department. (2 marks)
  • (ii) several students make several applications to different universities. (2 marks)
  • (iii) a department has several employees. (2 marks)

Answers

(a) Object-oriented database factors:

  • Complex data types needed
  • Inheritance requirements
  • Need for object-oriented programming integration
  • Better modeling of real-world entities

(b) Database design approaches:

  • Top-down approach:
    • Start with general requirements
    • Decompose into details
  • Bottom-up approach:
    • Start with atomic attributes
    • Combine into larger structures

(d) Cardinality representations:

  • (i) One-to-one (1:1)
  • (ii) Many-to-many (M:N)
  • (iii) One-to-many (1:M)

Question 7

  • (ii) intersection; (2 marks)
  • (iii) minus. (2 marks)

(b) Write equivalent SQL statements for each respective operation in (a). (6 marks)

(c) Describe two types of quantifiers that may be used in relational calculus expressions stating the symbol in each case. (4 marks)

(d) Given a relation named staff; write a relational algebra expression for the following statements:

  • (i) All the staff earn a salary of at least 9000; (2 marks)
  • (ii) There are less than 3 staff whose positions are managers. (2 marks)

Answers

(b) SQL set operations:

-- Union
SELECT * FROM table1
UNION
SELECT * FROM table2;

-- Intersection
SELECT * FROM table1
INTERSECT
SELECT * FROM table2;

-- Minus
SELECT * FROM table1
EXCEPT
SELECT * FROM table2;

(c) Relational calculus quantifiers:

  • Universal quantifier (∀): For all
  • Existential quantifier (∃): There exists

(d) Relational algebra expressions:

σsalary≥9000(staff)

σcount(position='manager')<3(staff)

Question 8

(a) Describe each of the following queries as used in Structured Query Language:

  • (i) Make table action query; (2 marks)
  • (ii) Append action query. (2 marks)

(b) Given the relations Tab1 and Tab2, perform each of the following in relational algebra:

  • (i) Tab1 ⋈ Tab2; (2 marks)
  • (ii) Tab1 ⟖ Tab2; (2 marks)
  • (iii) Tab1 ⟕ Tab2; (2 marks)
  • (iv) Tab1 ⟗ Tab2. (2 marks)

(c) Write the relational algebra statements for the following SQL statements:

(i)  SELECT id, name
    FROM table2
    WHERE hobby="swimming" OR hobby="travelling"; (3 marks)

(ii) SELECT s1.name, s2.name
    FROM students AS s1, students AS s2
    WHERE s1.telephone=s2.telephone; (3 marks)

(iii)SELECT fname, lname
    FROM customers
    WHERE balance < 0; (2 marks)

Answers

(a) Query types:

  • Make table query: Creates new table from query results
  • Append query: Adds records to existing table

(b) Join operations:

  • Tab1 ⋈ Tab2: Natural join
  • Tab1 ⟖ Tab2: Right outer join
  • Tab1 ⟕ Tab2: Left outer join
  • Tab1 ⟗ Tab2: Full outer join

(c) Relational algebra expressions:

(i) πid,name(σhobby="swimming"∨hobby="travelling"(table2))

(ii) πs1.name,s2.name(σs1.telephone=s2.telephone(ρs1(students)×ρs2(students)))

(iii) πfname,lname(σbalance<0(customers))
