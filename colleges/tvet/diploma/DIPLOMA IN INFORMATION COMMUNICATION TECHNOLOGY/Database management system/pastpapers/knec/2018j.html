Database Management Systems

July 2018 Past Paper

Paper Code: 2920/206

Duration: 3 hours

Module II

INSTRUCTIONS TO CANDIDATES

  • This paper consists of EIGHT questions.
  • Answer any FIVE of the following EIGHT questions in the answer booklet provided.
  • All questions carry equal marks.
  • Candidates should answer the questions in English.

Question 1

(a) Explain how a Database Management System accomplishes the following:

  • (i) Security management;
  • (ii) Multi-User Access Control;
  • (iii) Data Integrity Management.

(b) Many business computing systems used to store records in a single file system. Explain three challenges of this type of approach.

(c) Figure 1 shows a database tier architecture. Use it to answer the questions that follow:

  • (i) Identify the architecture.
  • (ii) Explain three benefits of this type of tier architecture in a database. (6 marks)

Answers

(a) DBMS accomplishments:

  • (i) Security management:
    • User authentication and authorization
    • Access control lists (ACLs)
    • Encryption of sensitive data
  • (ii) Multi-User Access Control:
    • Concurrency control mechanisms
    • Transaction management
    • Lock management
  • (iii) Data Integrity Management:
    • Constraint enforcement
    • ACID properties
    • Validation rules

(b) Single file system challenges:

  • Data redundancy and inconsistency
  • Difficulty in accessing data
  • Data isolation - multiple files and formats

(c) Three-tier architecture:

(i) Architecture: Three-tier client-server architecture

(ii) Benefits:

  • Better scalability through distributed processing
  • Improved data security by separating business logic
  • Enhanced maintenance and modification capabilities

Question 2

(a) Describe the classification of databases based on each of the following criteria:

  • (i) number of users;
  • (ii) location (6 marks)

(b) Distinguish between internal and external schema as used in database design. (4 marks)

(c) The following is a table named hostelworkers in a database. Use it to answer the questions that follow: (10 marks)

Write an SQL statement that would:

  • (i) Sort the hostel table by the name and salary in descending order;
  • (ii) Find the average salary for single employees;
  • (iii) Find salary more than twenty one thousand and less than thirty thousand and in red hostel;
  • (iv) Display all the records having month letters "dy";
  • (v) Increase salary for all employees.

Answers

(a) Database classifications:

(i) Number of users:

  • Single-user database: Only one user at a time
  • Multi-user database: Multiple concurrent users

(ii) Location:

  • Centralized: Single location
  • Distributed: Multiple locations
  • Cloud: Hosted on cloud servers

(b) Internal vs External Schema:

  • Internal Schema: Physical storage details, file organization
  • External Schema: User view of database, logical structure

(c) SQL Statements:

(i) Sort by name and salary:

SELECT * FROM hostelworkers 
ORDER BY name DESC, salary DESC;

(ii) Average salary for single employees:

SELECT AVG(salary) 
FROM hostelworkers 
WHERE marital_status = 'Single';

(iii) Salary range in red hostel:

SELECT * FROM hostelworkers 
WHERE salary BETWEEN 21000 AND 30000 
AND hostel = 'red';

(iv) Records with 'dy' in month:

SELECT * FROM hostelworkers 
WHERE month LIKE '%dy%';

(v) Increase salary:

UPDATE hostelworkers 
SET salary = salary + 1000;

Question 3

(a) Outline the function of each of the following SQL commands: (4 marks)

  • (i) IN-OPERATOR
  • (ii) UNIQUE
  • (iii) NULL

(b) Explain three routine maintenance jobs that can be carried by a database administrator in an organisation. (6 marks)

(c) Distinguish between system privileges and object privileges as used in SQL (4 marks)

(d) Describe each of the following modules that operate in a database system: (6 marks)

  • (i) Transaction manager;
  • (ii) Recovery manager;
  • (iii) Cache manager;

Answers

(a) SQL Commands:

  • (i) IN-OPERATOR: Tests if a value matches any in a list or subquery
  • (ii) UNIQUE: Ensures no duplicate values in a column
  • (iii) NULL: Represents missing or unknown data

(b) DBA maintenance jobs:

  • Backup and recovery procedures
  • Performance monitoring and tuning
  • Security auditing and updates

(c) Privileges in SQL:

  • System privileges: Rights to perform database-wide operations
  • Object privileges: Rights to perform actions on specific database objects

(d) Database modules:

  • (i) Transaction manager: Controls transaction execution and ensures ACID properties
  • (ii) Recovery manager: Handles system failures and restores database to consistent state
  • (iii) Cache manager: Optimizes memory usage and disk I/O operations

Question 4

(a) Outline three rules to be observed when creating sub queries in SQL. (3 marks)

(b) Distinguish between a schema and an instance as used in databases. (4 marks)

(c) The goal of abstraction in DBMS is to separate the user interface from the physical storage of data in the database. Describe three levels of abstraction that can perform these goals in a database. (6 marks)

(d) A car company dealing with importation of cars on behalf of clients is based in Mombasa. Imported vehicles are characterized by; unique car identification number, car weight, car type, destination and delivery date.

Shipped vehicles are received at the company go-downs. Each go-down is characterized by; a unique go-down id, location and telephone number. Shipped vehicles make their way to their clients via standard gauge railways or by road.

Each client is characterised by; a unique client id, clients name, address and the types of transport used to deliver the cargo.

Draw an entity relationship diagram for the above information. (7 marks)

Answers

(a) Subquery rules:

  • Must be enclosed in parentheses
  • Should be placed on the right side of comparison operator
  • Cannot contain ORDER BY clause

(b) Schema vs Instance:

  • Schema: Database structure definition, like a template
  • Instance: Actual data in database at a particular moment

(c) Levels of abstraction:

  • Physical level: How data is stored physically
  • Logical level: What data is stored and relationships
  • View level: How users see the data

(d) ER Diagram components:

  • Entities:
    • Vehicle (car_id, weight, type, destination, delivery_date)
    • Go-down (godown_id, location, telephone)
    • Client (client_id, name, address, transport_type)
  • Relationships:
    • Vehicle to Go-down (M:1)
    • Vehicle to Client (M:1)

Question 5

(a) Explain the function of each of the following components of a database management system: (4 marks)

  • (i) Procedures;
  • (ii) Database Engine.

(b) Outline four types of procedures that are would require to help operate and manage a DBMS. (4 marks)

(c) State two differences between relational algebra and relational calculus. (4 marks)

(d) Describe two query languages that can be used by an application developer in a database. (4 marks)

Answers

(a) DBMS components:

  • (i) Procedures: Stored programs that perform specific tasks
  • (ii) Database Engine: Core software managing data storage and retrieval

(b) Types of procedures:

  • Backup procedures
  • Recovery procedures
  • Security procedures
  • Maintenance procedures

(c) Relational algebra vs calculus:

  • Algebra is procedural, calculus is declarative
  • Algebra specifies operations, calculus specifies conditions

(d) Query languages:

  • SQL (Structured Query Language)
  • QBE (Query By Example)

Question 6

(a) State two wildcard operators that can be used in conjunction with a Like statement in SQL. (2 marks)

(b) Outline four SQL statements that can be used to define data. (4 marks)

(c) Describe each of the following terms as used in SQL: (6 marks)

  • (i) Primary key;
  • (ii) View;
  • (iii) Nested query.

(d) State the fields that would be given when each of the following relational algebra operations are performed: (8 marks)

  • (i) Inner join A and B;
  • (ii) Left outer join A and B;
  • (iii) Right outer Join A and B;
  • (iv) Full outer Join A and B.

Answers

(a) Wildcard operators:

  • % (matches any sequence of characters)
  • _ (matches any single character)

(b) SQL DDL statements:

  • CREATE - creates new database objects
  • ALTER - modifies existing objects
  • DROP - removes objects
  • TRUNCATE - removes all records

(c) SQL terms:

  • (i) Primary key: Unique identifier for each record
  • (ii) View: Virtual table based on result set of SQL query
  • (iii) Nested query: Query within another query

(d) Join operations:

  • (i) Inner join: Only matching records from both tables
  • (ii) Left outer join: All records from left table plus matching from right
  • (iii) Right outer join: All records from right table plus matching from left
  • (iv) Full outer join: All records from both tables

Question 7

(a) Describe denormalisation as used in databases. (2 marks)

(b) A database administrator encountered a fault while implementing database management system. Explain three likely causes of this failure. (6 marks)

(c) With the aid of an example distinguish between a composite attribute and a derived attribute as used in ER diagrams. (6 marks)

(d) Anne would like to model some database operations using laws in relational algebra. Describe two laws that would enable her meet her goals giving an example in each case. (6 marks)

Answers

(a) Denormalization:

Process of adding redundant data to improve query performance by reducing joins

(b) Database implementation failures:

  • Insufficient hardware resources
  • Software configuration errors
  • Data migration issues

(c) Attribute types:

  • Composite attribute: Name (firstName, middleName, lastName)
  • Derived attribute: Age (calculated from DateOfBirth)

(d) Relational algebra laws:

  • Commutative law: A ⋈ B = B ⋈ A
  • Associative law: (A ⋈ B) ⋈ C = A ⋈ (B ⋈ C)

Question 8

(a) Outline four advantages of client server architecture as used in databases. (4 marks)

(b)

  • (i) Explain concurrency control as used in databases. (2 marks)
  • (ii) Distinguish between read phase and write phase as used in concurrency control phase validation in a database. (4 marks)

(c) A learning institution maintains details of its lecturers who are teaching various units as follows:

Lecturer no, lecturer name, grade code, department name, subject code, subject name, subject level.

Each lecturer may teach many subjects but may not belong to more than one department.

Normalise this data to 3rd normal form. (10 marks)

Answers

(a) Client-server advantages:

  • Centralized data management
  • Better security control
  • Efficient resource sharing
  • Easier maintenance

(b) Concurrency control:

(i) Definition: Mechanism to ensure correct results when multiple transactions execute simultaneously

(ii) Phases:

  • Read phase: Transaction reads database values without making changes
  • Write phase: Transaction writes changes to database after validation

(c) Normalization to 3NF:

First Normal Form:

LECTURER(lecturer_no, lecturer_name, grade_code, department_name, subject_code, subject_name, subject_level)

Second Normal Form:

LECTURER(lecturer_no, lecturer_name, grade_code, department_name)
SUBJECT(subject_code, subject_name, subject_level)
LECTURER_SUBJECT(lecturer_no, subject_code)

Third Normal Form:

LECTURER(lecturer_no, lecturer_name, grade_code, dept_id)
DEPARTMENT(dept_id, department_name)
SUBJECT(subject_code, subject_name, subject_level)
LECTURER_SUBJECT(lecturer_no, subject_code)
