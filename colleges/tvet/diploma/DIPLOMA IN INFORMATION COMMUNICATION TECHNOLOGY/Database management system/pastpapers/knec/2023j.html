Question 1

(a) Outline four characteristics of a primary key in a database. (4 marks)

Answer

1. Uniqueness: Each value in the primary key column must be unique - no duplicates are allowed

2. Not Null: Primary key values cannot be null (must always contain a value)

3. Immutable: Values should not change over time to maintain data integrity

4. Minimal: Should use minimum number of columns necessary to ensure uniqueness

(b) Distinguish between associated record and orphan record as used in a database. (4 marks)

Answer

Associated Record Orphan Record
Has valid reference to parent record No valid reference to parent record
Maintains referential integrity Violates referential integrity

(c) Describe each of the following terms as used in a database:

(i) meta data; (2 marks)

Answer

Meta Data: Data that describes other data. It includes information about:

  • Table structures and relationships
  • Data types and constraints
  • Database object properties

(ii) data dictionary; (2 marks)

Answer

Data Dictionary: A centralized repository of information about:

  • Database structure and content
  • Data element definitions and relationships
  • System catalog information

(iii) database instance. (2 marks)

Answer

Database Instance: The actual data stored in a database at a particular moment in time. It represents:

  • Current state of the database
  • Collection of information stored in the database

(d) John, a system analyst, used an entity relationship diagram to design a database. Explain three benefits he would derive from the use of this design tool. (6 marks)

Answer

1. Clear Visual Representation: ERDs provide a clear visual representation of database structure, making it easier to understand relationships between entities and communicate design concepts to stakeholders.

2. Design Verification: ERDs help identify potential design flaws, redundancies, or missing relationships before implementation, saving time and resources in the development process.

3. Documentation: ERDs serve as comprehensive documentation of the database structure, facilitating future maintenance, updates, and onboarding of new team members.

Question 2

(a) Outline four challenges posed by integrating Artificial Intelligence in databases. (4 marks)

Answer

1. Data Quality: AI requires high-quality, consistent data for accurate results. Poor data quality can lead to incorrect insights and decisions.

2. System Performance: AI algorithms can be computationally intensive, potentially impacting database performance and response times.

3. Data Privacy: AI analysis may expose sensitive information, requiring additional security measures and compliance considerations.

4. Integration Complexity: Integrating AI systems with existing database infrastructure can be technically challenging and resource-intensive.

(b) Distinguish between relational model and hierarchical model as used in database design. (4 marks)

Answer

Relational Model Hierarchical Model
Data organized in tables (relations) Data organized in tree-like structure
Supports many-to-many relationships Supports one-to-many relationships
Flexible data access paths Fixed parent-child relationships
Based on set theory Based on parent-child relationships

Question 3

(a)(i) State the meaning of null value as used in a database. (1 mark)

Answer

Null Value: Represents the absence of a value or an unknown value in a database field. It is not the same as zero or an empty string.

(ii) Outline three circumstances that may lead to the occurrence of a null value in a database. (3 marks)

Answer

  1. Optional Data: When information is not required or unavailable at the time of data entry
  2. Not Applicable: When a field doesn't apply to a particular record
  3. Unknown Values: When the actual value exists but is not known at the time of entry

(b) With the aid of an example in each case, explain the following quantifiers in a relational algebra:

(i) existential quantifier;

(ii) universal quantifier. (4 marks)

Answer

1. Existential Quantifier (∃):

  • Tests if at least one tuple satisfies a condition
  • Example: Find departments that have at least one employee with salary > 50000
    • ∃ salary > 50000 (Employee ⨝ Department)

2. Universal Quantifier (∀):

  • Tests if all tuples satisfy a condition
  • Example: Find departments where all employees have salary > 30000
    • ∀ salary > 30000 (Employee ⨝ Department)

(c) Differentiate between IN clause and Between clause as used in structured query language. (4 marks)

Answer

IN Clause BETWEEN Clause
Tests for membership in a set of values Tests if value falls within a range
Can test against discrete values Tests against continuous range
Example: WHERE dept IN ('IT', 'HR') Example: WHERE salary BETWEEN 30000 AND 50000

(d) Using the Stproject table given in 2nd normal form:

(i) Explain two reasons why the relation violates the 2NF in normalization. (4 marks)

Answer

  1. Partial Dependencies: Non-key attributes depend on only part of the composite primary key, violating 2NF requirements.
  2. Redundant Data: Same information is repeated across multiple rows, leading to data redundancy and potential update anomalies.

(ii) Represent the Stproject relation to its 2nd normal form. (4 marks)

Answer

Split the table into multiple tables to achieve 2NF:

Student(StudentID, StudentName, Course)

Benefits:

  • Eliminates partial dependencies
  • Reduces data redundancy
  • Maintains data integrity

Question 4

(a) Outline four reasons that can lead to loss of data in a database. (4 marks)

Answer

  1. Hardware Failure: Disk crashes, storage device malfunction, or system crashes
  2. Software Errors: Application bugs, system crashes during transactions
  3. Human Error: Accidental deletion, incorrect updates, or unauthorized modifications
  4. Natural Disasters: Power outages, floods, fires affecting physical storage

(b) Outline three reasons why a database administrator is required to constantly audit an organisation database. (3 marks)

Answer

  1. Security Monitoring: Detect unauthorized access attempts and ensure data security policies are followed
  2. Performance Optimization: Identify performance bottlenecks and resource usage patterns
  3. Compliance Verification: Ensure database operations meet regulatory requirements and industry standards

(c) John entered data in a database and performed some operations that changed the state of the database. Explain three such operations. (6 marks)

Answer

  1. INSERT Operation:
    • Adds new records to the database
    • Creates new entries in tables
  2. UPDATE Operation:
    • Modifies existing data values
    • Changes field values in records
  3. DELETE Operation:
    • Removes records from the database
    • Permanently eliminates data entries

Question 5

(a) Outline four access privileges that may be granted to a database user. (4 marks)

Answer

  1. SELECT: Permission to read/view data from tables
  2. INSERT: Permission to add new records to tables
  3. UPDATE: Permission to modify existing records
  4. DELETE: Permission to remove records from tables

(b) State two differences between truncate and drop commands as used in structured query language. (4 marks)

Answer

TRUNCATE DROP
Removes all records but keeps table structure Removes entire table including structure
Can be rolled back in some DBMSs Cannot be rolled back once executed

(c) Explain three benefits of using a structured query language to manage data in database. (6 marks)

Answer

  1. Standardization:
    • Provides consistent syntax across different database systems
    • Reduces learning curve for users
  2. Data Integrity:
    • Enforces data consistency through constraints
    • Maintains referential integrity
  3. Flexibility:
    • Allows complex queries and data manipulation
    • Supports various data operations efficiently

(d) State the category of each of the following SQL commands, giving a reason for your answer:

(i) CREATE

(ii) INSERT INTO

(iii) ROLLBACK (6 marks)

Answer

  1. CREATE: Data Definition Language (DDL)
    • Used to define database structure
    • Creates database objects like tables and views
  2. INSERT INTO: Data Manipulation Language (DML)
    • Used to modify data content
    • Adds new records to tables
  3. ROLLBACK: Transaction Control Language (TCL)
    • Controls transaction operations
    • Undoes changes in a transaction

(i) Parametric; (2 marks)

Answer

Parametric Users: Users who perform only predefined operations using fixed parameters.

  • Use canned queries and reports
  • Have limited database interaction capabilities
  • Example: Bank tellers, clerks

Answer

Sophisticated Users: Users with extensive knowledge of database systems.

Casual Users: Occasional users who access the database intermittently.

  • Use simple queries
  • Need user-friendly interfaces
  • Example: Managers, executives

  2. Data Security:
    • Access control mechanisms
    • Data encryption requirements
    • Backup and recovery procedures
  3. Scalability:
    • Future growth considerations
    • Performance requirements

Question 7

(a) Outline four properties of a transaction in a database. (4 marks)

Answer

  1. Atomicity: Transaction must be completed entirely or not at all
  2. Consistency: Database must remain in a consistent state after transaction
  3. Isolation: Transactions must execute independently of each other
  4. Durability: Changes must persist after transaction completion

(b) Distinguish between sorting and indexing as used in databases. (4 marks)

Answer

Sorting Indexing
Temporary arrangement of data Permanent data structure
Affects query output display Improves search performance
Uses ORDER BY clause Uses CREATE INDEX statement
No additional storage needed Requires additional storage space

(c) Explain three types of data protection methods that may be applied in a database management system. (6 marks)

Answer

  1. Access Control:
    • User authentication and authorization
    • Role-based access permissions
  2. Data Encryption:
    • Encryption of sensitive data
    • Secure transmission protocols
  3. Backup and Recovery:
    • Regular data backups
    • Recovery procedures for data loss

(d) An organisation intends to use a database management system to support its operations. Explain three challenges the organisation may face as a result of using the application. (6 marks)

Answer

  1. Implementation Cost:
    • Hardware and software expenses
    • Training and maintenance costs
  2. Data Migration:
    • Converting existing data
    • Ensuring data integrity during transfer
  3. System Integration:
    • Compatibility with existing systems
    • User adaptation to new processes

Question 8

(a) State four elements of a database schema. (2 marks)

Answer

  1. Tables/Relations: Basic structures for storing data
  2. Attributes/Columns: Data fields within tables
  3. Constraints: Rules enforcing data integrity
  4. Relationships: Connections between tables

(b) Explain three disadvantages of flat file approach over relational database approach in respect to data management. (6 marks)

Answer

  1. Data Redundancy:
    • Same data stored multiple times
    • Wastes storage space
    • Increases risk of inconsistencies
  2. Limited Data Independence:
    • Changes to data structure affect applications
    • Poor separation of data and programs
  3. Poor Data Integrity:
    • No built-in constraints
    • Difficult to maintain consistency
    • Limited validation capabilities

(c) Using the Tenant relation provided, write an SQL statement that would display:

(i) all records for the latest occupation date in the field named latest_date; (3 marks)

Answer

SELECT *
FROM Tenant
WHERE occupation_date = (
    SELECT MAX(occupation_date)
    FROM Tenant
) AS latest_date;

(ii) average House_rent for all apartments with the name Balozi in the field named avgbalozi; (2 marks)

Answer

SELECT AVG(House_rent) AS avgbalozi
FROM Tenant
WHERE apartment_name = 'Balozi';

(iii) minimum house rent in a field named Lowest_rent; (3 marks)

Answer

SELECT MIN(House_rent) AS Lowest_rent
FROM Tenant;

(iv) all records with no comment in descending order of house number. (4 marks)

Answer

SELECT *
FROM Tenant
WHERE comment IS NULL
ORDER BY house_number DESC;
