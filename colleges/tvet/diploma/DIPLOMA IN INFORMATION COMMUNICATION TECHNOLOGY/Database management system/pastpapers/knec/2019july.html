Question 1

(a) Describe three types of Database Management System users. (6 marks)

Answer

  1. Database Administrator (DBA):
    • Manages and maintains the database system
    • Controls access rights and security
  2. Application Programmers:
    • Develops database applications
    • Creates and maintains database queries
  3. End Users:
    • Interacts with database through applications
    • Retrieves and updates data as needed

(b) With the aid of an example in each case distinguish between physical and logical data independence as used in database design (4 marks)

Answer

Physical Data Independence Logical Data Independence
Example: Changing storage device or file organization without affecting application programs Example: Adding a new column to a table without modifying existing applications
Ability to modify physical storage without changing logical schema Ability to modify logical schema without changing application programs

(c) Table 1 shows information from a booksDetails relation in a relational database. Use it to answer the questions that follow.

Table 1
Book_Id BookTitle BookAuthor
B2235 The Worlds Deity Maxwell
B2234 Database Made Easy Charles

Write a Structured Query Language statement that would:

(i) display the authors of books from the booksDetails relation whose Book_Id is greater than or equal to B2235 (2 marks)

Answer

SELECT BookAuthor
FROM booksDetails
WHERE Book_Id >= 'B2235';

(ii) add a new record; B2236 Database Applications Peter into the relation (2 marks)

Answer

INSERT INTO booksDetails (Book_Id, BookTitle, BookAuthor)
VALUES ('B2236', 'Database Applications', 'Peter');

(iii) remove the book titled The Worlds Deity from the relation; (2 marks)

Answer

DELETE FROM booksDetails
WHERE BookTitle = 'The Worlds Deity';

(iv) sort the table by Book_Id in descending order; (2 marks)

Answer

SELECT *
FROM booksDetails
ORDER BY Book_Id DESC;

(v) display the total number of books authored by Havi in the relation. (2 marks)

Answer

SELECT COUNT(*)
FROM booksDetails
WHERE BookAuthor = 'Havi';

Answer

Schema: The logical structure and organization of a database that defines:

  • Tables, fields, and their relationships
  • Constraints and rules for data integrity

(b) Distinguish between distributed architecture and client/server architecture as used in Database Management Systems. (4 marks)

Question 3

(a) State four functions of Data Definition Language statements used in Structured Query Language. (4 marks)

Answer

  1. CREATE: Creates database objects like tables, views, indexes
  2. ALTER: Modifies the structure of existing database objects
  3. DROP: Removes database objects from the system
  4. TRUNCATE: Removes all records from a table

(b) Explain three characteristics of relational databases (6 marks)

Answer

  1. Table-Based Structure:
    • Data organized in tables (relations)
    • Each table consists of rows and columns
  2. Data Integrity:
    • Enforces data consistency through constraints
    • Maintains referential integrity between tables
  3. Data Independence:
    • Physical independence from storage details
    • Logical independence from application programs

(c) Distinguish between domain constraint and referential integrity constraint as used in database design. (4 marks)

Answer

Domain Constraint Referential Integrity Constraint
Defines valid values for a column Ensures relationships between tables
E.g., data type, range, format E.g., foreign key relationships

(i) Explain the output from each of the following algebraic expression from the relation:

σ AdNumber='T-adCbc-W' (StudentDetails)

Answer

Question 4

(a) Outline the meaning of each of the following terms as used in databases:

(i) domain; (1 mark)

Answer

Domain: The set of allowable values that can be assigned to an attribute (column) in a relation.

(ii) attribute; (1 mark)

Answer

Attribute: A column in a database table that represents a specific characteristic or property of an entity.

(iii) entity set. (1 mark)

Answer

Entity Set: A collection of similar entities that share the same properties or attributes.

(b) Explain each of the following relational algebra statements used in database:

(i) set difference; (2 marks)

Answer

Set Difference (-):

  • Returns all tuples that appear in first relation but not in second relation
  • Relations must be union-compatible (same attributes)

(ii) Cartesian product. (2 marks)

Answer

Cartesian Product (×):

  • Combines each row from first relation with every row from second relation
  • Results in all possible combinations of rows

(c) (i) Distinguish between generalisation and specialisation as used in entity relationship diagrams (4 marks)

Answer

Generalisation Specialisation
Bottom-up approach Top-down approach
Combines similar entities into a higher-level entity Divides entity into more specific sub-entities

(ii) With the aid of an example explain the term functional dependency as used in database normalisation (3 marks)

Answer

Functional Dependency: When one attribute uniquely determines another attribute.

Example:

StudentID → StudentName
(StudentID uniquely determines StudentName)
  • For each StudentID, there can be only one StudentName
  • Used to identify primary keys and normalize relations

(d) A company with branches in different cities opted to use a distributed database system to manage their data. Explain three benefits that the company will gain from this approach. (6 marks)

Answer

  1. Improved Performance:
    • Local access to data reduces network traffic
    • Faster query response times
  2. Increased Reliability:
    • System continues to function if one site fails
    • Data replication provides backup
  3. Better Scalability:
    • Easy to add new locations
    • System grows with business needs

Question 5

(a) Outline four states of transactions in a database management system. (4 marks)

Answer

  1. Active: Initial state, transaction is being executed
  2. Partially Committed: After last operation, before permanent changes
  3. Committed: After successful completion and permanent changes
  4. Failed: After unsuccessful execution or abortion

(b) Explain a circumstance under which each of the following anomalies may occur in an unnormalized database:

(i) update; (2 marks)

Answer

Update Anomaly: When the same data appears in multiple rows and needs to be updated, requiring changes in multiple places to maintain consistency.

Example: Changing a customer's address requires updates in all rows containing that customer's information.

(ii) deletion; (2 marks)

Answer

Deletion Anomaly: When deleting data about one entity unintentionally removes data about another entity.

Example: Deleting the last course for a professor accidentally removes the professor's information.

(iii) insert. (2 marks)

Answer

Insert Anomaly: When unable to add new data because some required information is not yet available.

Example: Cannot add a new department until at least one employee is assigned to it.

(c) Explain the term data replication as used in Database Management system. (2 marks)

Answer

Data Replication: The process of creating and maintaining multiple copies of the same data at different locations to:

  • Improve data availability and reliability
  • Enable faster local access to data

(d) A company's database experienced system failure.

(i) Explain two recovery techniques which the company should have used in order to recover the data. (4 marks)

Answer

  1. Database Backup:
    • Regular copying of database to secure storage
    • Enables restoration to previous state
  2. Transaction Log:
    • Records all database modifications
    • Allows replay of transactions since last backup

(ii) During the recovery there was a transaction failure. Explain a reason that may have caused each of the following types of errors:

1. Logical errors;

2. System errors (4 marks)

Answer

1. Logical Errors:

  • Incorrect SQL statements or business rules
  • Data validation failures

2. System Errors:

  • Hardware failures or crashes
  • Operating system or memory issues

Question 6

(a) Explain a circumstance under which a database transaction may achieve each of the following properties:

(i) atomicity; (2 marks)

Answer

Atomicity: When transferring money between accounts:

  • Either both debit and credit operations complete
  • Or neither operation occurs (rollback)

(ii) consistency; (2 marks)

Answer

Consistency: In a banking system:

  • Total balance before equals total after transfer
  • All account balances remain valid

(iii) isolation. (2 marks)

Answer

Isolation: When multiple users update inventory:

  • Each transaction sees a consistent database state
  • Concurrent updates don't interfere with each other

(b) Distinguish between union and rename operators as used in relational algebra. (4 marks)

Answer

Union Operator (∪) Rename Operator (ρ)
Combines tuples from two relations Changes name of relation or attributes
Requires union compatibility No change to actual data

(i) With the aid of an example from the table, explain the foreign key constraint in a database. (3 marks)

Answer

Foreign Key Constraint Example:

  • P_Id in Orders table references P_Id in Persons table
  • Ensures every P_Id in Orders exists in Persons
  • Maintains referential integrity between tables

(ii) Write a Structured Query Language statement to:

1. create the orders table with all its constraints; (4 marks)

Answer

CREATE TABLE Orders (
    O_Id INT PRIMARY KEY,
    OrderNo INT NOT NULL,
    P_Id INT,
    FOREIGN KEY (P_Id) REFERENCES Persons(P_Id)
);

Answer

SELECT p.P_Id, p.LastName, o.OrderNo
FROM Persons p
JOIN Orders o ON p.P_Id = o.P_Id;

Question 7

(a) Outline four advantages of using views in a database. (4 marks)

Answer

  1. Data Security: Restrict access to sensitive data
  2. Data Abstraction: Hide complex table relationships
  3. Data Independence: Changes to base tables don't affect applications
  4. Simplified Queries: Complex queries can be encapsulated in views

(b) Describe two components of a database. (4 marks)

Answer

  1. Data Dictionary/System Catalog:
    • Contains metadata about database objects
    • Stores schema definitions and constraints
  2. Data Files:
    • Store actual data records
    • Organized according to storage structure

(c) A programmer opted to create an application that used file system as a means of storing data. Explain three challenges that the system is likely to experience from using this approach (6 marks)

Answer

  1. Data Redundancy:
    • Same data stored in multiple files
    • Wastes storage space and risks inconsistency
  2. Data Integrity:
    • Difficult to maintain data consistency
    • No built-in constraint mechanisms
  3. Concurrent Access:
    • Poor handling of multiple user access
    • Risk of data corruption

Question 8

(a) Outline four advantages of using views in a database. (4 marks)

Answer

  1. Data Security: Restrict unauthorized access to sensitive data
  2. Data Abstraction: Hide complex table relationships and structures
  3. Customized Data Presentation: Present data in different formats for different users
  4. Data Independence: Changes to base tables don't affect applications using views

(b) Describe two components of a database. (4 marks)

Answer

  1. Data Dictionary:
    • Contains metadata about the database structure
    • Includes information about tables, views, and relationships
  2. Storage Engine:
    • Manages data storage and retrieval
    • Handles physical organization of data

(c) A programmer opted to create an application that used file system as a means of storing data. Explain three challenges that the system is likely to experience from using this approach (6 marks)

Answer

  1. Data Redundancy:
    • Multiple copies of the same data in different files
    • Increases storage costs and inconsistency risks
  2. Data Dependency:
    • Programs are tightly coupled with file structures
    • Changes to file structure require program modifications
  3. Limited Data Sharing:
    • Difficult to share data between applications
    • Poor support for concurrent access
