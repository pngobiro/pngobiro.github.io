Database Management Systems

November 2017 Past Paper

Paper Code: 2920/206

Duration: 3 hours

Module II

INSTRUCTIONS TO CANDIDATES

  • Answer any FIVE of the following EIGHT questions in the answer booklet provided.
  • All questions carry equal marks.
  • Candidates to answer the questions in English.
  • This paper consists of 6 printed pages.
  • Candidates should check the question paper to ascertain that all pages are printed.

Question 1

(a) Explain each of the following terms as used in databases: (4 marks)

  • (i) Data Definition Language
  • (ii) Database Catalogue

(b) Table 1 is an extract from a database. Use it to answer the questions that follow.

Write an algebraic expression to:

  • (i) display the NationalID, FarmerName for all farmers who earn a bonus of less than 25000; (3 marks)
  • (ii) rename the table Farmer to FarmerDetail, the columns NationalID to ID and Farmername to Fname; (3 marks)
  • (iii) display the average bonus earned by the farmers. (2 marks)

(c) Explain two circumstances that may prompt an organisation to implement a centralised database. (4 marks)

(d) Distinguish between one to one and one to many cardinality as used in databases. (4 marks)

Answers

(a) Database terms:

  • (i) Data Definition Language (DDL): Language used to define and modify database structure and schema
  • (ii) Database Catalogue: Repository of metadata that contains information about database objects

(b) Algebraic expressions:

(i) σbonus<25000 π(NationalID, FarmerName)(Farmer)

(ii) ρFarmerDetail(ID←NationalID, Fname←FarmerName)(Farmer)

(iii) γAVG(bonus)(Farmer)

(c) Circumstances for centralized database:

  • Need for consistent data access and management
  • Better security control and monitoring
  • Reduced data redundancy and maintenance costs
  • Simplified backup and recovery procedures

(d) Cardinality:

  • One-to-One (1:1): Each entity in one set corresponds to exactly one entity in another set
  • One-to-Many (1:M): One entity in one set can be related to multiple entities in another set

Question 2

(a) State the database users who perform each of the following tasks: (2 marks)

  • (i) Interacts with database systems without writing programs;
  • (ii) Interacts with database systems through previously created programs.

(b) Study the following narrative and use it to answer the questions that follow:

A car insurance company has customers. The customer can be identified by the identification number, name and location. The customer can own one or more cars and can buy one or more Insurance premiums. Each car has been associated with one or more accidents. A car can be identified by the registration number, model and capacity.

Represent the information in an ER diagram. (6 marks)

(c) Explain two limitations of the file-based database approach. (4 marks)

(d)

  • (i) Distinguish between commit and rollback statements as used in SQL. (4 marks)
  • (ii) Explain the use of each of the following SQL statements: (4 marks)
    • Drop table Users Restrict;
    • Drop table Users Cascade.

Answers

(a) Database users:

  • (i) Casual users/End users
  • (ii) Application users/Naive users

(b) ER Diagram Components:

  • Entities:
    • Customer (id_number, name, location)
    • Car (reg_number, model, capacity)
    • Insurance (premium_id)
    • Accident (accident_id)
  • Relationships:
    • Customer owns Car (1:M)
    • Customer buys Insurance (1:M)
    • Car has Accident (1:M)

(c) File-based limitations:

  • Data redundancy and inconsistency
  • Difficult data sharing
  • Lack of data integrity
  • Complex application development

(d) SQL statements:

(i) COMMIT vs ROLLBACK:

  • COMMIT: Makes database changes permanent
  • ROLLBACK: Undoes changes since last COMMIT

(ii) DROP statements:

  • RESTRICT: Prevents table deletion if dependencies exist
  • CASCADE: Deletes table and all dependent objects

Question 3

(a) Explain each of the following terms as used in distributed database systems: (4 marks)

  • (i) partitioned tables;
  • (ii) replicated tables.

(b) With the aid of an example distinguish between the terms relation and relation schema as used in databases. (4 marks)

(c) John, a database administrator has been experiencing challenges manipulating data stored in a database. Explain three ways in which data normalization can resolve some of the challenges. (6 marks)

(d)

  • (i) Outline the function of the cartesian product as used in algebraic expression. (2 marks)
  • (ii) Write the equivalent algebraic expression for each of the following tuple calculus expressions: (4 marks)

Answers

(a) Distributed database terms:

  • (i) Partitioned tables: Tables split across multiple locations based on specific criteria
  • (ii) Replicated tables: Copies of tables maintained at multiple locations

(b) Relation vs Relation Schema:

  • Relation: Actual data in table (e.g., specific student records)
  • Relation Schema: Structure definition (e.g., Student(id, name, age))

(c) Benefits of normalization:

  • Eliminates data redundancy
  • Ensures data consistency
  • Simplifies data maintenance
  • Reduces update anomalies

(d) Cartesian product:

  • (i) Combines each row of first table with every row of second table
  • Example: A × B creates all possible combinations of rows from tables A and B

Question 4

(a) Define the term file server as used in a client server database. (2 marks)

(b) Describe two components of a database. (4 marks)

(c) Table 2 is an Employee table represented in 1NF. Use it to answer the questions that follow.

Represent the table in its 3NF. (8 marks)

(d) Using an example in each case explain the following operators as used in relational algebra: (6 marks)

  • (i) union;
  • (ii) intersection;
  • (iii) difference.

Answers

(a) File server:

Computer that manages file operations and shared access to files in a network

(b) Database components:

  • Data (actual stored information)
  • Data dictionary (metadata about database)
  • Indexes (improve data retrieval)
  • Query processor (handles data requests)

(d) Relational algebra operators:

  • (i) Union (∪): Combines all unique rows from two tables 
    A ∪ B = {elements in A OR B}
  • (ii) Intersection (∩): Returns only rows common to both tables 
    A ∩ B = {elements in both A AND B}
  • (iii) Difference (-): Returns rows in first table not in second 
    A - B = {elements in A but NOT in B}

Question 5

(a) Distinguish between the physical and logical stages of the database life cycle. (4 marks)

(b) Explain two functions of the Database Administrator in a database environment. (4 marks)

(c)

  • (i) Explain the use of "Grant Select on projects to User1;" SQL statement when applied in a database. (4 marks)
  • (ii) Write SQL statements to perform each of the following:
    • display StudentID, StudentName for all students whose CourseID is C240; (2 marks)
    • determine the total marks for the student Named Agnes and store the results in a field named TotalMarks; (2 marks)
    • display the lowest mark scored for CourseID is C236. (2 marks)

(d) Describe each of the following types of functional dependency: (4 marks)

  • (i) fully functional;
  • (ii) transitive.

Answers

(a) Database life cycle stages:

  • Physical stage: Deals with actual implementation on hardware
  • Logical stage: Focuses on database design and structure

(b) DBA functions:

  • Database security management
  • Performance monitoring and tuning
  • Backup and recovery procedures
  • User access management

(c) SQL statements:

(i) GRANT statement: Gives SELECT permission on projects table to User1

(ii) SQL queries:

SELECT StudentID, StudentName 
FROM Students 
WHERE CourseID = 'C240';

UPDATE Students 
SET TotalMarks = (SELECT SUM(marks) 
FROM Marks 
WHERE StudentName = 'Agnes')
WHERE StudentName = 'Agnes';

SELECT MIN(mark) 
FROM Marks 
WHERE CourseID = 'C236';

(d) Functional dependencies:

  • (i) Fully functional: Non-key attributes depend on entire primary key
  • (ii) Transitive: Attribute depends on another through intermediate attribute

Question 6

(a) Outline two concurrency controls applied in database management systems. (2 marks)

(b) State two differences between object oriented and relational database models. (4 marks)

(c)

  • (i) Define the term SQL injection as used in databases. (2 marks)
  • (ii) Write an SQL statement that would perform each of the following:
    • display all records from the table with records having status IN and arrange them based on the field name BookCategory in descending order; (3 marks)
    • remove all records with BookID labelled B02001 from the table; (2 marks)
    • display fields BookID, BookName for the books with BookID starting with B04 or BookCategory fiction. (3 marks)

(d) Explain two ways in which organisations can cope with challenges that arise from database emerging technologies. (4 marks)

Answers

(a) Concurrency controls:

  • Locking mechanisms
  • Timestamp ordering

(b) Object-oriented vs Relational:

  • Data structure (objects vs tables)
  • Inheritance support (supported vs not supported)

(c) SQL injection and queries:

(i) SQL injection: Attack technique where malicious SQL code is inserted into application queries

(ii) SQL statements:

SELECT * FROM Books 
WHERE status = 'IN' 
ORDER BY BookCategory DESC;

DELETE FROM Books 
WHERE BookID = 'B02001';

SELECT BookID, BookName 
FROM Books 
WHERE BookID LIKE 'B04%' 
OR BookCategory = 'fiction';

(d) Coping with emerging technologies:

  • Regular staff training
  • Phased implementation of new technologies
  • Maintaining documentation
  • Regular technology assessment

Question 7

(a) Define each of the following terms as used in databases: (4 marks)

  • (i) authorization;
  • (ii) authentication.

(b) Explain two circumstances that would prompt an organisation to use a special purpose database. (4 marks)

(c) Distinguish between a composite key and a compound key as used in databases. (4 marks)

(d) Write an SQL statement that would perform each of the following:

  • (i) create the table Car with the relevant fields; (3 marks)
  • (ii) make CarID the primary Key; (2 marks)
  • (iii) add fields named Make and Price that store 30 characters and numbers respectively. (3 marks)

Answers

(a) Security terms:

  • (i) Authorization: Determines what operations a user can perform
  • (ii) Authentication: Verifies user identity

(b) Special purpose database circumstances:

  • Need for specific data type handling (multimedia, spatial)
  • Specialized performance requirements
  • Industry-specific compliance requirements
  • Complex data relationships

(c) Key types:

  • Composite key: Multiple columns form a unique identifier
  • Compound key: Special type of composite key where all parts are foreign keys

(d) SQL statements:

CREATE TABLE Car (
    CarID VARCHAR(10),
    Model VARCHAR(50),
    Year INT
);

ALTER TABLE Car
ADD PRIMARY KEY (CarID);

ALTER TABLE Car
ADD COLUMN Make VARCHAR(30),
ADD COLUMN Price DECIMAL(10,2);

Question 8

(a) Define the term database security as used in databases. (2 marks)

(b) Explain two ways in which the role of a database designer complements that of a database developer. (4 marks)

(c)

  • (i) State an appropriate data type for the field named GradeName. (1 mark)
  • (ii) Write SQL statements to perform each of the following:
    • display the LecturerNo, LecturerName and Salary of all the lecturers with GradeName "Assistant Lecturer". (3 marks)
    • display the LectureNo, GradeID and Salary for all Lecturers above age 50 and YearEmployed is between 1980 and 1992. (3 marks)
    • display all details of the lecturers whose age is not captured in the table Lecturer. (3 marks)

(d) Jane intends to commence the requirements gathering phase for the development of a database management system in her organisation. Outline two reasons why she should consider each of the following during the phase:

  • (i) Reviewing of existing organisational documentation. (2 marks)
  • (ii) Understanding the current operating environment. (2 marks)

Answers

(a) Database security:

Protection of database from unauthorized access, modification, or destruction

(b) Designer and developer roles:

  • Designer focuses on structure, developer on implementation
  • Designer creates schema, developer builds functionality

(c) SQL queries:

(i) GradeName data type: VARCHAR(50)

(ii) SQL statements:

SELECT LecturerNo, LecturerName, Salary
FROM Lecturer
WHERE GradeName = 'Assistant Lecturer';

SELECT LectureNo, GradeID, Salary
FROM Lecturer
WHERE Age > 50 
AND YearEmployed BETWEEN 1980 AND 1992;

SELECT *
FROM Lecturer
WHERE Age IS NULL;

(d) Requirements gathering considerations:

(i) Reviewing documentation:

  • Understand existing processes
  • Identify data requirements

(ii) Understanding operating environment:

  • Identify technical constraints
  • Determine integration requirements
