Database Management Systems

November 2015 Past Paper

Paper Code: 2920/206

Duration: 3 hours

Module II

INSTRUCTIONS TO CANDIDATES

  • Write your name and index number in the spaces provided above.
  • Sign and write the date of examination in the spaces provided above.
  • Answer any FIVE of the following EIGHT questions.
  • Candidates must answer the questions in English.
  • This paper consists of 8 printed pages.

Question 1

(a)

  • (i) List six examples of Database Management Systems available in the market. (3 marks)
  • (ii) Describe a relational database management system. (2 marks)

(b) Describe each of the following components of a database:

  • (i) stored procedures; (2 marks)
  • (ii) tables; (2 marks)
  • (iii) triggers. (2 marks)

(c) With the aid of a diagram describe the three schema database architectures. (9 marks)

Answers

(a)

  • (i) Database Management Systems:
    • Oracle
    • MySQL
    • Microsoft SQL Server
    • PostgreSQL
    • MongoDB
    • IBM Db2
  • (ii) RDBMS: System that manages data using tables with rows and columns, enforcing relationships between tables using primary and foreign keys

(b) Database components:

  • Stored procedures: Pre-compiled SQL statements stored in database for repeated execution
  • Tables: Basic storage structures that hold data in rows and columns
  • Triggers: Special procedures that automatically execute in response to specific database events

(c) Three schema architecture:

  • External Schema (View Level):
    • User views of database
    • Different views for different users
    • Hides complexity
  • Conceptual Schema:
    • Logical structure of entire database
    • Entities, relationships, constraints
    • Independent of physical implementation
  • Internal Schema:
    • Physical storage structure
    • Access methods and indexing
    • Storage allocation

Question 2

(a) Outline three characteristics of an entity in a database. (3 marks)

(b) Describe each of the following criteria for classification of database management systems:

  • (i) based on data models; (2 marks)
  • (ii) based on the number of users; (2 marks)
  • (iii) based on database distribution. (2 marks)

(c) With the aid of an example distinguish between integrity and domain constraints as used in databases. (5 marks)

(d) With the aid of an example in each case describe each of the following attributes as used in databases:

  • (i) simple; (2 marks)
  • (ii) composite; (2 marks)
  • (iii) derived. (2 marks)

Answers

(a) Entity characteristics:

  • Has attributes that describe it
  • Must be uniquely identifiable
  • Can participate in relationships

(b) DBMS classification:

  • Data models:
    • Relational
    • Object-oriented
    • Network
    • Hierarchical
  • Number of users:
    • Single-user systems
    • Multi-user systems
  • Database distribution:
    • Centralized
    • Distributed

(c) Constraints:

  • Integrity constraints: 
    PRIMARY KEY, FOREIGN KEY constraints
  • Domain constraints: 
    Data type restrictions (e.g., age INT CHECK (age >= 0))

(d) Attribute types:

  • Simple: Single value (e.g., employee_id)
  • Composite: Multiple components (e.g., address = street + city + state)
  • Derived: Calculated from other attributes (e.g., age from date_of_birth)

Question 3

(a) Outline four advantages of using indexes in Structured Query Language. (4 marks)

(b) Write a Structured Query Language statements equivalent for each of the following relational algebra operations:

  • (i) projection; (2 marks)
  • (ii) difference; (2 marks)
  • (iii) union. (2 marks)

(c) Distinguish between a data administrator and a database administrator as used in databases. (4 marks)

(d) Alex is in the process of designing a database. Explain three phases that he should consider during design. (6 marks)

Answers

(a) Index advantages:

  • Faster data retrieval
  • Improved query performance
  • Enforces data uniqueness
  • Supports efficient sorting

(b) SQL equivalents:

(i) Projection:

SELECT DISTINCT column1, column2 FROM table;

(ii) Difference:

SELECT * FROM table1 
EXCEPT 
SELECT * FROM table2;

(iii) Union:

SELECT * FROM table1 
UNION 
SELECT * FROM table2;

(c) Administrator roles:

  • Data Administrator:
    • Strategic data planning
    • Data standards and policies
  • Database Administrator:
    • Technical implementation
    • Day-to-day operations

(d) Database design phases:

  • Requirements analysis:
    • Gather user requirements
    • Document data needs
  • Conceptual design:
    • Create ER diagrams
    • Define entities and relationships
  • Logical design:
    • Transform to relational schema
    • Normalize tables

Question 4

(a) With the aid of an example describe a hypermedia database. (3 marks)

(b) Explain why each of the following is a threat to databases:

  • (i) loss of integrity; (2 marks)
  • (ii) loss of availability; (2 marks)
  • (iii) loss of confidentiality. (2 marks)

(c) In a college a lecturer may teach many subjects but may not belong to more than one department. The college maintains information of its lecturers' subject area as follows:

Lecturer Number, Lecturer Name, Lecturer Grade, Department Code, Department Name, Subject Code, Subject Name and Subject Level

Represent this information to 3rd Normal Form. (11 marks)

Answers

(a) Hypermedia database:

Database that stores and manages multimedia content (text, images, audio, video) with hyperlinks between them. Example: Wikipedia database with linked articles and media

(b) Database threats:

  • Loss of integrity:
    • Data becomes inaccurate or inconsistent
    • Compromises decision-making reliability
  • Loss of availability:
    • Users cannot access needed data
    • Business operations disrupted
  • Loss of confidentiality:
    • Unauthorized access to sensitive data
    • Privacy breaches

(c) Normalization to 3NF:

1NF: Remove repeating groups
LECTURER_INFO(Lecturer_Number, Lecturer_Name, Lecturer_Grade, Department_Code, 
    Department_Name, Subject_Code, Subject_Name, Subject_Level)

2NF: Remove partial dependencies
LECTURER(Lecturer_Number, Lecturer_Name, Lecturer_Grade, Department_Code)
DEPARTMENT(Department_Code, Department_Name)
SUBJECT(Subject_Code, Subject_Name, Subject_Level)
LECTURER_SUBJECT(Lecturer_Number, Subject_Code)

3NF: Remove transitive dependencies
- Already in 3NF as no transitive dependencies exist

Question 5

(a)

  • (i) Describe the term repeating group as used in normalization. (2 marks)
  • (ii) Outline three goals that would be achieved from using a normalized table. (3 marks)

(b) Describe each of the following terms as used in database management:

  • (i) domain relational calculus; (2 marks)
  • (ii) query optimizer; (2 marks)
  • (iii) relational algebra. (2 marks)

Answers

(a) Normalization concepts:

  • (i) Repeating group: Multiple values of the same type in a single record (e.g., multiple phone numbers in one field)
  • (ii) Normalization goals:
    • Minimize data redundancy
    • Ensure data consistency
    • Simplify data maintenance

(b) Database terms:

  • Domain relational calculus:
    • Non-procedural query language
    • Specifies what to retrieve, not how
  • Query optimizer:
    • Component that determines most efficient query execution plan
    • Analyzes possible execution strategies
  • Relational algebra:
    • Procedural query language
    • Defines sequence of operations to get results
