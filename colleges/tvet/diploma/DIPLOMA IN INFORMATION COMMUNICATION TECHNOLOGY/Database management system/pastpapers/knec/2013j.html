Database Management Systems

July 2013 Past Paper

Paper Code: 2920/206

Duration: 3 hours

Module II

INSTRUCTIONS TO CANDIDATES

  • Write your name and index number in the spaces provided above.
  • Sign and write the date of examination in the spaces provided above.
  • Answer any FIVE of the EIGHT questions in the spaces provided.
  • All questions carry equal marks.

  • Candidates should check to ascertain that all pages are printed as indicated and that no questions are missing.

Question 1

(a) Define the term data dictionary as used in database management systems. (2 marks)

(b) Explain three reasons why database security is important in an organization. (6 marks)

(c) State four considerations that should be made when choosing a database management system. (4 marks)

(d) With the aid of a diagram, explain the three-schema architecture. (8 marks)

Answers

(a) Data dictionary:

A data dictionary is a centralized repository of information about data such as meaning, relationships to other data, origin, usage, and format. It contains metadata about the database structure, tables, columns, indexes, and other database objects.

(b) Importance of database security:

  1. Protect confidential information from unauthorized access
    • Prevent data breaches
    • Maintain privacy of sensitive data
  2. Ensure data integrity
    • Prevent unauthorized modifications
    • Maintain accuracy of data
  3. Comply with regulations
    • Meet legal requirements
    • Avoid penalties

(c) DBMS selection considerations:

  1. Cost (licensing, maintenance)
  2. Scalability requirements
  3. Technical support availability
  4. Performance capabilities

(d) Three-schema architecture:

External Schema (View Level)
        ↑
    Conceptual Schema
        ↑
Internal Schema (Physical Level)
  • External Schema: User's view of the database
  • Conceptual Schema: Community view of the database
  • Internal Schema: Physical storage specifications

Question 2

(a) Explain three properties of a relation. (6 marks)

(b) Using examples, distinguish between:

  • (i) entity and attribute; (2 marks)
  • (ii) weak and strong entity. (2 marks)

(c) Describe the purpose of each of these operations in relational algebra:

  • (i) projection; (2 marks)
  • (ii) selection. (2 marks)

(d) State three advantages of using views in a database. (6 marks)

Answers

(a) Properties of a relation:

  1. Each cell contains exactly one atomic value
  2. Each attribute has a unique name
  3. Each tuple is unique (no duplicate rows)

(b) Distinctions:

Entity Attribute
Real-world object (e.g., Student) Property of entity (e.g., student_name)
Weak Entity Strong Entity
Depends on another entity Exists independently

(c) Relational algebra operations:

  • Projection (π): Selects specific columns
  • Selection (σ): Filters rows based on condition

(d) Advantages of views:

  1. Enhanced security through data hiding
  2. Simplified complex queries
  3. Data independence

Question 3

(a) State three functions of a Database Administrator (DBA). (6 marks)

(b) Define each of the following terms as used in database management:

  • (i) primary key; (2 marks)
  • (ii) foreign key; (2 marks)
  • (iii) composite key. (2 marks)

(c) Explain two advantages of using distributed databases. (4 marks)

(d) Describe two data manipulation operations performed on a database. (4 marks)

Answers

(a) DBA functions:

  1. Database design and implementation
  2. Security administration
  3. Performance monitoring and tuning

(b) Key definitions:

  • Primary Key: Unique identifier for each record in a table
  • Foreign Key: References primary key in another table
  • Composite Key: Multiple columns together forming a unique identifier

(c) Advantages of distributed databases:

  • Improved reliability and availability
  • Better performance through local access

(d) Data manipulation operations:

  • Insert: Adding new records
  • Update: Modifying existing records

Question 4

(a) Define the term referential integrity. (2 marks)

(b) List four examples of integrity constraints in a database. (4 marks)

(c) Explain three methods used to maintain data integrity in a database. (6 marks)

(d) Describe four types of database users. (8 marks)

Answers

(a) Referential integrity:

A rule that ensures relationships between tables remain consistent. It ensures that foreign key values always reference existing primary key values.

(b) Integrity constraints:

  1. Entity integrity (Primary Key)
  2. Referential integrity (Foreign Key)
  3. Domain integrity (Data type, range)
  4. User-defined integrity

(c) Methods to maintain data integrity:

  1. Input validation
  2. Transaction management
  3. Backup and recovery procedures

(d) Database users:

  1. Database Administrator (DBA)
  2. Application Programmers
  3. Sophisticated Users
  4. Naive Users

Question 5

(a) List four characteristics of a database. (4 marks)

(b) Explain three advantages of using a database approach over file processing approach. (6 marks)

(c) State three types of relationships that can exist between entities in a database. (6 marks)

(d) Define the term data independence. (4 marks)

Answers

(a) Database characteristics:

  1. Self-describing nature
  2. Data independence
  3. Data integrity
  4. Concurrent access

(b) Advantages of database approach:

  1. Reduced data redundancy
    • Centralized data storage
    • Better data consistency
  2. Data independence
    • Physical independence
    • Logical independence
  3. Better security
    • Centralized control
    • Access control

(c) Types of relationships:

  1. One-to-One (1:1)
  2. One-to-Many (1:M)
  3. Many-to-Many (M:N)

(d) Data independence:

The ability to modify the schema at one level without affecting the schema at the next higher level. This includes:

  • Physical data independence
  • Logical data independence

Question 6

(a) Outline two rules applied on tables in their First Normal Form (1NF) tables. (2 marks)

(b) Describe each of the following types of attributes as used in Entity Relationship models:

  • (i) simple; (2 marks)
  • (ii) derived. (2 marks)

(c) In an organisation a consultant cannot exist without an employee while an employee can exist without a consultant. An employee can be identified with the following: employeeno, name, age, gender and salary while the consultant can be identified with the consultantno and designation. Represent this information in an ER diagram. (6 marks)

(d)

  • (i) Identify the type of database organisation approach used in Figure 2 giving a reason for your answer. (2 marks)
  • (ii) Explain three benefits an organisation would achieve when using this approach. (6 marks)

Answers

(a) 1NF rules:

  • No repeating groups of columns
  • All attributes must be atomic (single-valued)

(b) Attribute types:

  • Simple attribute:
    • Single, atomic value
    • Example: employee ID, phone number
  • Derived attribute:
    • Value calculated from other attributes
    • Example: age from date of birth, total from quantity×price

(c) ER diagram components:

  • Entities:
    • Employee (Strong entity): employeeno, name, age, gender, salary
    • Consultant (Weak entity): consultantno, designation
  • Relationship:
    • Employee to Consultant: One-to-One
    • Total participation from Consultant side
    • Partial participation from Employee side

Question 7

(a) State two rules which should be considered when using the following SQL statements:

  • (i) sum(); (2 marks)
  • (ii) null. (2 marks)

(b) Maurice would like to denormalize the tables he created in a database. Explain two reasons that may have led him to perform denormalisation. (4 marks)

(c) Write an SQL statement in each case that would be used to extract all the records in a table that meet each of the following criteria:

  • (i) whose field name starts with letter a; (2 marks)
  • (ii) whose field name has a pattern letters la; (2 marks)
  • (iii) whose last letter in the field name is m. (2 marks)

(d) Using Tables 2 and 3, state the algebraic operation that would be used to generate each of the given outputs from the tables. Justify your answer. (6 marks)

Answers

(a) SQL function rules:

  • SUM():
    • Works only on numeric columns
    • Ignores NULL values
  • NULL:
    • Cannot be compared using = operator
    • Must use IS NULL or IS NOT NULL

(b) Denormalization reasons:

  • Improve query performance
  • Reduce number of joins needed
  • Optimize read operations
  • Meet specific business requirements

(c) SQL pattern matching:

SELECT * FROM table 
WHERE name LIKE 'a%';

SELECT * FROM table 
WHERE name LIKE '%la%';

SELECT * FROM table 
WHERE name LIKE '%m';

Question 8

(a) With the aid of a diagram describe three levels of data abstraction in relational databases. (8 marks)

(b) Distinguish between Boyce code normal form (BCNF) and third normal form (3NF) in relation to normalization. (4 marks)

(c) Using the invoices shown in Figure 4 and Figure 5 from Nairobi Hardware Company Ltd, represent the invoices in un-normalized table. (8 marks)

Answers

(a) Data abstraction levels:

  • Physical level:
    • Describes how data is stored
    • Deals with storage structures
  • Logical level:
    • Describes what data is stored
    • Defines relationships between data
  • View level:
    • User's perspective of data
    • Hides complexity of lower levels

(b) Normalization forms:

  • BCNF:
    • Stricter than 3NF
    • Every determinant must be a candidate key
  • 3NF:
    • Eliminates transitive dependencies
    • Allows non-prime attributes to determine other non-prime attributes
