Database Management Systems

July 2016 Past Paper

Paper Code: 2920/206

Duration: 3 hours

Module II

INSTRUCTIONS TO CANDIDATES

  • This paper consists of EIGHT questions.
  • Answer any FIVE questions in the answer booklet provided.
  • All questions carry equal marks.
  • Candidates should answer the questions in English.
  • This paper consists of 3 printed pages.

Question 1

(a) Describe two components of a relational database. (4 marks)

(b) With the aid of an example distinguish between a system privilege and an object privilege as used in databases. (4 marks)

(c) Explain each of the following terms as used in data recovery in a database:

  • (i) checkpoint; (2 marks)
  • (ii) manual reprocessing; (2 marks)
  • (iii) immediate update. (2 marks)

(d) Peter would like to create a database system. Explain three stages in the database design development that the ER diagram would be most applicable. (6 marks)

Answers

(a) Relational database components:

  • Tables (Relations): Store data in rows and columns
  • Relationships: Define connections between tables

(b) Privileges:

  • System privilege: 
    CREATE USER, CREATE TABLE

    (affects entire system)

  • Object privilege: 
    SELECT ON employees, UPDATE ON customers

    (affects specific objects)

(c) Data recovery terms:

  • Checkpoint: Point in time where database state is saved to disk
  • Manual reprocessing: Human intervention to recover data after failure
  • Immediate update: Changes written to disk immediately after transaction

(d) ER diagram applicable stages:

  • Requirements analysis: Modeling business rules and entities
  • Conceptual design: Creating high-level data model
  • Logical design: Translating conceptual model to database schema

Question 2

(a)

  • (i) Outline two wild card characters used in structured query language. (2 marks)
  • (ii) Explain a reason for using If Exist command in a structured query language. (2 marks)

(b) Distinguish between active data dictionary and passive data dictionary as used in databases. (4 marks)

(c) Describe each of the following components of three-tier database architecture:

  • (i) application tier; (2 marks)
  • (ii) presentation tier; (2 marks)
  • (iii) data tier. (2 marks)

(d) Figure 1 shows a filing approach that an educational institution uses to manage its applications. Explain three advantages that the institution will benefit from this approach. (6 marks)

Answers

(a) SQL wildcards and IF EXISTS:

  • (i) Wildcards:
    • % (matches any sequence of characters)
    • _ (matches any single character)
  • (ii) IF EXISTS: Prevents errors by checking if object exists before operation

(b) Data dictionary types:

  • Active: Automatically maintains and updates metadata
  • Passive: Requires manual updates and maintenance

(c) Three-tier architecture:

  • Application tier: Business logic and data processing
  • Presentation tier: User interface and data presentation
  • Data tier: Database storage and data access

(d) Filing approach advantages:

  • Simple implementation
  • Direct access to records
  • Easy to understand and maintain

Question 3

(a) Outline four factors to consider when developing a user interface for a database system. (4 marks)

(b) Distinguish between the database engine and a database schema as used in database management system. (4 marks)

(c) Martin would like to enforce security in a multiple user database environment using data control commands. Describe two command statements that he is likely to use giving an example in each case. (6 marks)

(d) An organisation intends to acquire a database application to manage its data. Describe three professionals that may be required. (6 marks)

Answers

(a) User interface factors:

  • User experience level
  • Ease of navigation
  • Response time
  • Error handling

(b) Engine vs Schema:

  • Database engine: Software that manages data storage and retrieval
  • Database schema: Logical structure and organization of data

(c) Security commands:

GRANT SELECT ON employees TO user1;
REVOKE UPDATE ON customers FROM user2;

(d) Required professionals:

  • Database Administrator: Manages and maintains database
  • Database Designer: Plans database structure
  • Application Developer: Creates user interface and applications

Question 4

(a) Outline four guidelines to consider when using sub queries in a structured query language. (4 marks)

(b) Distinguish between Embedded and Dynamic structured query language. (4 marks)

(c) Describe two levels of data manipulation language giving an example in each case. (6 marks)

(d) Write the equivalent of each of the following logical operations in relational calculus:

  • (i) P1∧P2 (2 marks)
  • (ii) ∀i∈P(i) (2 marks)
  • (iii) P1=P2 (2 marks)

Answers

(a) Subquery guidelines:

  • Enclose in parentheses
  • Use appropriate comparison operator
  • Match number of columns in comparison
  • Consider performance impact

(b) SQL types:

  • Embedded SQL: Hard-coded queries in application code
  • Dynamic SQL: Queries built and executed at runtime

(c) DML levels:

  • Procedural: 
    SELECT * FROM emp WHERE salary > 5000
  • Non-procedural: 
    FIND EMPLOYEE WHERE SALARY > 5000

(d) Relational calculus:

  • (i) P1∧P2: {x | P1(x) AND P2(x)}
  • (ii) ∀i∈P(i): {x | ∀i(P(i) → Q(i,x))}
  • (iii) P1=P2: {x | P1(x) ↔ P2(x)}

Question 5

(a) A teacher noted that a database created by Andrew had data redundancy. Outline three problems that this may cause. (3 marks)

(b) In an organisation an employee is identified by a unique number, salary and telephone contact. Each employee works in a department. Each department is managed by an Employee and is identified by a department number, name and its budgetary allocation. Each employee may have a dependent child who is identified uniquely by employee's unique number, a name and age.

  • (i) Draw an ER diagram to represent the narrative. (7 marks)
  • (ii) Write a structured query language used to create one of the entities in the ER diagram in (i). (4 marks)

(c) Describe three F. Armstrong's Axioms set of rules that generates functional dependencies as applied in normalisation of tables. (6 marks)

Answers

(a) Data redundancy problems:

  • Inconsistent data updates
  • Increased storage costs
  • Data integrity issues

(b) ER Diagram and SQL:

CREATE TABLE Employee (
    emp_id INT PRIMARY KEY,
    salary DECIMAL(10,2),
    phone VARCHAR(15),
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES Department(dept_id)
);

(c) Armstrong's Axioms:

  • Reflexivity: If Y ⊆ X, then X → Y
  • Augmentation: If X → Y, then XZ → YZ
  • Transitivity: If X → Y and Y → Z, then X → Z

Question 6

(a) State two similarities between a hierarchical and network database models. (4 marks)

(b) Distinguish between relational algebra and relational calculus as used in databases. (4 marks)

(c) Write a relational algebraic statement to perform each of the following:

  • (i) Delete all loans with loan numbers between 1300 and 1500 from a deposit table. (2 marks)
  • (ii) Increase all balances by 5% in a deposit table. (2 marks)
  • (iii) Display empname, department where salary is greater than 50000 from the deposit table. (2 marks)

(d) Explain three categories of anomalies that may be experienced when working with tables that are not normalised in a database. (6 marks)

Answers

(a) Hierarchical and Network similarities:

  • Record-based models
  • Use physical pointers between records

(b) Algebra vs Calculus:

  • Relational algebra: Procedural, specifies operations
  • Relational calculus: Declarative, specifies what to retrieve

(c) Relational algebra:

σloan_no BETWEEN 1300 AND 1500(deposit)

ρbalance←balance*1.05(deposit)

πempname,department(σsalary>50000(deposit))

(d) Anomalies:

  • Update anomalies: Inconsistent changes
  • Insertion anomalies: Cannot add certain data
  • Deletion anomalies: Unintended data loss

Question 7

(a) Outline four structured query language commands that may be used to enhance search capabilities in a database. (4 marks)

(b) Explain two reasons for using online database. (4 marks)

(c) State three differences between a primary key and a foreign key as used in database. (6 marks)

(d) Outline three characteristics of each of the following distributed databases:

  • (i) Homogeneous. (3 marks)
  • (ii) Heterogeneous. (3 marks)

Answers

(a) SQL search commands:

  • LIKE: Pattern matching
  • BETWEEN: Range search
  • IN: Multiple value search
  • EXISTS: Subquery search

(b) Online database reasons:

  • Remote access capability
  • Real-time data updates
  • Shared access for multiple users
  • Centralized data management

(c) Primary vs Foreign key:

  • Primary key: Uniquely identifies records in its table
  • Foreign key: References primary key in another table
  • Primary key cannot be null, foreign key can be null

(d) Distributed database characteristics:

(i) Homogeneous:

  • Same DBMS at all sites
  • Consistent data model
  • Unified management

(ii) Heterogeneous:

  • Different DBMSs at different sites
  • Multiple data models
  • Complex integration needed

Question 8

(a) Outline four characteristics of a well designed database. (4 marks)

(b) Describe each of the cardinalities represented in the given ER diagram. (4 marks)

(c) Write tuple relational calculus statements for each of the following:

  • (i) find the name of all employees in the employee table who work for the bank named Mini Bank; (2 marks)
  • (ii) find all customers having both a loan and an account at the Kenyatta branch. (2 marks)
  • (iii) display all fields from teaching (T) table where teaching code="K2000". (2 marks)

(d) Table 1 shows details about students recorded in un-normalised table. Represent the information to 2nd normal form. (6 marks)

Answers

(a) Well-designed database characteristics:

  • Minimal data redundancy
  • Data integrity maintained
  • Efficient data retrieval
  • Easy to maintain and modify

(c) Tuple relational calculus:

{e.name | employee(e) ∧ e.bank_name = 'Mini Bank'}

{c | customer(c) ∧ ∃l,a(loan(l) ∧ account(a) ∧ 
     l.customer_id = c.id ∧ a.customer_id = c.id ∧
     l.branch = 'Kenyatta' ∧ a.branch = 'Kenyatta')}

{t | T(t) ∧ t.teaching_code = 'K2000'}

(d) 2NF normalization steps:

  • Identify partial dependencies
  • Create separate tables for each dependency
  • Establish relationships between tables
